## 一、功能概述

集成LanceDB向量数据库，实现OpenClaw mem9 L2温记忆架构，支持章节向量、知识库向量、风格向量的存储和检索。

### 核心能力

- **章节向量存储**: 长篇小说连贯性召回
- **知识库向量存储**: 分类知识库（科幻/玄幻/历史）+ 通用知识库（哲学/逻辑/语言学）
- **风格向量存储**: 经典文学作品风格特征
- **向量检索**: 余弦相似度top-k召回
- **增量更新**: 新章节追加，无需重建索引

## 三、数据模型

### 3.1 Pydantic模型

#### ChapterVector（章节向量）

```python
class ChapterVector(LanceModel):
    """章节向量模型"""
    chapter_id: str          # 章节ID（如chapter-001）
    content: str             # 章节文本内容
    metadata: Dict[str, Any] # 元数据（标题、字数等）
    created_at: str          # 创建时间
    updated_at: str          # 更新时间
    # vector字段由LanceDB自动管理
```

#### KnowledgeVector（知识向量）

```python
class KnowledgeVector(LanceModel):
    """知识库向量模型"""
    knowledge_id: str        # 知识点ID（如scifi-physics-001）
    category: str            # 分类（科幻/玄幻/历史）
    domain: str              # 领域（物理/化学/生物/宗教/神话）
    title: str               # 知识点标题
    content: str             # 知识点详细内容
    keywords: List[str]      # 关键词列表
    metadata: Dict[str, Any] # 元数据（引用、来源等）
    created_at: str
    updated_at: str
```

#### StyleVector（风格向量）

```python
class StyleVector(LanceModel):
    """风格向量模型"""
    style_id: str            # 风格ID（如style-chinese-modern）
    author: str              # 作者/风格流派
    content: str             # 风格样本文本
    style_features: Dict[str, Any] # 风格特征
    metadata: Dict[str, Any]
    created_at: str
    updated_at: str
```

#### VectorSearchResult（检索结果）

```python
class VectorSearchResult(BaseModel):
    """向量检索结果"""
    id: str                  # ID
    content: str             # 文本内容
    score: float             # 相似度分数（0-1）
    metadata: Dict[str, Any] # 元数据
```

## 四、关键技术实现

### 4.1 Embedding函数封装

支持两种Embedding方式：

#### OpenAI Embedding API

```python
# 使用OpenAI text-embedding-3-small
embed_func = EmbeddingFunction(
    model_type="openai",
    model_name="text-embedding-3-small",
    api_key="sk-...",
    base_url="https://api.openai.com/v1"
)

# 维度：1536（small）或 3072（large）
```

#### 本地模型（sentence-transformers）

```python
# 使用本地模型all-MiniLM-L6-v2
embed_func = EmbeddingFunction(
    model_type="local",
    model_name="all-MiniLM-L6-v2"
)

# 维度：384
```

**关键API**:

- `embed(text: str) -> List[float]`: 生成单个文本的向量
- `embed_batch(texts: List[str]) -> List[List[float]]`: 批量生成向量
- `get_dimension() -> int`: 获取向量维度

```python
# 添加章节
vector_store.add_chapter(
    chapter_id="chapter-001",
    content="第一章的内容...",
    metadata={"title": "开篇", "word_count": 3000}
)

# 批量添加
chapters = [
    {"chapter_id": "chapter-001", "content": "...", "metadata": {...}},
    {"chapter_id": "chapter-002", "content": "...", "metadata": {...}}
]
count = vector_store.add_chapters_batch(chapters)

# 召回相似章节（长篇连贯性）
results = vector_store.recall_similar_chapters(
    query="新章节大纲：星际飞船发现外星文明",
    top_k=10
)

for result in results:
    print(f"{result.id}: {result.content[:50]}... (score: {result.score})")
```

## 五、架构设计

### 5.1 分层架构

```
┌─────────────────────────────────────────────┐
│         应用层（iterative_generator_v2）     │
├─────────────────────────────────────────────┤
│         向量存储层（NovelVectorStore）       │
├─────────────────────────────────────────────┤
│    Embedding层（OpenAI/本地模型）            │
├─────────────────────────────────────────────┤
│    数据库层（LanceDB嵌入式向量数据库）        │
└─────────────────────────────────────────────┘
```

### 5.2 表结构

**chapters表**（章节向量）:
- chapter_id: 章节ID（主键）
- content: 章节内容
- vector: 向量（1536维或384维）
- metadata: 元数据（JSON）
- created_at: 创建时间
- updated_at: 更新时间

**knowledge表**（知识向量）:
- knowledge_id: 知识点ID（主键）
- category: 分类（科幻/玄幻/历史）
- domain: 领域（物理/化学/生物/宗教/神话）
- title: 标题
- content: 内容
- vector: 向量
- keywords: 关键词（JSON数组）
- metadata: 元数据（JSON）
- created_at: 创建时间
- updated_at: 更新时间

**styles表**（风格向量）:
- style_id: 风格ID（主键）
- author: 作者/风格流派
- content: 风格样本文本
- vector: 向量
- style_features: 风格特征（JSON）
- metadata: 元数据（JSON）
- created_at: 创建时间
- updated_at: 更新时间

### 5.3 检索流程

```
用户查询文本
    ↓
调用Embedding API生成向量
    ↓
LanceDB向量检索（余弦相似度）
    ↓
返回top-k结果（按相似度排序）
    ↓
应用层处理结果
```

## 八、使用示例

### 8.1 长篇连贯性召回

```python
# 在iterative_generator_v2中集成
from infrastructure.vector_store import get_vector_store

class IterativeGenerator:
    def __init__(self):
        self.vector_store = get_vector_store()
    
    def generate_with_recall(self, new_chapter_outline: str):
        # 1. 召回相似章节
        recalled = self.vector_store.recall_similar_chapters(
            query=new_chapter_outline,
            top_k=10
        )
        
        # 2. 构建上下文
        context = self.build_context_from_recall(recalled)
        
        # 3. 生成新章节
        new_chapter = self.generate_with_context(
            outline=new_chapter_outline,
            context=context
        )
        
        # 4. 添加到向量库
        self.vector_store.add_chapter(
            chapter_id=f"chapter-{self.chapter_count}",
            content=new_chapter,
            metadata={"outline": new_chapter_outline}
        )
        
        return new_chapter
```

### 8.2 知识库召回集成

```python
# 在enhanced_weighted_validator中集成
from infrastructure.vector_store import get_vector_store

class EnhancedWeightedValidator:
    def __init__(self):
        self.vector_store = get_vector_store()
    
    def validate_with_knowledge(self, content: str, genre: str):
        # 1. 召回相关知识
        knowledge = self.vector_store.recall_knowledge(
            query=content,
            category=genre,
            top_k=10
        )
        
        # 2. 检测知识一致性
        consistency_score = self.check_knowledge_consistency(
            content=content,
            knowledge=knowledge
        )
        
        return {
            "knowledge_consistency": consistency_score,
            "recalled_knowledge": knowledge
        }
```
