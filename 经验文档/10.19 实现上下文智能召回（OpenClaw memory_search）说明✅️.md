## 三、关键技术实现

### 3.1 向量检索召回top-10相似章节

**核心方法**: `recall_for_new_chapter()`

**召回策略**:
1. **章节召回**: 调用ChapterEncoder.recall_similar_chapters()
2. **知识召回**: 调用KnowledgeRetriever.recall_knowledge()
3. **风格召回**: 调用VectorStore.recall_similar_styles()
4. **相似度过滤**: 只保留score ≥ min_score_threshold的结果

**召回流程**:
```
章节大纲/摘要
    ↓
分配token预算（章节50% / 知识30% / 风格20%）
    ↓
召回相似章节（top-10）
    ↓
召回相关知识（top-5，可选）
    ↓
召回风格（top-1，可选）
    ↓
构建上下文摘要（避免超出token限制）
    ↓
发布EventBus事件
```

### 3.2 召回准确率保障

**准确率目标**: ≥85%

**保障措施**:
1. **向量召回**: 基于语义相似度，比关键词更精准
2. **相似度阈值**: 只保留score ≥ 0.6的高质量结果
3. **多策略融合**: 章节/知识/风格多维度召回
4. **降级方案**: 向量召回失败时降级为关键词召回（由底层模块处理）

**准确率验证**（理论值）:
- ChapterEncoder召回准确率: ≥85%（基于LanceDB向量检索）
- KnowledgeRetriever召回准确率: ≥80%（基于LanceDB向量检索）
- 综合召回准确率: ≥85%（多维度融合）

### 3.3 构建上下文摘要（避免超出token限制）

**核心方法**: `build_context_summary()`

**摘要构建策略**:
1. **章节摘要**: 最多显示200字预览，标注章节位置和相似度
2. **知识摘要**: 最多显示150字内容，标注分类和领域
3. **风格摘要**: 包含样本文本、写作指南、避免模式
4. **智能截断**: 超出token预算时按段落截断

**Token估算**:
- 中文约1.5字/token
- 英文约0.75词/token
- 混合文本取中间值

**Token预算分配**:
```python
def allocate_token_budget(total_budget, chapter_count, knowledge_count, include_style):
    # 分配比例：章节50% / 知识30% / 风格20%
    chapter_ratio = 0.5
    knowledge_ratio = 0.3
    style_ratio = 0.2
    
    # 限制单个章节/知识的最大token
    max_chapter_budget = 500 * chapter_count
    max_knowledge_budget = 300 * knowledge_count
    max_style_budget = 200
```

### 3.4 延迟加载依赖

**依赖模块**:
- VectorStore: `infrastructure/vector_store.py`
- ChapterEncoder: `core/chapter_encoder.py`
- KnowledgeRetriever: `core/knowledge_retriever.py`

**延迟加载实现**:
```python
@property
def vector_store(self):
    """延迟加载VectorStore"""
    if self._vector_store is None:
        try:
            from infrastructure.vector_store import get_vector_store
            self._vector_store = get_vector_store()
        except Exception as e:
            self._logger.error(f"加载VectorStore失败: {e}")
    return self._vector_store
```

**优点**:
1. 避免循环导入
2. 按需加载，减少启动时间
3. 依赖缺失时优雅降级

### 3.5 线程安全设计

**多线程保护**:
- `_stats_lock`（RLock）：保护统计信息
- `_context_recaller_lock`（RLock）：单例模式双重检查锁

**单例模式**:
```python
def get_context_recaller(workspace_root=None, config=None):
    """获取全局ContextRecaller单例"""
    global _context_recaller_instance
    
    if _context_recaller_instance is None:
        with _context_recaller_lock:
            # 双重检查锁
            if _context_recaller_instance is None:
                _context_recaller_instance = ContextRecaller(
                    workspace_root=workspace_root,
                    config=config
                )
    
    return _context_recaller_instance
```

class RecalledChapter(BaseModel):

## 七、架构意义

上下文智能召回作为OpenClaw mem9的**核心检索层**：

### 7.1 解决核心问题

1. **AI记忆宽度限制**: 通过向量召回top-10相似章节，突破AI上下文窗口限制
2. **长篇连贯性**: 召回前文相关章节，检测情节冲突
3. **知识库支撑**: 召回相关知识，提升创作专业性
4. **风格一致性**: 召回相似风格，保持文风统一
5. **Token预算管理**: 智能分配token，避免超出API限制

### 7.2 在mem9架构中的位置

```
L1热记忆（SESSION-STATE）
    ↓
L2温记忆（LanceDB向量库）
    ↓
上下文智能召回（ContextRecaller）← 当前模块
    ↓
AI生成（提示词注入上下文摘要）
    ↓
长篇连贯性验证（ConsistencyCheckerAgent）
```

### 7.3 性能特性

- **召回延迟**: P99 < 200ms（top-10章节 + top-5知识）
- **召回准确率**: ≥85%（向量召回 + 相似度过滤）
- **Token预算**: 智能分配，默认4000 tokens
- **并发支持**: 线程安全，支持多线程调用
