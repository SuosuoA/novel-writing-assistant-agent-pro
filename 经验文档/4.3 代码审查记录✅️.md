### 2.1 🔴 P0-1: LLM客户端缓存无容量限制

**位置**: `services/llm_client_with_resilience.py:76-78`

**问题描述**:
```python
# 请求缓存
self._cache: Dict[str, Dict[str, Any]] = {}
self._cache_ttl = 3600  # 1小时
```

缓存字典没有大小限制，长时间运行可能导致内存泄漏。

**影响**:
- 内存无限增长
- 长时间运行后可能OOM

**修复建议**:
```python
from collections import OrderedDict

class LRUCache:
    """带容量限制的LRU缓存"""
    def __init__(self, max_size: int = 1000):
        self._cache: OrderedDict[str, Dict[str, Any]] = OrderedDict()
        self._max_size = max_size
        self._lock = threading.Lock()
    
    def get(self, key: str) -> Optional[Any]:
        with self._lock:
            if key not in self._cache:
                return None
            # LRU: 移到最后
            self._cache.move_to_end(key)
            return self._cache[key]
    
    def set(self, key: str, value: Any) -> None:
        with self._lock:
            if key in self._cache:
                self._cache.move_to_end(key)
            else:
                if len(self._cache) >= self._max_size:
                    self._cache.popitem(last=False)  # 移除最旧
            self._cache[key] = value
```

### 2.3 🔴 P0-3: TaskQueue任务状态不一致

**位置**: `agents/master_agent.py:173-178`

**问题描述**:
```python
# 标记为进行中
if not self._dependency_state.mark_in_progress(task.task_id):
    logger.warning(f"任务 {task.task_id} 状态异常，跳过执行")
    return

# 从队列移除
self._task_queue.pop()
```

先标记状态再pop队列，如果`mark_in_progress`失败会return，但队列中该任务仍存在，造成状态不一致。

**影响**:
- 任务可能被重复执行或丢失
- 队列状态与依赖状态不同步

**修复建议**:
```python
# 先从队列移除
task = self._task_queue.pop()
if task is None:
    return

# 再标记状态
if not self._dependency_state.mark_in_progress(task.task_id):
    logger.warning(f"任务 {task.task_id} 状态异常，重新入队")
    self._task_queue.push(task)  # 回滚
    return
```

**负责人**: Agent系统工程师
**优先级**: 立即修复

### 3.1 🟡 P1-1: EventBus线程池shutdown兼容性问题

**位置**: `core/event_bus.py:419-425`

**问题描述**:
```python
def shutdown(self) -> None:
    try:
        self._executor.shutdown(wait=True, cancel_futures=False)
    except Exception:
        # Python 3.8 不支持 cancel_futures 参数
        self._executor.shutdown(wait=True)
```

使用裸露的`except Exception`会掩盖真实错误，且注释描述不准确（Python 3.9+才支持cancel_futures）。

**修复建议**:
```python
def shutdown(self) -> None:
    import sys
    if sys.version_info >= (3, 9):
        self._executor.shutdown(wait=True, cancel_futures=False)
    else:
        self._executor.shutdown(wait=True)
```

**优先级**: 本周修复

### 3.2 🟡 P1-2: ConnectionPool连接未正确关闭

**位置**: `core/database.py:95-102`

**问题描述**:
```python
def close_all(self) -> None:
    if hasattr(self._thread_local, "connection") and self._thread_local.connection:
        try:
            self._thread_local.connection.close()
        except Exception:
            pass
        self._thread_local.connection = None
```

只关闭当前线程的连接，其他线程的连接会泄漏。

**影响**:
- 多线程环境下SQLite连接泄漏
- 可能导致数据库锁定

**修复建议**:
```python
class ConnectionPool:
    def __init__(self, db_path: str, pool_size: int = 5, timeout: float = 30.0):
        # ...
        self._all_connections: List[sqlite3.Connection] = []
        self._connections_lock = threading.Lock()
    
    def get_connection(self) -> sqlite3.Connection:
        # ...
        with self._connections_lock:
            self._all_connections.append(conn)
        return conn
    
    def close_all(self) -> None:
        with self._connections_lock:
            for conn in self._all_connections:
                try:
                    conn.close()
                except Exception:
                    pass
            self._all_connections.clear()
```

**负责人**: 后端开发工程师
**优先级**: 本周修复
