### P0-3: ConnectionPool连接泄漏风险

**位置**: `core/database.py:103-115`

**问题**:
```python
def close_all(self) -> None:
    """关闭所有连接"""
    with self._connections_lock:
        for conn in self._all_connections:
            try:
                conn.close()
            except Exception:
                pass
        self._all_connections.clear()
    
    # 清理线程本地连接
    if hasattr(self._thread_local, "connection"):
        self._thread_local.connection = None
```

**为什么**: 
1. `close_all()` 调用后，其他线程的 `_thread_local.connection` 可能仍然引用已关闭的连接
2. 关闭连接后没有通知其他线程，可能导致使用已关闭连接

**影响**: 多线程环境下可能出现"使用已关闭连接"错误

**建议**: 
```python
def close_all(self) -> None:
    """关闭所有连接"""
    with self._connections_lock:
        # 标记所有连接为已关闭
        for conn in self._all_connections:
            try:
                conn.close()
            except Exception as e:
                logger.warning(f"Error closing connection: {e}")
        
        self._all_connections.clear()
    
    # 修复
    with self._connections_lock:
        # 1. 标记所有连接为关闭状态
        for conn in self._all_connections:
            try:
                conn.close()
            except Exception as e:
                logger.warning(f"Error closing connection: {e}")
        
        self._all_connections.clear()
    
    # 2. 清理线程本地连接引用
    with self._lock:
        self._thread_local.connection = None
    
    logger.info("All database connections closed")
```

### P0-4: GUI日志级别配置可能失败 ✅ 已修复

**位置**: `gui_main.py:65-68`

**问题**:
```python
log_level = os.getenv('LOG_LEVEL', 'INFO').upper()
logging.basicConfig(
    level=getattr(logging, log_level, logging.INFO),
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

**为什么**: 
1. 如果环境变量配置错误的日志级别（如 `LOG_LEVEL=DEBUGx`），会静默使用INFO
2. 没有任何警告日志提示配置错误

**影响**: 生产环境可能错误使用DEBUG级别，或期望DEBUG却实际使用INFO

**建议**: 
```python
log_level = os.getenv('LOG_LEVEL', 'INFO').upper()
if not hasattr(logging, log_level):
    logger.warning(f"Invalid LOG_LEVEL '{log_level}', using INFO")
    log_level = 'INFO'

logging.basicConfig(
    level=getattr(logging, log_level),
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
```

### P1-5: MasterAgent任务重入问题

**位置**: `agents/master_agent.py:173-182`

**问题**:
```python
# 先从队列移除
popped_task = self._task_queue.pop()
if popped_task is None:
    logger.warning(f"任务队列为空，跳过执行")
    return

# 再标记状态
if not self._dependency_state.mark_in_progress(task.task_id):
    logger.warning(f"任务 {task.task_id} 状态异常，重新入队")
    self._task_queue.push(task)  # 回滚
    return
```

**为什么**: 参数task与实际取出的popped_task可能不一致，逻辑混乱

**建议**: 
```python
# 从队列取出任务
popped_task = self._task_queue.pop()
if popped_task is None:
    logger.warning("任务队列为空，跳过执行")
    return

# 使用实际取出的任务
task = popped_task

# 标记状态
if not self._dependency_state.mark_in_progress(task.task_id):
    logger.warning(f"任务 {task.task_id} 状态异常，重新入队")
    self._task_queue.push(task)
    return
```
