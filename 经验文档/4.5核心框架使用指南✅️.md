## 一、快速开始

### 1.1 核心框架概述

Agent Pro 采用微内核架构，核心框架提供四大基础服务：

| 服务 | 说明 | 核心文件 |
|------|------|----------|
| EventBus | 插件间发布-订阅通信 | `core/event_bus.py` |
| PluginRegistry | 插件注册与状态管理 | `core/plugin_registry.py` |
| ServiceLocator | 依赖注入与服务定位 | `core/service_locator.py` |
| ConfigManager | 配置管理 | `core/config_manager.py` |

### 1.2 30秒快速上手

```python
from core import EventBus, PluginRegistry, ServiceLocator

# 1. 创建核心组件
event_bus = EventBus()  # 默认max_workers=10
registry = PluginRegistry()
locator = ServiceLocator()

# 2. 注册服务到ServiceLocator
locator.register(EventBus, event_bus)
locator.register(PluginRegistry, registry)

# 3. 订阅事件
def on_generation_started(event):
    print(f"生成开始: {event.data}")

event_bus.subscribe("generation.started", on_generation_started)

# 4. 发布事件
event_bus.publish("generation.started", {"chapter": "第一章"})
```

## 二、模块注册

### 2.1 注册新插件

插件注册通过 `PluginRegistry.register()` 方法完成：

```python
from core import PluginRegistry
from core.models import PluginMetadata

# 创建插件注册表
registry = PluginRegistry()

# 方式1：使用Pydantic PluginMetadata
metadata = PluginMetadata(
    # 必填字段
    id="my-plugin-v1",
    name="我的插件",
    version="1.0.0",
    description="示例插件",
    author="开发者",
    plugin_type="tool",
    dependencies=["outline-parser-v3"],  # 依赖其他插件
    permissions=["file.read", "file.write"],
    
    # 可选字段
    api_version="1.0",              # API版本
    priority=100,                    # 优先级（数值越小越高）
    enabled=True,                    # 是否启用
    conflicts=[],                    # 冲突插件列表
    min_platform_version="6.0.0",   # 最低平台版本
    entry_class="MyPlugin",          # 入口类名
)

# 注册插件（直接进入LOADED状态）
success = registry.register(
    plugin_id="my-plugin-v1",
    metadata=metadata,
    instance=my_plugin_instance  # 可选：插件实例
)
```

### 2.2 插件状态管理

插件使用5状态模型：

```
UNLOADED → LOADED → ACTIVE → UNLOADING → UNLOADED
              ↓         ↓
            ERROR ←←←←←←
```

| 状态 | 说明 | 可转换状态 |
|------|------|-----------|
| `UNLOADED` | 未加载（初始状态） | → LOADED |
| `LOADED` | 已加载，未激活 | → ACTIVE, ERROR, UNLOADING |
| `ACTIVE` | 已激活，可执行 | → LOADED, ERROR, UNLOADING |
| `ERROR` | 错误状态 | → LOADED, UNLOADING |
| `UNLOADING` | 卸载中 | → 从注册表移除 |

**状态转换示例**：

```python
# 激活插件
registry.activate("my-plugin-v1")  # LOADED → ACTIVE

# 停用插件
registry.deactivate("my-plugin-v1")  # ACTIVE → LOADED

# 错误恢复
registry.set_error("my-plugin-v1", "初始化失败")  # → ERROR
registry.load("my-plugin-v1", new_instance)  # ERROR → LOADED

# 卸载插件
registry.unregister("my-plugin-v1")  # → UNLOADING → 从注册表移除
```

### 2.3 V5核心模块保护

以下模块被保护，禁止卸载/覆盖：

```python
V5_PROTECTED_MODULES = frozenset([
    "outline-parser-v3",       # 大纲解析
    "style-learner-v5",        # 风格学习
    "character-manager-v1",    # 人物管理
    "worldview-parser-v1",     # 世界观解析
    "context-builder-v1",      # 上下文构建
    "iterative-generator-v2",  # 迭代生成
    "quality-validator-v1",    # 加权验证
    "novel-generator-v3",      # 生成入口
    "hot-ranking-v1",          # 热榜功能
])
```

尝试操作保护模块会抛出 `PluginProtectionError`：

```python
try:
    registry.unregister("outline-parser-v3")
except PluginProtectionError as e:
    print(f"保护模块无法卸载: {e}")
```

### 2.4 插槽绑定

插件可以绑定到命名插槽，实现插件替换：

```python
# 绑定插件到插槽
registry.bind_slot("novel-generator", "my-generator-v2")

# 获取插槽中的插件
generator = registry.get_slot_plugin("novel-generator")
```

## 三、事件总线使用

### 3.1 订阅事件

```python
from core import EventBus, EventPriority

event_bus = EventBus()  # 默认max_workers=10

# 基本订阅
handler_id = event_bus.subscribe(
    event_type="generation.completed",
    handler=lambda event: print(f"生成完成: {event.data}"),
)

# 带优先级的订阅
event_bus.subscribe(
    event_type="generation.started",
    handler=my_handler,
    priority=EventPriority.HIGH,  # 高优先级先执行
    handler_id="my-custom-id",    # 自定义ID便于管理
)

# 一次性订阅（触发后自动取消）
event_bus.subscribe_once(
    event_type="app.shutdown",
    handler=lambda e: print("应用关闭"),
)
```

### 3.2 发布事件

**异步发布（推荐）**：

```python
# publish() 使用线程池异步执行，立即返回
event_bus.publish(
    event_type="generation.started",
    data={
        "chapter_title": "第一章",
        "word_count_target": 2000,
    },
    source="UI"  # 可选：事件来源标识
)
```

**同步发布（需等待）**：

```python
# publish_sync() 等待所有处理器完成
results = event_bus.publish_sync(
    event_type="validation.request",
    data={"content": chapter_content},
    timeout=5.0  # 超时时间
)
```

### 3.3 事件优先级

```python
from core import EventPriority

# 优先级数值越小越先执行
class EventPriority(IntEnum):
    HIGHEST = 0    # 最高优先级
    HIGH = 10
    NORMAL = 20    # 默认优先级
    LOW = 30
    LOWEST = 40
```

### 3.4 线程安全规范

**重要**：Tkinter UI操作必须在主线程执行！

```python
# ❌ 错误：在事件回调中直接操作UI
def bad_handler(event):
    label.config(text=f"状态: {event.data['status']}")  # 可能崩溃！

event_bus.subscribe("status.update", bad_handler)

# ✅ 正确：使用after()调度到主线程
def safe_handler(event):
    root.after(0, lambda: label.config(
        text=f"状态: {event.data['status']}"
    ))

event_bus.subscribe("status.update", safe_handler)
```

### 3.5 死信队列

事件处理失败会进入死信队列：

```python
# 获取死信队列统计
stats = event_bus.get_dead_letter_stats()
# {
#     "current_size": 5,
#     "max_size": 100,
#     "total_added": 12,
#     "by_type": {"generation.started": 3, "validation.completed": 2}
# }

# 清空死信队列
event_bus.clear_dead_letter_queue()
```

## 四、热插拔机制

### 4.1 安全权限分级

热插拔使用L0-L3四级安全控制：

| 等级 | 类型 | 热重载 | 卸载 | 进程隔离 |
|------|------|--------|------|----------|
| L0 | 官方插件 | ✅ | ✅ | ❌ |
| L1 | 第三方签名 | ✅ | ❌ | ❌ |
| L2 | 未知来源 | ❌ | ❌ | ✅ |
| L3 | 风险插件 | ❌ | ❌ | ❌ |

### 4.2 启用热插拔监控

```python
from core import HotSwapManager

# 创建热插拔管理器
hot_swap = HotSwapManager(
    plugin_loader=loader,
    plugin_registry=registry,
    event_bus=event_bus,
)

# 启用文件监控（开发环境）
hot_swap.enable_monitoring(
    plugin_directories=["plugins/", "custom_plugins/"],
    debounce_delay=1.0,  # 防抖延迟（秒）
)
```

### 4.3 热插拔事件

```python
# 订阅热插拔事件
event_bus.subscribe("plugin.hot_swap", lambda e: print(
    f"插件 {e.data['plugin_id']} {e.data['action']} 完成"
))

# 热插拔事件类型
# - plugin.loaded: 插件加载
# - plugin.unloaded: 插件卸载
# - plugin.reloaded: 插件重载
# - plugin.error: 热插拔失败
```

### 4.4 热插拔最佳实践

**开发环境**：
- 启用热插拔监控
- 设置适当的防抖延迟（1-2秒）
- 使用日志跟踪重载事件

**生产环境**：
- 禁用热插拔监控
- 或仅监控特定目录
- 设置更长的防抖延迟

```python
# 生产环境配置
hot_swap.enable_monitoring(
    plugin_directories=["plugins/official/"],  # 仅监控官方目录
    debounce_delay=5.0,  # 更长的防抖
)
```

## 五、异步处理最佳实践

### 5.1 EventBus异步模式

```python
# ✅ 推荐：异步发布，不阻塞调用线程
event_bus.publish("long.running.task", {"data": large_data})

# ✅ 推荐：耗时操作在handler中异步执行
def async_handler(event):
    # 在线程池中执行耗时操作
    result = expensive_computation(event.data)
    # 结果通过事件传递
    event_bus.publish("computation.done", {"result": result})

event_bus.subscribe("start.computation", async_handler)

# ⚠️ 谨慎使用：同步发布会等待所有handler
results = event_bus.publish_sync("need.immediate.result", data)
```

### 5.2 线程池管理

EventBus 内置线程池，默认最大10个工作线程：

```python
# 自定义线程池大小
event_bus = EventBus()  # 默认 max_workers=10
event_bus = EventBus(max_workers=20)  # 高并发场景

# 关闭时优雅终止
event_bus.shutdown()  # 等待所有任务完成
```

### 5.3 熔断器机制

单个handler连续失败5次后自动熔断：

```python
# 熔断后handler不再接收事件
# 可手动重置熔断器
event_bus.reset_circuit(handler_id="my_handler")
```

### 5.4 异步任务模式

**Fire-and-Forget模式**：
```python
# 发出事件后不关心结果
event_bus.publish("log.write", {"message": "操作完成"})
```

**Request-Response模式**：
```python
# 通过事件ID关联请求和响应
request_id = str(uuid.uuid4())

def on_response(event):
    if event.data.get("request_id") == request_id:
        print(f"收到响应: {event.data}")

event_bus.subscribe_once("analysis.response", on_response)
event_bus.publish("analysis.request", {"request_id": request_id})
```

**Pipeline模式**：
```python
# 事件链式处理
def stage1_handler(event):
    result = process_stage1(event.data)
    event_bus.publish("pipeline.stage2", result)

def stage2_handler(event):
    result = process_stage2(event.data)
    event_bus.publish("pipeline.stage3", result)

def stage3_handler(event):
    save_final_result(event.data)

event_bus.subscribe("pipeline.stage1", stage1_handler)
event_bus.subscribe("pipeline.stage2", stage2_handler)
event_bus.subscribe("pipeline.stage3", stage3_handler)

# 启动流水线
event_bus.publish("pipeline.stage1", initial_data)
```

## 六、常见问题

### 6.1 事件订阅后没收到事件？

**检查清单**：
1. 确认事件类型字符串完全匹配（区分大小写）
2. 确认订阅在发布之前完成
3. 检查handler是否被熔断
4. 查看死信队列是否有失败记录

```python
# 调试：列出所有订阅
for event_type in event_bus.list_event_types():
    count = event_bus.get_handler_count(event_type)
    print(f"{event_type}: {count} handlers")
```

### 6.2 插件无法注册？

**可能原因**：
1. 插件ID已存在
2. 依赖的插件未加载
3. 元数据格式不正确
4. 与已加载插件冲突

```python
# 检查插件状态
if registry.get_plugin("my-plugin-v1") is not None:
    print("插件已存在")

# 检查依赖
metadata = PluginMetadata(
    id="my-plugin-v1",
    dependencies=["non-existent-plugin"],  # 会失败
    ...
)
```

### 6.3 UI操作导致崩溃？

**原因**：Tkinter不是线程安全的，UI操作必须在主线程执行。

**解决方案**：

```python
# 方案1：使用after()调度
def safe_ui_update(data):
    root.after(0, lambda: update_ui(data))

# 方案2：使用publish_sync在主线程调用
# 在主线程中
results = event_bus.publish_sync("ui.update", data)
```

### 6.4 如何处理超时？

```python
# 设置handler超时
event_bus.subscribe(
    event_type="slow.operation",
    handler=my_handler,
    timeout=60.0,  # 60秒超时
)

# 同步发布设置超时
results = event_bus.publish_sync(
    event_type="need.fast.response",
    data=data,
    timeout=5.0  # 5秒总超时
)
```

### 6.5 如何优雅关闭？

```python
# 关闭EventBus（等待所有任务完成）
event_bus.shutdown()

# 关闭连接池
database_pool.close(timeout=10.0)

# 释放所有服务
service_locator.dispose_all()
```

## 附录：核心API速查表

### EventBus

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `subscribe(event_type, handler, ...)` | 订阅事件 | handler_id: str |
| `subscribe_once(event_type, handler)` | 一次性订阅 | handler_id: str |
| `unsubscribe(handler_id)` | 取消订阅 | bool |
| `publish(event_type, data, source)` | 异步发布 | None |
| `publish_sync(event_type, data, timeout)` | 同步发布 | List[Any] |
| `reset_circuit(handler_id)` | 重置熔断器 | None |
| `get_handler_count(event_type)` | 获取处理器数 | int |
| `list_event_types()` | 列出所有事件类型 | List[str] |
| `get_dead_letter_stats()` | 获取死信队列统计 | Dict |
| `clear_dead_letter_queue()` | 清空死信队列 | None |
| `shutdown()` | 关闭事件总线 | None |

### PluginRegistry

| 方法 | 说明 | 返回值 |
|------|------|--------|

| `activate(plugin_id)` | 激活插件 | bool |
| `deactivate(plugin_id)` | 停用插件 | bool |
| `load(plugin_id, instance)` | 加载插件 | bool |

| `get_active_plugins()` | 获取所有活跃插件 | List[Any] |
| `bind_slot(slot_id, plugin_id)` | 绑定插槽 | bool |
| `get_slot_plugin(slot_id)` | 获取插槽插件 | Optional[Any] |
