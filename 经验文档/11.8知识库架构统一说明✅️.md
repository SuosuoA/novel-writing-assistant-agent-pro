## 一、背景问题

### 1.1 发现的问题

用户反馈"知识库从1000+条减少"，实际分析发现：

1. **两个知识库目录并存**：
   - `data/knowledge/` - JSON源文件目录（10个文件）
   - `data/knowledge_base/` - LanceDB向量数据库（659条知识点）

2. **架构不清晰**：
   - 代码中混用两个路径
   - 用户期望看到向量库数量，但看到的是JSON文件数量

3. **引用问题**：
   - 部分代码只引用JSON源文件
   - 部分代码只引用向量库
   - 导致知识点不可达

### 1.2 根本原因

- **ADR-003（知识库双层设计）**未完整实施
- KnowledgeManager未自动同步JSON→向量库
- 文档说明不够清晰

---

## 二、解决方案

### 2.1 架构统一

**双层知识库架构**：

```
┌─────────────────────────────────────────────────────────┐
│                  知识库统一架构                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  用户输入/编辑                                          │
│       ↓                                                 │
│  JSON源文件（data/knowledge/）                         │
│   - 手动编辑                                            │
│   - 批量导入                                            │
│   - 备份存储                                            │
│       ↓                                                 │
│  KnowledgeManager.create_knowledge()                   │
│       ├─ 保存到JSON文件                                │
│       └─ 自动同步到向量库                              │
│       ↓                                                 │
│  LanceDB向量库（data/knowledge_base/）                │
│   - 运行时检索                                          │
│   - 向量召回                                            │
│   - 智能推荐                                            │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**核心原则**：
1. **JSON源文件** = 编辑层（用于手动编辑和备份）
2. **LanceDB向量库** = 运行层（用于实际检索）
3. **自动同步** = KnowledgeManager负责双向同步

2. **添加VectorStore连接方法**：
```python
def _get_vector_store(self):
    """延迟获取VectorStore连接（用于向量检索）"""
    if self._vector_store is None:
        try:
            from infrastructure.vector_store import VectorStore
            self._vector_store = VectorStore(workspace_root=self.workspace_root)
        except Exception as e:
            print(f"[KnowledgeManager] 初始化VectorStore失败: {e}")
    return self._vector_store
```

3. **create_knowledge自动同步**：
```python
def create_knowledge(self, ...):
```
