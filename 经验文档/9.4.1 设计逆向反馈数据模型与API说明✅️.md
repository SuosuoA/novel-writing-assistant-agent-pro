## 一、概述

### 1.1 功能定义

逆向反馈功能用于分析已生成的小说章节内容与项目设定（大纲、人物、世界观）的一致性，检测冲突并生成修正建议。核心价值在于：

1. **设定一致性保障**：确保后续生成内容与已有章节保持一致
2. **冲突自动检测**：识别角色行为、情节发展、世界观设定的矛盾
3. **智能修正建议**：为每个冲突提供可执行的修正方案

### 1.2 核心组件

| 组件 | 文件位置 | 职责 |
|------|----------|------|
| ConsistencyIssue | core/plugin_interface.py | 冲突项数据结构 |
| ConsistencyReport | core/plugin_interface.py | 一致性分析报告 |
| ReverseFeedbackPlugin | core/plugin_interface.py | 逆向反馈插件接口 |
| project.schema.json | schemas/project.schema.json | 项目文件格式定义 |

## 二、数据模型设计

### 2.1 ConsistencyIssue（冲突项）

**定义位置**: `core/plugin_interface.py`

**枚举类型**:

```python
class ConsistencyIssueType(Enum):
    """冲突类型枚举"""
    OUTLINE = "outline"        # 大纲冲突
    CHARACTER = "character"    # 人物设定冲突
    WORLDVIEW = "worldview"    # 世界观冲突


class ConsistencySeverity(Enum):
    """冲突严重程度枚举"""
    LOW = "low"       # 轻微冲突，不影响整体逻辑
    MEDIUM = "medium" # 中等冲突，需要注意
    HIGH = "high"     # 严重冲突，必须修正
```

**数据结构**:

```python
@dataclass
class ConsistencyIssue:
    """章节内容与设定的冲突项"""
    
    issue_id: str               # 冲突项唯一标识（自动生成）
    issue_type: ConsistencyIssueType   # 冲突类型
    severity: ConsistencySeverity      # 严重程度
    description: str            # 冲突描述
    suggested_fix: str          # 建议修正方案
    original_content: str       # 原始设定内容
    chapter_reference: str      # 引发冲突的章节ID或标题
    detected_at: float          # 检测时间戳
    element_name: str           # 冲突涉及的元素名称（如角色名）
    confidence: float           # 检测置信度（0.0-1.0）
```

**JSON Schema 定义**（嵌入 project.schema.json）:

```json
{
  "issue_id": "issue-a1b2c3d4",
  "issue_type": "character",
  "severity": "high",
  "description": "角色'林逸'在第3章中使用了'御剑术'，但人物设定中其能力仅为'基础剑法'",
  "suggested_fix": "将人物设定中的能力修改为'御剑术'，或修改第3章内容",
  "original_content": "能力：基础剑法",
  "chapter_reference": "chap_003",
  "detected_at": "2026-03-24T10:30:00Z",
  "element_name": "林逸",
  "confidence": 0.92
}
```

**序列化方法**:

| 方法 | 说明 |
|------|------|
| `to_dict()` | 转换为字典，用于JSON序列化 |
| `from_dict(data)` | 从字典创建实例 |

### 2.2 ConsistencyReport（一致性分析报告）

**定义位置**: `core/plugin_interface.py`

**数据结构**:

```python
@dataclass
class ConsistencyReport:
    """一致性分析报告"""
    
    report_id: str              # 报告唯一标识（自动生成）
    project_name: str           # 项目名称
    chapters_analyzed: int      # 分析的章节数量
    issues: List[ConsistencyIssue]  # 冲突项列表
    summary: str                # 分析摘要
    analyzed_at: float          # 分析时间戳
    high_priority_count: int    # 高优先级冲突数量
    medium_priority_count: int  # 中等优先级冲突数量
    low_priority_count: int     # 低优先级冲突数量
```

**核心方法**:

| 方法 | 说明 |
|------|------|
| `add_issue(issue)` | 添加冲突项并自动更新统计 |
| `to_dict()` | 转换为字典 |
| `from_dict(data)` | 从字典创建实例 |

**JSON 示例**:

```json
{
  "report_id": "report-e5f6g7h8",
  "project_name": "修仙传奇",
  "chapters_analyzed": 5,
  "issues": [...],
  "summary": "发现12个冲突项，其中2个高优先级需要立即修正",
  "analyzed_at": "2026-03-24T10:35:00Z",
  "high_priority_count": 2,
  "medium_priority_count": 5,
  "low_priority_count": 5
}
```

## 三、插件接口设计

### 3.1 ReverseFeedbackPlugin 接口

**继承关系**: `ReverseFeedbackPlugin(AnalyzerPlugin)`

**定义位置**: `core/plugin_interface.py`

**核心方法**:

```python
class ReverseFeedbackPlugin(AnalyzerPlugin):
    """逆向反馈分析插件接口"""
    
    @abstractmethod
    def analyze_chapter_vs_settings(
        self,
        chapter_text: str,
        current_settings: Dict[str, Any],
        chapter_id: str = "",
    ) -> ConsistencyReport:
        """
        分析单章节与设定的冲突
        
        Args:
            chapter_text: 章节文本内容
            current_settings: 当前项目设定
                - outline: 大纲文本
                - characters: 人物设定列表
                - worldview: 世界观设定文本
            chapter_id: 章节ID或标题
        
        Returns:
            ConsistencyReport: 包含冲突列表的报告
        """
        pass
    
    @abstractmethod
    def analyze_project(
        self,
        project_data: Dict[str, Any],
        options: Optional[Dict[str, Any]] = None,
    ) -> ConsistencyReport:
        """
        分析整个项目的一致性
        
        Args:
            project_data: 完整项目数据
                - project_name: 项目名称
                - chapters: 章节列表
                - outline: 大纲文本
                - characters: 人物设定列表
                - worldview: 世界观设定文本
            options: 分析选项
        
        Returns:
            ConsistencyReport: 综合分析报告
        """
        pass
    
    @abstractmethod
    def generate_corrections(
        self,
        report: ConsistencyReport,
        current_settings: Dict[str, Any],
        options: Optional[Dict[str, Any]] = None,
    ) -> Dict[str, Any]:
        """
        根据冲突报告生成修正后的设定
        
        Args:
            report: 一致性分析报告
            current_settings: 当前项目设定
            options: 修正选项
                - auto_fix_low: 是否自动修正低优先级冲突
                - preserve_original: 是否保留原始设定备份
        
        Returns:
            {
                "updated_outline": str,
                "updated_characters": List[Dict],
                "updated_worldview": str,
                "suggestions": List[str],
                "backup": Optional[Dict]
            }
        """
        pass
```

**继承的方法**（来自 AnalyzerPlugin）:

| 方法 | 实现说明 |
|------|----------|
| `get_supported_formats()` | 返回 `["txt", "md", "json"]` |
| `get_analysis_types()` | 返回 `["consistency_check", "conflict_detection", ...]` |
| `analyze(content, options)` | 由子类实现，调用 `analyze_chapter_vs_settings` |

## 五、使用示例

### 5.1 单章节一致性分析

```python
from core.plugin_interface import ReverseFeedbackPlugin, ConsistencyReport

# 获取插件实例
plugin = plugin_registry.get_plugin("reverse-feedback-analyzer")

# 准备数据
chapter_text = "林逸拔出长剑，施展御剑术..."
current_settings = {
    "outline": "第一章：主角觉醒...",
    "characters": [
        {"name": "林逸", "ability": "基础剑法"}
    ],
    "worldview": "修仙世界..."
}

# 执行分析
report = plugin.analyze_chapter_vs_settings(
    chapter_text=chapter_text,
    current_settings=current_settings,
    chapter_id="chap_003"
)

# 检查结果
print(f"发现 {len(report.issues)} 个冲突")
for issue in report.issues:
    if issue.severity == "high":
        print(f"高优先级: {issue.description}")
```

### 5.2 项目整体分析

```python
# 加载项目数据
project_data = load_project("修仙传奇")

# 执行全项目分析
report = plugin.analyze_project(
    project_data=project_data,
    options={"include_low_severity": True}
)

# 保存报告
project_data["consistency_reports"].append(report.to_dict())
save_project(project_data)
```

### 5.3 生成修正建议

```python
# 根据报告生成修正
corrections = plugin.generate_corrections(
    report=report,
    current_settings=current_settings,
    options={
        "auto_fix_low": True,
        "preserve_original": True
    }
)

# 应用修正
if corrections["backup"]:
    save_backup(corrections["backup"])

project_data["outline"] = corrections["updated_outline"]
project_data["characters"] = corrections["updated_characters"]
project_data["worldview"] = corrections["updated_worldview"]
```

## 六、实现要点

### 6.1 核心流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     逆向反馈分析流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 输入准备                                                     │
│     ├── 章节文本（从 project.json 或文件读取）                    │
│     └── 项目设定（outline/characters/worldview）                 │
│                                                                  │
│  2. 信息提取（调用 AnalyzerPlugin）                              │
│     ├── 从章节提取角色行为、情节发展、世界观元素                  │
│     └── 从设定提取规则、约束、属性                                │
│                                                                  │
│  3. 一致性对比（调用 LLM）                                       │
│     ├── 角色行为 vs 人物设定                                     │
│     ├── 情节发展 vs 大纲                                         │
│     └── 世界观元素 vs 世界观设定                                 │
│                                                                  │
│  4. 冲突检测                                                     │
│     ├── 识别冲突类型                                             │
│     ├── 评估严重程度                                             │
│     └── 计算置信度                                               │
│                                                                  │
│  5. 修正建议生成                                                 │
│     ├── 为每个冲突生成建议                                       │
│     └── 综合生成修正后的设定                                     │
│                                                                  │
│  6. 输出                                                         │
│     ├── ConsistencyReport（报告对象）                            │
│     └── 更新 project.json（可选）                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 依赖关系

| 依赖 | 用途 |
|------|------|
| AnalyzerPlugin | 基类，提供分析方法框架 |
| LLM API | 语义理解和冲突检测 |
| EventBus | 发布分析进度事件 |
| 项目管理模块 | 读取/写入 project.json |

### 6.3 异常处理

| 异常场景 | 处理策略 |
|----------|----------|
| LLM调用超时 | 返回部分结果，标记未完成章节 |
| 设定文件缺失 | 跳过对应类型的检测 |
| 章节内容为空 | 记录警告，继续处理其他章节 |
