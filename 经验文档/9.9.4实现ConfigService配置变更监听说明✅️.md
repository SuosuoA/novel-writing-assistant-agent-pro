## 1. 设计目标

根据《9.9大模型集成.txt》设计文档，扩展ConfigService支持监听器模式，实现配置变更时自动发布`config.changed`事件到EventBus，供AIServiceManager订阅并动态切换AI服务提供商。

### 核心需求

1. **监听器模式**：支持多个监听器订阅配置变更
2. **EventBus集成**：配置变更时发布`config.changed`事件
3. **AI配置便捷方法**：提供`get_ai_config()`和`update_ai_config()`方法
4. **线程安全**：监听器注册/移除/通知需要线程安全

---

## 2. 架构设计

### 2.1 配置变更通知链

```
用户修改配置
    ↓
ConfigManager.set()
    ↓
ConfigService._on_config_changed_internal()
    ↓
    ├→ 清除配置缓存
    ├→ 通知所有监听器
    └→ 发布EventBus事件
        ↓
    AIServiceManager订阅
        ↓
    动态切换AIProvider
```

## 3. 实现细节

### 3.1 监听器模式实现

```python
# core/config_service.py

class ConfigService:
    def __init__(self, config_path: Optional[Path] = None):
        # ...原有代码...
        
        # V1.2新增：监听器列表
        self._listeners: List[Callable[[str, Any, Any], None]] = []
        self._listeners_lock = threading.RLock()
        
        # V1.2新增：EventBus实例（延迟初始化）
        self._event_bus: Optional[EventBus] = None
        
        # V1.2新增：向ConfigManager注册内部观察者
        self._config_manager.add_observer(self._on_config_changed_internal)
    
    def add_listener(self, listener: Callable[[str, Any, Any], None]) -> None:
        """添加配置变更监听器"""
        with self._listeners_lock:
            if listener not in self._listeners:
                self._listeners.append(listener)
    
    def remove_listener(self, listener: Callable[[str, Any, Any], None]) -> bool:
        """移除配置变更监听器"""
        with self._listeners_lock:
            try:
                self._listeners.remove(listener)
                return True
            except ValueError:
                return False
    
    def clear_listeners(self) -> None:
        """清除所有监听器"""
        with self._listeners_lock:
            self._listeners.clear()
```

### 3.2 EventBus集成实现

```python
def _on_config_changed_internal(self, key_path: str, old_value: Any, new_value: Any) -> None:
    """
    内部观察者：ConfigManager变更时触发
    """
    # 1. 清除配置缓存
    with self._lock:
        self._config_cache = None
    
    # 2. 通知所有监听器
    self._notify_listeners(key_path, old_value, new_value)
    
    # 3. 发布EventBus事件
    self._publish_config_changed_event(key_path, old_value, new_value)

def _publish_config_changed_event(self, key_path: str, old_value: Any, new_value: Any) -> None:
    """发布配置变更事件到EventBus"""
    # 延迟初始化EventBus（避免循环依赖）
    if self._event_bus is None:
        self._event_bus = get_event_bus()
    
    event_data = {
        "key_path": key_path,
        "old_value": old_value,
        "new_value": new_value,
        "timestamp": datetime.now().isoformat(),
    }
    
    event = Event(
        type="config.changed",
        data=event_data,
        source="ConfigService",
    )
    
    try:
        self._event_bus.publish(event)
    except Exception as e:
        logging.getLogger(__name__).error(
            f"Failed to publish config.changed event: {e}"
        )
```

### 3.3 AI配置便捷方法

```python
def get_ai_config(self) -> Dict[str, Any]:
    """
    获取AI相关配置（供AIServiceManager使用）
    
    Returns:
        AI配置字典，包含：
        - service_mode: "local" 或 "remote" 或 "online"
        - provider: 服务提供商（首字母大写，如 "DeepSeek"、"OpenAI"、"Anthropic"）
        - model: 模型名称
        - api_key: API密钥
        - base_url: API基础URL（可选）
        - temperature: 温度参数
        - max_tokens: 最大token数（可选）
        - local: 本地配置（可选）
    """
    config = self.get_config()
    
    ai_config = {
        "service_mode": config.service_mode,
        "provider": config.provider,
        "model": config.model,
        "api_key": config.api_key,
        "temperature": config.temperature,
        "local_url": config.local_url,
    }
    
    # 从config.yaml获取可选配置
    all_config = self.get_all()
    
    if "base_url" in all_config:
        ai_config["base_url"] = all_config["base_url"]
    
    if "max_tokens" in all_config:
        ai_config["max_tokens"] = all_config["max_tokens"]
    
    if "local" in all_config:
        ai_config["local"] = all_config["local"]
    
    return ai_config

def update_ai_config(self, new_config: Dict[str, Any]) -> None:
    """
    更新AI配置并发布变更事件
    
    Args:
        new_config: 新的AI配置字典
    """
    # 映射到顶层配置键
    key_mapping = {
        "service_mode": "service_mode",
        "provider": "provider",
        "model": "model",
        "api_key": "api_key",
        "base_url": "base_url",
        "temperature": "temperature",
        "max_tokens": "max_tokens",
        "local_url": "local_url",
    }
    
    with self._lock:
        for config_key, value in new_config.items():
            if config_key in key_mapping:
                self.set(key_mapping[config_key], value, source="ai_config_update")
            elif config_key == "local":
                # local配置块单独处理
                if isinstance(value, dict):
                    for sub_key, sub_value in value.items():
                        self.set(f"local.{sub_key}", sub_value, source="ai_config_update")
```

## 5. 技术要点

### 5.1 线程安全

| 操作 | 保护机制 |
|------|----------|
| 监听器注册/移除 | `_listeners_lock` (RLock) |
| 监听器通知 | 快照拷贝 + try-except隔离 |
| EventBus发布 | EventBus内部线程池 |
| 配置缓存清除 | `_lock` (RLock) |

### 5.2 避免循环依赖

**问题**：ConfigService初始化时EventBus可能未创建

**解决方案**：延迟初始化EventBus

```python
def _publish_config_changed_event(self, ...):
    # 延迟初始化EventBus
    if self._event_bus is None:
        self._event_bus = get_event_bus()
    # ...发布事件...
```

### 5.3 监听器异常隔离

**问题**：单个监听器异常不应阻塞其他监听器

**解决方案**：try-except捕获 + 日志记录

```python
def _notify_listeners(self, key_path, old_value, new_value):
    for listener in listeners_snapshot:
```
