## 三、问题列表

### 🔴 P0 阻塞问题

#### P0-1: agents/__init__.py 未导出 plugins 模块

**位置**: `agents/__init__.py`

**问题**:
文档中说明了通过 `from agents.plugins import ...` 导入Agent，但 `agents/__init__.py` 的 `__all__` 列表中没有导出 plugins 子模块。

**影响**: 用户无法通过 `from agents import OutlineAnalysisAgent` 导入，必须使用完整路径 `from agents.plugins import ...`。

**建议修复**:
```python
# 在 agents/__init__.py 末尾添加
from . import plugins

# 或在 __all__ 中添加
__all__ = [
    # ... 现有导出 ...
    "plugins",  # 添加插件模块
]
```

---

### 🟡 P1 建议问题

#### P1-1: AgentPool 缺少 _register_builtin_agents 方法

**位置**: `agents/agent_pool.py`

**问题**:
文档第4.1节说明了在 `AgentPool._register_builtin_agents()` 中注册4个Agent，但实际代码中没有这个方法。AgentPool 只提供了通用的 `register_agent()` 方法。

**影响**: 需要手动注册Agent，无法自动初始化内置Agent。

**建议修复**:
```python
# 在 AgentPool 类中添加
def _register_builtin_agents(self):
    """注册内置Agent"""
    from agents.plugins import (
        OutlineAnalysisAgent,
        StyleLearningAgent,
        NovelGenerationAgent,
        QualityValidationAgent,
    )
    
    adapters = [
        OutlineAnalysisAgent(),
        StyleLearningAgent(),
        NovelGenerationAgent(),
        QualityValidationAgent(),
    ]
    
    for agent in adapters:
        self.register_agent(agent)
```

#### P1-2: 缺少 Agent 插件的单元测试

**位置**: `tests/`

**问题**:
现有测试文件 `test_agent_core.py` 只测试了 `agents/core/` 下的核心模块，没有针对 `agents/plugins/` 中4个Agent的专门测试。

**影响**: 无法验证Agent与插件的集成是否正常工作。

**建议**: 创建 `tests/test_agent_plugins.py` 测试文件，测试：
- Agent初始化
- 插件加载
- execute执行
- 错误处理
- 清理资源

#### P1-3: 任务类型映射未实现

**位置**: 文档第4.2节

**问题**:
文档定义了任务类型映射（如 `outline_analysis` → `outline_analysis`），但代码中没有实现任务类型到Agent的自动路由。

**当前状态**: Agent 通过 `can_handle()` 方法自行判断能否处理任务。

**建议**: 考虑在 AgentPool 或 MasterAgent 中添加任务类型映射表，简化路由逻辑。

### 💭 P2 挑剔问题

#### P2-1: 插件导入路径使用 try-except 静默处理

**位置**: 各 Agent 的 `initialize()` 方法

**问题**:
当插件导入失败时，只记录警告日志，然后尝试创建本地实例。如果两者都失败，才设置 ERROR 状态。

**建议**: 提供更详细的错误信息，包括可能的解决方案。

#### P2-2: NovelGenerationAgent 的 dependencies 字段未实际使用

**位置**: `agents/plugins/novel_generation_agent.py:77`

**问题**:
定义了 `dependencies=["context-builder-v1", "iterative-generator-v2", "quality-validator-v1"]`，但在执行时没有检查这些依赖是否可用。

**建议**: 在 `initialize()` 中验证依赖插件是否已加载。
