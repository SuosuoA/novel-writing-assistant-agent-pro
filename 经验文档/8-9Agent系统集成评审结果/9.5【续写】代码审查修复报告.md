## 二、P0阻塞问题修复

### 🔴 P0-1: LLM API调用无超时保护机制

**问题描述**: `_call_llm_api()`方法虽然有`timeout=120`参数，但没有使用`concurrent.futures`实现真正的超时控制。

**修复方案**:
1. 引入`concurrent.futures.ThreadPoolExecutor`
2. 实现强制超时控制机制
3. 超时后取消Future任务

**修复代码**:
```python
import concurrent.futures

def _call_llm_api(self, prompt: str, max_tokens: int, temperature: float) -> str:
    # 使用concurrent.futures实现强制超时
    with concurrent.futures.ThreadPoolExecutor(max_workers=1) as executor:
        future = executor.submit(
            self._do_api_call,
            system_prompt,
            prompt,
            max_tokens,
            temperature
        )
        try:
            content = future.result(timeout=self._timeout)
            return content
        except concurrent.futures.TimeoutError:
            future.cancel()
            raise LLMTimeoutError(f"LLM调用超时（{self._timeout}秒）")
```

**验证结果**: ✅ 测试通过

**修复方案**:

3. 不同异常类型采用不同处理策略

```python

class LLMError(Exception):
    """LLM调用基础异常"""
    pass

class LLMTimeoutError(LLMError):
    """LLM调用超时异常"""
    pass

class LLMAuthenticationError(LLMError):
    """LLM认证异常（API Key无效或过期）"""
    pass

class LLMConnectionError(LLMError):
    """LLM连接异常（网络问题）"""
    pass

class LLMRateLimitError(LLMError):
    """LLM速率限制异常"""
    pass
```

**异常处理逻辑**:
- `LLMTimeoutError`: 直接抛出，不重试
- `LLMAuthenticationError`: 直接抛出，不重试
- `LLMRateLimitError`: 指数退避后重试
- `LLMConnectionError`: 线性等待后重试

### 🟡 P1-4: 章节概要生成算法简单

**问题描述**: 仅取前500字符截断，不是真正的"概要"。

**修复方案**: 智能概要生成算法

**算法步骤**:
1. 提取关键句子（包含动作动词、转折词、情感词汇）
2. 评分筛选高分句子
3. 组合为概要文本

**修复代码**:
```python
def _extract_key_sentences(self, content: str) -> List[str]:
    """提取关键句子"""
    sentences = [s.strip() for s in content.split('。') if s.strip()]
    key_sentences = []
    
    # 关键词列表
    action_verbs = ['说', '走', '跑', '看', '想', ...]
    transition_words = ['但是', '然而', '可是', ...]
    emotion_words = ['愤怒', '悲伤', '高兴', ...]
    
    for sentence in sentences:
        score = 0
        if any(verb in sentence for verb in action_verbs):
            score += 2
        if any(word in sentence for word in transition_words):
            score += 3
        if any(word in sentence for word in emotion_words):
            score += 2
        if '"' in sentence or '"' in sentence:  # 对话
            score += 2
        if 20 <= len(sentence) <= 100:  # 长度适中
            score += 1
        
        if score >= 3:
            key_sentences.append(sentence)
    
    return key_sentences
```
