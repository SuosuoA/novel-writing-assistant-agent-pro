## 一、需求总结

### 1.1 核心需求

在知识库中添加新题材"写作技巧"，固定领域为六大分类。区分两种知识库调用模式：

1. **传统知识库**：灵活引用（隐式/显式/约束）
2. **写作技巧库**：强制遵循（AI必须100%遵守）

### 1.2 用户期望

- 创作界面有两个多选框：知识库关联、写作技巧
- 传统知识库内容灵活使用
- 写作技巧约定内容，AI生成百分百要遵循
- 修改底层逻辑，构建开始创作按钮的真实关联

### 2.2 叙事技巧（9项）

| 技巧名称 | 说明 |
|----------|------|
| 第一人称叙事 | 使用"我"作为叙述主体，增强代入感 |
| 第三人称叙事 | 使用"他/她"叙述，视角灵活 |
| 多视角叙事 | 多个角色的视角交替叙述 |
| 倒叙 | 先展示结果，再回溯过程 |
| 插叙 | 在叙述中插入回忆或背景 |
| 平行叙事 | 多条线索同时推进 |
| 螺旋叙事 | 同一场景或情感反复回归、层层深入 |
| 多线叙事 | 多条情节线并行推进，最后交汇 |
| 意识流 | 内心意识连续流动的叙事，打破时空限制 |

### 2.6 特殊句式（13项）

| 句式名称 | 说明 | 典型例子 |
|----------|------|----------|
| 列锦句式 | 纯名词组合，意象叠加 | "鸡声茅店月，人迹板桥霜" |
| 倒装句式 | 语序倒装，突出强调 | "香稻啄余鹦鹉粒" |
| 紧缩句式 | 复句紧缩为单句形式 | "天高地厚" |
| 排比句式 | 三个以上结构相似的句子 | "山朗润起来了，水涨起来了，太阳的脸红起来了" |
| 对偶句式 | 字数相等、结构对称 | "两个黄鹂鸣翠柳，一行白鹭上青天" |
| 反复句式 | 相同词语或句子反复出现 | "盼望着，盼望着，东风来了" |
| 设问句式 | 自问自答 | "什么是幸福？幸福就是..." |
| 反问句式 | 只问不答，答案在问句中 | "难道这不值得我们深思吗？" |
| 感叹句式 | 表达强烈感情 | "多么美丽的景色啊！" |
| 祈使句式 | 表示请求或命令 | "请让一让。" |
| 省略句式 | 省略某些成分 | "他走了，她哭了。" |
| 独词句式 | 单个词成句 | "火！" |
| 意象组合 | 多个意象并置营造意境 | "枯藤老树昏鸦，小桥流水人家" |

**跨分类持久化多选**：

```python
# 存储所有写作技巧的选择状态（跨分类持久化）
self._all_writing_tech_vars = {}  # {技巧名: BooleanVar}

# 预先创建所有技巧的BooleanVar
for category, techniques in self._writing_techniques.items():
    for tech in techniques:
        self._all_writing_tech_vars[tech] = tk.BooleanVar(value=False)
```

**获取选中技巧**：

```python
def get_selected_writing_techniques(self) -> List[str]:
    """获取选中的写作技巧列表（跨分类）"""
    selected = []
    for tech, var in self._all_writing_tech_vars.items():
        if var.get():
            selected.append(tech)
    return selected
```
