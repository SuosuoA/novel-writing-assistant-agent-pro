## 一、设计概述

### 1.1 设计目标

续写插件接口（ContinuationPlugin）用于智能小说续写功能，核心目标：

1. **灵活的续写方向**：支持自然续写、特定方向、情感导向等多种模式
2. **丰富的上下文支持**：整合大纲、人设、世界观、风格、前文等上下文
3. **标准化接口**：继承GeneratorPlugin，与现有插件系统无缝集成
4. **元数据追踪**：返回模型、耗时、token等诊断信息

### 1.2 架构位置

```
GeneratorPlugin (生成器插件基类)
    ├── NovelGeneratorPlugin (小说生成)
    ├── ContextBuilderPlugin (上下文构建)
    └── ContinuationPlugin (续写插件) ← 新增
```

### 1.3 设计原则

| 原则 | 说明 |
|------|------|
| **继承复用** | 继承GeneratorPlugin，复用现有接口 |
| **最小参数** | 必需参数仅起始文本+字数，其他可选 |
| **上下文丰富** | 支持大纲/人设/世界观/风格/前文 |
| **方向灵活** | 5种续写方向满足不同场景 |
| **元数据透明** | 返回完整诊断信息 |

## 二、接口定义

### 2.1 ContinuationPlugin类

```python
class ContinuationPlugin(GeneratorPlugin):
    """续写插件接口
    
    继承自 GeneratorPlugin，扩展了专门的续写能力。
    """
    
    @abstractmethod
    def generate_continuation(
        self, 
        request: ContinuationRequest
    ) -> ContinuationResult:
        """执行续写 - 核心方法"""
        pass
    
    def get_continuation_options(self) -> Dict[str, Any]:
        """获取续写选项定义"""
        pass
    
    def estimate_continuation(
        self, 
        request: ContinuationRequest
    ) -> Dict[str, Any]:
        """预估续写效果（可选实现）"""
        pass
```

### 2.2 核心方法

| 方法 | 类型 | 说明 |
|------|------|------|
| `generate_continuation()` | 抽象方法 | 续写核心实现，必须实现 |
| `get_continuation_options()` | 默认实现 | 返回支持的续写选项定义 |
| `estimate_continuation()` | 默认实现 | 预估续写效果和资源消耗 |

### 2.3 继承方法适配

ContinuationPlugin继承GeneratorPlugin，通过适配实现父类方法：

```python
def generate(self, request: GenerationRequest) -> GenerationResult:
    """适配GenerationRequest到ContinuationRequest"""
    continuation_request = ContinuationRequest(
        starting_text=request.title,
        word_count=request.word_count,
        direction="natural",
        outline=request.outline,
        characters=request.character_profiles.get("characters"),
        worldview=request.worldview_config.get("worldview"),
        style_profile=request.style_profile,
        request_id=request.request_id
    )
    result = self.generate_continuation(continuation_request)
    return GenerationResult(
        request_id=request.request_id,
        content=result.text,
        word_count=result.word_count,
        iteration_count=result.metadata.iterations,
        error=result.error
    )
```

## 三、参数结构

### 3.1 ContinuationRequest模型

```python
class ContinuationRequest(BaseModel):
    """续写请求参数"""
    
    # === 必需参数 ===
    starting_text: str          # 起始文本（续写起点）
    word_count: int = 500       # 目标字数（100-5000）
    
    # === 续写方向 ===
    direction: str = "natural"  # 续写方向
    direction_hint: Optional[str] = None  # 方向提示
    
    # === 上下文配置 ===
    outline: Optional[str] = None           # 章节大纲
    characters: Optional[List[Dict]] = None # 人物设定
    worldview: Optional[str] = None         # 世界观设定
    style_profile: Optional[Dict] = None    # 风格档案
    previous_chapters: Optional[List[str]] = None  # 前文章节
    
    # === 高级选项 ===
    temperature: float = 0.8      # 生成温度（0.0-2.0）
    preserve_ending: bool = True  # 是否保留自然结尾
    request_id: Optional[str] = None  # 请求ID
```

### 3.2 参数分类

| 分类 | 参数 | 必需 | 说明 |
|------|------|------|------|
| **核心参数** | starting_text | ✅ | 续写起点文本 |
| | word_count | ✅ | 目标字数 |
| **方向参数** | direction | ❌ | 续写方向（默认natural） |
| | direction_hint | ❌ | 特定方向提示 |
| **上下文** | outline | ❌ | 章节大纲 |
| | characters | ❌ | 人物设定列表 |
| | worldview | ❌ | 世界观设定 |
| | style_profile | ❌ | 风格档案 |
| | previous_chapters | ❌ | 前文章节文本 |
| **高级选项** | temperature | ❌ | 生成温度 |
| | preserve_ending | ❌ | 自然结尾 |
| | request_id | ❌ | 请求追踪ID |

### 3.3 参数约束

```python
# 字数约束

# 温度约束
```

## 四、返回结果结构

### 4.1 ContinuationResult模型

```python
class ContinuationResult(BaseModel):
    """续写结果"""
    
    # === 核心结果 ===
    text: str               # 续写生成的文本内容
    word_count: int = 0     # 实际生成字数
    
    # === 元数据 ===
    metadata: ContinuationMetadata  # 生成元数据
    
    # === 状态 ===
    success: bool = True    # 是否成功
    error: Optional[str] = None  # 错误信息
    
    # === 建议（可选）===
    suggestions: Optional[List[str]] = None  # 后续建议
    
    def get_full_text(self, starting_text: str) -> str:
        """获取完整文本（起始文本 + 续写内容）"""
        return starting_text + self.text
```

### 4.2 ContinuationMetadata模型

```python
class ContinuationMetadata(BaseModel):
    """续写元数据"""
    
    # === 模型信息 ===
    model_name: str = ""    # 使用的模型名称
    provider: str = ""      # 模型提供商
    
    # === 性能指标 ===
    generation_time: float = 0.0  # 生成耗时（秒）
    tokens_used: int = 0          # 消耗的token数
    iterations: int = 1           # 迭代次数
    
    # === 质量指标 ===
    coherence_score: float = 0.0      # 连贯性评分（0-1）
    style_match_score: float = 0.0    # 风格匹配度（0-1）
    
    # === 上下文信息 ===
    context_length: int = 0           # 上下文长度
    starting_text_length: int = 0     # 起始文本长度
    
    # === 时间戳 ===
    timestamp: datetime = Field(default_factory=datetime.now)
```

### 4.3 结果字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| text | str | 续写生成的文本内容 |
| word_count | int | 实际生成字数 |
| metadata.model_name | str | 使用的模型名称 |
| metadata.provider | str | 模型提供商（deepseek/openai/anthropic） |
| metadata.generation_time | float | 生成耗时（秒） |
| metadata.tokens_used | int | 消耗的token数 |
| metadata.iterations | int | 迭代次数（如有重试） |
| metadata.coherence_score | float | 连贯性评分（0-1） |
| metadata.style_match_score | float | 风格匹配度（0-1） |
| success | bool | 是否成功 |
| error | str | 错误信息（如有） |
| suggestions | List[str] | 后续续写建议 |

## 五、续写方向设计

### 5.1 ContinuationDirection枚举

```python
class ContinuationDirection(str, Enum):
    """续写方向枚举"""
    
    NATURAL = "natural"       # 自然续写
    SPECIFIC = "specific"     # 特定方向
    EMOTION = "emotion"       # 情感导向
    ACTION = "action"         # 动作导向
    DIALOGUE = "dialogue"     # 对话导向
```

### 5.2 方向说明

| 方向 | 说明 | 适用场景 |
|------|------|----------|
| **natural** | 按大纲和上下文自然发展 | 常规续写，无特殊要求 |
| **specific** | 用户指定情节走向 | 有明确情节目标 |
| **emotion** | 侧重情感描写 | 情感场景、内心独白 |
| **action** | 侧重场景和动作 | 战斗、运动、场景描写 |
| **dialogue** | 侧重人物对话 | 人物互动、对话推进 |

### 5.3 方向实现建议

```python
# 自然续写
prompt = f"根据以下文本自然续写：\n{starting_text}"

# 特定方向
prompt = f"根据以下文本，按照方向提示续写：\n方向提示：{direction_hint}\n文本：{starting_text}"

# 情感导向
prompt = f"以情感描写为主续写以下内容：\n{starting_text}"

# 动作导向
prompt = f"以场景和动作描写为主续写以下内容：\n{starting_text}"

# 对话导向
prompt = f"以人物对话为主续写以下内容：\n{starting_text}"
```

## 六、上下文管理

### 6.1 上下文优先级

```
1. starting_text (必需)     - 续写起点，最高优先级
2. outline                   - 章节大纲，决定情节走向
3. characters                - 人物设定，保持角色一致性
4. style_profile             - 风格档案，匹配写作风格
5. worldview                 - 世界观设定，背景一致性
6. previous_chapters         - 前文章节，连贯性参考
```

### 6.2 上下文完整度评估

```python
def _evaluate_context_completeness(request: ContinuationRequest) -> float:
    """评估上下文完整度（0-1）"""
    score = 0.0
    if request.outline:
        score += 0.25
    if request.characters:
        score += 0.25
    if request.worldview:
        score += 0.2
    if request.style_profile:
        score += 0.2
    if request.previous_chapters:
        score += 0.1
    return score
```

### 6.3 上下文优化建议

```python
def _generate_context_suggestions(request: ContinuationRequest) -> List[str]:
    """生成上下文优化建议"""
    suggestions = []
    if not request.outline:
        suggestions.append("建议提供章节大纲以获得更好的情节连贯性")
    if not request.characters:
        suggestions.append("建议提供人物设定以保持角色一致性")
    if not request.style_profile:
        suggestions.append("建议提供风格档案以匹配写作风格")
    return suggestions
```

## 七、使用示例

### 7.1 基本续写

```python
# 创建续写请求
request = ContinuationRequest(
    starting_text="月光洒在窗台上，她轻轻叹了口气。",
    word_count=500,
    direction="natural"
)

# 调用续写
result = continuation_plugin.generate_continuation(request)

# 获取结果
print(f"续写内容: {result.text}")
print(f"实际字数: {result.word_count}")
print(f"生成耗时: {result.metadata.generation_time}秒")
```

### 7.2 带上下文的续写

```python
request = ContinuationRequest(
    starting_text="他推开门，发现屋内一片狼藉。",
    word_count=800,
    direction="action",
    outline="第一章：主角发现密室，寻找线索",
    characters=[
        {"name": "李明", "role": "主角", "traits": ["冷静", "观察力强"]}
    ],
    worldview="现代都市悬疑背景",
    style_profile={"tone": "紧张", "pacing": "紧凑"},
    previous_chapters=["上一章结尾：他站在门口犹豫了许久..."],
    temperature=0.9
)

result = continuation_plugin.generate_continuation(request)
```

### 7.3 特定方向续写

```python
request = ContinuationRequest(
    starting_text="两人对视良久，空气中弥漫着尴尬。",
    word_count=600,
    direction="specific",
    direction_hint="主角主动打破沉默，询问对方来意",
    outline="第二章：初次交锋，暗藏机锋"
)

result = continuation_plugin.generate_continuation(request)
```

### 7.4 预估续写效果

```python
# 预估续写
estimate = continuation_plugin.estimate_continuation(request)
print(f"预估token消耗: {estimate['estimated_tokens']}")
print(f"预估耗时: {estimate['estimated_time']}秒")
print(f"上下文完整度: {estimate['context_completeness']}")
```

```python
    # 访问V5模块
    style_learner = self._context.v5_modules.get("style-learner-v5")
    character_manager = self._context.v5_modules.get("character-manager")
    
    # 使用V5模块增强续写
    if style_learner and request.style_profile:
        style_features = style_learner.extract_features(request.style_profile)
        # 应用风格特征...
    
    if character_manager and request.characters:
        char_context = character_manager.build_context(request.characters)
        # 构建人物上下文...
```

### 8.2 V5保护机制

续写插件ID应为：`continuation-v1`（非V5保护模块，可自由扩展）

