## 一、核心目标（用户诉求驱动）

**"越用越聪明"的最终目标**：生成更符合用户诉求的小说

### 1.1 五大核心能力

| 能力 | 描述 | 实现方式 |
|------|------|----------|
| **用户反馈窗口** | 提供对话窗口，让用户直接提意见 | GUI反馈面板 + 反馈收集器 |

| 指标 | 目标值 | 说明 |
|------|:------:|------|

### 3.2 AI反馈提纯器（新增）

**功能**：从用户反馈中提取可执行知识

**提纯流程**：
```
用户反馈："这个对话太生硬了，不像角色会说的话"
    ↓
NLP分析：
    - 问题类型：风格问题
    - 涉及维度：自然度、人设一致性
    - 关键词：对话、生硬、不像角色
    ↓
提取知识点：
    - 类型：风格知识
    - 内容：对话要符合人物性格，避免书面化表达
    - 标签：对话技巧/人物塑造
    ↓
结构化存储：
    - 知识库：新增"对话自然度"知识点
    - 标签：风格/对话/人物
    - 权重：高（用户直接反馈）
    ↓
应用：
    - 下次生成时召回此知识点
    - Prompt中加入"对话要符合人物性格"的要求
```

**核心算法**：
```python
def purify_feedback(feedback_text: str, feedback_type: str) -> List[KnowledgePoint]:
    """
    从反馈中提取知识点
    
    Args:
        feedback_text: 用户反馈文本
        feedback_type: 反馈类型（内容/风格/AI感/其他）
    
    Returns:
        提取的知识点列表
    """
    knowledge_points = []
    
    # 1. NLP分析
    analysis = analyze_feedback(feedback_text)
    # analysis = {
    #     "problem_type": "风格问题",
    #     "involved_dimensions": ["自然度", "人设一致性"],
    #     "keywords": ["对话", "生硬", "不像角色"],
    #     "sentiment": "负面"
    # }
    
    # 2. 提取知识点
    if feedback_type == "风格问题":
        # 提取风格相关知识点
        kp = KnowledgePoint(
            category="style",
            content=f"对话要符合人物性格，避免书面化表达",
            tags=["对话技巧", "人物塑造"],
            source="用户反馈",
            weight=0.9,  # 用户反馈权重高
            context={
                "original_feedback": feedback_text,
                "problem_type": analysis["problem_type"],
                "keywords": analysis["keywords"]
            }
        )
        knowledge_points.append(kp)
    
    elif feedback_type == "AI感问题":
        # 提取AI痕迹知识点
        kp = KnowledgePoint(
            category="ai_feeling",
            content=f"避免{analysis['keywords'][0]}等AI痕迹表达",
            tags=["AI痕迹", "自然度"],
            source="用户反馈",
            weight=0.95,
            context={
                "original_feedback": feedback_text,
                "ai_words": extract_ai_words(feedback_text)
            }
        )
        knowledge_points.append(kp)
    
    # 3. 去重和合并
    knowledge_points = deduplicate_and_merge(knowledge_points)
    
    return knowledge_points
```

### 3.3 策略调整器（新增）

**功能**：根据反馈自动优化生成策略

**调整维度**：

| 调整项 | 触发条件 | 调整方式 |
|--------|----------|----------|
| Prompt优化 | 用户反馈风格问题 | 在Prompt中增加风格约束 |
| 权重调整 | 某维度持续低分 | 提升该维度权重 |
| 模板更新 | AI感评分高 | 更新生成模板，避免套路 |
| 知识召回 | 检测到冲突 | 召回更多相关知识 |

**调整示例**：
```python
class StrategyAdjuster:
    def adjust_from_feedback(self, feedback: Feedback):
        """根据反馈调整策略"""
        
        # 1. 分析反馈类型
        if feedback.type == "风格问题":
            # 优化Prompt
            self.optimize_prompt(
                add_constraints=["对话要自然，符合人物性格"]
            )
        
        elif feedback.type == "AI感问题":
            # 更新AI痕迹词库
            self.update_ai_feeling_words(
                add_words=feedback.extracted_ai_words
            )
        
        elif feedback.type == "内容问题":
            # 增加知识召回
            self.increase_knowledge_recall(
                related_tags=feedback.related_tags
            )
    
    def adjust_from_score_trend(self, trend: ScoreTrend):
        """根据评分趋势调整策略"""
        
        # 识别薄弱维度
        weak_dims = trend.get_weak_dimensions(threshold=0.75)
        
        for dim in weak_dims:
            # 提升该维度权重（+5%）
            self.adjust_dimension_weight(
                dimension=dim,
                delta=0.05
            )
            
            # 记录调整原因
            self.log_adjustment(
                reason=f"{dim}维度平均分{trend.get_avg(dim):.2f}低于阈值",
                action=f"提升{dim}权重5%"
            )
```

#### 3.5.1 Prompt优化引擎

**自动优化Prompt**：
```python
class PromptOptimizer:
    def optimize_prompt(self, base_prompt: str, 
                       knowledge_points: List[KnowledgePoint],
                       user_feedbacks: List[Feedback]) -> str:
        """
        根据知识库和用户反馈优化Prompt
        """
        optimized = base_prompt
        
        # 1. 添加知识约束
        if knowledge_points:
            knowledge_constraints = self._extract_constraints(knowledge_points)
            optimized += f"\n\n【写作要求】\n{knowledge_constraints}"
        
        # 2. 添加用户偏好
        if user_feedbacks:
            user_preferences = self._extract_preferences(user_feedbacks)
            optimized += f"\n\n【用户偏好】\n{user_preferences}"
        
        # 3. 添加AI痕迹规避
        ai_feeling_words = self._get_ai_feeling_words()
        if ai_feeling_words:
            optimized += f"\n\n【避免使用】\n{ai_feeling_words}"
        
        return optimized
```

**示例**：
```

"请根据大纲生成下一章内容..."

"请根据大纲生成下一章内容...

【写作要求】
- 对话要自然，符合人物性格，避免书面化表达
- 环境描写要有细节感，形成画面感
- 避免过度解释，让读者自己领悟

【用户偏好】
- 喜欢简洁明快的叙事风格

- 偏爱对话推动剧情

【避免使用】
- 仿佛、似乎、宛如、如同
- 不禁、不由得、忍不住
- 一股、一种、一阵
"
```

#### 3.5.2 Fine-tuning数据积累

**积累训练数据**：
```python
class FinetuningDataAccumulator:
    def collect_data(self, chapter: Chapter, 
                    user_feedback: Optional[Feedback] = None):
        """
        收集Fine-tuning数据
        """
        data = {
            "input": chapter.outline,  # 大纲作为输入
            "output": chapter.content,  # 生成内容作为输出
            "score": chapter.scores.total,  # 评分
            "iterations": chapter.iterations,  # 迭代次数
        }
        
        # 如果有用户修改，使用修改后的内容
        if user_feedback and user_feedback.corrected_content:
            data["output"] = user_feedback.corrected_content
            data["user_modified"] = True  # 标记为用户修改
        
        # 只收集高质量数据（评分≥0.85）
        if data["score"] >= 0.85:
            self.save_data(data)
            logger.info(f"收集Fine-tuning数据：评分{data['score']:.2f}")
    
    def export_dataset(self, output_path: str, min_samples: int = 100):
        """
        导出训练数据集
        """
        data = self.load_all_data()
        
        if len(data) >= min_samples:
            # 转换为训练格式
            dataset = []
            for item in data:
                dataset.append({
                    "instruction": "根据大纲生成小说章节",
                    "input": item["input"],
                    "output": item["output"]
                })
            
            # 保存为JSONL
            with open(output_path, 'w', encoding='utf-8') as f:
                for item in dataset:
                    f.write(json.dumps(item, ensure_ascii=False) + '\n')
            
            logger.info(f"导出训练数据集：{len(dataset)}条，保存至{output_path}")
        else:
            logger.warning(f"数据量不足：{len(data)}条，需要至少{min_samples}条")
```

**预期效果**：
- 积累100条高质量数据后，可以开始Fine-tuning
- Fine-tuning后，模型会更符合用户的写作风格偏好
- 减少迭代次数，提升生成质量

```
   - 风格：19%
   - 人设：19%
   - 世界观：12%
   - 逆向反馈：11%
   - 自然度：10%
    ↓
5. AI感检测
   - 词汇多样性
   - 结构自然度
   - 情感真实度
   - 节奏流畅度
    ↓
6. 一致性检查
   - 对照知识库
   - 对照前文设定
   - 检查已知冲突模式
    ↓
    判断：总分≥0.8 且 有【本章完】？
    ├─ 是 → 输出保存，进入学习阶段
    └─ 否 → 构建反馈，返回步骤3（最多5次）
```

### 4.2 用户反馈流程（手动）

```
用户打开【反馈与建议】窗口
    ↓
1. 查看当前章节内容
    ↓
2. 输入反馈
   - 选择反馈类型（内容/风格/AI感/其他）
   - 输入文本或语音
    ↓
3. 提交反馈
    ↓
4. 系统自动处理
   - NLP提纯反馈
   - 提取知识点
   - 更新知识库
   - 应用策略调整
    ↓
5. 展示学习结果
   - 显示已学习的知识点
   - 显示下次生成会应用的优化
```

## 七、关键规则保护

### 7.1 评分反馈循环保护（强制）

| 规则 | 说明 | 违反后果 |
|------|------|----------|
| 【本章完】强制 | 章节结束必须有此标记 | 未达标视为未完成 |
| 评分阈值 | 总分≥0.8才输出 | 低于阈值继续迭代 |
| 迭代上限 | 最多5次迭代 | 超过上限强制输出最佳版本 |
| 上下文记忆 | 保持前5章 | 丢失则一致性下降 |
| 权重总和 | 必须100% | 配置错误自动归一化 |

## 八、常见问题

### Q1: 用户反馈如何影响生成？

**答**：用户反馈会经过NLP提纯，提取出知识点和偏好，然后：
1. 更新知识库，下次生成时召回
2. 优化Prompt，增加约束和偏好
3. 调整策略，提升相关维度权重

### Q2: 如何确保系统不会"学偏"？

**答**：系统有多重保护：
1. 知识点去重和合并
2. 权重调整有上限（±10%）
3. 定期报告供用户审查
4. 用户可以手动干预

### Q3: 多久能看到效果？

**答**：
- **即时效果**：提交反馈后，下次生成立即应用
- **明显效果**：1-2周后，八大维度提升3-5%
- **显著效果**：1-2月后，八大维度提升8-12%，AI感降低30%

### Q4: 如何查看系统学到了什么？

**答**：
1. 在反馈窗口查看"系统已学习的知识点"
2. 查看定期报告（周报/月报）
3. 查看知识库（LanceDB）

### Q5: 可以手动干预吗？

**答**：可以，用户可以：
1. 手动添加知识点
2. 手动调整权重
3. 手动标记AI痕迹词汇
4. 删除错误的知识点
