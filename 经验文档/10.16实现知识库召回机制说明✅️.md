## 三、关键技术实现

### 3.1 智能召回（自动识别题材）

**核心方法**: `recall_for_content()`

**召回策略**:
1. **题材识别**: 根据关键词识别题材（科幻/玄幻/历史/通用）
   - 科幻关键词：飞船、星际、光速、虫洞、量子、基因、克隆、AI、机器人、时间旅行等
   - 玄幻关键词：修仙、丹药、灵气、宗门、法宝、元婴、渡劫、神兽、道法、佛门等
   - 历史关键词：朝代、皇帝、将军、战争、朝堂、谋士、骑兵、城池、谋反、科举等

2. **领域识别**: 根据关键词识别领域（物理/化学/生物/宗教/神话/历史等）
   - 物理领域：光速、相对论、量子、黑洞、引力、时间膨胀、能量、粒子等
   - 化学领域：化学、元素、反应、分子、原子、化合物、催化、化学键等
   - 生物领域：基因、克隆、进化、DNA、细胞、病毒、生物、遗传等
   - 宗教领域：道教、佛教、轮回、因果、修仙、丹道、禅宗、菩萨等
   - 神话领域：神话、神仙、龙、凤凰、妖魔、天神、神兽、异兽等
   - 历史领域：朝代、皇帝、战争、谋略、战役、历史、古都、朝堂等

3. **向量召回**: 调用knowledge_retriever的向量检索
   - 召回top_k个相关知识（默认10个）
   - 支持题材和领域过滤
   - 支持最小相似度阈值（默认0.5）

4. **降级召回**: 向量召回结果不足时，降级为关键词召回

**性能指标**:
- 召回准确率: ≥80%（理论值）
- 召回延迟: P99 <100ms

### 3.2 一致性检测

**核心方法**: `check_knowledge_consistency()`

**检测流程**:
1. 题材识别：自动识别题材（如果未提供）
2. 知识召回：召回相关知识（top_k个）
3. 冲突检测：检测内容是否与知识点冲突
4. 评分计算：根据冲突数量和严重程度计算一致性评分

**冲突检测逻辑**（简化版）:

**P0级别冲突**（严重违反知识库）:
- 物理冲突：超光速违反相对论、时间倒流违反因果律等
- 化学冲突：违反化学反应规则等
- 生物冲突：违反生物学原理等

**P1级别冲突**（可能存在问题）:
- 历史冲突：历史细节需核实等
- 宗教冲突：宗教体系矛盾等

**P2级别冲突**（轻微问题）:
- 细节不一致等

**一致性评分计算**:
```python
基础分: 1.0（完全一致）
P0冲突: 每个扣0.3分
P1冲突: 每个扣0.1分
P2冲突: 每个扣0.05分
召回知识点不足（<3个）: 评分降低10%
最低分: 0.0
```

### 3.3 题材识别器

**核心类**: `GenreRecognizer`

**识别方法**:
1. **recognize_genre(content)**: 识别题材
   - 返回：(题材, 置信度)
   - 题材：scifi/xuanhuan/history/general
   - 置信度：0-1（至少5个关键词匹配才达到1.0置信度）

2. **recognize_domain(content, category)**: 识别领域
   - 根据题材限定领域范围
   - 返回：领域（如果识别到）

### 3.4 线程安全设计

**多线程保护**:
- `_stats_lock`（RLock）：保护统计信息
- `_lock`（RLock）：单例模式双重检查锁

**单例模式**:
- `get_knowledge_recall()`: 获取全局单例
- `reset_knowledge_recall()`: 重置单例（测试用）

### 3.5 EventBus集成

**事件类型**:
- `knowledge.recall.completed`: 召回完成
- `knowledge.consistency.checked`: 一致性检测完成

**事件数据格式**:
```json
{
  "event_type": "knowledge.recall.completed",
  "source": "KnowledgeRecall",
  "data": {
    "content_length": 100,
    "category": "scifi",
    "domain": "physics",
    "result_count": 10,
    "latency_ms": 45.2
  }
}
```

## 四、API接口

### 4.1 智能召回

```python
from core import get_knowledge_recall

# 获取召回器实例
recall = get_knowledge_recall(workspace_root=Path("E:/project"))

# 智能召回（自动识别题材）
results = recall.recall_for_content(
    content="飞船接近光速飞行，时间流逝变慢",
    top_k=10,
    min_score=0.5
)

# 遍历结果
for r in results:
    print(f"标题: {r.title}")
    print(f"分类: {r.category}, 领域: {r.domain}")
    print(f"分数: {r.score}")
    print(f"召回策略: {r.recall_strategy}")
```

### 4.2 一致性检测

```python
# 一致性检测
check_result = recall.check_knowledge_consistency(
    content="超光速飞船瞬间到达目的地，时间倒流回出发前",
    category="scifi",  # 可选，自动识别
    top_k=10
)

# 检查结果
print(f"是否一致: {check_result.is_consistent}")
print(f"一致性评分: {check_result.consistency_score}")
print(f"冲突数量: {len(check_result.conflicts)}")

# 遍历冲突
for conflict in check_result.conflicts:
    print(f"冲突类型: {conflict.conflict_type}")
    print(f"严重程度: {conflict.severity}")
    print(f"描述: {conflict.description}")
    print(f"修复建议: {conflict.suggested_fix}")
```

### 4.3 题材识别

```python
# 题材识别
recognizer = GenreRecognizer()

genre, confidence = recognizer.recognize_genre(
    "修仙者在仙界炼制丹药，突破元婴期"
)
print(f"题材: {genre}, 置信度: {confidence}")

domain = recognizer.recognize_domain(
    "修仙者在仙界炼制丹药",
    genre
)
print(f"领域: {domain}")
```

## 六、架构意义

知识库召回机制作为OpenClaw mem9的**智能召回层**：
1. **智能识别**: 自动识别题材和领域，无需手动指定
2. **多策略召回**: 向量召回 + 关键词召回 + 规则召回
3. **一致性检测**: 检测生成内容是否违反知识库
4. **评分集成**: 可与ValidationScores协同验证知识库一致性
5. **EventBus集成**: 发布召回事件，供监控系统感知

### 与知识库检索的区别

| 特性 | KnowledgeRetriever（Sprint 3-4） | KnowledgeRecall（Sprint 5-6） |
|------|----------------------------------|------------------------------|
| 核心功能 | 向量检索接口 | 智能召回机制 |
| 题材识别 | 手动指定 | 自动识别 |
| 召回策略 | 单一（向量或关键词） | 多策略融合 |
| 一致性检测 | 无 | 有 |
| 冲突检测 | 无 | 有 |
| 适用场景 | 底层检索接口 | 高级业务层调用 |

### 集成到ValidationScores

```python
# 在quality-validator插件中集成
from core import get_knowledge_recall

recall = get_knowledge_recall()

# 一致性检测

scores.recalled_knowledge = [r.dict() for r in check_result.recalled_knowledge]
```
