## 一、改造背景

根据《9.9大模型集成.txt》要求，将快捷创作插件中的AI调用统一替换为AIServiceManager调用，实现本地AI和线上AI的动态切换。

### 2.2 改造前后对比

#### 改造前（直接调用OpenAI API）

```python
def _do_api_call(self, system_prompt: str, user_prompt: str, temperature: float, max_tokens: int) -> str:
    response = self.api_client.chat.completions.create(
        model=self.config.get("model", "deepseek-chat"),
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt}
        ],
        temperature=temperature,
        max_tokens=max_tokens,
        stream=False
    )
    
    return response.choices[0].message.content
```

#### 改造后（使用AIServiceManager）

```python
def _do_api_call(self, system_prompt: str, user_prompt: str, temperature: float, max_tokens: int) -> str:
    # 使用AIServiceManager统一调用
    from core.ai_service_manager import get_ai_service_manager
    from core.ai_provider import GenerationConfig

    ai_manager = get_ai_service_manager()
    
    config = GenerationConfig(
        temperature=temperature,
        max_tokens=max_tokens
    )
    
    result = ai_manager.generate_text(
        prompt=user_prompt,
        config=config,
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt}
        ]
    )
    
    return result.text
```

## 三、改造优势

### 3.1 统一接口

所有AI调用通过AIServiceManager统一管理，插件不再直接依赖OpenAI SDK。

### 3.2 动态切换

根据config.yaml配置自动选择Provider：
- **线上模式**: DeepSeek / OpenAI
- **本地模式**: Qwen2.5-14B-GPTQ-Int4

```yaml
service_mode: local          # 或 online
provider: Qwen               # 或 DeepSeek / OpenAI
model: Qwen2.5-14B-GPTQ-Int4
temperature: 0.7
```

### 3.3 实时生效

配置变更后通过EventBus发布`config.changed`事件，AIServiceManager订阅事件并动态切换Provider，无需重启应用。

### 3.4 容错降级

AIServiceManager内置四级容错机制：
1. **模型回退**: 主模型失败自动切换备用模型
2. **Provider降级**: 本地AI失败降级到线上API
3. **缓存兜底**: API失败返回缓存结果
4. **优雅降级**: 返回降级提示而非报错

## 六、技术细节

### 6.1 插件调用流程

```
用户请求 → QuickCreationPlugin
         → _call_llm() → concurrent.futures超时保护
         → _do_api_call() → AIServiceManager.generate_text()
         → LocalProvider / OnlineProvider
         → 返回结果
```

### 6.2 AIServiceManager架构

```
AIServiceManager
  ├─ ConfigService (读取配置)
  ├─ EventBus (订阅配置变更)
  ├─ Provider管理
  │   ├─ LocalProvider (本地AI)
  │   └─ OnlineProvider (线上API)
  └─ 容错机制
      ├─ 模型回退
      ├─ Provider降级
      ├─ 缓存兜底
      └─ 优雅降级
```

### 6.3 兼容性设计

保留`set_api_client()`方法作为兼容接口：

```python
def set_api_client(self, client: Any) -> None:
    """设置大模型API客户端（兼容接口）"""
    self.api_client = client
    logger.info("大模型API客户端已设置")
```

实际调用时不再使用`api_client`，而是通过AIServiceManager获取Provider。

## 八、注意事项

### 8.1 本地AI特性

- **贪婪解码**: 使用`do_sample=False`避免CUDA采样错误
- **GPU显存**: 7GB显存限制，超出部分自动offload到CPU
- **加载时间**: 首次调用时加载模型，约15秒

### 8.2 性能优化

- **缓存预热**: 登录时预加载常用缓存
- **全局缓存**: 使用CacheManager提升缓存命中率
- **并发控制**: RLock保护生成调用，避免并发问题

### 8.3 错误处理

```python
try:
    result = ai_manager.generate_text(prompt)
except QuickCreationTimeoutError:
    # 超时处理
    logger.error("生成超时")
except QuickCreationAPIError as e:
    # API错误处理
    logger.error(f"API调用失败: {e}")
```
