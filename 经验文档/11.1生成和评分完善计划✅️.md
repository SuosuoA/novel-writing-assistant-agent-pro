## 一、任务背景

### 1.1 需求来源

用户反馈：原评分权重体系中缺少对"上下文衔接（逆向反馈）"和"知识点引用（知识库功能）"的评分维度。

**原权重体系（7维度，总和110%）**：
- 字数: 10% / 大纲: 15% / 风格: 25% / 人设: 25% / 世界观: 20% / 自然度: 5% / 知识库一致性: 10%

**问题**：
1. 知识库一致性维度与实际知识库功能脱节
2. 逆向反馈功能已实现但未纳入正式评分维度
3. 总权重110%需要归一化处理

### 1.2 需求目标

**新权重体系（8维度，总和100%）**：
- 字数: 8% / 知识点引用: 8% / 大纲: 13% / 风格: 19% / 人设: 19% / 世界观: 12% / 逆向反馈: 11% / 自然度: 10%

## 三、新维度说明

### 3.1 知识点引用评分（knowledge_reference_score）

**用途**: 验证生成内容是否正确引用知识库知识点

**评分逻辑**:
- 基础分：0.5（默认分）
- 成功召回知识点：+0.2（每个+0.05，上限0.2）
- 知识点在文本中被引用：+0.3（每个+0.1，上限0.3）
- 最高分：1.0

**依赖模块**:
- `core/knowledge_retriever.py`: 知识检索器
- `core/knowledge_recall.py`: 知识召回

### 3.2 逆向反馈评分（reverse_feedback_score）

**用途**: 检查章节与已设定内容的一致性（上下文衔接）

**评分逻辑**:
- 基础分：1.0（无冲突）
- 高优先级冲突：每个-0.20
- 中优先级冲突：每个-0.10
- 低优先级冲突：每个-0.05
- 最低分：0.4（保底）

**依赖模块**:
- `plugins/reverse-feedback-analyzer/`: 逆向反馈分析器插件
