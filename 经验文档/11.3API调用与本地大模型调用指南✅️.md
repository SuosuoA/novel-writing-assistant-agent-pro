## 概述

本项目支持**线上API**和**本地模型**两种调用模式，通过统一的接口实现无缝切换。

### 核心优势

- **成本控制**: 线上API按需付费，本地模型免费使用
- **隐私保护**: 本地模型完全离线运行
- **质量保障**: 线上API提供更高质量的生成结果
- **容错机制**: 四级容错确保服务稳定

## 双模式设计

### 线上API模式（remote）

**适用场景**:
- 生产环境
- 高质量内容生成
- 复杂推理任务

- ✅ 高质量输出
- ✅ 持续优化
- ✅ 丰富功能
- ⚠️ 需要网络
- ⚠️ 按量计费

**适用场景**:

- 隐私敏感内容

**特点**:
- ✅ 完全免费
- ✅ 隐私保护
- ✅ 离线运行
- ⚠️ 质量略低
- ⚠️ 需要本地资源

## 配置方法

### 方法一：GUI配置（推荐）

1. 打开应用，进入**设置**页面
2. 选择**AI配置**标签页
3. 选择Provider类型（DeepSeek/OpenAI/Anthropic/Ollama）
4. 输入API Key（本地模型无需）
5. 点击**测试连接**验证配置
6. 点击**保存配置**

**安全提示**：
- API Key会自动加密存储在 `.secrets/api_keys.enc` 中
- 不会明文保存在 `config.yaml` 中
- 首次保存时会自动生成主密钥 `.secrets/.master_key`

### 方法二：配置文件

编辑 `config.yaml`:

```yaml
# AI 服务配置
provider: DeepSeek
model: deepseek-chat
service_mode: remote  # remote（线上API）或 local（本地模型）
temperature: 0.7

# 本地模型配置（仅当 service_mode: local 时生效）
local_url: http://localhost:11434/v1
```

**重要说明**：
- ⚠️ API Key 已加密存储在 `.secrets/api_keys.enc` 中
- ⚠️ `config.yaml` 中的 `api_key` 字段仅为占位符
- ⚠️ 请勿在 `config.yaml` 中填写真实API Key

## 调用示例

### 基础对话

```python
from openai import OpenAI

# 创建客户端（自动读取加密存储的API Key）
client = OpenAI(
    api_key=get_api_key_from_encrypted_storage(),
    base_url="https://api.deepseek.com"
)

# 发送请求
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=[
        {"role": "system", "content": "你是一个小说创作助手"},
        {"role": "user", "content": "写一段科幻小说的开头"}
    ]
)

print(response.choices[0].message.content)
```

### 流式响应

```python
# 流式调用
stream = client.chat.completions.create(
    model="deepseek-chat",
    messages=[{"role": "user", "content": "生成一个人物设定"}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

## 错误处理与降级

### 四级容错机制

```
Level 1: 重试（RetryManager）
    ↓ 失败
Level 2: 熔断器（CircuitBreaker）
    ↓ 触发
Level 3: 降级到备用模型
    ↓ 失败
Level 4: 缓存回退
```

### 错误类型与处理

| 错误类型 | HTTP状态码 | 处理策略 |
|----------|-----------|----------|
| 速率限制 | 429 | 指数退避重试 |
| 服务器错误 | 500/502/503/504 | 重试+降级 |
| 认证失败 | 401 | 停止重试，提示用户 |
| 请求超时 | Timeout | 重试+降级 |
| 网络错误 | ConnectionError | 重试+本地模型 |

## 性能优化

### 线上API优化

#### 1. 请求批处理

```python
# 批量处理多个请求
responses = []
for messages in messages_list:
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=messages
    )
    responses.append(response)
```

#### 2. 缓存策略

```python
from core.cache_manager import CacheManager

# 启用缓存
cache = CacheManager(ttl=3600)

# 缓存响应
cache.set(cache_key, response)
cached_response = cache.get(cache_key)
```

#### 3. 流式响应

```python
# 流式响应降低首字延迟
stream = client.chat.completions.create(
    model="deepseek-chat",
    messages=messages,
    stream=True
)

for chunk in stream:
    process_chunk(chunk)
```

## 故障排查

### API故障

#### 问题1: 认证失败

```
错误: AuthenticationError: Invalid API key
```

**解决方案**:
1. 检查API Key是否正确
2. 检查API Key是否过期
3. 检查账户余额是否充足

#### 问题2: 速率限制

```
错误: RateLimitError: Rate limit exceeded
```

**解决方案**:
1. 等待一段时间后重试
2. 降低请求频率
3. 升级API套餐

#### 问题3: 超时

```
错误: TimeoutError: Request timeout
```

**解决方案**:
1. 检查网络连接
2. 增加超时时间
3. 切换到本地模型

### 本地模型故障

#### 问题1: Ollama未启动

```
错误: ConnectionError: Cannot connect to localhost:11434
```

**解决方案**:
```bash
# 启动Ollama服务
ollama serve

# 检查服务状态
curl http://localhost:11434/api/tags
```

#### 问题2: 模型未找到

```
错误: ModelNotFoundError: Model not found
```

**解决方案**:
```bash
# 下载模型
ollama pull qwen2:7b

# 验证模型
ollama list
```

#### 问题3: 内存不足

```
错误: OutOfMemoryError: CUDA out of memory
```

**解决方案**:
1. 使用量化模型
2. 减少并发请求数
3. 增加系统交换空间

### 配置问题

#### 问题1: API Key未加密存储

```
警告: API Key is stored in plaintext
```

**解决方案**:
1. 使用GUI设置页面重新保存API Key
2. 系统会自动加密存储
3. 明文API Key会自动移除

#### 问题2: API Key泄露

```
警告: API Key detected in git commit
```

**解决方案**:
```bash
# 撤销最近提交
git reset --soft HEAD~1

# 从历史中删除敏感信息
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch config.yaml" \
  --prune-empty --tag-name-filter cat -- --all

# 强制推送
git push origin --force --all
```
