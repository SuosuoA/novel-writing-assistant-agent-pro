## 三、关键技术实现

### 3.1 向量检索召回（OpenClaw memory_search）

**核心方法**: `recall_knowledge()`

```python
def recall_knowledge(
    self,
    query: str,
    category: Optional[str] = None,
    domain: Optional[str] = None,
    top_k: int = 10,
    min_score: float = 0.5
) -> List[RetrievalResult]:
    """
    向量检索召回top-10相关知识（OpenClaw memory_search）
    
    性能指标：
    - 召回准确率: ≥80%
    - 检索延迟: P99 <100ms
    """
```

**检索流程**:
```
用户查询（"飞船接近光速会发生什么"）
    ↓
检查缓存（命中直接返回）
    ↓
向量检索（LanceDB recall_knowledge）
    ↓
降级关键词检索（向量检索结果不足时）
    ↓
去重 + 排序
    ↓
缓存结果
    ↓
返回top-10结果
```

**关键技术**:
1. **向量检索**: 调用LanceDB的recall_knowledge()方法
2. **题材过滤**: 支持category参数过滤（scifi/xuanhuan/history/general）
3. **领域过滤**: 支持domain参数过滤（physics/chemistry/religion等）
4. **降级方案**: 向量检索失败或结果不足时降级为关键词检索
5. **LRU缓存**: 缓存检索结果，减少重复计算

### 3.2 混合检索（向量+关键词）

**核心方法**: `hybrid_search()`

```python
def hybrid_search(
    self,
    query: str,
    category: Optional[str] = None,
    domain: Optional[str] = None,
    top_k: int = 10,
    vector_weight: float = 0.7,
    keyword_weight: float = 0.3
) -> List[RetrievalResult]:
    """
    混合检索（向量+关键词）
    
    混合策略：
    - 向量检索召回语义相似知识点
    - 关键词检索召回精确匹配知识点
    - 加权融合分数：final_score = vector_score * 0.7 + keyword_score * 0.3
    """
```

**混合策略**:
1. **向量检索**: 召回语义相似的知识点（理解"飞船"和"星际飞船"是同义词）
2. **关键词检索**: 召回精确匹配的知识点（标题/关键词完全匹配）
3. **加权融合**: `final_score = vector_score * 0.7 + keyword_score * 0.3`
4. **去重合并**: 同一知识点在两个结果中时，合并分数

## 四、性能优化

### 4.1 LRU缓存机制

```python
# 缓存配置
self._cache: Dict[str, List[RetrievalResult]] = {}
self._cache_lock = threading.RLock()
self._cache_max_size = 100

# 缓存键构建
def _build_cache_key(self, query: str, category: Optional[str], domain: Optional[str], top_k: int) -> str:
    return f"{query}|{category}|{domain}|{top_k}"

# LRU淘汰策略
if len(self._cache) >= self._cache_max_size:
    # 删除最旧的条目
    oldest_key = next(iter(self._cache))
    del self._cache[oldest_key]
```

**缓存效果**:
- 重复查询直接从缓存返回，无需重新计算
- 减少向量计算开销，提升响应速度

### 4.2 降级策略

```python
# 向量检索失败或结果不足时降级为关键词检索
if len(results) < top_k:
    keyword_results = self._keyword_search(
        query=query,
        category=category,
        domain=domain,
        top_k=top_k - len(results)
    )
    results.extend(keyword_results)
```

**降级场景**:
1. LanceDB未初始化或不可用
2. 向量检索结果不足（< top_k）
3. 向量检索异常（网络超时、内存不足等）

### 4.3 线程安全设计

```python
# 多个RLock保护共享资源
self._cache_lock = threading.RLock()  # 保护缓存
self._stats_lock = threading.RLock()  # 保护统计信息

# 单例模式双重检查锁
def __new__(cls, *args, **kwargs):
    if cls._instance is None:
        with cls._lock:
            if cls._instance is None:
                cls._instance = super().__new__(cls)
    return cls._instance
```

### 5.1 事件类型

| 事件类型 | 说明 | 数据字段 |
|---------|------|---------|
| `knowledge.retrieval.cache_hit` | 缓存命中 | query, category, domain |
| `knowledge.retrieval.completed` | 检索完成 | query, category, domain, result_count, latency_ms |
| `knowledge.retrieval.hybrid_completed` | 混合检索完成 | query, category, domain, result_count, latency_ms |

```python
})
```
