## 一、背景与目标

### 1.1 当前问题

**gui_main.py 文件过大**：13,760行代码，严重超标

**核心问题**：
1. **业务逻辑混杂在GUI层**：项目保存、加载、同步等业务逻辑直接写在GUI类中
2. **违反微内核架构原则**：GUI层承担了本应由服务层或插件层承担的职责
3. **数据管理逻辑分散**：项目数据管理代码散落在GUI类的多个方法中
4. **难以测试和维护**：业务逻辑与UI逻辑耦合，无法独立测试

### 1.2 外迁目标

遵循《项目总体架构设计说明书V1.9》和《插件接口定义V2.3》，将项目数据管理逻辑外迁到独立模块：

| 目标 | 说明 | 预期效果 |
|------|------|----------|
| **分离关注点** | GUI只负责用户交互，业务逻辑由服务层处理 | GUI文件减少~200行 |
| **符合架构原则** | 遵循微内核+插件化架构设计 | 提高架构一致性 |
| **提高可测试性** | 项目管理逻辑可独立测试 | 测试覆盖率提升 |
| **事件驱动通信** | 通过EventBus解耦GUI和服务层 | 降低耦合度 |

### 2.2 数据流设计

#### 保存流程

```
用户点击保存按钮
    │
    ▼
GUI._on_gen_save()
    │
    ▼
ProjectManager.sync_module_data('outline', outline_content)
ProjectManager.sync_module_data('characters', character_data)
ProjectManager.sync_module_data('worldview', worldview_content)
...
    │
    ▼
ProjectManager.save_project()
    │
    ├─▶ 保存到文件系统 (project.json)
    └─▶ 发布事件 ProjectSavedEvent
            │
            ▼
        EventBus.dispatch()
            │
            ▼
        GUI._on_project_saved() [订阅事件]
            │
            ▼
        更新UI（状态栏、提示框）
```

#### 加载流程

```
用户打开项目文件
    │
    ▼
ProjectManager.load_project(file_path)
    │
    ├─▶ 从文件系统读取 (project.json)
    └─▶ 发布事件 ProjectLoadedEvent
            │
            ▼
        EventBus.dispatch()
            │
            ▼
        GUI._on_project_loaded() [订阅事件]
            │
            ▼
        _restore_project_data_to_ui(project_data)
            │
            ▼
        更新UI（大纲、人物、世界观等）
```

## 三、实现方案

### 3.1 服务层实现

**文件**: `services/project_manager.py` (~350行)

**核心类**:
- `ProjectManager`: 项目管理器主类
- `ProjectDataEvent`: 数据变更事件
- `ProjectSavedEvent`: 项目保存事件
- `ProjectLoadedEvent`: 项目加载事件

**核心方法**:

| 方法 | 说明 |
|------|------|
| `create_project(name, path)` | 创建新项目 |
| `load_project(path)` | 加载项目文件 |
| `save_project(project)` | 保存项目 |
| `sync_module_data(type, data)` | 同步模块数据 |
| `get_module_data(type)` | 获取模块数据 |
| `get_current_project()` | 获取当前项目 |
| `is_project_open()` | 检查是否有打开的项目 |

**事件机制**:
- 数据变更时发布 `ProjectDataEvent`
- 项目保存时发布 `ProjectSavedEvent`
- 项目加载时发布 `ProjectLoadedEvent`

### 3.2 插件层实现

**文件**: `plugins/project-manager-v1/plugin.py` (~150行)

**插件类型**: `PluginType.TOOL`

**核心职责**:
1. 创建并初始化 `ProjectManager` 实例
2. 注册 `project_manager` 服务到 `ServiceLocator`
3. 订阅项目相关事件（用于日志）
4. 提供便捷方法供其他插件调用

**生命周期**:
- `initialize()`: 创建项目管理器，注册服务，订阅事件
- `shutdown()`: 自动保存项目，清理资源

### 3.3 GUI层修改

**修改策略**: 保持向后兼容，逐步迁移

def _init_project_manager(self):
    """初始化项目管理器"""
    try:
        from core import ServiceLocator
        self._project_manager = ServiceLocator.get("project_manager")

    except Exception as e:

        self._project_manager = None

**第二步**: 修改保存方法

```python
def _on_gen_save(self):
    """保存项目 - 使用项目管理器"""
    if not self._project_manager or not self._project_manager.is_project_open():
        messagebox.showwarning("保存项目", "当前没有打开的项目")
        return
    
    try:
        self._set_status("正在保存项目...")
        
        # 同步所有模块数据到项目管理器
        self._sync_all_data_to_manager()
        
        # 调用项目管理器保存
        success = self._project_manager.save_project()
        
        if success:
            self._set_status("项目保存完成")
            messagebox.showinfo("成功", "项目已保存！")
        else:
            self._set_status("保存项目失败")
            messagebox.showwarning("保存失败", "项目保存失败，请查看日志获取详情")
            
    except Exception as e:
        from core.user_friendly_errors import convert_file_error
        title, full_message = convert_file_error(e, "保存项目")
        messagebox.showerror(title, full_message)
        self._set_status("保存项目失败")

def _sync_all_data_to_manager(self):
    """同步所有模块数据到项目管理器"""
    if not self._project_manager:
        return
    
    # 同步各模块数据
    if hasattr(self, '_outline_content') and self._outline_content:
        self._project_manager.sync_module_data('outline', self._outline_content)
    
    if hasattr(self, '_character_data') and self._character_data:
        self._project_manager.sync_module_data('characters', self._character_data)
    
    if hasattr(self, '_worldview_content') and self._worldview_content:
        self._project_manager.sync_module_data('worldview', self._worldview_content)
    
    # ... 其他模块
```

**第三步**: 修改加载方法

```python
def _on_open_project(self):
    """打开项目 - 使用项目管理器"""
    file_path = filedialog.askopenfilename(
        title="打开项目文件",
        filetypes=[("项目文件", "*.json"), ("所有文件", "*.*")]
    )
    
    if not file_path:
        return
    
    try:
        self._set_status("正在加载项目...")
        
        # 使用项目管理器加载
        project_data = self._project_manager.load_project(file_path)
        
        if project_data:
            self.current_project = project_data
            self.project_file = file_path
            
            # 更新项目名称标签
            if hasattr(self, '_project_name_label'):
                self._project_name_label.config(
                    text=f"项目: {project_data.get('name', '未命名项目')}"
                )
            
            # 恢复项目数据到UI
            self._restore_project_data_to_ui(project_data)
            
            self._set_status(f"项目已加载: {project_data.get('name', '未命名项目')}")
        else:
            messagebox.showerror("错误", "加载项目失败，请检查文件格式")
            self._set_status("加载项目失败")
            
    except Exception as e:
        from core.user_friendly_errors import convert_file_error
        title, full_message = convert_file_error(e, "打开项目")
        messagebox.showerror(title, full_message)
        self._set_status("打开项目失败")
```

**第四步**: 订阅事件监听

```python
def _setup_event_listeners(self):
    """设置事件监听器"""
    if not self._services or not self._services.event_bus:
        return
    
    event_bus = self._services.event_bus
    
    # 订阅项目保存事件
    event_bus.subscribe("project.saved", self._on_project_saved)
    
    # 订阅项目加载事件
    event_bus.subscribe("project.loaded", self._on_project_loaded)
    
    # 订阅数据变更事件（可选，用于实时更新）
    event_bus.subscribe("project.data.*", self._on_data_changed)

def _on_project_saved(self, event):
    """项目保存事件回调"""
    self._set_status("项目已保存")
    logger.info(f"[MainWindow] 项目已保存: {event.project_name}")

def _on_project_loaded(self, event):
    """项目加载事件回调"""
    self._set_status(f"项目已加载: {event.project_name}")
    logger.info(f"[MainWindow] 项目已加载: {event.project_name}")

def _on_data_changed(self, event):
    """数据变更事件回调"""
    logger.info(f"[MainWindow] 数据变更: {event.data_type} ({event.operation})")
```

## 七、风险与缓解

### 风险1: 插件加载失败

**风险**: 项目管理器插件初始化失败，导致无法保存/加载项目

**缓解**:
1. 插件设置较高优先级（priority: 1）
2. GUI提供降级方案：如果项目管理器不可用，使用原有逻辑
3. 清晰的错误提示和日志

### 风险2: 事件总线未初始化

**风险**: EventBus未正确初始化，导致事件发布失败

**缓解**:
1. 在 `ProjectManager.__init__()` 中接受可选的EventBus参数
2. 事件发布前检查EventBus是否存在
3. 提供 `set_event_bus()` 方法用于延迟设置

### 风险3: 向后兼容性

**风险**: 修改破坏了现有功能

**缓解**:
1. 保留 `_restore_project_data_to_ui()` 方法
2. 逐步迁移，不一次性重写所有代码
3. 充分测试后再删除旧代码
