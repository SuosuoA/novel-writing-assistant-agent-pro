> **版本**: V1.0  
> **创建日期**: 2026-03-24

> **适用插件**: quick-creator-v1 (v1.3.0)

## 一、功能概述

快捷创作功能帮助小说创作者快速生成各类设定，包括世界观、章节大纲、人物设定、关键情节等。只需提供关键词或参考文本，即可在数分钟内获得完整的创作框架。

### 1.1 核心能力

| 能力 | 描述 | 适用场景 |
|------|------|----------|
| 世界观生成 | 基于关键词生成完整世界设定 | 新项目启动、设定扩展 |
| 大纲生成 | 生成章节规划和剧情走向 | 项目规划、大纲调整 |
| 人物生成 | 生成立体丰满的人物形象 | 新角色创建、角色丰富 |
| 情节生成 | 生成关键剧情和转折点 | 剧情设计、高潮设计 |
| 参考文本解析 | 从已有文本提取设定元素 | 参考学习、设定融合 |
| 结果存储 | 保存到项目或导入当前项目 | 成果固化、设定复用 |

### 1.2 功能入口

在GUI界面中，快捷创作功能位于：

```
工作台 → 快捷创作
```

## 二、快速开始

### 2.1 基本流程

```
输入关键词/参考文本 → 选择生成类型 → 点击生成 → 预览结果 → 保存/导入
```

### 2.2 5分钟上手示例

**场景**：想写一本修仙小说，但没有具体设定

**步骤**：

1. **打开快捷创作**：进入工作台 → 快捷创作

2. **输入关键词**：
   ```
   修仙、逆袭、系统、复仇
   ```

3. **选择生成类型**：
   - 目标：全部生成
   - 详细程度：标准
   - 题材：玄幻

4. **点击生成**，等待约2-3分钟

5. **查看结果**：
   - 世界观：修仙境界、势力分布、规则法则
   - 大纲：50章剧情规划
   - 人物：主角+3个核心配角
   - 情节：开篇、转折、高潮设计

6. **保存结果**：
   - 保存到新项目：创建新项目文件夹
   - 导入当前项目：合并到正在创作的项目中

## 三、生成类型详解

### 3.1 世界观生成

**用途**：构建小说的世界背景设定

**输入参数**：
- 关键词（必填）：如"修仙、仙界、灵气"
- 题材类型（可选）：玄幻、都市、科幻等
- 参考文本（可选）：已有世界观设定或参考小说片段
- 详细程度：快速/标准/详细

**输出内容**：
- 世界名称
- 时代背景
- 地理环境
- 社会结构
- 力量体系（修仙等级、魔法体系等）
- 主要势力
- 规则与法则
- 特殊元素
- 背景故事

**示例**：

输入关键词：`修仙、宗门、灵根、飞升`

输出结果：
```
世界名称：苍澜大陆
时代背景：上古修仙时期
力量体系：练气→筑基→金丹→元婴→化神→渡劫→大乘
主要势力：
  - 青云宗（正道第一宗门）
  - 天魔教（魔道霸主）
  - 灵药谷（炼丹圣地）
规则法则：
  - 灵根决定修炼资质
  - 渡劫成功方可飞升仙界
特殊元素：
  - 天材地宝可提升修为
  - 古修洞府藏有传承
```

### 3.2 大纲生成

**用途**：规划章节内容和剧情走向

**输入参数**：
- 主题（必填）：如"少年修仙成神"
- 世界观摘要（可选）：已生成的世界观简介
- 参考文本（可选）：已有大纲或剧情框架
- 章节数量（可选）：默认10章
- 目标字数（可选）：默认10万字
- 详细程度：快速/标准/详细

**输出内容**：
- 作品标题
- 主题
- 故事梗概
- 章节列表（每章标题+摘要+关键事件）
- 主线情节
- 支线情节
- 高潮点
- 结局方向

**示例**：

输入主题：`少年获得系统，从废物逆袭成仙帝`

输出结果：
```
标题：系统之仙帝崛起
主题：逆袭、成长、复仇

梗概：
主角林凡本是青云宗外门弟子，因天赋低下备受欺凌。
意外激活修仙系统，从此逆袭崛起，最终成为一代仙帝...

章节规划：
第1章：废物弟子
  - 林凡被同门嘲笑
  - 系统激活
第2章：初显锋芒
  - 外门大比
  - 击败天才弟子
第3章：神秘任务
  - 系统发布首个任务
  - 发现古修洞府
...
```

**输入参数**：

- 角色定位（必填）：主角/男主/女主/配角/反派
- 世界观摘要（可选）
- 参考文本（可选）：已有人物设定
- 详细程度：快速/标准/详细

**输出内容**：
- 姓名、年龄、性别
- 外貌描述
- 性格特点
- 背景故事
- 能力技能
- 目标动机
- 弱点缺陷
- 人物关系
- 语言风格
- 成长弧线

**示例**：

输入：姓名=林凡，角色定位=主角

输出结果：
```
姓名：林凡
年龄：16岁（开篇）
性别：男

外貌：
```

### 3.5 全部生成

**用途**：一键生成所有设定，自动处理依赖关系

**生成顺序**：
1. 世界观（其他设定依赖）
2. 大纲（基于世界观）
3. 人物（基于世界观）
4. 情节（基于以上所有）

**输入参数**：
- 关键词（必填）
- 题材类型（可选）
- 章节数量（可选）：默认10章
- 人物数量（可选）：默认3人
- 目标字数（可选）：默认10万字
- 详细程度：快速/标准/详细

**优点**：
- 自动处理依赖关系
- 确保各设定协调一致
- 节省时间，一次完成

## 四、参考文本功能

### 4.1 功能说明

参考文本功能允许用户提供已有文本（小说片段、设定文档等），系统会自动提取关键元素并融合到生成结果中。

### 4.2 支持的文本类型

| 类型 | 说明 | 提取内容 |
|------|------|----------|
| 小说文本 | 已有小说正文 | 风格、人物、世界观元素 |
| 大纲文本 | 剧情大纲文档 | 章节规划、关键事件 |
| 人物设定 | 人物设定文档 | 姓名、性格、能力、关系 |
| 世界观设定 | 世界观文档 | 时代、力量体系、势力 |

### 4.3 使用方法

1. 准备参考文本（可从已有小说复制或上传设定文档）
2. 粘贴到"参考文本"输入框
3. 系统会自动识别文本类型并提取元素
4. 生成时会将提取的元素融合到结果中

### 4.4 示例

**参考文本**：
```
世界观设定：
时代：上古修仙时期
力量体系：修仙境界，从练气到化神
势力：青云宗、天剑门、魔教

人物设定：
姓名：李云
角色定位：主角
性格：坚韧不拔、重情重义
能力：剑道天才、领悟剑意
```

**生成效果**：
- 世界观生成会参考"上古修仙时期"、"练气到化神"等设定
- 人物生成会参考"坚韧不拔"、"剑道天才"等性格和能力
- 确保生成内容与参考文本风格一致

## 五、结果存储功能

### 5.1 保存到新项目

将生成结果保存为独立的项目文件夹：

**项目结构**：
```
新建项目/
├── project.json           # 项目元数据
├── 世界观/
│   └── 世界观设定.json
├── 大纲/
│   └── 大纲设定.json
├── 人物设定/
│   ├── 主角.json
│   ├── 配角1.json
│   └── ...
└── 情节设定/
    ├── 主线情节.json
    └── ...
```

**操作步骤**：
1. 点击"保存到新项目"按钮
2. 输入项目名称
3. 选择保存位置
4. 确认保存

### 5.2 导入到当前项目

将生成结果合并到正在创作的项目中：

**冲突处理策略**：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| 保留原有设定（默认） | 跳过冲突项，保留用户已有设定 | 保护用户工作成果 |
| 覆盖原有设定 | 用新生成的内容替换原有设定 | 需要更新设定 |
| 合并设定 | 智能合并新旧内容 | 扩展已有设定 |
| 重命名新设定 | 新文件添加时间戳后缀 | 保留两份设定 |

**操作步骤**：
1. 确保已打开目标项目
2. 点击"导入到当前项目"按钮
3. 选择冲突处理策略
4. 确认导入

### 5.3 JSON Schema兼容性

生成的设定文件遵循统一的JSON Schema格式，确保与项目其他功能兼容：

- 世界观：`WorldviewSetting` Schema
- 大纲：`OutlineSetting` Schema
- 人物：`CharacterSetting` Schema
- 情节：`PlotSetting` Schema

## 六、详细程度选择

### 6.1 三种详细程度对比

| 维度 | 快速 | 标准 | 详细 |
|------|------|------|------|
| 生成时间 | 30-60秒 | 1-3分钟 | 3-5分钟 |
| Token消耗 | 2000-3000 | 3000-5000 | 5000-8000 |
| 细节丰富度 | 基础框架 | 较完整 | 非常详细 |
| 适用场景 | 快速原型 | 正式创作 | 精细设定 |

### 6.2 选择建议

- **快速**：灵感验证、快速原型、尝试不同方向
- **标准**（推荐）：正式创作、大多数场景
- **详细**：重要设定、需要丰富细节的核心元素

## 七、最佳实践

### 7.1 关键词输入技巧

**推荐做法**：
```
修仙、逆袭、系统、复仇          ✅ 多个关键词用顿号分隔
都市、重生、商业、首富          ✅ 包含题材、主题、核心元素
```

**避免做法**：
```
我想写一个修仙小说              ❌ 描述性语言，关键词不明确
修仙逆袭系统复仇                ❌ 没有分隔符，难以解析
```

### 7.2 参考文本使用建议

1. **提供结构化内容**：如已有的人物设定、世界观文档
2. **包含关键信息**：力量等级、势力名称、人物关系等
3. **保持一致性**：参考文本内容应与创作目标相关

### 7.3 生成结果优化

如果对生成结果不满意：

1. **调整关键词**：尝试不同的关键词组合
2. **提供参考文本**：给出更明确的参考信息
3. **提高详细程度**：使用"详细"模式获得更多内容
4. **单项生成**：先全部生成框架，再针对单项细化

### 7.4 工作流程建议

**新项目启动流程**：
```
1. 全部生成（快速模式） → 验证整体方向
2. 调整关键词/参考文本 → 重新生成不满意的部分
3. 全部生成（标准模式） → 获得完整设定
4. 保存到新项目 → 固化成果
5. 后续创作中随时生成新人物/情节
```

**已有项目扩展流程**：
```
1. 单项生成（人物/情节） → 补充新元素
2. 导入到当前项目 → 合并到现有设定
3. 选择"合并"策略 → 保留原有+新增内容
```

## 八、常见问题

### 8.1 生成失败怎么办？

**可能原因**：
- API调用超时：检查网络连接，稍后重试
- 关键词过少：至少提供2-3个关键词
- 参考文本格式问题：确保文本可读

**解决方案**：
1. 检查网络连接
2. 简化关键词，去掉特殊字符
3. 缩短参考文本长度
4. 尝试"快速"模式减少超时风险

### 8.2 生成结果不满意？

**常见问题及解决**：

| 问题 | 解决方案 |
|------|----------|
| 世界观太普通 | 提供更具体的关键词，如"赛博朋克修仙" |
| 人物不够立体 | 使用"详细"模式，提供参考文本 |
| 大纲逻辑混乱 | 先生成世界观，再生成大纲 |
| 情节缺乏张力 | 明确情节类型（开篇/转折/高潮） |

### 8.3 如何保证一致性？

1. **使用全部生成**：自动处理依赖关系
2. **提供世界观摘要**：生成人物和情节时引用
3. **使用参考文本**：从已有内容中提取元素

### 8.4 导入时发生冲突？

默认策略为"保留原有设定"，保护用户已有工作成果。

如需覆盖：
1. 确认要覆盖的内容
2. 选择"覆盖原有设定"策略
3. 确认导入

## 九、快捷键和操作技巧

### 9.1 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+Enter | 开始生成 |
| Ctrl+S | 保存结果 |
| Ctrl+I | 导入到项目 |
| Esc | 取消生成 |

### 9.2 效率技巧

1. **批量生成**：一次生成多个设定，系统会缓存结果
2. **模板复用**：保存常用的关键词组合
3. **渐进式细化**：先用快速模式确定方向，再用详细模式丰富

## 十、技术细节

### 10.1 插件信息

| 属性 | 值 |
|------|------|
| 插件ID | quick-creator-v1 |
| 版本 | 1.3.0 |
| 类型 | generator |
| 权限 | llm.call, cache.readwrite, file.write |

### 10.2 配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| default_generation_type | standard | 默认生成详细程度 |
| max_retries | 3 | API调用最大重试次数 |
| timeout | 120 | 超时时间（秒） |
| default_conflict_strategy | keep_original | 默认冲突处理策略 |

### 10.3 API调用说明

快捷创作使用LLM（大语言模型）进行生成，每次生成会消耗Token：

| 生成类型 | 预估Token消耗 |
|----------|--------------|
| 世界观（快速） | ~2000 |
| 世界观（标准） | ~3000 |
| 世界观（详细） | ~5000 |
| 全部生成（标准） | ~10000-15000 |
