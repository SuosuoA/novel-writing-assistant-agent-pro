## 1. 概述

逆向反馈分析器（ReverseFeedbackAnalyzer）是一个分析已生成章节与项目设定一致性的插件，用于检测冲突并生成修正建议。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| 人物一致性检测 | 检测角色行为、能力、性格与设定的偏离 |
| 大纲一致性检测 | 检测情节发展与大纲规划的矛盾 |
| 世界观一致性检测 | 检测场景、规则与世界观设定的冲突 |
| LLM深度分析 | 接入大模型进行语义级别的比对 |
| 缓存机制 | 避免重复调用AI，提高性能 |

### 1.2 插件信息

```json
{
  "id": "reverse-feedback-analyzer",
  "name": "逆向反馈分析器",
  "version": "1.0.0",
  "type": "analyzer",
  "priority": 100
}
```

## 3. 核心类设计

### 3.1 ReverseFeedbackAnalyzer

继承自 `AnalyzerPlugin`，实现逆向反馈分析接口。

```python
class ReverseFeedbackAnalyzer(AnalyzerPlugin):
    """逆向反馈分析器"""
    
    @classmethod
    def get_metadata(cls) -> PluginMetadata:
        """获取插件元数据"""
        
    def initialize(self, context: PluginContext) -> bool:
        """初始化插件"""
        
    def analyze_chapter_vs_settings(
        self,
        chapter_text: str,
        current_settings: Dict[str, Any],
        chapter_id: str = "",
    ) -> ConsistencyReport:
        """分析章节与设定的冲突"""
```

### 3.2 AnalysisCache

缓存分析结果，避免重复AI调用。

```python
class AnalysisCache:
    """分析结果缓存"""
    
    def get(self, chapter_text: str, settings: Dict, analysis_type: str) -> Optional[Dict]:
        """获取缓存结果"""
        
    def set(self, chapter_text: str, settings: Dict, result: Dict, ttl: float = None):
        """设置缓存结果"""

        """清空缓存"""
```

## 4. 工作流程

```
┌─────────────────────────────────────────────────────────┐
│                    analyze_chapter_vs_settings          │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 1. 检查缓存（避免重复分析）                               │
│    - 生成缓存键：SHA256(chapter_text + settings)         │
│    - 命中则直接返回缓存结果                               │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 2. 提取章节语义特征（SemanticExtractor）                 │
│    - 人物提取：识别人物、行为、特征                       │
│    - 大纲提取：识别情节元素、事件                         │
│    - 世界观提取：识别场景、规则                           │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 3. 与设定进行对比（LLMAnalyzer）                         │
│    - 人物设定对比 → 人物冲突检测                         │
│    - 大纲设定对比 → 大纲冲突检测                         │
│    - 世界观设定对比 → 世界观冲突检测                     │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 4. LLM深度语义分析（可选）                               │
│    - 调用大模型进行语义级别的一致性分析                   │
│    - 降级策略：LLM不可用时使用规则检测                   │
└─────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────┐
│ 5. 生成分析报告（ConsistencyReport）                    │
│    - 冲突列表：每个冲突包含类型、严重度、建议             │
│    - 保存缓存供后续复用                                  │
└─────────────────────────────────────────────────────────┘
```

## 5. 使用示例

### 5.1 基本使用

```python
from plugins.reverse_feedback_analyzer import ReverseFeedbackAnalyzer

# 创建分析器
analyzer = ReverseFeedbackAnalyzer()

# 初始化（需要PluginContext）
analyzer.initialize(context)

# 分析章节
report = analyzer.analyze_chapter_vs_settings(
    chapter_text="第一章内容...",
    current_settings={
        "outline": {...},
        "characters": {...},
        "worldview": {...}
    },
    chapter_id="chapter_1"
)

# 处理冲突
for issue in report.issues:
    print(f"[{issue.severity.value}] {issue.description}")
    print(f"  建议: {issue.suggested_fix}")
```

### 5.2 设置LLM客户端

```python
from openai import OpenAI

# 创建LLM客户端
client = OpenAI(api_key="...", base_url="https://api.deepseek.com")

# 设置到分析器
analyzer.set_llm_client(client)
```

### 5.3 通过analyze接口调用

```python
# 通用接口调用
result = analyzer.analyze(
    content="章节内容...",
    options={
        "settings": {...},

    }
)
```

## 6. 冲突类型定义

| 冲突类型 | 枚举值 | 说明 |
|----------|--------|------|
| 人物偏离 | `CHARACTER_DEVIATION` | 角色性格、能力、行为与设定不符 |
| 大纲偏离 | `OUTLINE_DEVIATION` | 情节发展与大纲规划矛盾 |
| 世界观偏离 | `WORLDVIEW_DEVIATION` | 场景、规则与世界观数据冲突 |
| 时间线冲突 | `TIMELINE_CONFLICT` | 时间顺序或时序逻辑问题 |
| 逻辑矛盾 | `LOGIC_CONTRADICTION` | 前后逻辑不一致 |

### 严重程度

| 级别 | 说明 |
|------|------|
| `LOW` | 轻微问题，不影响整体理解 |
| `MEDIUM` | 中等问题，建议修正 |
| `HIGH` | 严重问题，必须修正 |

## 7. 缓存机制

### 7.1 缓存策略

- **键生成**: SHA256(analysis_type + chapter_text[:500] + settings_json[:1000])
- **默认TTL**: 3600秒（1小时）
- **最大容量**: 500条
- **淘汰策略**: LRU（最近最少使用）

### 7.2 缓存配置

```json
{
  "config": {
    "cache_ttl_seconds": 3600,
    "max_cache_size": 500,
    "llm_timeout_seconds": 60,
    "max_retries": 3,
    "default_severity_threshold": "medium"
  }
}
```

## 9. LLM降级策略

当LLM不可用时，插件使用规则检测：

1. **人物关键词检测**: 检测角色是否出现，能力是否体现
2. **大纲关键词检测**: 检测大纲关键事件是否在章节中体现
3. **世界观规则检测**: 检测场景设定是否符合世界观
