---

## 1. 功能概述

逆向反馈模块用于分析已生成的小说章节与项目设定的一致性，检测冲突并生成修正建议。支持：
- 章节 upload（单个/批量上传文件或粘贴文本）
- 章节列表管理（显示字数、状态、删除）
- 运行分析按钮（调用逆向反馈分析插件）
- 分析结果展示（冲突列表，带修正建议）
- 应用修正按钮（调用设定修正生成器，更新当前项目设定）

## 3. 核心组件

### 3.1 章节上传区域

**功能按钮**:
- `📁 上传文件`: 选择单个或多个TXT/DOCX文件
- `📁 批量上传`: 选择文件夹，批量上传所有章节文件
- `📋 粘贴文本`: 切换粘贴文本区域显示/隐藏
- `🔄 刷新列表`: 重新统计章节字数并更新状态

**粘贴文本区域**:
- 标题输入框
- 内容文本框
- 添加章节按钮

### 3.2 章节列表

**列定义**:
| 列名 | 宽度 | 说明 |
|------|------|------|
| 序号 | 50 | 自动编号（第N章） |
| 标题 | 180 | 章节标题 |
| 字数 | 70 | 章节字数统计 |
| 状态 | 70 | 未完成/已完成 |
| 来源 | 80 | 文件名/粘贴 |

**右键菜单**:
- ✅ 标记为完成
- ⏳ 标记为未完成
- 👁️ 查看内容
- 🗑️ 删除章节

**操作按钮**:
- 删除选中
- 全部标记完成
- 清空列表

### 3.3 分析配置

**分析维度（复选框）**:
- ✓ 一致性检查（章节间逻辑一致性）
- ✓ 逻辑漏洞检测（情节合理性）
- ✓ 人设偏离检测（角色行为一致性）
- ✓ 风格匹配度（写作风格一致性）
- ✓ 世界观冲突（设定一致性）

**分析范围（单选框）**:
- 仅分析选中章节
- 分析所有已完成章节
- 分析所有章节

### 3.4 分析结果展示

**冲突列表（Treeview）**:
| 列名 | 宽度 | 说明 |
|------|------|------|
| 类型 | 80 | 人物/大纲/世界观 |
| 优先级 | 60 | 🔴高/🟡中/🟢低 |
| 元素 | 100 | 冲突元素名称 |
| 章节 | 100 | 章节引用 |
| 描述 | 200 | 问题描述摘要 |

**修正建议详情（Text）**:
显示选中冲突项的详细信息：
- 冲突类型
- 严重程度
- 涉及元素
- 章节引用
- 问题描述
- 原始设定
- 修正建议
- 置信度

### 3.5 底部操作按钮

| 按钮 | 功能 |
|------|------|
| 🔧 应用修正 | 将修正建议应用到当前项目设定 |
| 📋 导出报告 | 导出分析报告为Markdown文件 |
| 🔄 重新分析 | 重新运行分析 |
| 🗑️ 清除结果 | 清除当前分析结果 |

## 6. 插件集成

### 6.1 获取插件实例

```python
def _get_reverse_feedback_plugin(self):
    # 1. 尝试从PluginRegistry获取
    registry = get_plugin_registry()
    if registry:
        plugin = registry.get_plugin("reverse-feedback-analyzer")
        if plugin:
            return plugin
    
    # 2. 动态创建插件实例
    from plugins.reverse_feedback_analyzer.reverse_feedback_analyzer import ReverseFeedbackAnalyzer
    plugin = ReverseFeedbackAnalyzer()
    
    # 3. 创建上下文并初始化
    context = PluginContext(...)
    plugin.initialize(context)
    return plugin
```

### 6.2 调用分析接口

```python
# 分析单个章节
result = plugin.analyze_chapter_vs_settings(
    chapter_text=chapter['content'],
    current_settings=settings,
    chapter_id=chapter['title']
)

# 生成修正
corrections = plugin.generate_corrections(report, settings)
```

## 7. 线程安全

所有异步操作使用 `threading.Thread` 执行，UI更新通过 `root.after(0, callback)` 调度：

```python
def run_analysis():
    try:
        # 异步执行分析
        result = plugin.analyze_chapter_vs_settings(...)
        
        # 主线程更新UI
        self.root.after(0, lambda: self._update_analysis_result())
    finally:
        self.root.after(0, lambda: self._run_analysis_btn.configure(state=tk.NORMAL))

threading.Thread(target=run_analysis, daemon=True).start()
```

## 8. 项目管理器集成

当应用修正时，自动刷新项目管理器：

```python
def _apply_corrections_to_project(self, corrections: Dict):
    # 更新大纲
    if corrections.get('updated_outline'):
        self._outline_content = corrections['updated_outline']
    
    # 更新人物
    if corrections.get('updated_characters'):
        self._character_data = corrections['updated_characters']
    
    # 刷新项目管理器
    self._refresh_project_manager()
```

## 9. 使用流程

1. **上传章节**
   - 点击"上传文件"选择章节文件
   - 或点击"批量上传"选择文件夹
   - 或点击"粘贴文本"直接粘贴内容

2. **标记章节状态**
   - 右键菜单标记已完成/未完成
   - 或点击"全部标记完成"

3. **配置分析参数**
   - 勾选分析维度
   - 选择分析范围

4. **运行分析**
   - 点击"运行分析"按钮
   - 等待分析完成

5. **查看结果**
   - 在冲突列表中选择问题
   - 查看详细修正建议

6. **应用修正**
   - 点击"应用修正"
   - 确认后更新项目设定

7. **导出报告**（可选）
   - 点击"导出报告"
   - 选择保存位置
