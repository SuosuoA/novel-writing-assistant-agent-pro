## 二、核心设计

### 2.1 模块定位

冲突修复建议生成作为OpenClaw多Agent架构的**修复建议层**：
- 接收ConsistencyCheckerAgent的冲突列表
- 为每个冲突生成多种修复方案（A/B/C）
- 支持用户选择接受/拒绝
- 记录用户决策，优化后续建议

### 2.2 核心类设计

```
ConflictFixer
├── 数据模型（5个Pydantic模型）
│   ├── FixOption: 单个修复方案选项
│   ├── ConflictFix: 冲突修复建议（含多方案）
│   ├── FixGenerationResult: 修复建议生成结果
│   └── FixStats: 修复建议统计
│
├── 主入口方法
│   ├── generate_fixes(): 为冲突列表生成修复建议
│   ├── record_user_decision(): 记录用户决策
│   └── get_acceptance_rate(): 获取用户接受率
│
├── 修复方案生成
│   ├── _generate_character_fix(): 人物冲突修复
│   ├── _generate_plot_fix(): 情节冲突修复
│   └── _generate_worldview_fix(): 世界观冲突修复
│
└── 辅助方法
    ├── _severity_rank(): 严重程度排序
    └── _publish_fix_event(): 发布修复事件
```

### 2.3 修复策略设计

**三种修复策略**：

| 策略 | 特点 | 适用场景 |
|------|------|----------|
| conservative（保守） | 最小修改，影响范围小 | P2冲突、细节问题 |
| balanced（平衡） | 适度修改，需要调整多处 | P1冲突、中等问题 |
| aggressive（激进） | 大幅重写，根本解决 | P0冲突、严重问题 |

**方案分配**：
- 方案A（推荐）: conservative策略
- 方案B（备选）: balanced策略
- 方案C（激进）: aggressive策略（仅P0冲突提供）

## 三、API接口

### 3.1 基本使用

```python
from core import get_conflict_fixer
from agents import get_consistency_checker_agent

# 获取一致性检查Agent
checker = get_consistency_checker_agent(workspace_root=Path("E:/project"))

# 执行一致性检查
check_result = checker.execute(
    chapter_number=3,
    chapter_title="星际战争爆发",
    chapter_content="..."
)

# 获取修复建议生成器
fixer = get_conflict_fixer(workspace_root=Path("E:/project"))

# 为冲突生成修复建议
fix_result = fixer.generate_fixes(
    conflicts=check_result.conflicts,
    chapter_content="...",
    chapter_number=3,
    chapter_title="星际战争爆发",
    genre="scifi"
)

# 查看修复建议
for fix in fix_result.fixes:
    print(f"[{fix.severity}] {fix.description}")
    print(f"方案A: {fix.option_a.title} - {fix.option_a.description}")
    if fix.option_b:
        print(f"方案B: {fix.option_b.title} - {fix.option_b.description}")
```

### 3.2 记录用户决策

```python
# 用户接受方案A
fixer.record_user_decision(
    fix_id="fix_conflict_1",
    decision="accept_a",
    reason="方案A修改最小，符合预期"
)

# 用户拒绝
fixer.record_user_decision(
    fix_id="fix_conflict_2",
    decision="reject",
    reason="该冲突可忽略"
)

# 获取用户接受率
acceptance_rate = fixer.get_acceptance_rate()
print(f"用户接受率: {acceptance_rate:.2%}")
```

### 3.3 获取统计信息

```python
stats = fixer.get_stats()
print(f"总生成次数: {stats.total_generations}")
print(f"总修复建议数: {stats.total_fixes}")
print(f"用户接受率: {stats.acceptance_rate:.2%}")
print(f"人物冲突修复: {stats.character_fixes}")
print(f"情节冲突修复: {stats.plot_fixes}")
print(f"世界观冲突修复: {stats.worldview_fixes}")
```

## 五、架构意义

冲突修复建议生成作为OpenClaw多Agent架构的**核心修复层**：
1. **智能修复**: 为每个冲突提供多种修复方案，用户可选择最适合的
2. **优先级处理**: P0冲突优先处理，提供激进方案
3. **用户主导**: 用户可选择接受、拒绝或延期
4. **学习优化**: 记录用户偏好，计算接受率
5. **EventBus集成**: 与其他Agent协同工作
