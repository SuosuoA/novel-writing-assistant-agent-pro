# 插件加载器与热插拔说明

## 概述

本文档描述插件加载器（PluginLoader）和热插拔管理器（HotSwapManager）的设计与实现，支持插件的动态发现、加载、卸载和热重载。

### 依赖说明

| 依赖包 | 必需性 | 说明 |
|--------|:------:|------|
| `watchdog` | **可选** | 用于热插拔文件监控。如未安装，热插拔功能将优雅降级，其他功能正常工作 |

**降级行为**：
- watchdog未安装时，`start_watch()`返回`False`并记录警告日志
- 插件加载、注册、状态管理等核心功能不受影响
- 安装watchdog后重启应用即可启用热插拔功能

```bash
# 安装watchdog以启用热插拔
pip install watchdog
```

## PluginLoader 实现细节

### 1. 插件发现

支持两种插件发现方式：

#### plugin.json 方式
```
plugins/
└── my-plugin/
    ├── plugin.json    # 元数据文件
    └── __init__.py    # 入口模块
```

#### 直接模块方式
```
plugins/
└── simple-plugin/
    └── __init__.py    # 包含 BasePlugin 子类
```

### 2. 核心API

```python
from core.plugin_loader import PluginLoader, get_plugin_loader

# 获取单例实例
loader = get_plugin_loader()

# 发现插件
discovered = loader.discover_plugins()
# 返回: {"plugin-id": PluginMetadata, ...}

# 加载插件
result = loader.load_plugin("my-plugin")
# 返回: PluginLoadResult(success, plugin_id, instance, error)

# 卸载插件（需权限检查）
success = loader.unload_plugin("my-plugin")

# 重载插件
result = loader.reload_plugin("my-plugin")
```

### 3. 依赖解析

```python
from core.plugin_loader import DependencyResolver

# 解析加载顺序
plugins = [metadata1, metadata2, metadata3]
sorted_ids, errors = DependencyResolver.resolve(plugins)

# 检测冲突
conflicts = DependencyResolver.check_conflicts(plugins)
```

依赖解析算法：
- 拓扑排序确定加载顺序
- 检测循环依赖
- 检测缺失依赖
- 检测插件冲突

## HotSwapManager 实现细节

### 1. 文件监控

使用 watchdog 库监控插件目录变更：

```python
from core.hot_swap_manager import HotSwapManager, get_hot_swap_manager

# 获取单例实例
manager = get_hot_swap_manager()

# 启动监控
manager.start_watch(["plugins/", "extensions/"])

# 停止监控
manager.stop_watch()

# 检查状态
if manager.is_enabled():
    print("热插拔监控已启用")
```

### 2. 防抖机制

防止频繁重载：
- 默认防抖延迟：1秒
- 状态机管理：IDLE → PENDING → PROCESSING → COMPLETED
- 支持取消待处理操作

```python
# 取消待处理的重载
manager.cancel_pending("my-plugin")

# 获取插件状态
state = manager.get_plugin_state("my-plugin")
```

### 3. 状态管理

```python
from core.hot_swap_manager import HotSwapState

class HotSwapState(Enum):
    IDLE = "idle"           # 空闲
    PENDING = "pending"     # 待处理（防抖中）
    PROCESSING = "processing"  # 处理中
    COMPLETED = "completed"    # 完成
    FAILED = "failed"          # 失败
```

## 权限分级系统

### 安全等级定义

| 等级 | 类型 | 加载 | 重载 | 卸载 | 进程隔离 | 说明 |
|------|------|------|------|------|----------|------|
| L0 | 官方插件 | ✅ | ✅ | ✅ | ❌ | 完全信任 |
| L1 | 第三方签名 | ✅ | ✅ | ❌ | ❌ | 有限信任 |
| L2 | 未知来源 | ✅ | ❌ | ❌ | ✅ | 需隔离 |
| L3 | 风险插件 | ❌ | ❌ | ❌ | ❌ | 禁止加载 |

### HotSwapPermission 类

```python
from core.plugin_loader import HotSwapPermission

permission = HotSwapPermission(
    plugin_id="my-plugin",
    signature_verified=True,
    is_official=False,

if permission.can_load():
    # 加载插件
    
if permission.can_reload():
    # 重载插件
```

## PluginRegistry 运行时接口

### 运行时加载/卸载

```python
from core.plugin_registry import get_plugin_registry

registry = get_plugin_registry()

# 运行时加载
success, error = registry.load_plugin_runtime("my-plugin", loader)

# 运行时卸载
success, error = registry.unload_plugin_runtime("my-plugin", loader)

# 运行时重载
success, error = registry.reload_plugin_runtime("my-plugin", loader)
```

### 状态转换

```
UNLOADED → LOADED → ACTIVE → UNLOADING → UNLOADED
              ↓         ↓
           ERROR ←───────
```

```python
from core.plugin_interface import PluginState

# 状态检查
state = registry.get_state("my-plugin")
# 返回: "unloaded" | "loaded" | "active" | "error" | "unloading"

# 状态变更
registry.activate("my-plugin")    # LOADED → ACTIVE
registry.deactivate("my-plugin")  # ACTIVE → LOADED
registry.set_error("my-plugin", "错误信息")  # → ERROR
registry.reset_error("my-plugin") # ERROR → LOADED
```

## 事件发布

插件加载/卸载事件通过 EventBus 发布：

```python
from core.event_bus import EventBus

event_bus = EventBus()

# 订阅插件事件
event_bus.subscribe("plugin.loaded", on_plugin_loaded)
event_bus.subscribe("plugin.unloaded", on_plugin_unloaded)
event_bus.subscribe("plugin.reloaded", on_plugin_reloaded)
event_bus.subscribe("plugin.error", on_plugin_error)
```

事件数据格式：
```python
{
    "plugin_id": "my-plugin",
    "timestamp": "2026-03-21T18:00:00Z",
    "action": "load",  # load | unload | reload
    "success": True,
    "error": None,  # 失败时的错误信息
}
```

## 使用示例

### 完整工作流

```python
from core.plugin_loader import get_plugin_loader
from core.hot_swap_manager import get_hot_swap_manager
from core.plugin_registry import get_plugin_registry

# 1. 获取实例
loader = get_plugin_loader()
manager = get_hot_swap_manager()
registry = get_plugin_registry()

# 2. 发现插件
discovered = loader.discover_plugins()
print(f"发现 {len(discovered)} 个插件")

# 3. 启动热插拔监控
manager.start_watch(["plugins/"])

# 4. 加载并激活插件
for plugin_id in discovered:
    result = loader.load_plugin(plugin_id)
    if result.success:
        registry.activate(plugin_id)
        print(f"插件 {plugin_id} 已激活")

# 5. 运行时重载（如果插件文件变更）
# HotSwapManager 会自动检测并触发重载

# 6. 停止监控
manager.stop_watch()
```

## 注意事项

1. **线程安全**：所有操作都有线程锁保护
2. **V5保护**：核心模块不可热重载/卸载
3. **权限检查**：加载/卸载前会验证权限级别
4. **防抖机制**：避免频繁重载导致资源浪费
5. **事件通知**：状态变更会发布事件供其他组件订阅

## P2级优化实现（2026-03-21）

### 1. 签名验证逻辑

新增 `PluginSignatureVerifier` 类，实现插件完整性验证：

```python
from core.plugin_loader import PluginSignatureVerifier

# 验证插件签名
is_valid, error = PluginSignatureVerifier.verify_plugin_signature(
    plugin_path=Path("plugins/my-plugin"),
    plugin_id="my-plugin",
    plugin_json={"signature": "sha256:abc123..."}
)

if is_valid:
    print("插件签名验证通过")
else:
    print(f"验证失败: {error}")
```

验证规则：
- 检查 `plugin.json` 中的 `signature` 字段
- 支持 `sha256:<hash>` 格式的签名
- 计算插件目录所有文件的 SHA256 哈希
- 与签名进行比对验证完整性

### 2. watchdog 依赖检测与优雅降级

`HotSwapManager` 支持在 watchdog 未安装时优雅降级：

```python
# hot_swap_manager.py
_WATCHDOG_AVAILABLE = False
try:
    from watchdog.observers import Observer
    from watchdog.events import FileSystemEventHandler
    _WATCHDOG_AVAILABLE = True
except ImportError:
    Observer = None
    FileSystemEventHandler = None

def is_watchdog_available() -> bool:
    """检查 watchdog 是否可用"""
    return _WATCHDOG_AVAILABLE

def start_watch(self, directories: List[str]) -> bool:
    if not _WATCHDOG_AVAILABLE:
        logger.warning("watchdog 未安装，热插拔监控已禁用")
        return False
    # ... 正常启动监控
```

降级行为：
- watchdog 未安装时 `start_watch()` 返回 `False`
- 记录警告日志而非抛出异常
- 其他功能（状态管理、事件发布）继续正常工作

### 3. 统一结构化日志器

使用 `infrastructure/logger.py` 的结构化日志器：

```python
from infrastructure.logger import get_logger

logger = get_logger("plugin_loader")

# 结构化日志记录
logger.info(
    "plugin_loaded",
    plugin_id="my-plugin",
    load_time=1.23,
    dependencies=["dep1", "dep2"]
)

# 敏感信息自动脱敏
logger.info(
    "api_key_configured",
    api_key="sk-abc123..."  # 自动脱敏为 sk-***
)
```

日志特性：
- JSON 格式输出，便于解析和分析
- 敏感信息（API密钥、密码、Token）自动脱敏
- 支持日志轮转和大小限制
- 兼容标准 logging 模块
