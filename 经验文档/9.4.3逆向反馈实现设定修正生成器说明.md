## 1. 概述

设定修正生成器（SettingsCorrector）是逆向反馈分析插件的核心模块，用于根据检测到的冲突自动生成修正后的设定。

### 1.1 核心功能

| 功能 | 说明 |
|------|------|
| 单个冲突修正 | 调用LLM生成针对单个冲突的修正方案 |
| 批量修正 | 一次处理多个冲突，按类型分组优化 |
| 结构化输出 | 输出修正后的大纲、人物设定、世界观 |
| 备份机制 | 保留原始设定备份，支持回滚 |

### 1.2 模块信息

| 属性 | 值 |
|------|-----|
| 文件位置 | `plugins/reverse-feedback-analyzer/settings_corrector.py` |
| 代码行数 | ~500行 |
| 主入口函数 | `settings_corrector()` |

## 2. 数据模型

### 2.1 CorrectionResult（修正结果）

```python
@dataclass
class CorrectionResult:
    issue_id: str                    # 对应的冲突ID
    issue_type: ConsistencyIssueType # 冲突类型
    element_name: str                # 修正的元素名称
    original_setting: str            # 原始设定
    corrected_setting: str           # 修正后的设定
    correction_reason: str           # 修正原因
    confidence: float                # 修正置信度
```

### 2.2 CorrectionReport（修正报告）

```python
@dataclass
class CorrectionReport:
    report_id: str
    project_name: str
    corrections: List[CorrectionResult]
    
    # 修正后的结构化数据
    updated_outline: str
    updated_characters: List[Dict[str, Any]]
    updated_worldview: str
    
    # 元数据
    total_issues: int
    corrected_count: int
    failed_count: int
    backup: Optional[Dict[str, Any]]
```

## 5. 使用示例

### 5.1 修正单个冲突

```python
from plugins.reverse_feedback_analyzer import correct_single_issue
from core.plugin_interface import ConsistencyIssue, ConsistencyIssueType, ConsistencySeverity

# 创建冲突
issue = ConsistencyIssue(
    issue_id="issue-001",
    issue_type=ConsistencyIssueType.CHARACTER,
    severity=ConsistencySeverity.HIGH,
    description="角色'张三'在章节中表现出外向性格，但设定为内向",
    suggested_fix="将性格修改为外向",
    element_name="张三",
    original_content="性格：内向",
)

# 原始设定
original_settings = {
    "outline": "第一章 开始",
    "characters": [{"name": "张三", "personality": "内向"}],
    "worldview": "现代都市",
}

# 执行修正
result = correct_single_issue(issue, original_settings)

print(f"修正后设定: {result.corrected_setting}")
print(f"置信度: {result.confidence}")
```

### 5.2 批量修正冲突

```python
from plugins.reverse_feedback_analyzer import settings_corrector

# 冲突列表
issues = [
    ConsistencyIssue(...),  # 人物冲突
    ConsistencyIssue(...),  # 大纲冲突
    ConsistencyIssue(...),  # 世界观冲突
]

# 原始设定
original_settings = {
    "project_name": "修仙传奇",
    "outline": "第一章 觉醒\n第二章 入门",
    "characters": [
        {"name": "林逸", "personality": "坚毅", "ability": "基础剑法"}
    ],
    "worldview": "修仙世界...",
}

# 执行修正
report = settings_corrector(
    issues, 
    original_settings,
    options={
        "auto_fix_low": True,
        "preserve_original": True,
        "batch_mode": True,
    }
)

# 查看结果
print(f"修正数量: {report.corrected_count}")
print(f"更新后大纲: {report.updated_outline}")
print(f"更新后人物: {report.updated_characters}")
```

## 6. 修正选项

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `auto_fix_low` | bool | True | 是否自动修正低优先级冲突 |
| `preserve_original` | bool | True | 是否保留原始设定备份 |
| `batch_mode` | bool | False | 是否批量处理（分组优化） |

## 8. 降级策略

当LLM不可用时，使用规则修正：

| 冲突类型 | 修正规则 |
|----------|----------|
| 人物冲突 | 使用`suggested_fix`作为修正内容 |
| 大纲冲突 | 添加`[修正]`前缀 + 建议内容 |
| 世界观冲突 | 添加`规则更新：`前缀 + 建议内容 |

规则修正置信度固定为 `0.6`。
