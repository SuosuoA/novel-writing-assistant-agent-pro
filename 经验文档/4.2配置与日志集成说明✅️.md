## 二、实现内容

### 2.1 ConfigService（配置服务）

**文件位置**: `core/config_service.py`

**版本**: V1.1（评审后修订版）

**核心功能**:
- 封装 ConfigManager，提供配置读取服务
- 支持 Pydantic 模型验证（AppConfig）
- **新增**: 字段验证器（API密钥长度/格式、温度范围、主题枚举、窗口大小格式）
- 线程安全实现（RLock）
- 通过 ServiceLocator 注册和获取

**AppConfig 配置模型（含验证器）**:
```python
class AppConfig(BaseModel):
    ai_learning: bool = Field(default=True)
    api_key: str = Field(default="")
    auto_save: bool = Field(default=True)
    backup_interval: str = Field(default="30")
    font_size: str = Field(default="19")
    local_url: str = Field(default="http://localhost:11434/v1")
    model: str = Field(default="deepseek-chat")
    provider: str = Field(default="DeepSeek")
    save_path: str = Field(default="")
    service_mode: str = Field(default="local")
    temperature: float = Field(default=0.7, ge=0.0, le=2.0)  # 新增范围验证
    theme: str = Field(default="dark")
    window_size: str = Field(default="1280x720")

    @field_validator("api_key")
    @classmethod
    def validate_api_key(cls, v: str) -> str:
        """验证API密钥格式：至少16字符，只允许字母数字下划线连字符"""

    @field_validator("theme")
    @classmethod
    def validate_theme(cls, v: str) -> str:
        """验证主题：必须是 dark 或 light"""

    @field_validator("service_mode")
    @classmethod
    def validate_service_mode(cls, v: str) -> str:
        """验证服务模式：必须是 local 或 remote"""

    @field_validator("window_size")
    @classmethod
    def validate_window_size(cls, v: str) -> str:
        """验证窗口大小：格式如 1280x720，范围 800x600 到 3840x2160"""
```

### 2.2 LoggingService（日志服务）

**文件位置**: `core/logging_service.py`

**版本**: V1.1（评审后修订版）

**核心功能**:
- 按日期滚动的日志文件（每天午夜滚动）
- 保留30天的日志
- 结构化日志格式（带事件类型标记）
- 记录关键事件（插件、AI请求、生成、配置、系统）
- **新增**: 集成 log_sanitizer 模块实现敏感信息脱敏
- **新增**: 动态日志级别调整（set_level/get_level）
- 线程安全实现

**脱敏示例**:
```
原始日志: [AI:DeepSeek/deepseek-chat] API Key: sk-test-api-key-12345
脱敏后: [AI:DeepSeek/deepseek-chat] API Key: ***REDACTED***
```

**新增API**:
```python
# 动态调整日志级别
logging_service.set_level("DEBUG")
current_level = logging_service.get_level()  # 返回 "DEBUG"
```

### 2.4 GUI主程序集成

**文件位置**: `gui_main.py`

**修改内容**:
1. 导入新服务：
```python
from core import (
    # ... 原有导入 ...
    initialize_core_services,
    dispose_core_services,
    get_config_service,
    get_logging_service,
)
```

2. 在 main() 函数中集成：
```python
def main():
    # 初始化核心服务（配置和日志）
    if CORE_AVAILABLE:
        try:
            init_results = initialize_core_services()
            if init_results.get("ConfigService") and init_results.get("LoggingService"):
                logging_service = get_logging_service()
                logging_service.log_system_event("startup", "应用程序启动 - 核心服务初始化完成")
        except Exception as e:
            logger.error(f"Failed to initialize core services: {e}")

    try:
        app = MainWindow()
        app.run()
    except Exception as e:
        logger.error(f"Application error: {e}")
        raise
    finally:
        # 释放核心服务
        if CORE_AVAILABLE:
            try:
                dispose_core_services()
            except Exception as e:
                logger.error(f"Failed to dispose core services: {e}")
```
