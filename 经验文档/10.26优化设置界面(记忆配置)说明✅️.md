## 二、实现内容

### 2.1 新增记忆配置区域

在设置页面的"长期记忆管理"模块之后，新增"记忆配置"区域，包含以下配置项：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| WAL自动保存 | Boolean | True | AI调用前自动保存会话状态 |
| 会话自动恢复 | Boolean | True | 程序启动时自动恢复上次会话 |
| 向量召回数量 | Int | 10 | 上下文召回时返回的相似章节数 |
| 每日冥想时间 | String | 23:00 | 自动提炼当日重要事件到MEMORY.md |
| Embedding模型 | Choice | local | 向量编码模型（local/openai/deepseek） |

### 2.2 代码修改

#### gui_main.py

**1. 新增配置变量**

```python
# 记忆配置（Sprint 9-10）
self._wal_auto_save_var = tk.BooleanVar(value=True)
self._auto_recovery_var = tk.BooleanVar(value=True)
self._vector_recall_topk_var = tk.IntVar(value=10)
self._daily_meditation_time_var = tk.StringVar(value="23:00")
self._embedding_model_var = tk.StringVar(value="local")
```

**2. 新增UI组件**

- WAL自动保存开关
- 会话自动恢复开关
- 向量召回数量SpinBox
- 每日冥想时间Entry
- Embedding模型ComboBox

**3. 更新`_on_save_settings()`方法**

在保存配置时，添加记忆配置到config.yaml：

```python
config_data = {
    # ... 其他配置 ...
    # 记忆配置（Sprint 9-10）
    "memory": {
        "wal_auto_save": self._wal_auto_save_var.get(),
        "auto_recovery": self._auto_recovery_var.get(),
        "vector_recall_topk": self._vector_recall_topk_var.get(),
        "daily_meditation_time": self._daily_meditation_time_var.get(),
        "embedding_model": self._embedding_model_var.get()
    }
}
```

**4. 更新`_load_settings_from_config()`方法**

在加载配置时，读取记忆配置并设置到UI：

```python
# 加载记忆配置（Sprint 9-10）
memory_config = config_data.get("memory", {})
if hasattr(self, '_wal_auto_save_var'):
    self._wal_auto_save_var.set(memory_config.get("wal_auto_save", True))
# ... 其他配置项 ...
```

#### config.yaml.example

新增记忆配置示例：

```yaml
# ========================================
# 记忆配置（OpenClaw mem9）
# ========================================
memory:
  wal_auto_save: true          # WAL自动保存
  auto_recovery: true          # 会话自动恢复
  vector_recall_topk: 10       # 向量召回数量
  daily_meditation_time: '23:00'  # 每日冥想时间
  embedding_model: local       # Embedding模型
```
