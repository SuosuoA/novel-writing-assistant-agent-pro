## 三、关键技术实现

### 3.1 章节向量编码

**核心方法**: `encode_chapter()`

**编码流程**:
```
用户完成章节创作
    ↓
提取章节文本
    ↓
调用NovelVectorStore.add_chapter()
    ↓
使用OpenAI Embedding API生成向量
    ↓
存储到LanceDB（chapter_id + content + vector + metadata）
    ↓
更新编码历史记录
    ↓
发布EventBus事件
```

**参数验证**:
- chapter_id不能为空
- content不能为空

**更新机制**:
- 检查章节是否已存在
- 已存在时更新向量（重新编码）

### 3.2 批量编码

**核心方法**: `encode_chapters_batch()`

**功能**:
- 一次性编码多个章节
- 减少API调用次数
- 统计成功/失败数量

### 3.3 增量更新

**核心方法**: `incremental_encode()`

**增量策略**:
1. 扫描章节目录（*.md / *.txt）
2. 检查编码历史记录
3. 对比文件修改时间
4. 仅编码新增/修改章节
5. 跳过未修改章节

**元数据提取**:
- 从文件名提取章节ID
- 从内容开头提取标题（如：# 第一章）
- 统计字数

### 3.4 召回相似章节

**核心方法**: `recall_similar_chapters()`

**功能**:
- 输入查询文本
- 召回top-k相似章节
- 用于长篇连贯性验证

### 3.5 编码历史记录

**存储位置**: `data/chapter_encoding_history.json`

**记录内容**:
```json
{
  "chapter-001": {
    "chapter_id": "chapter-001",
    "content_length": 5000,
    "metadata": {"title": "开篇", "word_count": 5000},
    "encoding_time_ms": 120.5,
    "vector_dimension": 1536,
    "timestamp": "2026-03-25T22:30:00"
  }
}
```

**用途**:
- 增量更新判断依据
- 避免重复编码
- 记录编码耗时

### 3.6 线程安全设计

**多线程保护**:
- `_encoding_lock`（RLock）：保护编码操作
- `_stats_lock`（RLock）：保护统计信息
- `_history_lock`（RLock）：保护历史记录

**单例模式**:
- `get_chapter_encoder()`: 获取全局单例
- `reset_chapter_encoder()`: 重置单例（测试用）

### 3.7 EventBus集成

**事件类型**:
| 事件类型 | 说明 |
|----------|------|
| `chapter.encoding.completed` | 章节编码成功 |
| `chapter.encoding.failed` | 章节编码失败 |

**事件数据**:
```json
{
  "chapter_id": "chapter-001",
  "content_length": 5000,
  "encoding_time_ms": 120.5,
  "vector_dimension": 1536
}
```

## 四、API接口

### 4.1 创建编码器实例

```python
from core import get_chapter_encoder
from pathlib import Path

# 获取编码器实例
encoder = get_chapter_encoder(
    workspace_root=Path("E:/project"),
    embedding_type="openai",
    embedding_model="text-embedding-3-small",
    api_key="sk-..."
)
```

### 4.2 编码单个章节

```python
# 编码单个章节
result = encoder.encode_chapter(
    chapter_id="chapter-001",
    content="第一章的内容...",
    metadata={"title": "开篇", "word_count": 3000}
)

print(f"编码成功: {result.success}")
print(f"向量维度: {result.vector_dimension}")
print(f"编码耗时: {result.encoding_time_ms}ms")
```

### 4.3 批量编码

```python
# 批量编码
results = encoder.encode_chapters_batch([
    {"chapter_id": "chapter-001", "content": "...", "metadata": {...}},
    {"chapter_id": "chapter-002", "content": "...", "metadata": {...}}
])

print(f"总数: {results.total_chapters}")
print(f"成功: {results.successful}")
print(f"失败: {results.failed}")
```

### 4.4 增量更新

```python
# 增量编码（检测新增/修改章节）
result = encoder.incremental_encode(
    chapters_dir=Path("大纲"),
    force=False  # 不强制重新编码
)

print(f"新增/修改章节: {result.total_chapters}")
```

### 4.5 召回相似章节

```python
# 召回相似章节
similar_chapters = encoder.recall_similar_chapters(
    query="星际飞船穿越虫洞",
    top_k=10,
    min_score=0.5
)

for chapter in similar_chapters:
    print(f"章节ID: {chapter['chapter_id']}")
    print(f"相似度: {chapter['score']}")
```

### 4.6 查询章节向量

```python
# 查询章节向量
chapter = encoder.get_chapter_vector("chapter-001")
if chapter:
    print(f"章节ID: {chapter['chapter_id']}")
    print(f"内容长度: {chapter['content_length']}")
```

### 4.7 删除章节向量

```python
# 删除章节向量
success = encoder.delete_chapter("chapter-001")
print(f"删除成功: {success}")
```

### 4.8 获取统计信息

```python
# 获取统计信息
stats = encoder.get_stats()

print(f"已编码总数: {stats.total_encoded}")
print(f"成功数: {stats.successful}")
print(f"失败数: {stats.failed}")
print(f"平均耗时: {stats.average_encoding_time_ms}ms")
```

## 五、与VectorStore的协同

### 5.1 VectorStore已有的章节向量支持

`infrastructure/vector_store.py`已实现：

- `add_chapter()`: 添加章节向量
- `add_chapters_batch()`: 批量添加章节向量
- `recall_similar_chapters()`: 召回相似章节

### 5.2 ChapterEncoder的增强功能

ChapterEncoder在VectorStore基础上提供：

| 功能 | VectorStore | ChapterEncoder |
|------|-------------|----------------|
| 添加章节 | add_chapter() | encode_chapter() + 编码结果封装 |
| 批量添加 | add_chapters_batch() | encode_chapters_batch() + 统计 |
| 召回章节 | recall_similar_chapters() | recall_similar_chapters() |
| 增量更新 | 无 | incremental_encode() |
| 编码历史 | 无 | 历史记录 + 持久化 |
| 统计信息 | 无 | EncodingStats |
| EventBus | 无 | 事件发布 |
| 元数据提取 | 无 | 自动提取标题/字数 |

## 七、架构意义

章节向量编码作为OpenClaw mem9的**L2温记忆层**核心功能：

1. **长篇连贯性**: 通过向量召回相似章节，检测情节冲突
2. **增量更新**: 新章节追加，不重建索引，高效处理
3. **语义检索**: 基于向量相似度召回，比关键词更精准
4. **上下文构建**: 为AI生成提供前文相关章节摘要
5. **EventBus集成**: 编码事件可被监控系统感知

---

## 八、性能特性

- **向量维度**: OpenAI 1536/3072维，本地模型384维
- **编码延迟**: 约100-200ms（OpenAI API）

- **增量更新**: 仅编码新增/修改章节
- **历史记录**: JSON文件持久化，轻量级
