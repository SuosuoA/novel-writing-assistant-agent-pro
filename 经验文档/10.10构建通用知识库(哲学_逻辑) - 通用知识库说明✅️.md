## 四、Schema规范

### 4.1 知识点数据结构

```json
{
  "knowledge_id": "general-philosophy-001",
  "category": "general",
  "domain": "philosophy",
  "title": "电车难题",
  "content": "电车难题是伦理学中最著名的思想实验之一...",
  "keywords": ["电车难题", "伦理学", "道德困境"],
  "difficulty": "intermediate",
  "tags": ["哲学", "伦理"],
  "references": [],
  "metadata": {
    "source": "expert",
    "confidence": 0.9,
    "language": "zh"
  },
  "created_at": "2026-03-25T20:24:00",
  "updated_at": "2026-03-25T20:24:00"
}
```

### 4.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| knowledge_id | string | ✅ | 知识点唯一标识，格式：`{category}-{domain}-{序号}` |
| category | string | ✅ | 知识库分类，固定为`general` |
| domain | string | ✅ | 知识领域：`philosophy`或`logic` |
| title | string | ✅ | 知识点标题，1-50字符 |
| content | string | ✅ | 知识点内容，10-2000字符 |
| keywords | array | ✅ | 关键词列表，3-10个 |
| difficulty | string | ❌ | 难度等级：`basic`/`intermediate`/`advanced` |
| tags | array | ❌ | 标签列表 |
| references | array | ❌ | 参考文献列表 |
| metadata | object | ❌ | 元数据（来源、置信度、语言等） |
| created_at | string | ✅ | 创建时间（ISO 8601格式） |
| updated_at | string | ✅ | 更新时间（ISO 8601格式） |

## 六、向量嵌入流程

### 6.1 向量化流程

```python
from infrastructure.vector_store import EmbeddingFunction

# 1. 初始化嵌入函数
embed_func = EmbeddingFunction(
    model_type="openai",  # 或 "local"
    model_name="text-embedding-3-small"
)

# 2. 批量生成向量
knowledge_points = load_json("data/knowledge/general/philosophy.json")
for kp in knowledge_points:
    embedding = embed_func.embed(kp["content"])
    kp["embedding"] = embedding

# 3. 存储到LanceDB
vector_store = VectorStore()
vector_store.add_knowledge_vectors(knowledge_points)
```

### 6.2 环境配置

**配置OPENAI_API_KEY（必需）**:
```bash
# Windows
set OPENAI_API_KEY=sk-your-api-key

# Linux/macOS
export OPENAI_API_KEY=sk-your-api-key
```

**运行向量嵌入**:
