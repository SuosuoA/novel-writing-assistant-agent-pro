## 三、关键技术实现

### 3.1 多Agent协作设计（参考OpenClaw）

**设计理念**:
- 单一职责：ConsistencyCheckerAgent专注于一致性检查
- 协作模式：调用ContextRecaller召回上下文，调用LLM客户端检测冲突
- 事件驱动：通过EventBus发布检查事件
- 延迟加载：依赖模块按需加载，避免循环导入

**协作流程**:
```
用户提交新章节
    ↓
ConsistencyCheckerAgent.execute()
    ↓
调用ContextRecaller.recall_for_new_chapter() → 召回相关上下文
    ↓
构建检查prompt（新章节 + 召回上下文）
    ↓
调用LLM客户端 → 检测冲突
    ↓
解析LLM响应 → 提取冲突列表和修复建议
    ↓
计算一致性准确率
    ↓
发布EventBus事件
    ↓
返回检查结果
```

### 3.2 冲突检测逻辑

**冲突类型**（3种）:
1. **character**: 人物行为矛盾
   - 性格突变
   - 前后行为不一致
   - 能力前后矛盾

2. **plot**: 情节冲突
   - 时间线错乱
   - 事件前后矛盾
   - 因果关系混乱

3. **worldview**: 世界观冲突
   - 设定前后不一致
   - 规则违反
   - 逻辑漏洞

**严重程度**（3级）:
- **P0**: 严重冲突，必须修复（如时间倒流违反因果律）
- **P1**: 中等冲突，建议修复（如人物性格突变）
- **P2**: 轻微冲突，可选修复（如细节不一致）

**准确率计算**:
```python
score = 1.0 - (p0_count * 0.3 + p1_count * 0.1 + p2_count * 0.05)
```

### 3.3 LLM检查Prompt设计

**Prompt结构**:
```
你是一位专业的小说编辑，负责检测长篇小说中的逻辑冲突。

## 参考上下文（前文相关章节摘要）
{recalled_context}

## 新章节内容
{new_chapter}

## 检查任务
请检测新章节与前文是否存在以下冲突：
1. 人物行为矛盾
2. 情节冲突
3. 世界观冲突

## 输出格式
[冲突类型]|[严重程度]|[冲突描述]|[冲突位置]|[证据]|[修复建议]

如果没有冲突，输出：无冲突
```

**输出格式示例**:
```
character|P1|主角性格突变，从冷静变得冲动|第三章第5段|第一章提到"他总是冷静分析局势"|建议增加心理转变的铺垫情节
plot|P0|时间线错误，事件顺序矛盾|第三章第10段|第一章说"三天后"，第二章说"两天后"|调整时间线，修正为"两天后"
worldview|P2|设定细节不一致|第三章第15段|第一章说"魔法需要消耗魔力"，本章未提及|建议补充魔力消耗的描述
```

### 3.4 延迟加载依赖

**依赖模块**:
- ContextRecaller: `core/context_recall.py`
- EventBus: `core/event_bus.py`
- LLM客户端: `services/llm_client_with_resilience.py`

**延迟加载优点**:
- 避免循环导入
- 按需加载，减少启动时间
- 依赖缺失时优雅降级

### 3.5 线程安全设计

**多线程保护**:
- `_lock`（RLock）：保护执行操作
- `_stats_lock`（RLock）：保护统计信息
- `_agent_lock`（RLock）：单例模式双重检查锁

**单例模式**:
- `get_consistency_checker_agent()`: 获取全局单例
- `reset_consistency_checker_agent()`: 重置单例（测试用）

### 3.6 EventBus集成

**事件类型**:

## 四、API接口

### 4.1 基本使用

```python
from agents import get_consistency_checker_agent

# 获取Agent实例
agent = get_consistency_checker_agent(workspace_root=Path("E:/project"))

# 执行一致性检查
result = agent.execute({
    "new_chapter": "第三章 星际战争爆发...",
    "existing_chapters": [...],  # 可选，如果为空则从上下文召回
    "genre": "scifi",  # 可选，默认自动识别
    "top_k": 10,  # 可选，默认10
    "max_tokens": 4000  # 可选，默认4000
})

# 获取检查结果
check_id = result["check_id"]
is_consistent = result["is_consistent"]
conflicts = result["conflicts"]
suggestions = result["suggestions"]
accuracy = result["accuracy"]
```

### 4.2 检查结果格式

```python
{
    "check_id": "check-1234567890",
    "is_consistent": False,
    "conflicts": [
        {
            "conflict_id": "conflict-1234567890-0",
            "conflict_type": "character",
            "severity": "P1",
            "description": "主角性格突变，从冷静变得冲动",
            "location": "第三章第5段",
            "evidence": "第一章提到\"他总是冷静分析局势\"",
            "suggestion": "建议增加心理转变的铺垫情节"
        }
    ],
    "suggestions": [
        "建议增加心理转变的铺垫情节"
    ],
    "recalled_context": "【第一章】故事开始...",
    "accuracy": 0.9,
    "check_duration_ms": 1500.0
}
```


## 五、与相关模块的协同

### 5.1 与ContextRecaller的协同

- ContextRecaller提供上下文召回功能
- ConsistencyCheckerAgent调用recall_for_new_chapter()召回相关章节
- 召回的上下文用于构建检查prompt

### 5.2 与LLM客户端的协同

- LLM客户端提供AI能力
- ConsistencyCheckerAgent调用chat_completion()检测冲突
- 使用低temperature（0.3）确保确定性输出

### 5.3 与EventBus的协同

- EventBus提供事件发布和订阅功能
- ConsistencyCheckerAgent发布consistency.check.completed事件
- 事件可被监控系统订阅，用于感知检查状态

- MasterAgent可调度ConsistencyCheckerAgent执行检查任务
- ConsistencyCheckerAgent可作为流水线中的一个阶段
- 与其他Agent协作形成完整的创作流程

## 七、架构意义

一致性检查Agent作为OpenClaw多Agent架构的**专家Agent**：

1. **长篇连贯性保障**: 检测长篇小说中的逻辑冲突
2. **多Agent协作**: 调用ContextRecaller、LLM客户端，形成协作流程
3. **智能冲突检测**: 基于LLM的语义理解，检测人物/情节/世界观冲突
4. **修复建议生成**: 为每个冲突提供具体的修复建议
5. **事件驱动架构**: 通过EventBus发布检查事件，支持监控和扩展
