## 一、优化背景

根据《升级方案 10.升级方案✅️.md》Sprint 9-10 第5条任务：
> 优化长篇检测UI反馈
> 验收: 反馈实时性测试通过

原有长篇检测面板存在以下问题：
1. **进度反馈缺失**：检测过程中没有实时进度指示
2. **阶段状态不明确**：用户不知道当前在做什么（召回上下文/LLM检测/解析结果）
3. **耗时显示缺失**：用户不知道检测需要多长时间
4. **视觉反馈不够明显**：状态标签变化不明显

---

## 二、优化内容

### 2.1 新增UI组件

在检测结果信息栏新增以下组件：

| 组件 | 变量名 | 说明 |
|------|--------|------|
| 阶段标签 | `_consistency_stage_label` | 显示当前检测阶段（初始化/召回相似章节/LLM冲突检测/解析结果） |
| 耗时标签 | `_consistency_time_label` | 显示检测耗时（格式：耗时：X.Xs） |
| 进度条 | `_consistency_progress` | 不确定模式进度条，显示检测进行中 |

### 2.2 检测阶段实时反馈

新增4个检测阶段的实时反馈：

| 阶段 | 触发时机 | 状态显示 |
|------|----------|----------|
| 初始化 | 开始检测时 | "阶段：初始化" |
| 准备上下文 | 构建检查上下文前 | "阶段：准备上下文" |
| 召回相似章节 | 向量召回前 | "阶段：召回相似章节" |
| LLM冲突检测 | 调用LLM检测前 | "阶段：LLM冲突检测" |
| 解析结果 | 解析LLM响应前 | "阶段：解析结果" |

### 2.3 视觉反馈优化

1. **状态标签颜色**：
   - 无冲突：绿色 `#10B981`
   - 发现冲突：橙色 `#F59E0B`
   - 检测失败：红色 `#EF4444`
   - 就绪：默认文本色

2. **按钮状态**：
   - 检测中：禁用"开始检测"按钮
   - 检测完成/失败：恢复按钮

3. **进度条动画**：
   - 检测开始：启动进度条动画（10ms间隔）
   - 检测结束：停止进度条

```python
start_time = time.time()

# 更新阶段状态的回调函数
def update_stage(stage: str):
    self.root.after(0, lambda: self._consistency_stage_label.config(text=f"阶段：{stage}"))

# 更新耗时的回调函数
def update_time():
    elapsed = time.time() - start_time
    self.root.after(0, lambda: self._consistency_time_label.config(text=f"耗时：{elapsed:.1f}s"))
```

### 3.4 新增错误处理方法（_handle_consistency_error）

```python
def _handle_consistency_error(self, error_msg: str, elapsed_time: float = 0.0):
    """处理一致性检查错误"""
    self._consistency_progress.stop()
    self._consistency_check_btn.config(state=tk.NORMAL)
    self._consistency_stage_label.config(text="")
    self._consistency_time_label.config(text=f"耗时：{elapsed_time:.1f}s")
    self._consistency_status_label.config(text="状态：❌ 检测失败", foreground="#EF4444")
    self._consistency_accuracy_label.config(text="准确率：-")
    self._set_status(f"一致性检查失败: {error_msg}")
```
