| 字数达标 | 8% | ≥0.7 | 实际字数/目标字数 |

1. **阅读生成内容**：完整阅读AI生成的章节内容
2. **对照大纲检查**：验证主要情节是否覆盖
3. **检查人物塑造**：对话、行为是否符合人设
4. **验证世界观一致性**：设定、术语是否准确
5. **评估风格匹配**：文笔、节奏是否符合预期
6. **记录问题清单**：标注需要修复的具体位置

## 三、常见问题识别

### 3.1 情节问题

| 问题类型 | 识别特征 | 严重程度 |
|----------|----------|----------|
| 情节遗漏 | 大纲要点未体现 | 高 |
| 逻辑矛盾 | 因果关系不合理 | 高 |
| 节奏失衡 | 重点情节篇幅不足 | 中 |
| 结局仓促 | 收尾不完整 | 中 |

### 3.2 人物问题

| 问题类型 | 识别特征 | 严重程度 |
|----------|----------|----------|
| OOC（角色崩坏） | 行为与性格设定不符 | 高 |
| 对话失真 | 口吻、用词不匹配 | 高 |
| 动机缺失 | 行为缺乏合理动机 | 中 |
| 形象模糊 | 描写不够生动 | 低 |

### 3.3 风格问题

| 问题类型 | 识别特征 | 严重程度 |
|----------|----------|----------|
| AI痕迹 | 过度修饰、堆砌词藻 | 高 |
| 风格割裂 | 前后风格不一致 | 中 |
| 现代词汇 | 古风题材出现现代用语 | 中 |
| 节奏拖沓 | 无效描写过多 | 低 |

### 3.4 世界观问题

| 问题类型 | 识别特征 | 严重程度 |
|----------|----------|----------|
| 设定冲突 | 与已建立设定矛盾 | 高 |

| 体系混乱 | 力量体系/等级混乱 | 中 |

## 四、修复操作指南

### 4.1 修复优先级

1. **P0（阻塞性问题）**：情节遗漏、逻辑矛盾、设定冲突
2. **P1（严重问题）**：OOC、对话失真、AI痕迹
3. **P2（一般问题）**：节奏失衡、形象模糊、风格割裂
4. **P3（轻微问题）**：措辞不当、标点错误

### 4.2 修复方法

#### 4.2.1 情节修复

```
步骤：
1. 定位问题段落
2. 分析缺失/冲突的具体内容
3. 手动补充或修改情节
4. 确保与前后文衔接自然
```

**示例**：
- 问题：大纲要求"主角发现隐藏密室"，但正文直接跳过
- 修复：补充发现过程描写，包含环境描写、心理活动

#### 4.2.2 人物修复

```
步骤：
1. 对比人物设定档案
2. 标注不符合人设的对话/行为
3. 重新设计符合性格的表达
4. 保持人物弧光的一致性
```

**示例**：
- 问题：高冷角色突然话痨
- 修复：精简台词，改为简短有力的表达

#### 4.2.3 风格修复

```
步骤：
1. 对比风格学习样本
2. 识别AI痕迹特征（如"璀璨""深邃"等高频词）
3. 替换为更自然的表达
4. 调整句式节奏
```

**示例**：
- 问题："他的眼眸深邃如星空"
- 修复："他盯着她，眼睛一动不动"

#### 4.2.4 世界观修复

```
步骤：
1. 查阅世界观设定文档
2. 标注设定冲突点
3. 修正为正确设定
4. 补充必要的设定说明
```

**示例**：
- 问题：金丹期使用了元婴期的能力
- 修复：调整为金丹期合理的能力范围

  - `"fanfiction"` vs `"tongren"`（同人）

- **管理层**（`knowledge_manager.py`）：题材验证白名单缺少 `"horror"`、`"mystery"`、`"sports"`

**问题3：领域选项绑定错误**
- AI生成对话框中题材选项硬编码，未使用`_category_map`
- 导致题材选项与实际映射不一致

#### 修复内容

**修复1：Unicode编码问题**
- 文件：`core/knowledge_generator.py`
- 替换所有 `✓` → `[OK]`，`✗` → `[FAIL]`，`⚠` → `[WARN]`
- 修复行：592、595、632、635、639行

**修复2：题材映射统一**
- 文件：`core/knowledge_generator.py`
- `KNOWLEDGE_DOMAINS`键名与GUI映射统一：

### 2026-03-31 项目数据管理外迁修复

**修复人**: 后端架构师

#### 任务背景

遵循架构文档要求，将项目数据管理逻辑从GUI外迁到服务层和插件层。

#### 核心问题

1. **职责混乱**: GUI直接处理项目I/O（文件读写）
2. **事件缺失**: 项目保存/加载没有事件通知机制
3. **可测试性差**: 业务逻辑与UI耦合，无法独立测试
4. **架构不一致**: 违反"微内核+插件化"设计原则

#### 架构设计

**三层架构**:
```
GUI层 (gui_main.py)
    ↓ 调用
服务层 (services/project_manager.py)
    ↓ 事件
插件层 (plugins/project-manager-v1/)
```

**数据流**:
- **保存**: GUI → `ProjectManager.sync_module_data()` → `ProjectManager.save_project()` → 文件系统
- **加载**: 文件系统 → `ProjectManager.load_project()` → `ProjectLoadedEvent` → GUI

**事件机制**:
- `project.saved`: 项目保存完成事件
- `project.loaded`: 项目加载完成事件
- `project.data.*`: 数据变更事件

#### 修复内容

**新建文件**:
1. `services/project_manager.py` (~350行) - 项目管理器服务
2. `plugins/project-manager-v1/plugin.py` (~150行) - 项目管理器插件
3. `plugins/project-manager-v1/plugin.json` - 插件元数据
4. `plugins/project-manager-v1/__init__.py` - 插件模块初始化
5. `tests/test_project_manager.py` (~350行) - 测试脚本

**修改文件**:
1. `gui_main.py` (~50行) - 集成项目管理器
2. `core/bootstrap.py` (~30行) - 核心启动服务注册

|--------|------|
| 项目管理器服务注册 | ✅ 通过 |
| 项目创建和保存 | ✅ 通过 |
| 项目加载和恢复 | ✅ 通过 |
| 核心启动服务集成 | ✅ 通过 |

**测试通过率**: 4/4 (100%)

| 改进项 | 修复前 | 修复后 |

# 新增方法_publish_ai_config_loaded()
def _publish_ai_config_loaded(self) -> None:
    """启动时发布AI配置加载事件，通知AI状态管理插件"""
    try:
        # 直接获取EventBus（避免依赖ConfigService的内部状态）
        from core.event_bus import get_event_bus, Event
        
        event_bus = get_event_bus()
        if not event_bus:
            logger.warning("[配置加载] EventBus不可用，跳过事件发布")
            return
        
        # 构建AI配置字典（从UI变量获取）
        ai_config = {
            "service_mode": self._service_mode_var.get(),
            "provider": self._provider_var.get(),
            "model": self._model_var.get(),
            "api_key": self._api_key_var.get(),
            "local_url": self._local_url_var.get(),
            "temperature": self._temp_var.get(),
        }
        
        # 发布config.changed事件
        event = Event(
            type='config.changed',
            data={
                'type': 'ai_config',
                'data': ai_config
            },
            source='MainWindow._load_settings_from_config'
        )
        event_bus.publish(event)
        logger.info(f"[配置加载] 已发布AI配置事件: provider={ai_config['provider']}, mode={ai_config['service_mode']}")
        
    except Exception as e:
        logger.warning(f"[配置加载] 发布AI配置事件失败: {e}")

    检查Qwen服务健康状态（V3.2.2新增）
    
    Args:
        base_url: API端点（如 http://localhost:8000/v1）
    
    Returns:
        bool: 服务是否健康
    """
    try:
        import requests
        
        # 提取根URL（移除/v1后缀）
        health_url = base_url.replace("/v1", "").rstrip("/") + "/health"
        
        print(f"[KnowledgeGenerator] 检查Qwen服务健康: {health_url}")
        
        response = requests.get(health_url, timeout=5)
        
        if response.status_code == 200:
            data = response.json()
            model_loaded = data.get("model_loaded", False)
            
            if model_loaded:
                print(f"[KnowledgeGenerator] ✓ Qwen服务正常: {data.get('model', 'unknown')}")
                return True
            else:
                print(f"[KnowledgeGenerator] ⚠ Qwen服务模型未加载")
                return False
        else:
            print(f"[KnowledgeGenerator] ✗ Qwen服务返回错误: {response.status_code}")
            return False
            
    except requests.exceptions.ConnectionError:
        print(f"[KnowledgeGenerator] ✗ Qwen服务未启动！")
        print(f"[KnowledgeGenerator] 请先启动Qwen服务：")

        return False
        
    except requests.exceptions.Timeout:
        print(f"[KnowledgeGenerator] ✗ Qwen服务响应超时")
        return False
        
    except Exception as e:
        print(f"[KnowledgeGenerator] ✗ Qwen服务健康检查失败: {e}")

**问题1：Event发布方法签名错误**
- **位置**: `gui_main.py` 第12446行
- **根因**: `EventBus.publish()` 方法签名是 `publish(event_type: str, data=None, source=None)`
- **错误代码**: 传入了 `Event` 对象而非参数：`event_bus.publish(event)`
- **正确代码**: `event_bus.publish(event_type='config.changed', data={...}, source=...)`

**问题2：AIProviderState枚举值不存在**
- **位置**: `core/qwen_provider.py` 第192行、196行
- **根因**: `AIProviderState` 枚举只有 `UNINITIALIZED`, `READY`, `ERROR`, `BUSY`
- **错误代码**: `self._state = AIProviderState.UNAVAILABLE`
- **正确代码**: `self._state = AIProviderState.ERROR`

#### 修复内容

**修复1：修正Event发布方式**

**文件**: `gui_main.py` 第12445-12454行

```python
# 修复前
event = Event(

event_bus.publish(event)

# 修复后（V3.2.2）
event_bus.publish(
    event_type='config.changed',
    data={'type': 'ai_config', 'data': ai_config},
    source='MainWindow._load_settings_from_config'
)
```

**修复2：使用正确的枚举值**

**文件**: `core/qwen_provider.py` 第192行、196行
