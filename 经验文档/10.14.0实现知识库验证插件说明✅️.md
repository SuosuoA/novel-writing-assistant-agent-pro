## 一、功能概述

知识库验证插件（knowledge-validator）是一个Validator类型插件，用于验证小说内容与知识库的一致性。该插件集成了OpenClaw mem9框架的L2温记忆（LanceDB向量库），通过向量检索召回相关知识，检测内容中的知识冲突。

### 核心功能

1. **题材识别**：自动识别科幻/玄幻/历史/通用题材
2. **知识检索**：向量检索召回top-10相关知识
3. **冲突检测**：检测科学事实、修仙设定、历史事件等冲突
4. **评分生成**：生成知识库一致性评分（0-1）

### 2.2 设计模式

### 3.2 验证维度

```python
def get_validation_dimensions(self) -> List[str]:
    """获取验证维度"""
    return [
        "knowledge_consistency",  # 知识库一致性
        "genre_accuracy",        # 题材准确性
        "fact_correctness",      # 事实正确性
        "logic_coherence",       # 逻辑连贯性
    ]
```

### 3.3 验证流程

```python
def validate(self, content: str, context: Optional[Dict[str, Any]] = None) -> ValidationScores:
    """验证内容并返回评分
    
    流程:
    1. 识别题材（科幻/玄幻/历史/通用）
    2. 提取知识点关键词（专业术语、设定名词等）
    3. 向量检索召回相关知识（top-10）
    4. 检测知识冲突（科学事实、设定一致性等）
    5. 生成评分（基础分 + 召回质量分 - 冲突惩罚）
    """
    # 1. 识别题材
    genre = context.get("genre") or self._detect_genre(content)
    
    # 2. 提取关键词
    keywords = self._extract_knowledge_keywords(content, context)
    
    # 3. 召回知识
    recalled_knowledge = self._recall_knowledge(keywords, genre)
    
    # 4. 检测冲突
    conflicts = self._check_knowledge_conflicts(content, recalled_knowledge, genre)
    
    # 5. 生成评分
    score = self._generate_score(recalled_knowledge, conflicts)
    
    # 6. 返回ValidationScores
    return ValidationScores(
        knowledge_consistency_score=score,
        knowledge_conflicts=[c.to_dict() for c in conflicts],
        recalled_knowledge=recalled_knowledge,
        # ...其他维度
    )
```

### 3.4 冲突检测规则

#### 科幻题材
- 光速/超光速旅行与相对论冲突
- 时间旅行逻辑悖论
- 基因编辑技术描述不准确

#### 玄幻题材
- 修仙等级顺序混乱
- 炼丹设定缺少细节
- 功法描述前后矛盾

#### 历史题材
- 历史人物时代错乱
- 地理位置描述错误
- 历史事件时间线混乱

## 六、使用示例

### 6.1 在iterative_generator_v2.py中集成

```python
from plugins.knowledge_validator.plugin import KnowledgeValidatorPlugin

# 初始化
validator = KnowledgeValidatorPlugin()
context = {"project_root": "/path/to/project"}
validator.initialize(context)

# 验证
test_content = """
飞船穿过虫洞，进入了一个陌生的星系。舰长命令启动量子引擎，准备进行星际跳跃。
"""

result = validator.validate(test_content, {"genre": "科幻"})

# 查看结果
print(f"知识库一致性评分: {result.knowledge_consistency_score}")
print(f"检测到冲突: {len(result.knowledge_conflicts)}条")
print(f"召回知识: {len(result.recalled_knowledge)}条")
```

## 九、已知限制

### 9.1 当前版本限制

1. **知识库依赖**：需要预先构建知识库向量索引
2. **题材覆盖**：仅支持科幻/玄幻/历史/通用四种题材
3. **冲突规则**：冲突检测规则较为简单，需要持续积累
4. **降级模式**：知识检索失败时降级为关键词匹配，准确率较低

### 9.2 后续优化方向

1. 扩充题材覆盖（都市、悬疑、言情等）
2. 增强冲突检测规则（基于LLM的深度分析）
3. 支持自定义知识库导入
4. 优化降级策略（混合检索）
