## 三、P0 阻塞级问题（必须修复）

### P0-001: EventBus.publish() 伪异步实现

**问题描述**：
```python
def publish(self, event_type: str, data: Any, source: str = None):
    """发布事件（异步）- 线程安全实现"""
    event = Event(type=event_type, data=data, source=source)
    
    with self._lock:
        handlers = list(self._handlers.get(event_type, []))
    
    for handler_info in handlers:
        try:
            handler_info["handler"](event)  # 同步调用！
        except Exception as e:
            print(f"事件处理失败 {event_type}: {e}")
```

**风险分析**：
- 注释声明"异步"但实际为同步顺序执行
- 单个handler阻塞会导致整个事件链阻塞
- 高并发场景下可能引发级联延迟
- 违背事件驱动架构的解耦初衷

**SRE影响**：
- 错误预算消耗不可控（阻塞时间无法预估）
- 无法设置合理的超时和熔断策略
- 故障定位困难（无法区分handler执行时间）

**修复建议**：
```python
def publish(self, event_type: str, data: Any, source: str = None):
    """发布事件（真异步）- 使用线程池"""
    event = Event(type=event_type, data=data, source=source)
    
    with self._lock:
        handlers = list(self._handlers.get(event_type, []))
    
    # 提交到线程池异步执行
    for handler_info in handlers:
        self._executor.submit(self._invoke_handler, handler_info, event)

def _invoke_handler(self, handler_info, event):
    """带超时和异常处理的handler调用"""
    handler_id = handler_info["id"]
    try:
        with self._timeout_context(5.0):  # 5秒超时
            handler_info["handler"](event)
    except TimeoutError:
        self._metrics.increment("event.handler.timeout", {"handler": handler_id})
    except Exception as e:
        self._metrics.increment("event.handler.error", {"handler": handler_id})
        self._logger.error(f"Handler {handler_id} failed", exc_info=e)
```

**验收标准**：
- [ ] publish()方法真正异步执行
- [ ] 每个handler有独立超时控制（默认5秒）
- [ ] handler异常不影响其他handler执行
- [ ] 提供同步版本publish_sync()用于特殊场景

### P0-002: PluginRegistry 状态机缺少ERROR恢复路径

**问题描述**：
当前状态机定义：
```python
class PluginState(Enum):
    LOADED = "loaded"      # 已加载
    ACTIVE = "active"      # 已激活
    ERROR = "error"        # 错误状态
    UNLOADING = "unloading"  # 卸载中
```

但文档中activate/deactivate方法仅处理LOADED↔ACTIVE转换，ERROR状态无恢复机制。

**风险分析**：
- 插件进入ERROR状态后无法自动或手动恢复
- 必须重启应用才能重新加载故障插件
- 违背高可用设计原则

**SRE影响**：
- MTTR（平均恢复时间）不可接受
- 需要人工介入的故障比例过高
- 无法实施自愈机制

**修复建议**：
```python
def recover(self, plugin_id: str) -> bool:
    """从ERROR状态恢复插件"""
    with self._lock:
        if self._states.get(plugin_id) != PluginState.ERROR:
            return False
        
        plugin = self._plugins.get(plugin_id)
        if not plugin:
            return False
        
        try:
            # 尝试重新初始化
            success = plugin.initialize(self._context)
            if success:
                self._states[plugin_id] = PluginState.LOADED
                self._event_bus.publish("plugin.recovered", {"plugin_id": plugin_id})
                return True
            else:
                self._event_bus.publish("plugin.recovery_failed", {"plugin_id": plugin_id})
                return False
        except Exception as e:
            self._logger.error(f"Plugin {plugin_id} recovery failed", exc_info=e)
            return False

def get_error_plugins(self) -> List[str]:
    """获取所有处于ERROR状态的插件ID"""
    with self._lock:
        return [pid for pid, state in self._states.items() if state == PluginState.ERROR]
```

**验收标准**：
- [ ] 提供recover()方法支持ERROR→LOADED转换
- [ ] 提供get_error_plugins()查询故障插件
- [ ] 恢复失败时保留错误信息供诊断
- [ ] 文档更新状态转换图

### P1-001: 熔断器设计缺少半开状态探测机制

**问题描述**：
文档7.2.1节提到熔断器模式，但仅描述状态转换，未定义HALF_OPEN状态的探测逻辑。

**风险分析**：
- 服务恢复后无法自动感知
- 需要人工干预才能恢复流量
- 恢复时间不可控

**修复建议**：
```python
class CircuitBreaker:
    def __init__(self, ...):
        self.state = CircuitState.CLOSED
        self.failure_count = 0
        self.last_failure_time = None
        self.half_open_attempts = 0
        
    def call(self, func, *args, **kwargs):
        if self.state == CircuitState.OPEN:
            if self._should_attempt_reset():
                self.state = CircuitState.HALF_OPEN
                self.half_open_attempts = 0
            else:
                raise CircuitBreakerOpenException()
        
        if self.state == CircuitState.HALF_OPEN:
            if self.half_open_attempts >= self.half_open_max_calls:
                raise CircuitBreakerOpenException()
            self.half_open_attempts += 1
        
        try:
            result = func(*args, **kwargs)
            self._on_success()
            return result
        except Exception as e:
            self._on_failure()
            raise
    
    def _should_attempt_reset(self) -> bool:
        """判断是否应尝试恢复"""
        if not self.last_failure_time:
            return False
        elapsed = time.time() - self.last_failure_time
        return elapsed >= self.recovery_timeout
    
    def _on_success(self):
        if self.state == CircuitState.HALF_OPEN:
            # 半开状态连续成功，关闭熔断器
            self.state = CircuitState.CLOSED
            self.failure_count = 0
            self.half_open_attempts = 0
        else:
            self.failure_count = max(0, self.failure_count - 1)
    
    def _on_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        
        if self.state == CircuitState.HALF_OPEN:
            # 半开状态失败，重新打开
            self.state = CircuitState.OPEN
        elif self.failure_count >= self.failure_threshold:
            self.state = CircuitState.OPEN
```

### P1-002: 健康检查系统缺少依赖健康传播

**问题描述**：
文档7.4节定义了组件级健康检查，但未定义依赖健康状态的传播机制。

**风险分析**：
- 下游服务故障无法及时感知
- 无法做故障隔离和优雅降级
- 级联故障风险

**修复建议**：
```python
class HealthChecker:
    def __init__(self):
        self._components: Dict[str, HealthCheckable] = {}
        self._dependencies: Dict[str, List[str]] = {}  # 组件依赖关系
        
    def register(self, name: str, component: HealthCheckable, 
                 dependencies: List[str] = None):
        self._components[name] = component
        self._dependencies[name] = dependencies or []
    
    def check_health(self) -> SystemHealth:
        """执行健康检查并传播依赖状态"""
        individual = {}
        
        # 首先检查所有组件
        for name, component in self._components.items():
            try:
                individual[name] = component.health_check()
            except Exception as e:
                individual[name] = HealthStatus(
                    status=HealthState.UNHEALTHY,
                    message=f"Health check failed: {e}"
                )
        
        # 传播依赖状态
        for name, deps in self._dependencies.items():
            if individual[name].status != HealthState.UNHEALTHY:
                for dep in deps:
                    if dep in individual and individual[dep].status == HealthState.UNHEALTHY:
                        individual[name] = HealthStatus(
                            status=HealthState.DEGRADED,
                            message=f"Dependency {dep} is unhealthy"
                        )
                        break
        
        # 计算整体状态
        overall = self._calculate_overall_status(individual.values())
        
        return SystemHealth(overall=overall, components=individual)
```

class MetricsCollector:
    """支持标签的指标采集器"""
    
    def increment(self, name: str, value: int = 1, 
                  tags: Dict[str, str] = None):
        """计数器递增"""
        tags = tags or {}
        key = self._make_key(name, tags)
        self._counters[key] = self._counters.get(key, 0) + value
    
    def record_histogram(self, name: str, value: float,
                        tags: Dict[str, str] = None):
        """记录直方图值"""
        tags = tags or {}
        key = self._make_key(name, tags)
        if key not in self._histograms:
            self._histograms[key] = []
        self._histograms[key].append(value)
    
    def _make_key(self, name: str, tags: Dict[str, str]) -> str:
        """生成带标签的指标键"""
        if not tags:
            return name
        tag_str = ",".join(f"{k}={v}" for k, v in sorted(tags.items()))
        return f"{name}{{{tag_str}}}"

### P2-001: 数据库连接池缺少大小限制

**问题描述**：
文档8.3节提到"连接池（最多10个连接）"，但代码示例中未体现连接池实现。

**修复建议**：
```python
class SQLiteConnectionPool:
    def __init__(self, db_path: str, max_connections: int = 10):
        self.db_path = db_path
        self.max_connections = max_connections
        self._pool = queue.Queue(maxsize=max_connections)
        self._semaphore = threading.Semaphore(max_connections)
        
        # 预创建连接
        for _ in range(max_connections):
            conn = sqlite3.connect(db_path, check_same_thread=False)
            conn.execute("PRAGMA journal_mode=WAL")
            self._pool.put(conn)
    
    def acquire(self, timeout: float = 30.0) -> sqlite3.Connection:
        if not self._semaphore.acquire(timeout=timeout):
            raise TimeoutError("Failed to acquire database connection")
        return self._pool.get()
    
    def release(self, conn: sqlite3.Connection):
        self._pool.put(conn)
        self._semaphore.release()
    
    @contextmanager
    def connection(self, timeout: float = 30.0):
        conn = self.acquire(timeout)
        try:
            yield conn
        finally:
            self.release(conn)
```

### P2-002: 事件总线缺少死信队列实现

**问题描述**：
文档3.1.3节提到"死信队列（最大1000条）"，但未在代码中实现。

**修复建议**：
```python
class ReliableEventBus(EventBus):
    def __init__(self, max_dlq_size: int = 1000):
        super().__init__()
        self._dlq = deque(maxlen=max_dlq_size)
        self._retry_policy = RetryPolicy(max_retries=3, backoff=ExponentialBackoff())
    
    def _invoke_handler_with_retry(self, handler_info, event):
        handler_id = handler_info["id"]
        
        for attempt in range(self._retry_policy.max_retries):
            try:
                handler_info["handler"](event)
                return  # 成功
            except Exception as e:
                if attempt < self._retry_policy.max_retries - 1:
                    wait_time = self._retry_policy.get_wait_time(attempt)
                    time.sleep(wait_time)
                else:
                    # 重试耗尽，加入死信队列
                    self._dlq.append({
                        "event": event,
                        "handler_id": handler_id,
                        "error": str(e),
                        "timestamp": datetime.now(timezone.utc).isoformat(),
                        "attempts": attempt + 1
                    })
                    self._metrics.increment("event.dlq.added")
    
    def get_dlq_size(self) -> int:
        return len(self._dlq)
    
    def replay_dlq(self, max_events: int = None):
        """重放死信队列中的事件"""
        events_to_replay = list(self._dlq)[:max_events]
        for item in events_to_replay:
            self.publish(item["event"].type, item["event"].data)
            self._dlq.remove(item)
```

### 8.1 总体评价

**架构设计质量**: 良好
