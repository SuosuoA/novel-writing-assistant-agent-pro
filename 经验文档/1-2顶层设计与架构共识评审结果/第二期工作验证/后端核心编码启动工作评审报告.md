```python
# event_bus.py - ThreadPoolExecutor真异步实现
def publish(self, event_type: str, data: Any = None, source: Optional[str] = None) -> None:
    """
    发布事件 - 真异步,使用线程池执行
    
    实现细节:
    1. 创建Event对象
    2. 在锁内拷贝handlers快照
    3. 提交到ThreadPoolExecutor异步执行
    4. 立即返回,不阻塞调用线程
    """
    # 1. 在锁内拷贝handlers快照(读拷贝模式)
    with self._lock:
        handlers = list(self._handlers.get(event_type, []))
    
    # 2. 提交到线程池异步执行(不阻塞调用线程)
    if handlers:
        self._executor.submit(self._dispatch_event, event, handlers)
    
    # 3. 立即返回,调用线程不被阻塞
```

## 九、风险提示

### 9.1 高风险项

#### 9.1.1 ThreadPoolExecutor线程泄漏(已缓解)

**描述**: EventBus使用的ThreadPoolExecutor在异常情况下可能未正确关闭。

**缓解措施**: 已注册atexit清理钩子,应用退出时自动调用`shutdown()`。

**监控建议**: 添加线程池活跃线程数监控。

#### 9.1.2 熔断器误触发

**描述**: 故障阈值设置不当可能导致熔断器频繁开启。

**缓解措施**: 默认阈值(5次失败)合理,提供手动重置接口。

**监控建议**: 添加熔断器状态变更告警。

### 9.2 中风险项

#### 9.2.1 死信队列内存增长

**描述**: 死信队列最大容量100条,长期运行可能占用内存。

**缓解措施**: 已设置容量上限,超出时移除旧记录。

**监控建议**: 定期检查死信队列大小。
