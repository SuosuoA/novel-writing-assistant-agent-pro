## 三、知识库Schema设计

### 3.1 数据结构

**必填字段** (8个):
- `knowledge_id`: 唯一标识符（格式：scifi-physics-001）
- `category`: 分类（scifi）
- `domain`: 领域（physics/chemistry/biology）
- `title`: 知识点标题
- `content`: 知识点详细内容
- `keywords`: 关键词列表
- `created_at`: 创建时间
- `updated_at`: 更新时间

**可选字段** (5个):
- `embedding`: 向量嵌入（待生成）
- `references`: 参考来源
- `difficulty`: 难度等级（basic/intermediate/advanced）
- `tags`: 自定义标签
- `metadata`: 元数据（来源、置信度、语言等）

### 3.2 ID命名规范

**格式**: `{category}-{domain}-{序号}`

**示例**:
- `scifi-physics-001`: 科幻-物理-第1条
- `scifi-chemistry-050`: 科幻-化学-第50条
- `scifi-biology-100`: 科幻-生物-第100条

## 四、向量嵌入与检索

### 4.1 向量嵌入方案

**技术选型**:
- Embedding模型: OpenAI text-embedding-3-small (1536维)
- 备选模型: 本地模型 all-MiniLM-L6-v2 (384维)

**实现方式**:
```python
# 使用KnowledgeManager自动生成向量嵌入
manager = KnowledgeManager(workspace_root)
result = manager.import_from_json("data/knowledge/scifi/physics.json")
```

**前置条件**:
- 配置环境变量: `OPENAI_API_KEY`
- 或在初始化时传入: `KnowledgeManager(workspace_root, openai_api_key="sk-...")`

### 4.2 向量检索流程

```
用户输入查询（如"量子力学"）
    ↓
调用OpenAI Embedding API生成查询向量
    ↓
LanceDB向量检索（余弦相似度）
    ↓
返回top-10相关知识点
    ↓
注入ValidationScores评分维度
```

### 4.3 检索准确率验证

**测试方法**:
```python
# 测试检索准确率
test_queries = ["量子力学", "相对论", "基因编辑", "化学反应", "黑洞"]
for query in test_queries:
    results = manager.search_by_vector(query, top_k=5)
    # 验证前5个结果的相关性
```

**目标**: 召回准确率 ≥ 85%

## 五、知识库应用场景

### 5.1 评分反馈优化

**当前问题**: ValidationScores完全依赖关键词匹配，缺乏知识库支撑

**解决方案**:
- 新增"知识库一致性"维度（10%权重）
- 检测生成内容是否违反物理/化学/生物知识常识
- 召回相关知识点作为评分依据

**示例**:
```python
# 用户创作科幻小说章节
content = "飞船以超光速飞行，穿越了时间..."

# 知识库检索
physics_knowledge = manager.search_by_vector("超光速 时间旅行", top_k=10)

# 一致性检查
# 发现"时间膨胀效应"知识点，验证设定合理性
# 评分：如果符合知识点描述，给高分；如果违反，扣分
```

### 5.2 风格学习支撑

**当前问题**: 风格库为空泛模板，缺乏学科知识支撑

**解决方案**:
- 根据题材加载对应知识库（科幻→物理/化学/生物）
- 将知识点注入风格学习模块
- 增强AI对专业术语和概念的理解

### 5.3 长篇连贯性保障

**当前问题**: AI无法记住前文细节，长篇小说创作连贯性差

**解决方案**:
- 章节向量编码存储到LanceDB
- 新章节创作时召回相似上下文
- 知识库验证情节设定的知识一致性

## 八、经验总结

### 8.1 成功经验

1. **分层构建策略**: 先生成核心知识点，再扩充覆盖面
2. **Schema优先**: 先定义数据结构，确保一致性
3. **模块化设计**: 知识管理器、向量存储、检索接口解耦
4. **渐进式完成**: JSON文件优先，向量嵌入可后续补充

### 8.2 遇到的问题

1. **Windows编码问题**: emoji字符导致GBK编码错误
   - 解决方案：去除emoji，使用纯文本输出

2. **JSON格式错误**: 手工编辑JSON文件导致格式错误
   - 解决方案：使用脚本自动生成，确保格式正确

3. **API密钥缺失**: OpenAI Embedding需要配置API密钥
   - 解决方案：文档说明，支持后续补充

### 8.3 改进建议
