## 一、概述

本文档描述小说生成流水线的实现，包括：
- `agents/pipeline_orchestrator.py`: 流水线编排器
- `agents/plugins/`: 插件Agent实现（调用实际插件）
- `agents/novel_generation_service.py`: 生成服务封装
- GUI集成与端到端测试

## 二、流水线架构

### 2.1 流水线阶段定义

```
小说生成流水线
├── outline_analysis (大纲解析) - agent_type: "outline_analysis"
│   └── 输出：大纲结构、章节要点、关键事件
├── style_learning (风格学习) - agent_type: "style_learning"
│   └── 输出：风格档案、关键词、句式模式
├── context_building (上下文构建) - agent_type: "context_building"
│   └── 输出：完整上下文、提示词片段（可选阶段）
├── content_generation (内容生成) - agent_type: "novel_generation"
│   └── 输出：章节内容
└── validation (质量验证) - agent_type: "quality_validation"
    └── 输出：评分、是否通过、改进建议
```

### 2.2 Agent实现架构（V1.1更新）

**统一使用 `agents/plugins/` 模块的Agent实现：**

| Agent类 | agent_type | 调用插件 |
|---------|------------|----------|
| `OutlineAnalysisAgent` | `outline_analysis` | outline-parser-v3 |
| `StyleLearningAgent` | `style_learning` | style-learner-v5 |
| `NovelGenerationAgent` | `novel_generation` | novel-generator-v3 |
| `QualityValidationAgent` | `quality_validation` | quality-validator-v1 |

**注意**: 原 `agents/adapters/novel_generation_adapters.py` 已删除，统一使用plugins模块。

### 2.3 验证失败重试机制

```
生成 → 验证
  ↑      │
  │      │ 未通过(分数 < 阈值)
  │      ↓
  └──── 优化反馈

最大迭代次数: 5
验证通过阈值: 0.8
```

## 四、Agent适配器

### 4.1 大纲分析Agent

**类**: `OutlineAnalysisAgent`（来自 `agents.plugins`）

**agent_type**: `outline_analysis`

**输入**:
- `content`: 大纲文本内容
- `file_path`: 大纲文件路径（可选）
- `options`: 解析选项（可选）

**输出**:
```python
{
    "chapter_number": 1,
    "target_word_count": 2000,
    "outline_structure": {"chapters": [...], "total_chapters": 10},
    "chapter_outline_parsed": {"scenes": [...], "key_points": [...]},
    "key_events": ["事件1", "事件2"],
    "characters_involved": ["人物1", "人物2"],
    "scene_settings": ["室内", "白天"],
}
```

### 4.2 风格学习Agent

**类**: `StyleLearningAgent`（来自 `agents.plugins`）

**agent_type**: `style_learning`

**输入**:
- `content`: 风格样本文本
- `file_path`: 风格样本路径
- `style_profile`: 已有风格档案

**输出**:
```python
{
    "style_profile": {
        "keywords": ["关键词1", "关键词2"],
        "sentence_patterns": ["句式模式1"],
        "tone": "neutral",
        "formality": "neutral",
        "description_style": "detailed",
        "dialogue_style": "natural",
    },
    "style_keywords": [...],
    "sentence_patterns": [...],
    "tone": "neutral",
    "formality": "neutral",
}
```

### 4.3 小说生成Agent

**类**: `NovelGenerationAgent`（来自 `agents.plugins`）

**agent_type**: `novel_generation`

**核心流程（强制保护）**:
1. 整理打包请求内容（强制附加【本章完】要求）
2. 向大模型发送请求
3. 接受返回文章
4. 多维度评分
5. 分数<0.8 → 发送评分+修改建议 → 再次生成
6. 评分≥0.8且有【本章完】→ 输出保存

**输出**:
```python
{
    "content": "章节内容...",
    "word_count": 2050,
    "chapter_number": 1,
    "chapter_title": "第一章",
    "has_chapter_end": True,  # 是否包含【本章完】
}
```

### 4.4 质量验证Agent

**类**: `QualityValidationAgent`（来自 `agents.plugins`）

**agent_type**: `quality_validation`

**评分维度与权重**:
| 维度 | 权重 | 说明 |
|------|------|------|
| word_count | 10% | 字数符合性 |
| outline | 15% | 大纲符合性 |
| style | 25% | 风格一致性 |
| character | 25% | 人物一致性 |
| worldview | 20% | 世界观一致性（一票否决） |
| naturalness | 5% | 自然度 |

**输出**:
```python
{
    "total_score": 0.85,
    "scores": {
        "word_count": 0.9,
        "outline": 0.8,
        "style": 0.85,
        "character": 0.9,
        "worldview": 0.8,
        "naturalness": 0.85,
    },
    "passed": True,  # total_score >= 0.8
    "suggestions": ["改进建议1", "改进建议2"],
    "word_count": 2050,
    "target_word_count": 2000,
}
```

## 五、上下文传递机制

### 5.1 AgentContext

```python
@dataclass
class AgentContext:
    task_id: str
    session_id: str = ""
    conversation_history: List[Dict] = field(default_factory=list)
    shared_memory: Dict[str, Any] = field(default_factory=dict)
    metadata: Dict[str, Any] = field(default_factory=dict)
    config: Dict[str, Any] = field(default_factory=dict)
```

### 5.2 传递流程

```
阶段1: outline_analysis
  └── context.set_shared("outline_analysis.outline_structure", {...})
  └── context.set_shared("outline_analysis.key_events", [...])

阶段2: style_learning
  └── 读取: context.get_shared("outline_analysis.key_events")
  └── 写入: context.set_shared("style_learning.style_profile", {...})

阶段3: context_building（可选）
  └── 读取: context.shared_memory (所有前序结果)
  └── 写入: context.set_shared("context_building.prompt_fragments", {...})

阶段4: content_generation (novel_generation)
  └── 读取: payload (包含shared_memory)
  └── 写入: context.set_shared("novel_generation.content", "生成内容")

阶段5: validation (quality_validation)
  └── 读取: context.get_shared("novel_generation.content")
  └── 输出: 评分和建议
```

### 5.3 验证失败时的反馈传递

```python
if stage_name == "validation" and validation_score < threshold:
    feedback = self._extract_validation_feedback(stage_result.data)
    context.set_shared("validation.feedback", feedback)
    context.set_shared("validation.score", validation_score)
    
    # 构建优化提示
    current_payload["optimization_feedback"] = feedback
    current_payload["previous_score"] = validation_score
    
    return stage_results, True  # 需要重试
```

## 十、使用示例

### 10.1 同步执行

```python
from agents import get_generation_service, NovelGenerationConfig

service = get_generation_service()

config = NovelGenerationConfig(
    chapter_title="第一章 开始",
    chapter_number=1,
    target_word_count=2000,
    outline_content="第一章：主角登场...",
    chapter_outline="场景：城市街道\n重点：主角首次出现",
    max_iterations=5,
    validation_threshold=0.8,
)

result = service.generate_chapter_sync(
    chapter_title=config.chapter_title,
    chapter_number=config.chapter_number,
    outline_content=config.outline_content,
    chapter_outline=config.chapter_outline,
    **config.__dict__
)

if result.success:
    print(f"生成成功: {result.final_output['word_count']}字")
else:
    print(f"生成失败: {result.error}")
```

## 十一、最佳实践

### 11.1 配置管理

- 使用`NovelGenerationConfig`统一管理生成参数
- 根据实际需求调整`max_iterations`和`validation_threshold`
- 建议从项目文件中读取大纲、人物、世界观等配置

### 11.2 进度反馈（V1.1更新）

```python
def on_progress(progress):
    print(f"[迭代{progress.iteration}/{progress.max_iterations}] {progress.stage_name}: {progress.progress_percent:.0f}%")
    print(f"  {progress.message}")

service.add_progress_callback(on_progress)
```

### 11.3 错误处理

- 所有Agent的execute方法返回标准化的`AgentResult`
- 使用`result.success`判断是否成功
- 使用`result.error`获取错误详情
