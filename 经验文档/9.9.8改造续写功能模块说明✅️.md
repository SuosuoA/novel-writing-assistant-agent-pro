## 一、改造背景

根据《9.9大模型集成.txt》要求，将续写插件中的AI调用统一替换为AIServiceManager调用，实现本地AI和线上AI的动态切换。

### 2.2 改造前后对比

#### 位置1：流式生成（stream_continuation方法）

**改造前（直接调用OpenAI API）**

```python
# 流式调用API
response = self._api_client.chat.completions.create(
    model=self.default_model,
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": prompt}
    ],
    temperature=request.temperature or self.default_temperature,
    max_tokens=max_tokens,
    stream=True,
    timeout=self._timeout
)

# 流式输出
for chunk in response:
    if chunk.choices and chunk.choices[0].delta.content:
        content = chunk.choices[0].delta.content
        full_text += content
        if callback:
            callback(content)
        yield content
```

**改造后（使用AIServiceManager）**

```python
# 使用AIServiceManager统一调用（流式）
from core.ai_service_manager import get_ai_service_manager
from core.ai_provider import GenerationConfig

ai_manager = get_ai_service_manager()

config = GenerationConfig(
    temperature=request.temperature or self.default_temperature,
    max_tokens=max_tokens,
    timeout=self._timeout
)

# 流式生成
result = ai_manager.generate_text_stream(
    prompt=prompt,
    callback=callback,
    config=config,
    messages=[
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": prompt}
    ]
)

full_text = result.text
```

## 三、改造优势

### 3.1 统一接口

所有AI调用通过AIServiceManager统一管理，插件不再直接依赖OpenAI SDK。

### 3.2 动态切换

根据config.yaml配置自动选择Provider：
- **线上模式**: DeepSeek / OpenAI
- **本地模式**: Qwen2.5-14B-GPTQ-Int4

```yaml
service_mode: local          # 或 online
provider: Qwen               # 或 DeepSeek / OpenAI
model: Qwen2.5-14B-GPTQ-Int4
temperature: 0.7
```

### 3.3 实时生效

配置变更后通过EventBus发布`config.changed`事件，AIServiceManager订阅事件并动态切换Provider，无需重启应用。

### 3.4 容错降级

AIServiceManager内置四级容错机制：
1. **模型回退**: 主模型失败自动切换备用模型
2. **Provider降级**: 本地AI失败降级到线上API
3. **缓存兜底**: API失败返回缓存结果
4. **优雅降级**: 返回降级提示而非报错

## 四、续写功能特性

### 4.1 核心功能

1. **基于起始文本智能续写**: 从用户提供的起始文本开始续写
2. **5种续写方向**: 自然/特定/情感/动作/对话
3. **上下文智能感知**: 自动读取大纲/人设/世界观/前文
4. **流式生成支持**: 实时返回生成内容
5. **多版本生成**: 生成多个版本并选择最佳

### 4.2 流式生成改造

续写功能最核心的特性是**流式生成**，改造后通过AIServiceManager的`generate_text_stream()`方法实现：

```python
# 流式生成
result = ai_manager.generate_text_stream(
    prompt=prompt,
    callback=callback,  # 实时回调
    config=config,
    messages=[...]
)
```

**回调函数**:
```python
def callback(chunk: str):
    """实时接收生成内容"""
    print(chunk, end="", flush=True)
    # 或更新UI显示
```

### 4.3 上下文感知

续写插件自动读取项目设定：

**项目上下文管理器**（ProjectContextManager）：
- 自动读取大纲文件（大纲/*.yaml）
- 自动读取人物设定（人物设定/*.yaml）
- 自动读取世界观设定（世界观/*.yaml）
- 自动读取前5章内容作为上下文

**章节概要生成**：
- 智能提取关键词
- 选择重要句子
- 避免重复加载

## 五、使用示例

### 5.1 基本使用

```python
from core.ai_service_manager import get_ai_service_manager

ai_manager = get_ai_service_manager()

# 流式续写
def on_chunk(chunk: str):
    print(chunk, end="", flush=True)

result = ai_manager.generate_text_stream(
    prompt="李明站在悬崖边，望着远方的云海...",
    callback=on_chunk,
    config={"temperature": 0.8, "max_tokens": 2000}
)

print(f"\n生成完成，Token使用: {result.usage}")
```

## 七、技术细节

### 7.1 插件调用流程

```
用户请求 → ContinuationGeneratorPlugin
         → stream_continuation()
         → _build_continuation_prompt() → 构建Prompt
         → AIServiceManager.generate_text_stream()
         → LocalProvider / OnlineProvider
         → 流式返回结果
```

### 7.2 AIServiceManager架构

```
AIServiceManager
  ├─ ConfigService (读取配置)
  ├─ EventBus (订阅配置变更)
  ├─ Provider管理
  │   ├─ LocalProvider (本地AI)
  │   └─ OnlineProvider (线上API)
  └─ 容错机制
      ├─ 模型回退
      ├─ Provider降级
      ├─ 缓存兜底
      └─ 优雅降级
```

### 7.3 流式生成原理

**改造前**:
```
OpenAI API → 流式响应 → for循环迭代 → 回调函数
```

**改造后**:
```
AIServiceManager → Provider → 流式响应 → 内部迭代 → 回调函数 → 返回完整结果
```

**关键改进**:
- AIServiceManager内部处理流式迭代
- 通过callback参数传递每个chunk
- 最终返回完整的GenerationResult对象（包含text和usage）

### 7.4 兼容性设计

保留`set_api_client()`方法作为兼容接口：

```python
def set_api_client(self, client: Any) -> None:
    """设置API客户端（兼容接口）"""
    self._api_client = client
    logger.info("API客户端已设置")
```

实际调用时不再使用`_api_client`，而是通过AIServiceManager获取Provider。

## 九、注意事项

### 9.1 本地AI特性

- **贪婪解码**: 使用`do_sample=False`避免CUDA采样错误
- **GPU显存**: 7GB显存限制，超出部分自动offload到CPU
- **加载时间**: 首次调用时加载模型，约15秒

### 9.2 流式生成注意事项

- **回调函数**: 必须是非阻塞的，避免耗时操作
- **异常处理**: 回调中不要抛出异常
- **线程安全**: 回调可能在子线程中执行

### 9.3 性能优化

- **缓存预热**: 登录时预加载常用缓存
- **全局缓存**: 使用CacheManager提升缓存命中率
- **并发控制**: RLock保护生成调用，避免并发问题

### 9.4 错误处理

```python
try:
    for chunk in plugin.stream_continuation(request):
        print(chunk, end="", flush=True)
except LLMTimeoutError:
    # 超时处理
    logger.error("续写超时")
except LLMConnectionError as e:
    # 连接错误处理
    logger.error(f"连接失败: {e}")
except LLMError as e:
    # 其他LLM错误
    logger.error(f"续写失败: {e}")
```
