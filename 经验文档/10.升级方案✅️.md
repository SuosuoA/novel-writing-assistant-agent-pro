## 二、升级目标：套用OpenClaw mem9框架

### 2.1 OpenClaw mem9框架核心设计

**设计哲学**: **文件即真相源**（File is the Source of Truth）

OpenClaw的记忆系统建立在纯文本Markdown文件之上，核心原则：
1. **纯文本驱动**: 所有记忆以Markdown格式存储，透明可编辑
2. **不重新训练模型**: 通过写入和检索记忆文件来适应，而非改动底层LLM
3. **向量索引非真相源**: 文件本身是唯一真相，向量索引仅用于加速检索

### 2.2 mem9的5层记忆架构

OpenClaw采用**5层记忆存储架构**（从热到冷）：

| 层级 | 存储介质 | 内容类型 | 更新频率 | 检索方式 | 作用 |
|:---:|:---|:---|:---|:---|:---|
| **L1: 热记忆** | `SESSION-STATE.md` | 当前会话状态、活跃任务、临时上下文 | 实时更新（每次AI响应前） | 直接读取 | 快速恢复会话，防崩溃丢失 |
| **L2: 温记忆** | LanceDB向量库 | 章节向量、知识库向量、风格向量 | 增量更新 | 向量检索（cosine similarity） | 语义召回，智能上下文构建 |
| **L3: 冷记忆** | Git-Notes | 分支感知的历史决策、里程碑事件 | 按需归档 | Git log检索 | 分支隔离，历史追溯 |
| **L4: 精选档案** | `MEMORY.md` | 精心提炼的长期事实、关键决策、重要人物 | 每日冥想提炼 | 手动维护 | 核心知识库，跨会话持久 |
| **L5: 云备份** | 外部存储 | 完整workspace镜像 | 每日备份 | 云端API | 灾难恢复，异地备份 |

**关键机制**:
- **WAL协议**（Write-Ahead Log）: 在AI响应前先写入SESSION-STATE.md，确保中断后可恢复
- **每日冥想**: 自动提炼当日重要事件到MEMORY.md，删除噪音
- **每周大冥想**: 深度精简MEMORY.md，归档旧日记到`memory/archive/`

**技术实现**:
```python
# core/wal_manager.py
import json
from pathlib import Path
from datetime import datetime

class WALManager:
    """WAL协议管理器 - 在AI响应前先写入状态"""
    
    def __init__(self, workspace: Path):
        self.session_state_file = workspace / ".workbuddy" / "session-state.md"
        self.session_state_file.parent.mkdir(parents=True, exist_ok=True)
    
    def write_before_ai_call(self, context: dict):
        """WAL协议：在调用AI前先持久化状态"""
        state = {
            "timestamp": datetime.now().isoformat(),
            "active_task": context.get("task"),
            "current_file": context.get("file"),
            "last_operation": context.get("operation"),
            "pending_data": context.get("pending")
        }
        # 同步写入文件（阻塞式，确保落盘）
        self._write_state(state)
    
    def recover_on_startup(self) -> dict:
        """启动时恢复上次会话状态"""
        if not self.session_state_file.exists():
            return {}
        return self._read_state()
```

#### L2: LanceDB向量库（温记忆）

**文件位置**: `data/vector_store/`

**向量类型**:
1. **章节向量**: 每章小说内容的语义向量（用于长篇连贯性召回）
2. **知识库向量**: 物理/化学/生物/宗教/哲学等知识点向量
3. **风格向量**: 经典文学作品的风格特征向量

**技术选型**（ADR-001）:
- **向量数据库**: LanceDB（嵌入式、零配置、支持向量检索）
- **嵌入模型**: OpenAI text-embedding-3-small（或本地all-MiniLM-L6-v2）
- **检索方式**: 余弦相似度（cosine similarity）top-10召回

**集成方案**:
```python
# infrastructure/vector_store.py
import lancedb
from lancedb.embeddings import EmbeddingFunctionRegistry, TextEmbeddingFunction
from lancedb.pydantic import LanceModel, Vector

class NovelVectorStore:
    """小说向量存储 - 实现长篇连贯性召回"""
    
    def __init__(self, db_path: str = "data/vector_store"):
        self.db = lancedb.connect(db_path)
        self.registry = EmbeddingFunctionRegistry.get_instance()
        self.embed_func = self.registry.get("openai").create()  # 或本地模型
    
    def create_chapter_table(self):
        """创建章节向量表"""
        class ChapterVector(LanceModel):
            chapter_id: str
            content: str
            vector: Vector(self.embed_func.ndims()) = self.embed_func.VectorField()
            metadata: dict
        
        self.db.create_table("chapters", schema=ChapterVector, mode="overwrite")
    
    def add_chapter(self, chapter_id: str, content: str, metadata: dict = None):
        """添加章节向量"""
        table = self.db.open_table("chapters")
        table.add([{
            "chapter_id": chapter_id,
            "content": content,
            "vector": self.embed_func(content),
            "metadata": metadata or {}
        }])
    
    def recall_similar_context(self, query: str, top_k: int = 10) -> list:
        """召回相似上下文 - 实现照相记忆"""
        table = self.db.open_table("chapters")
        results = table.search(query).limit(top_k).to_pandas()
        return results.to_dict('records')
```

```python
import subprocess
from pathlib import Path

class GitNotesManager:
    """Git-Notes分支感知记忆管理"""
    
    def __init__(self, repo_root: Path):
        self.repo_root = repo_root
    
    def add_note(self, commit_hash: str, note: str, ref: str = "memory"):
        """添加Git笔记"""
        subprocess.run([
            "git", "notes", "--ref", ref, "add", "-m", note, commit_hash
        ], cwd=self.repo_root, check=True)
    
    def get_notes_for_branch(self, branch: str, ref: str = "memory") -> list:
        """获取分支特定的记忆"""
        result = subprocess.run([
            "git", "log", branch, "--notes=" + ref, "--pretty=%H %N"
        ], cwd=self.repo_root, capture_output=True, text=True)
        return self._parse_notes(result.stdout)
```

#### 3.4.1 章节向量编码

#### 3.4.2 上下文智能召回

**召回策略**（参考OpenClaw的memory_search）:
```python
# core/context_recall.py
class ContextRecaller:
    """上下文智能召回 - 实现照相记忆"""
    
    def __init__(self, vector_store: NovelVectorStore):
        self.vector_store = vector_store
    
    def recall_for_new_chapter(self, chapter_outline: str) -> str:
        """为新章节召回相关上下文"""
        # 1. 从大纲提取关键词
        keywords = self.extract_keywords(chapter_outline)
        
        # 2. 向量检索召回 top-10 相似章节
        similar_chapters = self.vector_store.recall_similar_context(
            query=chapter_outline,
            top_k=10
        )
        
        # 3. 构建上下文摘要
        context_summary = self.build_context_summary(similar_chapters)
        
        return context_summary
    
    def build_context_summary(self, chapters: list) -> str:
        """构建上下文摘要 - 避免超出token限制"""
        summary_parts = []
        for chapter in chapters:
            summary_parts.append(f"【第{chapter['chapter_id']}章】{chapter['content'][:200]}...")
        return "\n".join(summary_parts)
```

```python
        self.recaller = ContextRecaller()
    
    def execute(self, context: dict) -> dict:
        """执行一致性检查"""
        new_chapter = context.get("new_chapter")
        existing_chapters = context.get("existing_chapters")
        
        # 1. 召回相似上下文
        recalled_context = self.recaller.recall_for_new_chapter(new_chapter)
        
        # 2. 构建检查prompt
        prompt = self.build_check_prompt(new_chapter, recalled_context)
        
        # 3. 调用LLM检查冲突
        conflicts = self.call_llm(prompt)
        
        return {
            "conflicts": conflicts,
            "recall_context": recalled_context
        }
    
    def build_check_prompt(self, new_chapter: str, recalled_context: str) -> str:
        """构建一致性检查prompt"""
        return f"""
你是一位专业的小说编辑，负责检测长篇小说中的逻辑冲突。

## 参考上下文（前文相关章节摘要）
{recalled_context}

## 新章节内容
{new_chapter}

## 检查任务
请检测新章节与前文是否存在以下冲突：
1. 人物行为矛盾（如性格突变、前后行为不一致）
2. 情节冲突（如时间线错乱、事件前后矛盾）
3. 世界观冲突（如设定前后不一致）

## 输出格式
如果发现冲突，请按以下格式输出：
- 冲突类型: [人物/情节/世界观]
- 冲突描述: [具体描述]
- 严重程度: [P0/P1/P2]
- 修复建议: [建议如何修复]

如果没有冲突，输出："无冲突"
"""
```

**冲突识别准确率目标**: ≥90%

## 五、关键决策记录（ADR）

### ADR-001: 采用LanceDB作为向量数据库

**决策时间**: 2026-03-25  
**决策者**: 后端架构师  
**状态**: 已批准  

**背景**:
- 需要向量数据库存储章节向量和知识库向量
- 备选方案: LanceDB / Pinecone / Milvus / ChromaDB

**决策理由**:
1. **零配置**: LanceDB是嵌入式数据库，无需启动独立服务
2. **性能优秀**: 基于Rust实现，检索速度快
3. **Python友好**: 官方提供完整的Python API
4. **成本低**: 无需云服务费用，本地存储
5. **符合技术栈锁定**: 不引入新的外部依赖

**后果**:
- 正面: 简化部署，降低成本，易于集成
- 负面: 不支持分布式部署（单机场景无影响）

### ADR-002: 采用OpenClaw 5层记忆架构

**决策时间**: 2026-03-25  
**决策者**: 高级项目经理  
**状态**: 已批准  

**背景**:
- 需要实现长期记忆，解决AI记忆宽度限制
- 备选方案: OpenClaw mem9 / Mem0 / 自研方案

**决策理由**:
1. **成熟方案**: OpenClaw已在生产环境验证（GitHub 310k+ Star）
2. **文件即真相**: 纯文本Markdown存储，透明可编辑
3. **WAL协议**: 确保数据不丢失，中断可恢复
4. **分层设计**: 5层架构从热到冷，检索效率高
5. **社区支持**: 大量实践经验和最佳实践文档

**后果**:
- 正面: 借鉴成熟架构，降低设计风险
- 负面: 需要新增多个管理模块（session_state、wal_manager等）

### ADR-003: 知识库双层设计（分类库+通用库）

**决策时间**: 2026-03-25  
**决策者**: 后端架构师  
**状态**: 已批准  

**背景**:
- 需要构建知识库支撑评分反馈和风格学习
- 备选方案: 单层知识库 vs 双层知识库

**决策理由**:
1. **精准召回**: 分类知识库按题材隔离，减少跨题材噪声
2. **效率优化**: 只加载当前题材的知识库，节省内存
3. **扩展灵活**: 新增题材只需添加新分类，不影响现有库
4. **通用共享**: 通用知识库（哲学/逻辑）跨题材复用

**后果**:
- 正面: 召回准确率提升，内存占用优化
- 负面: 需要维护多个知识库文件

### ADR-004: 本地Embedding模型部署

**决策时间**: 2026-03-25  
**决策者**: 后端架构师  
**状态**: 已批准  

**背景**:
- 向量召回需要Embedding模型生成向量
- 备选方案: OpenAI API vs 本地模型

**决策理由**:
1. **成本控制**: 本地模型免费，OpenAI API按token收费
2. **无网络依赖**: 本地模型离线可用，适合断网环境
3. **隐私保护**: 数据不上传第三方，符合安全要求
4. **性能稳定**: 无API限流和延迟问题

**技术方案**:
- 模型: all-MiniLM-L6-v2（384维向量）
- 缓存目录: `F:\sentence_transformers_cache\`
- 镜像加速: `https://hf-mirror.com`

**后果**:
- 正面: 零成本、离线可用、隐私安全
- 负面: 首次下载需约90MB，占用磁盘空间

### ADR-005: 模型部署与备份策略

**决策时间**: 2026-03-25  
**决策者**: 高级项目经理

**背景**:

| `项目/Qwen/` | 备份分发 | Qwen-7B-Chat |
