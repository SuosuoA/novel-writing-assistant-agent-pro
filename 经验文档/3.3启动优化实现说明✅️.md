## 2. 架构设计

### 2.1 分层加载策略

```
┌─────────────────────────────────────────────────────┐
│                    启动流程                          │
├─────────────────────────────────────────────────────┤
│  1. 创建主窗口（隐藏）                                │
│  2. 加载核心层（CORE）← 同步，必须完成                │
│  3. 显示主窗口                                       │
│  4. 异步加载其他层（PLUGIN/AGENT/UI）                │
└─────────────────────────────────────────────────────┘
```

### 2.2 优先级定义

| 优先级 | 名称 | 模块示例 | 加载时机 |
|--------|------|----------|----------|
| 0 | CORE | ConfigManager, EventBus, ServiceLocator | 同步，必须完成 |
| 10 | PLUGIN | PluginRegistry, PluginLoader | 异步加载 |
| 20 | AGENT | MasterAgent, AsyncHandler | 异步加载 |
| 30 | UI | UI API, Database | 异步加载 |

### 2.3 核心组件

#### LazyLoader（延迟加载管理器）

- **职责**：模块注册、依赖解析、加载调度
- **特性**：
  - 拓扑排序解析依赖关系
  - 支持同步/异步加载
  - 加载状态跟踪
  - 统计信息收集

#### OptimizedLauncher（优化启动器）

- **职责**：启动流程管理、窗口控制
- **特性**：
  - 隐藏窗口启动
  - 核心层同步加载
  - 非核心层异步加载
  - 加载完成回调

## 5. 最佳实践

### 5.1 模块注册原则

1. **核心层模块**：无依赖或仅依赖同层模块
2. **插件层模块**：可依赖核心层模块
3. **Agent层模块**：可依赖核心层和插件层模块
4. **UI层模块**：可依赖所有层模块

### 5.2 延迟初始化模式

对于需要运行时参数的模块，采用延迟初始化：

```python
# 方案1：延迟到实际使用时初始化
def load_master_agent():
    from .event_bus import get_event_bus
    from agents.master_agent import MasterAgent
    event_bus = get_event_bus()  # 获取已加载的单例
    return MasterAgent(event_bus)

# 方案2：注册占位实例，运行时初始化
def load_ui_api():
    from .ui_api import CoreServiceManager
    return CoreServiceManager.__new__(CoreServiceManager)
```

### 5.3 避免循环依赖

1. 使用拓扑排序自动检测循环依赖
2. 通过依赖注入（ServiceLocator）解耦
3. 使用事件总线（EventBus）异步通信
