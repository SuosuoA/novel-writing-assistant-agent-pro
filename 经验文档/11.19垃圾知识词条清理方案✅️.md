## 一、需求概述

### 1.1 核心需求

**目标**: 创建独立的知识库验证Agent，负责批量检查和清理低质量、重复的知识词条。

**关键特性**:
1. ✅ **强查重能力** - 检测重复、相似度过高的词条
2. ✅ **质量评估** - 基于多维度标准评估词条质量
3. ✅ **自动清理** - 删除不合格词条（不删除文件/文件夹）
4. ✅ **Claw集成** - 删除操作传输数据给Claw，优化AI生成
5. ✅ **GUI集成** - 知识库界面添加"清除垃圾"+"恢复清理"按钮
6. ✅ **模块化设计** - 独立运行，不影响其他模块流畅度

### 1.3 设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| **模块化** | 独立运行，不与其他模块强耦合 | ToolPlugin接口，最小化依赖 |
| **性能优化** | 不影响系统流畅度 | 异步执行，进度反馈，SimHash算法 |
| **安全性** | 不删除文件/文件夹 | 只清理JSON内词条，备份机制，路径安全检查 |
| **可扩展性** | 保留扩展接口 | 策略模式，支持自定义规则，配置化阈值 |
| **错误容忍** | 异常不中断流程 | Try-except保护，日志记录 |
| **架构一致性** | 符合微内核架构 | ToolPlugin + EventBus + Pydantic v2 |

## 四、查重算法设计

### 4.1 三层查重机制

```
┌─────────────────────────────────────────────────────────┐
│                   查重策略组合                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 完全匹配去重 (快速, O(n))                        │
│     └─→ 标题 + 关键词完全相同                       │
│                                                         │
│  2. 语义相似度去重 (中等)                           │
│     └─→ 使用LanceDB向量检索，阈值0.85            │
│                                                         │
│  3. 内容重叠度去重 (SimHash优化, O(n log n))        │
│     └─→ Jaccard相似度，阈值0.70                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
