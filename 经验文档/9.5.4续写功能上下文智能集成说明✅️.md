## 一、概述

### 1.1 功能定位

续写功能上下文智能集成是续写生成器插件V1.1版本的核心增强功能，实现了自动读取项目设定和智能加载前文概要的能力。

### 1.2 核心能力

| 能力 | 描述 |
|------|------|
| 自动读取设定 | 自动从项目管理器或文件系统读取大纲、人设、世界观 |
| 前文概要加载 | 选择章节时自动加载该章节之前的剧情概要，避免重复 |
| 多源数据融合 | 支持项目管理器接口、文件系统扫描、直接传入三种数据源 |
| 智能上下文构建 | 根据续写需求自动构建完整的上下文Prompt |


## 三、ProjectContext数据结构

### 3.1 字段定义

```python
@dataclass
class ProjectContext:
    """项目上下文数据结构"""
    project_path: str = ""                    # 项目根目录路径
    outline: str = ""                         # 大纲内容
    characters: List[Dict] = field(...)       # 人物设定列表

    recent_chapters: List[str] = field(...)   # 最近章节内容
    chapter_summaries: Dict[str, str] = ...   # 章节ID -> 概要
    current_chapter_id: Optional[str] = None  # 当前章节ID
    last_updated: Optional[datetime] = None   # 最后更新时间
```

## 四、核心功能实现

### 4.1 自动读取项目设定

**数据源优先级**:

1. **项目管理器接口**（最高优先级）
   ```python
   if self._project_manager:
       outline = self._project_manager.get_outline()
       characters = self._project_manager.get_characters()
       worldview = self._project_manager.get_worldview()
   ```

2. **文件系统扫描**（次优先级）
   ```python
   # 扫描目录模式
   outline_patterns = ["大纲", "outline", "章节大纲"]
   character_patterns = ["人物设定", "characters", "人物"]
   worldview_patterns = ["世界观", "worldview", "设定"]
   
   # 支持的文件格式
   extensions = ['.txt', '.md', '.yaml', '.json']
   ```

3. **直接传入数据**（最低优先级）
   ```python
   # 在ContinuationRequest中直接传入
   request = ContinuationRequest(
       outline="第一章：主角登场...",
       characters=[{"name": "张三", ...}],
       worldview="现代都市背景..."
   )
   ```

### 4.2 章节选择时自动加载前文概要

**实现逻辑**:

```python
def _get_chapter_summaries_before(self, chapter_id: str) -> Dict[str, str]:
    """
    获取指定章节之前的所有章节概要
    
    用于续写时避免重复已发生的内容。
    """
    summaries = {}
    
    # 1. 获取章节列表
    chapter_list = self._project_manager.list_chapters()
    
    # 2. 找到当前章节位置
    current_index = find_chapter_index(chapter_list, chapter_id)
    
    # 3. 获取之前章节的概要
    for chapter in chapter_list[:current_index]:
        content = self._project_manager.get_chapter_content(chapter['title'])
        summary = self._generate_chapter_summary(content, max_length=500)
        summaries[chapter['id']] = summary
    
    return summaries
```

**概要生成策略**:

```python
def _generate_chapter_summary(self, content: str, max_length: int = 500) -> str:
    """
    生成章节概要
    
    当前实现：取前500字符并在句号处截断
    TODO: 后续可以使用LLM生成更智能的概要
    """
    summary = content[:max_length]
    last_period = summary.rfind('。')
    if last_period > max_length // 2:
        summary = summary[:last_period + 1]
    return summary + "..."
```

**格式2**: - 姓名 | 角色 | 性格

## 五、插件集成

### 5.1 初始化集成

```python
class ContinuationGeneratorPlugin(ContinuationPlugin):
    def __init__(self):
        # V1.1新增: 项目上下文管理器
        self._context_manager: Optional[ProjectContextManager] = None
    
    def initialize(self, context: PluginContext) -> bool:
        # 初始化上下文管理器
        self._context_manager = ProjectContextManager()
        self._context_manager.set_logger(self._logger)
        
        # 从服务定位器获取项目管理器
        if context.service_locator:
            project_manager = context.service_locator.get("project_manager")
            if project_manager:
                self._context_manager.set_project_manager(project_manager)
        
        # 从配置获取项目路径
        save_path = config.get('save_path', '')
        if save_path:
            self._context_manager.set_project_path(save_path)
```

### 5.3 使用示例

**基本使用**:

```python
from plugins.continuation_generator_v1.plugin import ContinuationGeneratorPlugin

# 创建插件实例
plugin = ContinuationGeneratorPlugin()
plugin.set_api_client(openai_client)
plugin.set_project_manager(project_manager)

# 自动加载上下文并生成续写
result = plugin.generate_continuation_with_auto_context(
    starting_text="月光洒在窗台上，她轻轻叹了口气。",
    word_count=500,
    chapter_id="ch5"  # 当前章节ID
)

print(result.text)
```

**手动控制上下文**:

```python
# 手动加载上下文
context = plugin.auto_load_context(
    chapter_id="ch5",
    include_recent_chapters=3
)

# 查看加载的上下文
print(f"大纲: {context.outline}")
print(f"人物: {[c['name'] for c in context.characters]}")
print(f"世界观: {context.worldview}")
print(f"前文概要: {context.chapter_summaries}")

# 使用上下文生成续写
request = ContinuationRequest(
    starting_text="...",
    word_count=500,
    outline=context.outline,
    characters=context.characters,
    worldview=context.worldview,
    previous_chapters=list(context.chapter_summaries.values())
)
result = plugin.generate_continuation(request)
```

## 六、Prompt构建

### 6.1 完整Prompt结构

```
【章节大纲】
第一章：主角登场
第二章：初遇对手
...

【人物设定】
- 张三（主角）：勇敢、善良
- 李四（配角）：聪明

【世界观设定】
现代都市背景，时间设定在2024年...

【写作风格】
基调：轻松，节奏：中等

【前文参考】
第1章概要：张三在城市中醒来...
第2章概要：张三遇到了李四...

【续写方向】自然续写
按照情节的自然发展进行续写...

【起始文本】
月光洒在窗台上，她轻轻叹了口气。

【续写要求】
1. 从起始文本自然衔接，保持语气和风格一致
2. 目标字数：约500字
3. 内容要充实，避免空洞描述
4. 注意情节推进和人物刻画
```

### 6.2 前文概要的作用

1. **避免重复**: 知道前面发生了什么，不会重复描写相同事件
2. **保持连贯**: 人物行为和情节发展符合前文逻辑
3. **提升质量**: 上下文信息越完整，AI生成的内容越连贯
