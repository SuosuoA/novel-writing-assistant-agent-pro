## 三、DeepSeek API配置修复

### 3.1 问题分析

**症状**: GUI显示"未配置API"，无法调用DeepSeek

**根因**:
1. `config_service.py`的`get_ai_config()`没有读取`deepseek.base_url`
2. `ai_service_manager.py`的`_get_ai_config()`使用了错误的默认值

### 3.2 修复内容

**config_service.py**（第394-409行）:
```python
# base_url（优先从provider配置块读取）
# DeepSeek配置
if "deepseek" in all_config and isinstance(all_config["deepseek"], dict):
    ai_config["base_url"] = all_config["deepseek"].get("base_url", "https://api.deepseek.com")
elif "base_url" in all_config:
    ai_config["base_url"] = all_config["base_url"]
```

**ai_service_manager.py**（第188-221行）:
```python
# 读取base_url（优先级：deepseek.base_url > base_url > local_url）
if "deepseek" in config and isinstance(config["deepseek"], dict):
    ai_config["base_url"] = config["deepseek"].get("base_url", "https://api.deepseek.com")
elif "base_url" in config:
    ai_config["base_url"] = config["base_url"]
else:
    ai_config["base_url"] = config.get("local_url", "http://localhost:11434/v1")
```
