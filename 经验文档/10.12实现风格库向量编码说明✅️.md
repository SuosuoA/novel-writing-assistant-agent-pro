## 三、风格库设计详解

### 3.1 经典文学风格（classical.json）

**风格定义**:

- **叙事模式**: 第三人称全知视角
- **节奏**: 慢节奏，注重铺垫
- **基调**: 庄重、深沉、内敛
- **词汇**: 书面语、文言词汇、成语典故
- **句式**: 长句为主，复杂句式，修辞丰富
- **描写风格**: 细腻描写，注重环境、心理、细节
- **对话风格**: 含蓄委婉，潜台词丰富
- **人物深度**: 人物立体，性格复杂，有深度
- **主题深度**: 社会批判，人性探索，哲学思考

**代表作家作品**:

1. 曹雪芹《红楼梦》（古典、含蓄、隐喻、象征）
2. 鲁迅《阿Q正传》（现实主义、讽刺、心理描写、社会批判）
3. 老舍《骆驼祥子》（现实主义、环境描写、细节刻画、悲悯）
4. 沈从文《边城》（诗意、田园、纯真、自然）
5. 钱钟书《围城》（讽刺、幽默、知识分子、婚姻）

**写作指南**:

- ✅ 注重细节描写，通过环境、动作、心理展现人物性格
- ✅ 使用含蓄委婉的表达方式，避免直白
- ✅ 运用象征、隐喻等修辞手法，增加文本深度
- ✅ 注重社会背景描写，体现时代特征
- ✅ 人物对话要有潜台词，言外之意丰富
- ✅ 适当使用文言词汇和成语典故，提升文学性
- ✅ 叙事节奏宜慢不宜快，给读者思考空间
- ✅ 主题要有社会批判或人性探索的深度

**避免模式**:

- ❌ 过于直白的情感表达
- ❌ 简单粗暴的情节推进
- ❌ 缺乏深度的表面描写
- ❌ 人物扁平化、脸谱化
- ❌ 快餐式的叙事节奏
- ❌ 网络流行语和现代俚语
- ❌ 过于戏剧化的冲突设计
- ❌ 缺乏现实逻辑的情节

### 3.2 现代文学风格（modern.json）

**风格定义**:

- **叙事模式**: 第一人称/第三人称限制视角/多视角切换
- **节奏**: 中等节奏，时空跳跃
- **基调**: 疏离、荒诞、内省、反讽
- **词汇**: 口语化、日常用语、新词汇
- **句式**: 长短句结合，碎片化句式，意识流
- **描写风格**: 主观感受，印象式描写，象征暗示
- **对话风格**: 日常口语，生活化，真实感
- **人物深度**: 内心世界复杂，存在主义困境
- **主题深度**: 个体困境，现代性批判，存在意义

**代表作家作品**:

1. 卡夫卡《变形记》（荒诞、存在主义、异化、疏离）
2. 村上春树《挪威的森林》（意识流、回忆、孤独、感伤）
3. 王小波《黄金时代》（黑色幽默、荒诞、反叛、自由）
4. 余华《活着》（口语化、生活化、朴素、真实）
5. 史铁生《我与地坛》（内省、哲思、沉静、超然）

**写作指南**:

- ✅ 运用意识流手法，展现人物内心世界的复杂性
- ✅ 采用碎片化叙事，时空跳跃，打破线性叙事
- ✅ 注重主观感受和印象式描写，而非客观现实
- ✅ 使用日常口语和生活化对话，增强真实感
- ✅ 探讨现代人的存在困境、孤独、异化等主题
- ✅ 适当运用荒诞、反讽、黑色幽默等手法
- ✅ 人物要有复杂的内心世界和存在主义困境
- ✅ 叙事视角可以灵活切换，展现多元视角

**避免模式**:

- ❌ 过于宏大的叙事和史诗般的结构
- ❌ 理想化的人物和完美的人格
- ❌ 直白的道德说教和主题表达
- ❌ 传统的线性叙事和封闭结局
- ❌ 过于华丽的辞藻和修辞堆砌
- ❌ 脸谱化的人物和简单的善恶对立
- ❌ 过于戏剧化的冲突和巧合
- ❌ 缺乏现代意识的传统叙事方式

**风格定义**:

- **叙事模式**: 第一人称/第三人称限制视角（强代入感）
- **节奏**: 快节奏，紧凑情节，频繁爽点
- **基调**: 轻松、热血、幽默、爽快
- **词汇**: 网络流行语、游戏术语、二次元词汇
- **句式**: 短句为主，简洁明快，易读性强
- **描写风格**: 点到为止，重在情节推进和爽点设置
- **对话风格**: 口语化，网络梗，幽默风趣
- **人物深度**: 主角光环明显，配角功能化，反派脸谱化
- **主题深度**: 升级打怪、逆袭复仇、爽文内核

**代表作家作品**:

1. 猫腻《庆余年》（穿越、权谋、爽文、轻松）
2. 唐家三少《斗罗大陆》（玄幻、战斗、爽点、升级）
3. 天蚕土豆《斗破苍穹》（装逼、打脸、逆袭、爽文）
4. 我吃西红柿《星辰变》（修真、突破、升级、爽点）
5. 烽火戏诸侯《雪中悍刀行》（武侠、幽默、轻松、热血）

**写作指南**:

- ✅ 设置频繁的爽点（升级、打脸、逆袭、装逼打脸）
- ✅ 节奏要快，情节紧凑，避免冗长的铺垫
- ✅ 主角要有明显的成长线和爽点设置
- ✅ 使用网络流行语和梗，增加趣味性
- ✅ 对话要轻松幽默，符合网文读者的阅读习惯
- ✅ 金手指设置要合理，避免过于逆天
- ✅ 反派要脸谱化，打脸要爽，节奏要快
- ✅ 代入感要强，让读者能够共情主角

**避免模式**:

- ❌ 过于沉重的主题和悲剧性结局
- ❌ 冗长的环境描写和心理刻画
- ❌ 复杂的社会批判和哲学思考
- ❌ 过于含蓄委婉的表达方式
- ❌ 慢节奏的叙事和漫长的铺垫
- ❌ 复杂的叙事结构和多元视角
- ❌ 过于文言化的词汇和句式
- ❌ 缺乏爽点的平淡叙事

## 五、向量编码集成方案

### 5.1 编码流程

```
风格库JSON文件（styles/*.json）
    ↓
读取sample_texts
    ↓
调用NovelVectorStore.add_style()
    ↓
使用OpenAI Embedding API生成向量
    ↓
存储到LanceDB的styles表
    ↓
风格匹配时使用recall_similar_styles()
```

### 5.2 集成代码示例

```python
from infrastructure.vector_store import get_vector_store
from pathlib import Path
import json

# 获取向量存储实例
vector_store = get_vector_store(
    embedding_type="openai",
    api_key="sk-..."
)

# 加载风格库
styles_dir = Path("styles")
for style_file in styles_dir.glob("*.json"):
    with open(style_file, 'r', encoding='utf-8') as f:
        style_data = json.load(f)
    
    # 添加风格向量
    for sample in style_data["sample_texts"]:
        vector_store.add_style(
            style_id=sample["text_id"],
            author=sample["author"],
            content=sample["excerpt"],
            style_features={
                "style_category": style_data["style_category"],
                "style_tags": sample["style_tags"]
            },
            metadata={
                "work": sample["work"],
                "word_count": sample["word_count"]
            }
        )

# 风格匹配
query_text = "那僧便念咒书符，大展幻术..."
similar_styles = vector_store.recall_similar_styles(
    query=query_text,
    top_k=3
)

for result in similar_styles:
    print(f"风格: {result.metadata['style_category']}")
    print(f"作者: {result.metadata['author']}")
    print(f"相似度: {result.score}")
```

## 七、与知识库检索的协同

### 7.1 双重检索机制

```
用户创作文本
    ↓
1. 风格检索：recall_similar_styles() → 确定创作风格
    ↓
2. 知识检索：recall_knowledge() → 召回相关知识
    ↓
3. 风格指导：应用writing_guidelines和avoid_patterns
    ↓
4. 知识验证：检测内容是否符合知识库
    ↓
输出符合风格且有知识支撑的创作内容
```

### 7.2 集成到ValidationScores

**新增维度**: 风格一致性（10%）

```python
from core.knowledge_retriever import get_knowledge_retriever
from infrastructure.vector_store import get_vector_store

# 风格验证
def validate_style(content: str, target_style: str) -> float:
    vector_store = get_vector_store()
    similar_styles = vector_store.recall_similar_styles(content, top_k=5)
    
    # 检查匹配的风格是否为目标风格
    match_count = sum(
        1 for s in similar_styles 
        if s.metadata.get('style_category') == target_style
    )
    
    return match_count / 5  # 返回匹配率
```
