## 一、功能概述

### 1.1 什么是续写功能

续写功能是Novel Writing Assistant-Agent Pro项目的智能写作辅助工具，能够根据已有的文本内容，自动生成连贯、符合风格的后续内容。该功能基于大语言模型（LLM），结合项目设定（大纲、人物、世界观、风格），生成高质量的续写文本。

### 1.2 核心能力

| 能力 | 描述 |
|------|------|
| 智能续写 | 根据起始文本和上下文自动生成后续内容 |
| 多版本生成 | 同时生成多个不同风格的版本供选择 |
| 方向控制 | 支持10种续写方向（自然/剧情推进/高潮铺垫等） |
| 上下文感知 | 自动读取项目设定，保持内容一致性 |
| 版本管理 | 可视化管理多个版本，快速切换和选择 |
| 灵活保存 | 支持保存为新章节、追加到章节、导出文件 |

### 1.3 适用场景

- 章节创作遇到瓶颈，需要AI辅助推进情节
- 快速生成多个版本，比较选择最佳方案
- 保持风格一致性的批量内容生成
- 世界观和人物设定的智能融合

## 二、快速开始

### 2.1 进入续写功能

1. 启动Novel Writing Assistant-Agent Pro
2. 在左侧导航栏点击"工作台"
3. 在工作台标签页中点击"续写"

### 2.2 基本操作流程

```
步骤1：输入原文 → 步骤2：设置参数 → 步骤3：开始续写 → 步骤4：查看结果 → 步骤5：保存
```

**步骤1：输入原文**

在"原文内容"文本框中输入或粘贴起始文本：
- 直接粘贴文本
- 点击"选择文件"导入文本文件
- 点击"选择章节"从项目中选取

**步骤2：设置参数**

| 参数 | 选项 | 说明 |
|------|------|------|
| 字数 | 500/1000/2000/自定义 | 目标生成字数（100-5000） |
| 续写方向 | 10种方向 | 选择情节发展方向 |
| 创意温度 | 0.0-1.0 | 数值越高创意性越强 |
| 生成模式 | 单次/多版本 | 生成单个或多个版本 |

**步骤3：开始续写**

点击"开始续写"按钮，系统将：
1. 自动加载项目设定（大纲、人物、世界观）
2. 根据参数构建提示词
3. 调用大模型API生成内容
4. 显示生成进度和结果

**步骤4：查看结果**

续写完成后：
- 在结果预览区查看生成内容
- 多版本模式下可切换查看不同版本
- 查看字数、耗时、评分等统计信息

**步骤5：保存结果**

点击"保存结果"，选择保存方式：
- 保存为新章节
- 追加到当前章节
- 导出为文件

## 四、续写方向详解

### 4.1 方向类型

续写功能提供10种续写方向，满足不同创作需求：

| 方向 | 说明 | 适用场景 |
|------|------|----------|
| **自然续写** | 按大纲和上下文自然发展 | 常规续写，无特殊要求 |
| **剧情推进** | 加速情节发展，推进主线 | 需要推动故事进展时 |
| **高潮铺垫** | 为后续高潮做铺垫和准备 | 章节末尾，过渡到重要情节 |
| **人物深入** | 深入刻画人物内心世界 | 人物成长、内心独白场景 |
| **对话展开** | 以人物对话为主推进情节 | 人物互动、信息揭示 |
| **场景描写** | 侧重环境和场景的描绘 | 氛围营造、场景转换 |
| **悬念设置** | 设置悬念，引发读者好奇 | 章节结尾、转折点 |
| **冲突升级** | 加剧人物或情节的冲突 | 矛盾激化、对抗场景 |
| **情感渲染** | 强化情感表达和氛围 | 感情戏、高潮场景 |
| **转折揭示** | 揭示关键转折或秘密 | 剧情反转、真相揭露 |

### 4.2 方向选择建议

```
情节发展阶段          推荐方向
──────────────────────────────────
开篇介绍             → 自然续写 / 场景描写
人物出场             → 人物深入 / 对话展开
矛盾初现             → 冲突升级 / 剧情推进
高潮前奏             → 高潮铺垫 / 悬念设置
高潮场景             → 情感渲染 / 冲突升级
转折点               → 转折揭示 / 悬念设置
章节结尾             → 悬念设置 / 高潮铺垫
```

## 五、多版本管理

### 5.1 版本生成

**单次生成模式**：
- 生成1个版本
- 使用当前设置的创意温度
- 适合明确需求时使用

**多版本生成模式**：
- 同时生成3个版本
- 自动使用不同温度分布（0.6/0.8/1.0）
- 适合探索不同可能性时使用

### 5.2 版本列表

版本历史列表显示以下信息：

| 列 | 说明 |
|------|------|
| 版本 | V1, V2, V3... |
| 温度 | 生成时的创意温度 |
| 字数 | 实际生成的字数 |
| 评分 | 综合评分（0-1） |
| 状态 | 最佳版本标记"⭐最佳" |

### 5.3 版本操作

| 操作 | 方法 |
|------|------|
| 切换版本 | 点击版本列表项，或使用◀ ▶按钮 |
| 快速保存 | 双击版本列表项 |
| 选择最佳 | 点击"选择最佳版本"按钮 |
| 重新生成 | 点击"重新生成"按钮累积新版本 |

### 5.4 版本数量限制

- 最大版本数量：5个
- 达到上限后需先保存或清空才能继续生成
- 重新生成会累积版本而非覆盖

## 六、评分机制

### 6.1 评分维度

续写结果通过5个维度进行评分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 连贯性 | 30% | 续写与起始文本的衔接自然度 |
| 风格匹配 | 25% | 与原始风格的匹配程度 |
| 字数达标 | 20% | 是否接近目标字数 |
| 创意性 | 15% | 词汇多样性、句式变化 |
| 可读性 | 10% | 段落结构、标点使用 |

### 6.2 评分计算

```
总分 = 连贯性 × 0.30 + 风格匹配 × 0.25 + 字数达标 × 0.20 + 创意性 × 0.15 + 可读性 × 0.10
```

### 6.3 评分示例

```
版本1: 连贯性0.85, 风格匹配0.80, 字数达标0.90, 创意性0.70, 可读性0.85
总分 = 0.85×0.30 + 0.80×0.25 + 0.90×0.20 + 0.70×0.15 + 0.85×0.10 = 0.82

版本2: 连贯性0.78, 风格匹配0.85, 字数达标0.85, 创意性0.80, 可读性0.80
总分 = 0.78×0.30 + 0.85×0.25 + 0.85×0.20 + 0.80×0.15 + 0.80×0.10 = 0.81

结论: 版本1评分更高，标记为最佳版本
```

## 七、上下文智能集成

### 7.1 自动读取设定

续写功能会自动读取以下项目设定：

| 设定类型 | 数据来源 | 用途 |
|----------|----------|------|
| 章节大纲 | 项目管理器/文件系统 | 控制情节走向 |
| 人物设定 | 项目管理器/文件系统 | 保持角色一致性 |
| 世界观设定 | 项目管理器/文件系统 | 背景一致性 |
| 写作风格 | 风格学习模块 | 匹配写作风格 |
| 前文章节 | 项目章节列表 | 保持剧情连贯 |

### 7.2 数据源优先级

```
1. 项目管理器接口（最高优先级）
   └── 通过 get_outline(), get_characters(), get_worldview() 获取

2. 文件系统扫描（次优先级）
   └── 扫描 大纲/, 人物设定/, 世界观/, 章节/ 目录

3. 直接传入数据（最低优先级）
   └── 在续写请求中直接提供设定内容
```

### 7.3 前文概要加载

选择章节时，系统会自动加载该章节之前的剧情概要：
- 提取前5章的剧情概要
- 每章概要约500字
- 避免续写内容与前文重复
- 保持情节连贯性

## 八、重新生成模式

### 8.1 模式类型

点击"重新生成"可选择4种模式：

| 模式 | 温度 | 特点 |
|------|------|------|
| 创意模式 | 1.0 | 高创意性，可能有意想不到的内容 |
| 平衡模式 | 0.8 | 创意与稳定兼顾 |
| 保守模式 | 0.6 | 更稳定，与原文风格更接近 |
| 聚焦模式 | 0.5 | 最稳定，严格遵循设定 |

### 8.2 模式选择建议

```
需求场景              推荐模式
──────────────────────────────────
希望有新意/惊喜      → 创意模式
常规续写            → 平衡模式
需要严格符合设定     → 保守模式 / 聚焦模式
风格敏感的章节       → 聚焦模式
```

## 九、保存选项

### 9.1 保存方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 保存为新章节 | 创建新章节，保存续写内容 | 独立的新内容 |
| 追加到当前章节 | 将续写内容追加到原文后 | 延续当前章节 |
| 导出为文件 | 保存为txt文件到本地 | 备份或外部使用 |

### 9.2 保存流程

**保存为新章节**：
1. 点击"选择最佳版本"或"保存结果"
2. 弹出章节命名对话框
3. 输入章节名称（默认"第X章 续写内容"）
4. 确认保存

**追加到章节**：
1. 选择"追加到当前章节"
2. 系统自动合并原文和续写内容
3. 更新项目中的章节内容

**导出文件**：
1. 选择"导出为文件"
2. 选择保存位置和文件名
3. 导出为txt文件

## 十、最佳实践

### 10.1 原文准备

- 建议输入200-500字的起始文本
- 确保起始文本有明确的场景和人物
- 避免在句子中间截断
- 保持原文风格一致

### 10.2 参数设置

| 场景 | 字数 | 温度 | 方向 |
|------|------|------|------|
| 日常描写 | 500-1000 | 0.6-0.8 | 自然续写 |
| 紧张情节 | 1000-2000 | 0.8-1.0 | 剧情推进 |
| 情感场景 | 500-1000 | 0.7-0.9 | 情感渲染 |
| 章节结尾 | 300-500 | 0.6-0.8 | 悬念设置 |

### 10.3 提高续写质量

1. **完善项目设定**：确保大纲、人物、世界观设定完整
2. **选择合适方向**：根据情节需要选择续写方向
3. **多版本比较**：使用多版本模式生成，选择最佳结果
4. **合理设置温度**：温度过高可能偏离设定，过低可能缺乏创意
5. **迭代优化**：对不满意的结果使用重新生成

### 10.4 常见问题

**Q: 续写内容与前文不一致？**

A: 检查项目设定是否正确加载，可手动在请求中提供设定内容。确保前文章节已正确关联。

**Q: 续写内容风格与原文差异大？**

A: 降低创意温度（0.5-0.6），使用保守模式或聚焦模式。确保风格学习模块已学习原文风格。

**Q: 字数与目标差异较大？**

A: 字数控制在±20%范围内属于正常。如需更精确控制，可在续写后手动调整。

**Q: 生成速度较慢？**

A: 多版本生成需要调用多次API，耗时较长。如需快速预览，可先使用单次生成模式。

## 十一、API参考

### 11.1 ContinuationRequest参数

| 参数 | 类型 | 必需 | 默认值 | 说明 |
|------|------|------|--------|------|
| starting_text | str | ✅ | - | 起始文本 |
| word_count | int | ✅ | 500 | 目标字数（100-5000） |
| direction | str | ❌ | natural | 续写方向 |
| direction_hint | str | ❌ | None | 特定方向提示 |
| outline | str | ❌ | None | 章节大纲 |
| characters | List[Dict] | ❌ | None | 人物设定 |
| worldview | str | ❌ | None | 世界观设定 |
| style_profile | Dict | ❌ | None | 风格档案 |
| previous_chapters | List[str] | ❌ | None | 前文章节 |
| temperature | float | ❌ | 0.8 | 生成温度（0.0-2.0） |
| preserve_ending | bool | ❌ | True | 是否保留自然结尾 |

### 11.2 ContinuationResult字段

| 字段 | 类型 | 说明 |
|------|------|------|
| text | str | 续写生成的文本内容 |
| word_count | int | 实际生成字数 |
| metadata.model_name | str | 使用的模型名称 |
| metadata.provider | str | 模型提供商 |
| metadata.generation_time | float | 生成耗时（秒） |
| metadata.tokens_used | int | 消耗的token数 |
| metadata.coherence_score | float | 连贯性评分（0-1） |
| metadata.style_match_score | float | 风格匹配度（0-1） |
| success | bool | 是否成功 |
| error | str | 错误信息（如有） |
| suggestions | List[str] | 后续续写建议 |

### 11.3 主要方法

```python
# 单次续写
result = plugin.generate_continuation(request)

# 多版本生成
results = plugin.generate_multiple_versions(request, num_versions=3)

# 选择最佳版本
best_result, best_index, scores = plugin.select_best_version(results)

# 带自动上下文的续写
result = plugin.generate_continuation_with_auto_context(
    starting_text="...",
    word_count=500,
    chapter_id="ch5"
)

# 重新生成
result = plugin.regenerate(request, variation="creative")

# 流式生成
for chunk in plugin.stream_continuation(request):
    print(chunk, end='')
```
