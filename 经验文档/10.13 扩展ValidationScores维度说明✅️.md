## 三、技术实现

### 3.1 评分计算逻辑

**V1.7版本**（8维度加权）:
```python
def calculate_total(self) -> float:
    """计算总分（8维度加权）
    
    权重分配（V1.7版本）：
    - 字数: 8%
    - 知识点引用: 8%
    - 大纲: 13%
    - 风格: 19%
    - 人设: 19%
    - 世界观: 12%
    - 逆向反馈: 11%
    - 自然度: 10%
    
    注意：总权重为100%，无需归一化处理
    """
    raw_score = (
        self.word_count_score * 0.08
        + self.knowledge_reference_score * 0.08
        + self.outline_score * 0.13
        + self.style_score * 0.19
        + self.character_score * 0.19
        + self.worldview_score * 0.12
        + self.reverse_feedback_score * 0.11
        + self.naturalness_score * 0.10
    )
    
    self.total_score = raw_score
    return self.total_score
```

## 四、知识库一致性验证流程

### 4.1 验证流程

```
创作章节内容
    ↓
提取知识点关键词（物理/化学/生物/宗教/历史等）
    ↓
识别题材（科幻/玄幻/历史）→ 加载分类知识库
    ↓
向量检索召回 top-10 相关知识点
    ↓
检测一致性：
    - 物理定律是否违反（如超光速是否违反相对论）
    - 化学反应是否合理（如燃烧是否需要氧气）
    - 生物过程是否科学（如基因编辑是否符合遗传学）
    - 历史事件是否准确（如赤壁之战的时间地点）
    ↓
生成知识库一致性评分（0-1分）
    ↓
记录检测到的冲突到knowledge_conflicts字段
    ↓
记录召回的知识到recalled_knowledge字段
    ↓
集成到ValidationScores.total_score
```

### 4.2 冲突检测示例

**冲突记录格式**:
```json
{
  "conflict_type": "physics",
  "description": "飞船以超光速行驶但未提及相对论时间膨胀效应",
  "severity": "medium",
  "knowledge_id": "scifi-physics-001",
  "suggestion": "建议补充时间膨胀的描述，或设定特殊技术规避该效应"
}
```

## 六、架构意义

ValidationScores扩展知识库一致性维度作为OpenClaw mem9的**评分反馈优化**：
1. **知识库支撑**: 评分不再完全依赖关键词匹配，引入知识库验证
2. **专业性提升**: 检测生成内容是否违反物理/化学/生物等知识常识
3. **题材适配**: 根据题材加载对应知识库，精准验证
4. **冲突追踪**: 记录检测到的知识冲突，便于修复
5. **召回审计**: 记录召回的相关知识，便于验证准确性
