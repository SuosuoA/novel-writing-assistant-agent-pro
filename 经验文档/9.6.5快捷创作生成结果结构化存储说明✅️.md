## 一、概述

### 1.1 功能目标

快捷创作结果存储模块（ResultStorageManager）提供以下核心能力：

1. **JSON Schema定义** - 确保生成结果与现有项目文件格式兼容
2. **保存结果** - 将生成结果保存为新项目文件
3. **导入当前项目** - 将生成结果合并到当前项目的设定中
4. **冲突处理** - 处理导入时的命名冲突（保留用户原有设定）

### 1.2 与现有项目文件格式的兼容性

| 设定类型 | 文件路径 | 格式 |
|----------|----------|------|
| 世界观 | `世界观/世界观设定.json` | JSON Schema V1.0 |
| 大纲 | `大纲/大纲设定.json` | JSON Schema V1.0 |
| 人物 | `人物设定/{人物名}.json` | JSON Schema V1.0 |
| 情节 | `情节设定/{情节名}.json` | JSON Schema V1.0 |
| 项目元数据 | `project.json` | 项目信息 |

## 三、冲突处理策略

### 3.1 ConflictStrategy枚举

```python
class ConflictStrategy(str, Enum):
    """冲突处理策略枚举"""
    KEEP_ORIGINAL = "keep_original"      # 保留原有设定（默认）
    OVERWRITE = "overwrite"              # 覆盖原有设定
    MERGE = "merge"                      # 合并设定
    RENAME_NEW = "rename_new"            # 重命名新设定
```

### 3.2 策略说明

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| `keep_original` | 保留用户原有设定，跳过导入 | 保护用户已有工作成果（默认） |
| `overwrite` | 直接覆盖原有文件 | 用户确认需要更新设定 |
| `merge` | 合并新旧设定 | 智能合并（保留双方内容） |
| `rename_new` | 新文件添加时间戳后缀 | 保留两份设定供用户选择 |

### 3.3 冲突检测

冲突检测发生在以下情况：

1. **世界观冲突** - `世界观/世界观设定.json` 已存在
2. **大纲冲突** - `大纲/大纲设定.json` 已存在
3. **人物冲突** - `人物设定/{人物名}.json` 已存在同名人物
4. **情节冲突** - `情节设定/{情节名}.json` 已存在同名情节

## 四、核心API

### 4.1 ResultStorageManager类

```python
class ResultStorageManager:
    """快捷创作结果存储管理器"""
    
    def __init__(self, project_path: Optional[Path] = None):
        """初始化，可选指定项目路径"""
        
    def save_result(
        self,
        result: QuickCreationResult,
        project_name: str = "新建项目",
        output_path: Optional[Path] = None,
        format: str = "json"
    ) -> Path:
        """
        保存生成结果到新项目文件
        
        参数:
            result: 快捷创作结果
            project_name: 项目名称
            output_path: 输出路径（可选）
            format: 保存格式（json/markdown）
            
        返回:
            项目目录路径
        """
        
    def import_to_project(
        self,
        result: QuickCreationResult,
        conflict_strategy: ConflictStrategy = ConflictStrategy.KEEP_ORIGINAL,
        merge_mode: str = "append"
    ) -> ImportResult:
        """
        将生成结果导入到当前项目
        
        参数:
            result: 快捷创作结果
            conflict_strategy: 冲突处理策略
            merge_mode: 合并模式（append追加/replace替换）
            
        返回:
            ImportResult对象，包含导入结果详情
        """
```

## 五、使用示例

### 5.1 保存到新项目

```python
from plugins.quick_creator_v1 import QuickCreationPlugin
from core.models import QuickCreationRequest

# 创建插件实例
plugin = QuickCreationPlugin()
plugin.set_api_client(openai_client)

# 生成设定
request = QuickCreationRequest(
    keywords="修仙,逆袭,系统",
    target="all",
    genre="玄幻"
)
result = plugin.generate_all(request)

# 保存到新项目
project_path = plugin.save_result_to_project(
    result,
    project_name="修仙之路",
    output_path="E:/小说项目",
    format="json"
)
print(f"项目已保存到: {project_path}")
```

### 5.2 导入到当前项目

```python
# 设置当前项目路径
plugin.set_storage_project_path("E:/小说项目/我的小说")

# 导入设定（保留原有设定）
import_result = plugin.import_to_current_project(
    result,
    conflict_strategy="keep_original"
)

if import_result.success:
    print(f"导入成功: {import_result.imported_items}")
    if import_result.conflicts:
        print(f"发现冲突: {len(import_result.conflicts)} 处")
else:
    print(f"导入失败: {import_result.error}")
```

### 5.3 使用便捷函数

```python
from plugins.quick_creator_v1 import (
    save_quick_creation_result,
    import_quick_creation_result
)

# 保存
project_path = save_quick_creation_result(
    result,
    output_path="E:/小说项目",
    project_name="测试项目"
)

# 导入
import_result = import_quick_creation_result(
    result,
    project_path=project_path,
    strategy="rename_new"  # 重命名新设定
)
```
