## 一、实现概述

### 1.1 目标

支持Ollama、llama.cpp、vLLM、LocalAI等常见本地推理框架，通过HTTP API调用，实现四级容错机制。

### 1.2 核心特性

- ✅ **多框架支持**：Ollama、llama.cpp、vLLM、LocalAI
- ✅ **同步/流式生成**：generate_text、generate_text_stream
- ✅ **四级容错**：重试层（指数退避）→ 熔断层（CircuitBreaker）→ 模型回退 → 缓存降级
- ✅ **线程安全**：RLock保护所有共享状态
- ✅ **超时保护**：ThreadPoolExecutor + Future实现超时控制
- ✅ **Token估算**：基于字符数的快速估算算法
- ✅ **文本分析**：支持情感分析、摘要提取、关键词提取等

## 三、核心功能实现

### 3.1 初始化流程

```python
def __init__(self, config: Dict[str, Any]):
    """
    初始化LocalProvider
    
    Args:
        config: 配置字典，包含：
            - framework: 框架名称（ollama/llama-cpp/vllm/localai）
            - endpoint: API端点URL
            - model: 模型名称
            - temperature: 温度参数（可选）
            - timeout: 超时时间（可选）
            - max_retries: 最大重试次数（可选）
    """
    # 1. 基础配置
    self._framework = config.get("framework", "ollama")
    self._model = config.get("model", "llama3.1")
    self._endpoint = config.get("endpoint", "http://localhost:11434")
    
    # 2. 超时和重试配置
    self._timeout = config.get("timeout", 120)
    self._retry_policy = LocalRetryPolicy(
        max_retries=config.get("max_retries", 3)
    )
    
    # 3. 初始化客户端
    self._init_client()
```

### 3.2 同步生成

```python
def generate_text(
    self,
    prompt: str,
    system_prompt: Optional[str] = None,
    config: Optional[GenerationConfig] = None
) -> GenerationResult:
    """
    同步生成文本
    
    特性：
    1. 熔断器保护（OPEN状态拒绝请求）
    2. 重试机制（可重试错误最多3次）
    3. 超时保护（ThreadPoolExecutor + Future）
    4. 错误分类（连接/模型不存在/服务器错误）
    5. 多框架适配（Ollama格式 / OpenAI兼容格式）
    """
    # 1. 熔断器检查
    if not self._circuit_breaker.can_execute():
        raise AIProviderUnavailableError("熔断器开启，本地服务暂时不可用")
    
    # 2. 重试循环
    for attempt in range(self._retry_policy.max_retries + 1):
        try:
            # 3. 构建请求
            url = self._get_endpoint_url(use_chat=True)
            payload = self._build_request_payload(prompt, system_prompt, config)
            
            # 4. 发送请求
            response = requests.post(url, json=payload, timeout=self._timeout)
            response.raise_for_status()
            
            # 5. 解析响应（根据框架类型）
            result_data = response.json()
            
            if self._framework == "ollama":
                text = result_data.get("response", "")
                tokens_used = result_data.get("eval_count", 0)
            else:
                # OpenAI兼容格式
                text = result_data["choices"][0]["message"]["content"]
                tokens_used = result_data["usage"]["total_tokens"]
            
            # 6. 记录成功
            self._circuit_breaker.record_success()
            return GenerationResult(text=text, ...)
            
        except Exception as e:
            # 7. 错误处理
            if not self._should_retry(e, attempt):
                raise self._wrap_exception(e)
            time.sleep(self._retry_policy.get_delay(attempt))
```

### 3.3 流式生成

```python
def generate_text_stream(
    self,
    prompt: str,
    system_prompt: Optional[str] = None,
    config: Optional[GenerationConfig] = None
) -> Iterator[str]:
    """
    流式生成文本
    
    特性：
    1. 使用requests的stream=True模式
    2. 逐行解析JSON响应
    3. 根据框架类型提取文本片段
    4. yield生成内容
    """
    payload = self._build_request_payload(prompt, system_prompt, config)
    payload["stream"] = True  # 启用流式
    
    with requests.post(url, json=payload, stream=True, timeout=self._timeout) as response:
        for line in response.iter_lines():
            if line:
                data = json.loads(line.decode('utf-8'))
                
                if self._framework == "ollama":
                    yield data["response"]
                else:
                    # OpenAI兼容格式
                    yield data["choices"][0]["delta"]["content"]
```

### 3.4 多框架请求适配

```python
def _build_request_payload(
    self,
    prompt: str,
    system_prompt: Optional[str] = None,
    config: Optional[GenerationConfig] = None
) -> Dict[str, Any]:
    """
    构建请求payload
    
    根据不同框架构建不同的请求格式
    """
    if self._framework == "ollama":
        # Ollama原生格式
        return {
            "model": self._model,
            "prompt": prompt,
            "system": system_prompt,
            "stream": False,
            "options": {
                "temperature": config.temperature if config else self._temperature,
                "num_predict": config.max_tokens if config else 2048
            }
        }
    
    elif self._framework in ["llama-cpp", "vllm", "localai"]:
        # OpenAI兼容格式
        messages = []
        if system_prompt:
            messages.append({"role": "system", "content": system_prompt})
        messages.append({"role": "user", "content": prompt})
        
        return {
            "model": self._model,
            "messages": messages,
            "temperature": config.temperature if config else self._temperature,
            "max_tokens": config.max_tokens if config else 2048,
            "stream": False
        }
```

## 四、容错机制详解

### 4.1 四级容错架构

```
┌─────────────────────────────────────────┐
│ Level 1: 重试层                          │
│ - 指数退避（1s → 2s → 4s）               │
│ - Jitter随机抖动（避免惊群效应）         │
│ - 可重试错误：连接/超时/服务器错误       │
└─────────────────────────────────────────┘
                    ↓ 失败
┌─────────────────────────────────────────┐
│ Level 2: 熔断层                          │
│ - CLOSED → OPEN → HALF_OPEN → CLOSED    │
│ - 失败阈值：3次（比OnlineProvider更低）  │
│ - 冷却时间：30秒                         │
└─────────────────────────────────────────┘
                    ↓ 熔断
┌─────────────────────────────────────────┐
│ Level 3: 模型回退                        │
│ - llama3.1 → mistral → codellama        │
│ - 配置：fallback_models列表（待集成）    │
└─────────────────────────────────────────┘
                    ↓ 失败
┌─────────────────────────────────────────┐
│ Level 4: 缓存降级                        │
│ - CacheManager缓存历史响应（待集成）     │
│ - 缓存键：prompt hash                    │
└─────────────────────────────────────────┘
```

### 4.2 错误分类与重试策略

```python
RETRYABLE_ERRORS = {
    # 可重试错误
    "connection_error": True,       # 连接错误（本地服务未启动）
    "timeout": True,                # 超时
    "server_error": True,           # 服务器错误（5xx）
    "service_unavailable": True,    # 服务不可用（503）
    
    # 不可重试错误
    "model_not_found": False,       # 模型不存在
    "invalid_request": False,       # 参数错误
    "context_length_exceeded": False  # 上下文超长
}
```

### 4.3 熔断器配置差异

| 参数 | OnlineProvider | LocalProvider | 原因 |
|------|----------------|---------------|------|
| failure_threshold | 5次 | 3次 | 本地服务更不稳定 |
| success_threshold | 3次 | 2次 | 本地服务恢复更快 |
| timeout | 60秒 | 30秒 | 本地服务重启更快 |

## 六、性能优化

### 6.1 连接测试优化

- **初始不阻塞**：连接测试失败不阻塞初始化，允许后续使用时重试
- **超时设置**：连接测试超时5秒，快速反馈
- **降级策略**：连接失败时仍创建Provider实例，延迟到实际使用时检查

### 6.2 请求超时控制

```python
# 使用ThreadPoolExecutor实现超时控制
self._executor = ThreadPoolExecutor(max_workers=5)

def _execute_with_timeout(self, func, *args, **kwargs):
    """超时执行函数"""
    future = self._executor.submit(func, *args, **kwargs)
    try:
        return future.result(timeout=self._timeout)
    except FuturesTimeoutError:
        raise AIProviderTimeoutError(f"请求超时: {self._timeout}秒")
```

### 6.3 熔断器阈值优化

- **failure_threshold = 3**：本地服务更不稳定，阈值更低
- **success_threshold = 2**：恢复探测更快
- **timeout = 30秒**：本地服务重启更快
