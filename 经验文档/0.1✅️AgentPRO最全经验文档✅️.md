## 二、微内核架构设计

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户界面层 (Tkinter)                      │
└─────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────┐
│                      应用层 (MasterAgent)                        │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐              │
│  │ Thinker │ │Optimizer│ │Validator│ │Planner  │              │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘              │
└─────────────────────────────────────────────────────────────────┘
                                    │
                    ┌───────────────┼───────────────┐
                    ▼               ▼               ▼
┌───────────────────────┐ ┌───────────────┐ ┌───────────────────────┐
│   微内核核心 (Kernel)   │ │  插件系统      │ │   服务层 (Services)    │
│  ┌──────────────────┐ │ │ ┌───────────┐ │ │  ┌─────────────────┐  │
│  │  EventBus       │ │ │ │ PluginMgr │ │ │  │   AI API服务     │  │
│  │  PluginRegistry │ │ │ │ Loader    │ │ │  └─────────────────┘  │
│  │  ServiceLocator │ │ │ │ HotSwap   │ │ │  ┌─────────────────┐  │
│  │  ConfigManager   │ │ │ └───────────┘ │ │  │   文件服务      │  │
│  └──────────────────┘ │ │               │ │  └─────────────────┘  │
└───────────────────────┘ └───────────────┘ └───────────────────────┘
```

### 2.2 核心设计原则

**微内核原则**: 核心系统仅提供基础调度服务，所有业务功能通过插件实现。

**榫卯式集成**: 插件与核心通过标准接口"咬合"，接口定义严格，实现灵活。

**事件驱动**: 插件间通过EventBus通信，解耦依赖。

> **详细设计见**: 《1.1项目总体架构设计说明书修订执行版✅️.md》

## 十二、状态栏AI状态显示修复（V1.42.3）

### 12.1 修复背景

用户启动程序后报错，状态栏无法正确显示AI状态。错误信息显示Event发布方法签名错误和AIProviderState枚举值不存在。

### 12.2 核心问题

**问题1：Event发布方法签名错误**
- **位置**: `gui_main.py` 第12446行
- **根因**: `EventBus.publish()` 方法签名是 `publish(event_type: str, data=None, source=None)`
- **错误代码**: 传入了 `Event` 对象而非参数：`event_bus.publish(event)`
- **正确代码**: `event_bus.publish(event_type='config.changed', data={...}, source=...)`

**问题2：AIProviderState枚举值不存在**
- **位置**: `core/qwen_provider.py` 第192行、196行
- **根因**: `AIProviderState` 枚举只有 `UNINITIALIZED`, `READY`, `ERROR`, `BUSY`
- **错误代码**: `self._state = AIProviderState.UNAVAILABLE`
- **正确代码**: `self._state = AIProviderState.ERROR`

### 12.3 修复内容

**修复1：修正Event发布方式**

文件：`gui_main.py` 第12445-12454行

```python
# 修复前
event = Event(
    type='config.changed',
    data={'type': 'ai_config', 'data': ai_config},
    source='MainWindow._load_settings_from_config'
)
event_bus.publish(event)

# 修复后（V3.2.2）
event_bus.publish(
    event_type='config.changed',
    data={'type': 'ai_config', 'data': ai_config},
    source='MainWindow._load_settings_from_config'
)
```

**修复2：使用正确的枚举值**

文件：`core/qwen_provider.py` 第192行、196行

```python
# 修复前
self._state = AIProviderState.UNAVAILABLE

# 修复后（V3.2.2）
self._state = AIProviderState.ERROR
```
