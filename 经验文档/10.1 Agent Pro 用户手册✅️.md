## 一、产品概述

### 1.1 产品简介

Novel Writing Assistant-Agent Pro 是一款智能小说写作辅助工具，集成了AI驱动的创作功能，帮助作者提高创作效率、保持作品一致性、激发创作灵感。

### 1.2 核心能力

| 功能模块 | 说明 |
|----------|------|
| **逆向反馈** | 分析章节与设定的一致性，自动检测冲突并生成修正建议 |
| **续写功能** | 根据原文智能续写，支持多种发展方向和多版本管理 |
| **快捷创作** | 快速生成世界观、大纲、人物、情节等创作设定 |
| **热榜功能** | 实时获取各大平台热门作品，激发创作灵感 |
| **风格学习** | 学习范文风格，保持创作风格一致性 |

### 1.3 系统要求

- 操作系统：Windows 10/11
- 内存：8GB以上（推荐16GB）
- 存储空间：500MB以上
- 网络：需要网络连接（AI功能需要调用API）

## 三、核心功能指南

### 3.1 逆向反馈功能

#### 功能概述

逆向反馈分析已生成的章节内容与项目设定的一致性，检测冲突并生成修正建议。

**核心价值**：
- 检测设定冲突：识别章节内容与大纲、人物、世界观的矛盾
- 生成修正建议：为每个冲突提供可执行的修正方案
- 自动评分反馈：将一致性检查结果纳入章节质量评分

#### 操作流程

```
上传章节 → 标记完成状态 → 配置分析参数 → 运行分析 → 查看结果 → 应用修正
```

#### 详细步骤

**步骤1：上传章节**

| 方式 | 操作方法 | 适用场景 |
|------|----------|----------|
| 上传文件 | 点击"上传文件"，选择TXT/DOCX文件 | 单个或少量章节 |
| 批量上传 | 点击"批量上传"，选择文件夹 | 大量章节文件 |
| 粘贴文本 | 点击"粘贴文本"，在文本框中粘贴 | 快速测试 |

**步骤2：标记章节状态**

- 右键点击章节，选择"标记为完成"或"标记为未完成"
- 或点击"全部标记完成"批量处理

**步骤3：配置分析参数**

**分析维度**（可多选）：
- ✓ 一致性检查（章节间逻辑一致性）
- ✓ 逻辑漏洞检测（情节合理性）
- ✓ 人设偏离检测（角色行为一致性）
- ✓ 风格匹配度（写作风格一致性）
- ✓ 世界观冲突（设定一致性）

**步骤4：运行分析**

点击"运行分析"按钮，等待分析完成。

**步骤5：查看结果**

在冲突列表中选择问题，查看详细的修正建议。

**步骤6：应用修正**

点击"应用修正"按钮，将修正建议应用到当前项目设定。

#### 冲突类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| 🔴 人物冲突 | 角色行为与设定不一致 | 角色使用了未设定的能力 |
| 🟡 大纲冲突 | 情节发展与大纲不一致 | 章节内容偏离主线 |
| 🟢 世界观冲突 | 内容违反世界观规则 | 使用了禁用的设定 |

#### 评分机制

| 冲突情况 | 一致性评分 |
|----------|-----------|
| 无冲突 | 1.0 |
| 1个低优先级 | 0.95 |
| 1个中优先级 | 0.90 |
| 1个高优先级 | 0.80 |
| 大量冲突（保底） | 0.40 |

### 3.2 续写功能

#### 功能概述

根据已有的文本内容，智能分析并生成后续内容。支持多种发展方向和多版本管理。

**核心能力**：
- 智能续写：根据起始文本和上下文自动生成后续内容
- 多版本生成：同时生成多个不同风格的版本供选择
- 方向控制：支持10种续写方向
- 上下文感知：自动读取项目设定，保持内容一致性

#### 操作流程

```
输入原文 → 设置参数 → 开始续写 → 查看结果 → 选择最佳 → 保存
```

#### 详细步骤

**步骤1：输入原文**

在"原文内容"文本框中输入或粘贴起始文本：
- 直接粘贴文本
- 点击"选择文件"导入文本文件
- 点击"选择章节"从项目中选取

**建议**：输入200-500字的起始文本，确保有明确的场景和人物。

**步骤2：设置参数**

| 参数 | 选项 | 说明 |
|------|------|------|
| 字数 | 500/1000/2000/自定义 | 目标生成字数（100-5000） |
| 续写方向 | 10种方向 | 选择情节发展方向 |
| 创意温度 | 0.0-1.0 | 数值越高创意性越强 |
| 生成模式 | 单次/多版本 | 生成单个或多个版本 |

**步骤3：开始续写**

点击"开始续写"按钮，系统将自动加载项目设定并生成内容。

**步骤4：查看结果**

续写完成后：
- 在结果预览区查看生成内容
- 多版本模式下可切换查看不同版本
- 查看字数、耗时、评分等统计信息

**步骤5：选择最佳版本**

多版本模式下，点击"选择最佳版本"自动选择评分最高的版本。

**步骤6：保存结果**

点击"保存结果"，选择保存方式：
- 保存为新章节
- 追加到当前章节
- 导出为文件

#### 续写方向详解

| 方向 | 说明 | 适用场景 |
|------|------|----------|
| 自然续写 | 按大纲和上下文自然发展 | 常规续写，无特殊要求 |
| 剧情推进 | 加速情节发展，推进主线 | 需要推动故事进展时 |
| 高潮铺垫 | 为后续高潮做铺垫和准备 | 章节末尾，过渡到重要情节 |
| 人物深入 | 深入刻画人物内心世界 | 人物成长、内心独白场景 |
| 对话展开 | 以人物对话为主推进情节 | 人物互动、信息揭示 |
| 场景描写 | 侧重环境和场景的描绘 | 氛围营造、场景转换 |
| 悬念设置 | 设置悬念，引发读者好奇 | 章节结尾、转折点 |
| 冲突升级 | 加剧人物或情节的冲突 | 矛盾激化、对抗场景 |
| 情感渲染 | 强化情感表达和氛围 | 感情戏、高潮场景 |
| 转折揭示 | 揭示关键转折或秘密 | 剧情反转、真相揭露 |

#### 多版本管理

| 模式 | 版本数 | 温度分布 | 适用场景 |
|------|--------|----------|----------|
| 单次生成 | 1个 | 当前设置温度 | 明确需求时使用 |
| 多版本生成 | 3个 | 0.6/0.8/1.0 | 探索不同可能性时使用 |

版本数量上限为5个，达到上限后需先保存或清空才能继续生成。

### 3.3 快捷创作功能

#### 功能概述

快速生成各类设定，包括世界观、章节大纲、人物设定、关键情节等。只需提供关键词或参考文本，即可在数分钟内获得完整的创作框架。

**核心能力**：
- 世界观生成：基于关键词生成完整世界设定
- 大纲生成：生成章节规划和剧情走向
- 人物生成：生成立体丰满的人物形象
- 情节生成：生成关键剧情和转折点
- 参考文本解析：从已有文本提取设定元素
- 结果存储：保存到新项目或导入当前项目

#### 操作流程

```
输入关键词/参考文本 → 选择生成类型 → 点击生成 → 预览结果 → 保存/导入
```

#### 详细步骤

**步骤1：输入关键词**

```
修仙、逆袭、系统、复仇          ✅ 多个关键词用顿号分隔
都市、重生、商业、首富          ✅ 包含题材、主题、核心元素
```

**避免做法**：
```
我想写一个修仙小说              ❌ 描述性语言，关键词不明确
修仙逆袭系统复仇                ❌ 没有分隔符，难以解析
```

**步骤2：选择生成类型**

| 类型 | 输入参数 | 输出内容 |
|------|----------|----------|
| 世界观 | 关键词、题材、参考文本 | 世界背景、力量体系、势力分布 |
| 大纲 | 主题、章节数、目标字数 | 章节规划、关键事件、主线情节 |
| 人物 | 姓名、角色定位、世界观 | 性格、背景、能力、关系 |
| 情节 | 大纲摘要、参与人物、类型 | 开端、发展、高潮、结局 |
| 全部生成 | 关键词、详细程度 | 以上所有内容（自动处理依赖） |

**步骤3：点击生成**

点击对应按钮开始生成，等待约1-3分钟。

**步骤4：预览结果**

在结果区查看生成内容（2×2布局）：
- 左上：世界观
- 右上：大纲
- 左下：人物设定
- 右下：情节设计

**步骤5：保存/导入**

**保存到新项目**：
1. 点击"保存到新项目"
2. 输入项目名称
3. 选择保存位置
4. 确认保存

**导入到当前项目**：

选择冲突处理策略：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| 保留原有设定（默认） | 跳过冲突项 | 保护用户已有设定 |
| 覆盖原有设定 | 替换原有设定 | 需要更新设定 |
| 合并设定 | 智能合并 | 扩展已有设定 |
| 重命名新设定 | 添加时间戳后缀 | 保留两份设定 |

#### 参考文本功能

支持自动识别以下文本类型：

| 类型 | 提取内容 |
|------|----------|
| 小说文本 | 风格、人物、世界观元素 |
| 大纲文本 | 章节规划、关键事件 |
| 人物设定 | 姓名、性格、能力、关系 |
| 世界观设定 | 时代、力量体系、势力 |

#### 详细程度选择

| 级别 | 生成时间 | Token消耗 | 适用场景 |
|------|----------|-----------|----------|
| 快速 | 30-60秒 | 2000-3000 | 灵感验证、快速原型 |
| 标准 | 1-3分钟 | 3000-5000 | 正式创作（推荐） |
| 详细 | 3-5分钟 | 5000-8000 | 重要设定、精细细节 |

## 五、常见问题

### 5.1 逆向反馈相关

**Q: 分析结果不准确怎么办？**

A: 可能原因及解决方案：
1. 设定信息不完整 → 补充完整的项目设定
2. LLM理解偏差 → 使用更强大的模型
3. 章节内容截断 → 检查章节是否被截断

**Q: 分析时间过长怎么办？**

A: 解决方案：
- 使用缓存机制（自动启用）
- 分批分析章节
- 配置本地模型加速

### 5.2 续写功能相关

**Q: 续写内容与前文不一致？**

A: 检查项目设定是否正确加载，可手动在请求中提供设定内容。确保前文章节已正确关联。

**Q: 续写内容风格与原文差异大？**

A: 降低创意温度（0.5-0.6），使用保守模式或聚焦模式。确保风格学习模块已学习原文风格。

**Q: 字数与目标差异较大？**

A: 字数控制在±20%范围内属于正常。如需更精确控制，可在续写后手动调整。

### 5.3 快捷创作相关

**Q: 生成失败怎么办？**

A: 可能原因及解决方案：
1. API调用超时 → 检查网络连接，稍后重试
2. 关键词过少 → 至少提供2-3个关键词
3. 参考文本格式问题 → 确保文本可读

**Q: 生成结果不满意？**

| 问题 | 解决方案 |
|------|----------|
| 世界观太普通 | 提供更具体的关键词，如"赛博朋克修仙" |
| 人物不够立体 | 使用"详细"模式，提供参考文本 |
| 大纲逻辑混乱 | 先生成世界观，再生成大纲 |
| 情节缺乏张力 | 明确情节类型（开篇/转折/高潮） |
