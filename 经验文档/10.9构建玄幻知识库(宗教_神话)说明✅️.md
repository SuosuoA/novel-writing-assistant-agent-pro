**第一阶段：手工精选核心知识点（34条）**
- 道教丹道核心概念：炼精化气、金丹、元婴等

- 修仙体系核心概念：等级体系、灵根、法宝等
- 西方神话核心内容：希腊神话、北欧神话等
- 东方神话核心内容：中国神话、山海经等
- 克苏鲁神话核心内容：外神、旧日支配者等

**第二阶段：扩充知识点（30条）**
- 道教丹道扩充：三花聚顶、九转金丹、辟谷等
- 佛教因果扩充：功德业力、净土世界、菩萨道等
- 修仙体系扩充：天劫雷罚、心魔劫难、灵兽伙伴等
- 西方神话扩充：泰坦之战、英雄传说、魔法体系等
- 东方神话扩充：天庭体系、地府冥界、妖族传承等
- 克苏鲁神话扩充：外神降临、旧日支配者、深潜者等

**第三阶段：自动生成知识点（236条）**
- 基于关键词模板自动生成
- 涵盖道教丹道、佛教因果、修仙体系相关术语
- 涵盖西方神话、东方神话、克苏鲁神话相关术语
- 自动生成符合Schema规范的知识点

### 3.2 知识点格式

所有知识点符合`knowledge_schema.json`规范：
- **必填字段**：knowledge_id、category、domain、title、content、keywords、created_at、updated_at
- **可选字段**：embedding、references、difficulty、tags、metadata
- **ID格式**：xuanhuan-{domain}-{序号}（如xuanhuan-religion-001）
- **分类标识**：category = "xuanhuan"（玄幻）

## 六、知识库检索

### 6.1 关键词检索

使用`KnowledgeManager.search_knowledge()`方法：
```python
from core.knowledge_manager import KnowledgeManager

manager = KnowledgeManager(workspace_root=Path("E:/project"))
results = manager.search_knowledge(
    query="金丹修炼",
    category="xuanhuan",
    domain="religion",
    top_k=10
)
```

### 6.2 向量检索

使用`NovelVectorStore.recall_similar_context()`方法：
```python
from infrastructure.vector_store import NovelVectorStore

vector_store = NovelVectorStore(db_path=Path("data/vector_store"))
results = vector_store.recall_similar_context(
    query="道教修炼方法",
    top_k=10,
    category="xuanhuan",
    domain="religion"
)
```

## 九、经验总结

### 9.1 构建策略

1. **分阶段构建**：手工精选 → 扩充 → 自动生成
2. **模板驱动**：基于关键词模板自动生成知识点

1. **Schema规范**：严格遵守knowledge_schema.json定义
2. **ID规范**：xuanhuan-{domain}-{序号}格式
3. **分类清晰**：宗教/神话两大领域清晰分离
4. **标签系统**：支持多种标签组合检索
