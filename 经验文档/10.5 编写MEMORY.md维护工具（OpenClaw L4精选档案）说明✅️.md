## ⚠️ 路径迁移说明（2026-03-26）

**重要变更**: L4 精选档案路径已从 `.workbuddy/memory/` 迁移至独立目录。

### 迁移原因

原 L4 精选档案路径 `.workbuddy/memory/` 与 WorkBuddy 系统的开发记忆路径混淆，导致：
- 项目长期记忆与开发临时记忆混在一起
- 用户难以区分哪些是项目记忆，哪些是开发记忆
- 不符合"项目记忆应独立于开发工具"的原则

### 迁移结果

| 层级 | 名称 | 新路径 | 说明 |
|------|------|--------|------|
| L1 | 热记忆 | `data/session-state/` | 当前会话状态、WAL日志 |
| L2 | 温记忆 | `data/vector_store/` | 向量数据库、章节编码历史 |
| L3 | 冷记忆 | `.git/refs/notes/` | Git-Notes分支感知历史 |
| **L4** | **精选档案** | **`Memory-Novel Writing Assistant-Agent Pro/`** | **MEMORY.md、长期记忆** |

### 代码修改

**已修改文件**:
- `tools/memory_backup.py` - L4 路径配置已更新
- `tools/memory_maintenance.py` - L4 路径配置已更新

### 与开发记忆的区分

| 类型 | 路径 | 用途 |
|------|------|------|

## 一、功能概述

### 1.1 定位

MEMORY.md 维护工具是 OpenClaw mem9 框架中 **L4 精选档案** 层的核心实现，负责：

1. **每日冥想**：自动提炼当日重要事件到 MEMORY.md
2. **每周大冥想**：深度精简 MEMORY.md，归档旧日记
3. **大小控制**：确保 MEMORY.md < 50KB

### 1.2 设计哲学

参考 OpenClaw mem9 框架核心设计：

- **文件即真相源**：所有记忆以 Markdown 格式存储，透明可编辑
- **精简原则**：每条记录问"6个月后还有用吗？"
- **自动归档**：超过30天的日记自动归档到 `Memory-Novel Writing Assistant-Agent Pro/archive/`

## 四、核心功能

### 4.1 每日冥想（daily_meditation）

**执行时机**：每日结束时（或手动触发）

**处理流程**：
```
1. 收集当日所有日记文件
   ├─ Memory-Novel Writing Assistant-Agent Pro/YYYY-MM-DD.md
   └─ Memory-Novel Writing Assistant-Agent Pro/{身份}/YYYY-MM-DD.md

2. 提取重要事件
   ├─ 按标题分割（### 事件标题）
   ├─ 判断重要性（high/medium/low）
   ├─ 判断类别（feature/bugfix/refactor/doc/decision）
   └─ 判断是否保留（6个月后还有用吗？）

3. 过滤值得保留的事件
   └─ 保留：importance=high 或 preserve=True

4. 更新 MEMORY.md
   └─ 在"工作记录"章节插入新事件
```

**重要性判断规则**：

| 级别 | 触发关键词 |
|------|-----------|
| **high** | ADR-、决策、锁定、保护、修复、完成、实现、重大、关键、核心、架构、安全、P0、P1 |
| **medium** | 优化、更新、新增、集成、测试、验证、评审 |
| **low** | 其他 |

**保留判断规则**（6个月后还有用吗？）：

保留关键词：ADR-、决策、锁定、保护、规则、规范、架构、技术栈、路径、清单、索引

**调用示例**：
```python
from tools.memory_maintenance import MemoryMaintenanceTool

tool = MemoryMaintenanceTool(workspace_root)
result = tool.daily_meditation(date="2026-03-25")

# 返回结果
{
    "date": "2026-03-25",
    "files_scanned": 3,
    "events_extracted": 38,
    "events_preserved": 31,
    "memory_size_kb": 17.09
}
```

### 4.2 每周大冥想（weekly_meditation）

**执行时机**：每周结束时（或 MEMORY.md > 50KB 时）

**处理流程**：
```
1. 检查 MEMORY.md 大小
   └─ 如果 <= 50KB，跳过精简

2. 解析 MEMORY.md
   └─ 按二级标题（##）分割章节

3. 精简工作记录章节
   ├─ 删除 low 重要性事件（🟢 标记）
   ├─ 保留 high（🔴）和 medium（🟡）事件
   └─ 统计删除/保留数量

4. 重写 MEMORY.md
   └─ 更新精简后的内容

5. 归档旧日记文件
   ├─ 遍历所有身份目录下的日记文件
   ├─ 超过30天的文件移动到 archive/YYYY-MM/
   └─ 命名格式：{身份}_{YYYY-MM-DD}.md
```

**归档目录结构**：
```
Memory-Novel Writing Assistant-Agent Pro/archive/
├── 2026-02/
│   ├── 后端架构师_2026-02-15.md
│   ├── AI工程师_2026-02-20.md
│   └── root_2026-02-28.md
└── 2026-03/
    └── ...
```

**调用示例**：
```python
from tools.memory_maintenance import MemoryMaintenanceTool

tool = MemoryMaintenanceTool(workspace_root)
result = tool.weekly_meditation(week_end_date="2026-03-25")

# 返回结果
{
    "date": "2026-03-25",
    "memory_size_before": 52.3,
    "memory_size_after": 45.1,
    "events_removed": 15,
    "events_preserved": 42,
    "files_archived": 8,
    "archive_dir": "E:\\...\\memory\\archive"
}
```

## 八、集成建议

### 8.1 自动化调度

建议在启动脚本中集成每日冥想：

```python
# 在程序启动时执行每日冥想
from tools.memory_maintenance import run_daily_meditation

# 获取上次冥想日期
last_meditation = get_last_meditation_date()
today = datetime.now().strftime("%Y-%m-%d")

if last_meditation != today:
    run_daily_meditation(workspace_root)
    save_last_meditation_date(today)
```

### 8.2 GUI 集成

可在"设置"页面添加手动触发按钮：

```python
# 设置页面
def on_daily_meditation_click():
    result = run_daily_meditation(workspace_root)
    show_info(f"每日冥想完成：提炼 {result['events_preserved']} 条事件")

def on_weekly_meditation_click():
    result = run_weekly_meditation(workspace_root)
    show_info(f"每周大冥想完成：精简至 {result['memory_size_after']:.2f}KB")
```

### 8.3 定时任务

建议每周一凌晨执行每周大冥想：

```python
# 使用 schedule 库
import schedule

def weekly_job():
    run_weekly_meditation(workspace_root)

schedule.every().monday.at("04:00").do(weekly_job)
```

## 十、注意事项

1. **编码问题**：Windows 控制台默认 GBK 编码，测试代码需要设置 `sys.stdout.encoding = 'utf-8'`
2. **线程安全**：所有操作使用 `threading.RLock` 保护，支持多线程环境
3. **文件大小**：MEMORY.md 最大 50KB，超过触发精简
4. **归档周期**：默认归档超过30天的日记文件
5. **幂等性**：每日冥想会检查是否已有当日记录，避免重复插入
