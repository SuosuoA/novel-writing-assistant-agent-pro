## 一、功能概述

### 1.1 设计目标

实现OpenClaw mem9框架的L3冷记忆层，基于Git-Notes机制实现：
- **分支特定决策记录**：不同分支拥有独立的决策记忆
- **里程碑事件存储**：记录项目重大事件
- **历史可追溯**：支持git log检索历史决策
- **分支切换同步**：自动加载对应分支的记忆

### 1.2 技术选型（ADR-004）

**决策**: 采用Git-Notes作为L3冷记忆存储机制

**理由**:
1. **原生集成**：Git-Notes是Git原生功能，无需额外依赖
2. **分支隔离**：不同分支的commit对应不同notes
3. **持久化**：与commit关联，不会丢失
4. **可推送**：支持push/pull同步到远程仓库
5. **可检索**：支持git log检索

## 二、架构设计

### 2.1 模块结构

```
core/git_notes_manager.py
├── GitNote            # 笔记数据模型
├── BranchMemory       # 分支记忆汇总
├── GitNotesState      # 管理器状态
├── GitNotesManager    # 核心管理类
│   ├── record_decision()     # 记录决策
│   ├── record_milestone()    # 记录里程碑
│   ├── record_lesson()       # 记录经验教训
│   ├── get_branch_memories() # 获取分支记忆
│   ├── sync_on_branch_switch() # 分支切换同步
│   └── search_notes()        # 搜索笔记
├── get_git_notes_manager()   # 单例工厂
└── reset_git_notes_manager() # 重置单例
```

### 2.2 数据模型

#### GitNote（笔记记录）

```python
class GitNote(BaseModel):
    note_id: str          # 笔记ID（时间戳格式）
    timestamp: str        # 记录时间
    note_type: str        # 类型：decision/milestone/lesson/context
    title: str            # 标题
    content: str          # 内容
    tags: List[str]       # 标签
    branch: str           # 所属分支
    commit_hash: str      # 关联的commit hash
```

#### BranchMemory（分支记忆）

```python
class BranchMemory(BaseModel):
    branch_name: str           # 分支名称
    last_sync: str             # 上次同步时间
    decisions: List[GitNote]   # 决策记录
    milestones: List[GitNote]  # 里程碑事件
    lessons: List[GitNote]     # 经验教训
    context: List[GitNote]     # 上下文记录
```

## 三、核心功能实现

### 3.1 Git-Notes基础操作

Git-Notes使用Git的notes命名空间存储额外信息：

```bash
# 添加note
git notes --ref memory add -m "note内容" <commit-hash>

# 查看note
git notes --ref memory show <commit-hash>

# 列出所有notes
git notes --ref memory list

# 推送notes
git push origin refs/notes/memory
```

### 3.2 记录决策（ADR风格）

```python
from core.git_notes_manager import get_git_notes_manager

git_notes = get_git_notes_manager()

# 记录架构决策
git_notes.record_decision(
    title="采用LanceDB作为向量数据库",
    content="ADR-001: 零配置、嵌入式、Python友好",
    tags=["ADR", "vector-db"]
)
```

### 3.3 记录里程碑

```python
# 记录项目里程碑
git_notes.record_milestone(
    title="V5.5人设评分修复完成",
    content="修复人设评分固定返回低分问题，新增语义扩展匹配"
)
```

### 3.4 记录经验教训

```python
# 记录踩坑经验
git_notes.record_lesson(
    title="git restore误删用户成果",
    content="破坏性命令必须先确认，不可绕过用户确认流程"
)
```

### 3.5 分支切换同步

```python
# 当检测到分支切换时
git_notes.sync_on_branch_switch(
    old_branch="develop",
    new_branch="feature/long-term-memory"
)
```

### 3.6 检索历史决策

```python
# 搜索关键词
results = git_notes.search_notes("LanceDB", note_type="decision")

# 获取最近决策
recent = git_notes.get_recent_decisions(limit=10)

# 获取所有里程碑
milestones = git_notes.get_milestones()
```

## 四、与OpenClaw mem9架构的集成

### 4.1 L3冷记忆定位

| 层级 | 存储介质 | 内容类型 | 更新频率 | 检索方式 |
|:---:|:---|:---|:---|:---|
| **L1热记忆** | SESSION-STATE.md | 当前会话状态 | 实时更新 | 直接读取 |
| **L2温记忆** | LanceDB向量库 | 章节向量、知识库向量 | 增量更新 | 向量检索 |
| **L3冷记忆** | Git-Notes | 分支决策、里程碑 | 按需归档 | git log检索 |
| **L4精选档案** | MEMORY.md | 长期事实、关键决策 | 每日冥想 | 手动维护 |
| **L5云备份** | 外部存储 | 完整workspace镜像 | 每日备份 | 云端API |

### 4.2 与其他层的协作

```
用户操作
    ↓
L1热记忆（SESSION-STATE）记录实时状态
    ↓
关键决策/里程碑
    ↓
L3冷记忆（Git-Notes）持久化存储
    ↓
每日冥想提炼
    ↓
L4精选档案（MEMORY.md）跨会话持久
```

## 六、使用示例

### 6.1 基本使用

```python
from core import get_git_notes_manager

# 获取管理器实例
git_notes = get_git_notes_manager()

# 记录决策
git_notes.record_decision(
    title="采用SQLite WAL模式",
    content="ADR-005: 提高并发读写性能",
    tags=["ADR", "database"]
)

# 获取当前分支记忆
memory = git_notes.get_branch_memories()
print(f"决策数: {len(memory.decisions)}")
print(f"里程碑数: {len(memory.milestones)}")

# 搜索
results = git_notes.search_notes("database")
for note in results:
    print(f"[{note.note_type}] {note.title}")
```

### 6.2 与其他模块协作

```python
# 与SESSION-STATE协作
from core import get_session_state_manager, get_git_notes_manager

session = get_session_state_manager()
git_notes = get_git_notes_manager()

# 完成重要决策时
session.update_state(
    pending_data={
        "memory_update_pending": "ADR-006: 采用OpenClaw mem9架构"
    }
)

# 持久化到Git-Notes
git_notes.record_decision(
    title="采用OpenClaw mem9架构",
    content="ADR-006: 5层记忆架构（L1-L5）",
    tags=["ADR", "memory"]
)
```

## 七、注意事项

### 7.1 Git仓库要求

- 必须在Git仓库根目录下运行
- 需要.git目录存在
- 首次使用会自动初始化notes ref

### 7.2 分支命名

- 建议使用清晰的分支命名（如feature/xxx, hotfix/xxx）
- 分支切换时会自动同步记忆

### 7.3 与远程仓库同步

```bash
# 推送notes到远程
git push origin refs/notes/memory

# 拉取远程notes
git fetch origin refs/notes/memory:refs/notes/memory
```
