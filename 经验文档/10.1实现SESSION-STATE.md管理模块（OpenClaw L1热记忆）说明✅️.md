### 2.2 数据模型定义

```python
class ActiveTask(BaseModel):
    """活跃任务状态"""
    current_function: str      # 当前功能
    current_file: str          # 当前文件
    last_operation: str        # 上次操作
    last_operation_time: str   # 操作时间
    task_id: str               # 任务ID

class TempContext(BaseModel):
    """临时上下文"""
    current_chapter: str       # 当前章节
    word_count: int            # 字数
    recent_generation: str     # 最近生成
    characters_involved: List[str]  # 涉及人物
    worldview_elements: List[str]   # 世界观元素

class ErrorState(BaseModel):
    """异常状态"""
    has_error: bool            # 是否有异常
    error_type: str            # 异常类型
    error_message: str         # 异常消息
    error_time: str            # 异常时间
    stack_trace: str           # 堆栈追踪
    recovery_hint: str         # 恢复提示

class PendingData(BaseModel):
    """待持久化数据"""
    latest_score: float        # 最新评分
    score_time: str            # 评分时间
    memory_update_pending: str # 待写入MEMORY
    archive_candidates: List[str]  # 归档候选
```

## 三、关键技术实现

### 3.1 WAL协议实现

```python
def write_before_ai_call(self, context: Dict[str, Any]) -> None:
    """
    WAL协议：在AI调用前写入状态
    
    确保中断后可恢复。必须在调用LLM API前调用此方法。
    """
    with self._wal_lock:
        updates = {
            "active_task": {
                "current_function": context.get("task", ""),
                "current_file": context.get("file", ""),
                "last_operation": context.get("operation", ""),
                "last_operation_time": datetime.now().isoformat(),
            }
        }
        # 同步写入（阻塞式，确保落盘）
        self.update_state(**updates)
```

### 3.2 线程安全设计

- 使用 `threading.RLock()` 保护所有状态操作
- WAL写入使用独立锁 `_wal_lock` 确保顺序
- EventBus延迟初始化避免循环依赖

### 3.3 文件即真相源

- 所有状态变更立即写入 `.workbuddy/session-state.md`
- 支持手动编辑Markdown文件
- 新实例启动时自动从文件恢复状态

### 3.4 EventBus集成

```python
def _publish_state_change_event(self, updates: Dict[str, Any]) -> None:
    """发布状态变更事件"""
    if self._event_bus is None:
        from .event_bus import get_event_bus
        self._event_bus = get_event_bus()
    
    self._event_bus.publish(
        event_type="session.state_changed",
        data={"updates": updates, "timestamp": datetime.now().isoformat()},
        source="SessionStateManager"
    )
```

## 四、使用示例

### 4.1 基本使用

```python
from core.session_state import get_session_state_manager

# 获取管理器
manager = get_session_state_manager()

# 设置活跃任务
manager.set_active_task(
    function="世界观编辑",
    file="世界观/科幻/星际文明.md",
    operation="保存成功"
)

# 更新生成上下文
manager.update_generation_context(
    chapter="第三章 星际战争",
    word_count=12345,
    characters=["主角A", "反派B"]
)

# 记录评分
manager.record_score(0.87)

# 清除活跃任务
manager.clear_active_task()
```

### 4.2 WAL协议使用

```python
# 在调用AI前写入状态
manager.write_before_ai_call({
    "task": "生成章节",
    "file": "章节/第三章.md",
    "operation": "调用AI生成",
    "pending": {"latest_score": 0.87}
})

# 然后调用LLM API
response = llm_client.generate(...)
```

### 4.3 异常处理

```python
try:
    # 执行操作
    ...
except Exception as e:
    manager.set_error(
        error_type=type(e).__name__,
        error_message=str(e),
        stack_trace=traceback.format_exc(),
        recovery_hint="请检查文件路径"
    )
```

### 4.4 启动恢复

```python
# 应用启动时调用
recovered = manager.recover_on_startup()
if recovered and recovered.is_task_active():
    print(f"检测到未完成任务: {recovered.active_task.current_function}")
    # 提示用户是否继续
```
