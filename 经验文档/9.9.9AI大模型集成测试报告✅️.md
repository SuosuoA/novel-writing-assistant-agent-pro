## Bug修复记录

### Bug #1: EventBus.publish()参数错误

**发现位置**: `core/config_service.py` 第360行  
**问题描述**: `_publish_config_changed_event()`方法调用`EventBus.publish()`时传递了Event对象，但实际签名应为`(event_type, data, source)`

**修复前**:
```python
event = Event(
    type="config.changed",
    data=event_data,
    source="ConfigService",
)
self._event_bus.publish(event)  # ❌ 错误：传递Event对象
```

**修复后**:
```python
# V1.3修复：EventBus.publish签名是(event_type, data, source)
self._event_bus.publish(
    event_type="config.changed",
    data=event_data,
    source="ConfigService"
)  # ✅ 正确：传递三个参数
```

**影响范围**: 所有通过ConfigService修改配置时的事件发布功能  
**修复状态**: ✅ 已修复并验证

### Bug #2: EventBus.unsubscribe()参数错误

**发现位置**: `tests/test_ai_integration.py` 第397行  
**问题描述**: `unsubscribe()`应该只接收`subscription_id`参数，但测试代码传递了两个参数

**修复前**:
```python
self.event_bus.unsubscribe("config.changed", on_config_changed)  # ❌ 错误：传递两个参数
```

**修复后**:
```python
# 订阅时保存subscription_id
subscription_id = self.event_bus.subscribe("config.changed", on_config_changed)

# 取消订阅时只传递subscription_id
self.event_bus.unsubscribe(subscription_id)  # ✅ 正确：只传递subscription_id
```

**影响范围**: 测试代码中的事件订阅清理逻辑  
**修复状态**: ✅ 已修复并验证
