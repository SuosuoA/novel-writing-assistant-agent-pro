## 一、背景与目标

### 问题诊断

用户反馈：**设置界面的长期记忆管理和记忆配置没有实际生效**

根因分析：
1. 配置保存到config.yaml，但代码没有读取执行
2. WAL协议、会话恢复、向量召回、每日冥想等功能设计完成但未实装
3. 缺少AI调用前的状态持久化机制

1. **WAL自动保存机制**：AI调用前自动写入状态，确保中断可恢复

4. **每日冥想定时任务**：定时总结经验，更新MEMORY.md
5. **DeepSeek Embedding支持**：支持配置DeepSeek API进行向量嵌入

## 二、核心实装内容

### 2.1 WAL自动保存机制

**文件**: `core/ai_call_wrapper.py` (新建，+220行)

**功能**：
- 自动在AI调用前执行WAL写入
- 自动在AI调用后标记完成/失败状态
- 提供同步和异步两种调用方式
- 无侵入式集成（包装器模式）

**核心代码**：
```python
class AICallWrapper:
    """AI调用包装器 - 自动集成WAL协议"""
    
    def call(
        self,
        prompt: str,
        context: Dict[str, Any],
        generator_func: Callable,
        operation: str = "",
        task_id: Optional[str] = None,
        **kwargs
    ) -> Any:
        # 1. WAL写入（如果启用）
        if self._wal_enabled:
            self.wal_manager.write_before_ai_call(
                context=context,
                operation=operation,
                task_id=task_id
            )
        
        # 2. 更新SessionState
        self.session_manager.write_before_ai_call({...})
        
        # 3. 执行AI调用
        try:
            result = generator_func(prompt, **kwargs)
            # 4. 标记成功
            self.wal_manager.mark_completed(result)
            return result
        except Exception as e:
            # 5. 标记失败
            self.wal_manager.mark_failed(str(e))
            raise
```

**使用方式**：
```python
from core.ai_call_wrapper import get_ai_call_wrapper

wrapper = get_ai_call_wrapper()

result = wrapper.call(
    prompt="生成第3章",
    context={"chapter_number": 3, "outline": "..."},
    generator_func=ai_api.generate,
    operation="生成章节"
)
```

**配置开关**：
```yaml
memory:
  wal_auto_save: true  # 启用WAL自动保存
```

### 2.2 会话自动恢复

**文件**: `gui_main.py` (修改)

**功能**：
- 启动时自动检查config.yaml中的auto_recovery配置
- 如果启用，尝试恢复上次未完成的任务
- 提供用户友好的恢复提示

**核心代码**：
```python
def _check_auto_recovery(self):
    """V2.19: 检查会话自动恢复配置"""
    try:
        config_path = Path.cwd() / "config.yaml"
        if not config_path.exists():
            return
        
        with open(config_path, 'r', encoding='utf-8') as f:
            config = yaml.safe_load(f)
        
        memory_config = config.get("memory", {})
        auto_recovery = memory_config.get("auto_recovery", False)
        
        if auto_recovery:
            from core.wal_manager import get_wal_manager
            wal_manager = get_wal_manager()
            recovered = wal_manager.recover_on_startup()
            
            if recovered:
                logger.info(f"[会话恢复] 发现未完成任务: {recovered['operation']}")
                # 提供恢复选项
                self._show_recovery_dialog(recovered)
                
    except Exception as e:
        logger.error(f"[会话恢复] 检查失败: {e}")
```

**配置开关**：
```yaml
memory:
  auto_recovery: true  # 启用会话自动恢复
```

### 2.4 每日冥想定时任务

**文件**: `core/daily_meditation.py` (新建，+450行)

**功能**：
- 定时总结当日工作（默认23:00执行）
- 分析工作效率和生成质量
- 提取经验教训并更新MEMORY.md
- 压缩冷数据（归档30天前的记录）
- 支持APScheduler精确调度和简单定时器降级

**核心机制**：
```python
class DailyMeditation:
    """每日冥想执行器"""
    
    def execute(self) -> Dict[str, Any]:
        """执行每日冥想"""
        # 1. 收集当日数据
        daily_data = self._collect_daily_data()
        
        # 2. 分析工作效率
        analysis = self._analyze_productivity(daily_data)
        
        # 3. 提取经验教训
        lessons = self._extract_lessons(daily_data, analysis)
        
        # 4. 更新MEMORY.md
        self._update_memory(lessons)
        
        # 5. 压缩冷数据
        self._compress_cold_data()
        
        return results


class DailyMeditationScheduler:
    """每日冥想调度器"""
    
    def start(self) -> bool:
        """启动调度器"""
        # 使用APScheduler（如果可用）
        if APSCHEDULER_AVAILABLE:
            hour, minute = map(int, self._schedule_time.split(":"))
            self._scheduler.add_job(
                func=self.meditation.execute,
                trigger=CronTrigger(hour=hour, minute=minute),
                id="daily_meditation"
            )
            self._scheduler.start()
        else:
            # 降级到简单定时器
            self._schedule_simple_timer()
```

**经验提取逻辑**：
- 效率等级判断：excellent / good / average / needs_improvement
- 成功率分析：低于50%时建议检查API配置

- 自动降级：DeepSeek暂不支持独立Embedding API，降级到local模型
- 从config.yaml自动读取embedding配置

```python
class EmbeddingFunction:
    def __init__(self, model_type: str = "local", model_name: str = None, api_key: str = None, base_url: str = None):
        # DeepSeek暂不支持embedding，降级到local
        if model_type == "deepseek":
            logger.warning("[EmbeddingFunction] DeepSeek暂不支持embedding，降级到local模型")
            model_type = "local"
        
        self.model_type = model_type
        # ...初始化对应模型
```

**配置方式**：
```yaml
memory:
  embedding_model: deepseek  # 或 local
```

**降级说明**：
- DeepSeek目前不提供独立的Embedding API
- 配置deepseek时会自动降级到local模型（sentence-transformers）

## 四、架构设计

### 4.1 OpenClaw mem9五层记忆架构

```
L1 热记忆 (SessionState)
  ↓ WAL协议
L2 温记忆 (VectorStore - LanceDB)
  ↓ 定期归档
L3 冷记忆 (GitNotes)
  ↓ 压缩存储
L4 档案 (MEMORY.md)
  ↓ 离线分析
L5 深度记忆 (经验文档)
```

### 4.2 实装功能映射

| 层级 | 功能 | 实装状态 | 文件 |
|------|------|---------|------|
| L1 | SessionState管理 | ✅ 已有 | core/session_state.py |
| WAL | WAL协议 | ✅ 已有 | core/wal_manager.py |
| WAL | 自动保存包装器 | ✅ 新增 | core/ai_call_wrapper.py |
| L2 | VectorStore | ✅ 已有 | infrastructure/vector_store.py |
| L2 | 向量召回配置 | ✅ 新增 | core/context_recall.py |
| L3 | GitNotes管理 | ✅ 已有 | core/git_notes_manager.py |
| L4 | MEMORY.md更新 | ✅ 新增 | core/daily_meditation.py |
| 定时任务 | 每日冥想 | ✅ 新增 | core/daily_meditation.py |

### 4.3 数据流

```
用户操作 → GUI
  ↓
AI调用包装器 (AICallWrapper)
  ↓
WAL写入 (wal_manager.write_before_ai_call)
  ↓
SessionState更新 (session_state_manager.write_before_ai_call)
  ↓
AI API调用 (online_provider.generate_text)
  ↓
WAL标记完成 (wal_manager.mark_completed)
  ↓
SessionState清除任务 (session_state_manager.clear_active_task)
  ↓
每日冥想 (daily_meditation.execute - 定时23:00)
  ↓
更新MEMORY.md (_update_memory)
```

## 七、踩坑经验

### 问题1：APScheduler未安装

**现象**：日志警告 `APScheduler未安装，定时任务功能不可用`

**解决**：
```bash
pip install apscheduler
```

**降级方案**：代码自动降级到简单定时器（threading.Timer）

### 问题2：DeepSeek Embedding不可用

**现象**：配置embedding_model: deepseek后，日志警告降级到local

**原因**：DeepSeek暂不提供独立的Embedding API

**解决**：代码自动降级到sentence-transformers，不影响其他功能

### 问题3：WAL文件过大

**现象**：长期使用后.wal.json文件变大

**解决**：每次AI调用成功后会自动清理，无需手动干预

### 问题4：冥想任务不执行
