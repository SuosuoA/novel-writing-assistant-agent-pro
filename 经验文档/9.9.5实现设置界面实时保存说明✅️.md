## 1. 设计目标

根据《9.9大模型集成.txt》设计文档，修改设置界面的保存逻辑，使其调用`ConfigService.update_ai_config()`更新AI配置，从而触发EventBus事件，实现AI服务配置的实时生效。

### 核心需求

1. **设置界面保存**：用户在设置界面修改AI配置后，调用ConfigService更新配置
2. **EventBus事件发布**：ConfigService更新配置后自动发布`config.changed`事件
3. **AIServiceManager响应**：订阅事件并动态切换AI Provider
4. **无需重启**：配置修改后立即生效，无需重启应用

## 2. 架构设计

```

    ↓

        ↓
    动态切换AIProvider
```

## 3. 实现细节

### 3.1 修改前的保存逻辑

```python
def _on_save_settings(self) -> None:
    """保存设置到config.yaml并应用"""
    # ...构建config_data...
    
    try:
        # 直接写入config.yaml
        with open(config_path, "w", encoding="utf-8") as f:
            yaml.dump(config_data, f, allow_unicode=True, default_flow_style=False)
        
        # 旧方式更新配置（不会触发EventBus事件）
        if self._services and self._services.config:
            self._services.config.set("provider", self._provider_var.get())
            self._services.config.set("model", self._model_var.get())
            self._services.config.set("temperature", self._temp_var.get())
        
        # 应用设置
        self._apply_settings(config_data)
        
        messagebox.showinfo("成功", "设置已保存并应用！\n部分设置可能需要重启程序才能完全生效。")
```

**问题**：
- 直接写入config.yaml，不会触发ConfigService的监听器
- 旧方式更新配置不会发布EventBus事件
- 用户需要重启应用才能切换AI服务

### 3.2 修改后的保存逻辑

```python
def _on_save_settings(self) -> None:
    """保存设置到config.yaml并应用
    
    V2.5更新：调用ConfigService.update_ai_config()更新AI配置
    """
    # ...构建config_data...
    
    try:
        # 1. 保存到config.yaml文件
        with open(config_path, "w", encoding="utf-8") as f:
            yaml.dump(config_data, f, allow_unicode=True, default_flow_style=False)
        
        # 2. 调用ConfigService更新AI配置（触发EventBus事件）
        try:
            from core import get_config_service
            config_service = get_config_service()
            
            # 构建AI配置字典
            ai_config = {
                "service_mode": self._service_mode_var.get(),
                "provider": self._provider_var.get(),
                "model": self._model_var.get(),
                "api_key": self._api_key_var.get(),
                "local_url": self._local_url_var.get(),
                "temperature": self._temp_var.get(),
            }
            
            # 更新AI配置（会发布config.changed事件）
            config_service.update_ai_config(ai_config)
            logger.info("AI配置已更新，EventBus事件已发布")
            
        except Exception as e:
            logger.warning(f"ConfigService更新失败，使用旧方式：{e}")
            # 降级：使用旧方式更新配置
            if self._services and self._services.config:
                self._services.config.set("provider", self._provider_var.get())
                self._services.config.set("model", self._model_var.get())
                self._services.config.set("temperature", self._temp_var.get())
        
        # 3. 应用设置
        self._apply_settings(config_data)
        
        self._set_status("设置已保存并应用")
        messagebox.showinfo("成功", "设置已保存并应用！\nAI服务配置已实时生效。")
```

**改进**：
- 调用`ConfigService.update_ai_config()`更新AI配置
- 自动发布`config.changed`事件到EventBus
- AIServiceManager订阅事件并动态切换AI Provider
- 配置修改后立即生效，无需重启应用
- 包含降级逻辑：如果ConfigService不可用，使用旧方式更新

## 6. 技术要点

### 6.1 降级机制

当ConfigService不可用时，自动降级到旧方式更新配置：

```python
try:
    # 优先使用ConfigService
    config_service.update_ai_config(ai_config)
except Exception as e:
    logger.warning(f"ConfigService更新失败，使用旧方式：{e}")
    # 降级：使用旧方式更新配置
    if self._services and self._services.config:
        self._services.config.set("provider", ...)
```

### 6.2 线程安全

- ConfigService内部使用RLock保护配置更新
- EventBus发布事件使用线程池异步处理
- UI操作在主线程执行，线程安全

### 6.3 日志记录

```python
logger.info("AI配置已更新，EventBus事件已发布")
logger.warning(f"ConfigService更新失败，使用旧方式：{e}")
```
