# 异步事件处理器说明

## 概述

AsyncHandler是Novel Writing Assistant-Agent Pro项目的异步任务处理核心模块，用于在GUI应用中执行耗时操作而不阻塞UI线程。

## 架构设计

### 核心组件

```
AsyncHandler
├── PriorityTaskQueue    # 优先级任务队列
├── ThreadPoolExecutor   # 线程池（3-5工作线程）
├── Worker Threads       # 后台工作线程
└── Main Thread Callback # 主线程回调（root.after）
```

### 任务生命周期

```
PENDING → RUNNING → COMPLETED/FAILED/TIMEOUT/CANCELLED
```

## 功能特性

### 1. 优先级任务队列

支持5级优先级（数值越小优先级越高）：

| 优先级 | 数值 | 用途 |
|--------|------|------|
| HIGHEST | 0 | 立即执行的关键任务 |
| HIGH | 10 | 高优先级任务 |
| NORMAL | 20 | 普通任务（默认） |
| LOW | 30 | 低优先级任务 |
| BACKGROUND | 40 | 后台任务 |

### 2. 线程池配置

- 工作线程数：可配置3-5个（推荐4个）
- 线程命名：AsyncHandler-Worker-{n}
- 守护线程：自动随主进程退出

### 3. 主线程回调

所有回调（成功/失败）通过`root.after(0, callback)`在主线程执行，确保：
- GUI操作线程安全
- 无需额外的线程同步
- 符合Tkinter编程规范

### 4. 任务管理

- 任务ID自动生成或自定义
- 任务状态实时查询
- 支持取消PENDING状态任务
- 超时自动中断（默认30秒）

## 使用示例

### 基本使用

```python
from core.async_handler import AsyncHandler, TaskPriority, init_async_handler

# 初始化（在GUI主窗口创建后）
handler = init_async_handler(root=tk_root, worker_count=4)

# 定义耗时任务
def long_running_task(file_path):
    import time
    time.sleep(5)  # 模拟耗时操作
    return "处理完成"

# 定义回调
def on_success(result):
    print(f"成功: {result}")

def on_error(error):
    print(f"失败: {error}")

# 提交任务
task_id = handler.submit(
    func=long_running_task,
    args=("data.txt",),
    callback=on_success,
    error_callback=on_error,
    priority=TaskPriority.HIGH,
    timeout=60.0
)
```

### GUI按钮点击不阻塞

```python
import tkinter as tk
from tkinter import ttk
from core.async_handler import init_async_handler, TaskPriority

class MyApplication:
    def __init__(self, root):
        self.root = root
        self.handler = init_async_handler(root=root, worker_count=4)
        
        # 创建按钮
        self.btn = ttk.Button(
            root,
            text="执行耗时任务",
            command=self.on_button_click
        )
        self.btn.pack(pady=20)
        
        # 状态标签
        self.status = ttk.Label(root, text="就绪")
        self.status.pack()
    
    def on_button_click(self):
        """按钮点击立即返回，不阻塞UI"""
        self.status.config(text="任务执行中...")
        self.btn.config(state="disabled")
        
        self.handler.submit(
            func=self.heavy_work,
            callback=self.on_complete,
            error_callback=self.on_error,
            priority=TaskPriority.NORMAL
        )
        # 按钮点击后立即返回，UI保持响应
    
    def heavy_work(self):
        """耗时操作"""
        import time
        time.sleep(3)
        return "处理完成"
    
    def on_complete(self, result):
        """成功回调（在主线程执行）"""
        self.status.config(text=result)
        self.btn.config(state="normal")
    
    def on_error(self, error):
        """错误回调（在主线程执行）"""
        self.status.config(text=f"错误: {error}")
        self.btn.config(state="normal")

if __name__ == "__main__":
    root = tk.Tk()
    app = MyApplication(root)
    root.mainloop()
```

### 小说生成任务

```python
def generate_chapter_task(outline, style, characters):
    """章节生成任务（在后台线程执行）"""
    # 调用V5生成逻辑
    from scripts.optimized_generator_v2 import generate_chapter
    result = generate_chapter(outline, style, characters)
    return result

def on_chapter_complete(result):
    """章节生成完成（在主线程更新UI）"""
    text_widget.insert(tk.END, result)
    text_widget.see(tk.END)
    progress_bar.stop()

def on_chapter_error(error):
    """章节生成失败"""
    messagebox.showerror("生成失败", str(error))
    progress_bar.stop()

# 提交生成任务
handler.submit(
    func=generate_chapter_task,
    args=(outline, style, characters),
    callback=on_chapter_complete,
    error_callback=on_chapter_error,
    priority=TaskPriority.HIGH,
    timeout=300.0  # 5分钟超时
)
```

## API参考

### AsyncHandler类

#### 构造函数

```python
AsyncHandler(
    root: Any = None,           # Tkinter root窗口
    worker_count: int = 4,       # 工作线程数
    default_timeout: float = 30.0 # 默认超时时间
)
```

#### 主要方法

| 方法 | 说明 | 返回值 |
|------|------|--------|
| `submit(func, args, kwargs, task_id, callback, error_callback, priority, timeout)` | 提交异步任务 | 任务ID |
| `cancel(task_id)` | 取消任务 | bool |
| `get_task_state(task_id)` | 获取任务状态 | TaskState |
| `get_task_result(task_id)` | 获取任务结果 | Any |
| `get_task_error(task_id)` | 获取任务错误 | Exception |
| `get_pending_count()` | 获取等待任务数 | int |
| `get_running_count()` | 获取运行任务数 | int |
| `get_statistics()` | 获取统计信息 | Dict |
| `clear_completed_tasks(max_age)` | 清理已完成任务 | int |
| `shutdown(wait, timeout)` | 关闭处理器 | None |
| `set_root(root)` | 设置root窗口 | None |

### Task数据类

```python
@dataclass
class Task:
    id: str                           # 任务ID
    func: Callable                    # 执行函数
    args: Tuple = ()                  # 位置参数
    kwargs: Dict = {}                 # 关键字参数
    priority: TaskPriority = NORMAL   # 优先级
    callback: Optional[Callable]      # 成功回调
    error_callback: Optional[Callable] # 错误回调
    timeout: float = 30.0             # 超时时间
    state: TaskState = PENDING        # 任务状态
    result: Any = None                # 执行结果
    error: Optional[Exception] = None # 错误信息
```

### TaskState枚举

| 状态 | 值 | 说明 |
|------|-----|------|
| PENDING | 0 | 等待执行 |
| RUNNING | 1 | 正在执行 |
| COMPLETED | 2 | 执行完成 |
| FAILED | 3 | 执行失败 |
| CANCELLED | 4 | 已取消 |
| TIMEOUT | 5 | 超时 |

## 与EventBus集成

AsyncHandler可与EventBus配合使用：

```python
from core import EventBus, AsyncHandler, init_async_handler

# 初始化
handler = init_async_handler(root=tk_root)
event_bus = EventBus()

# 异步事件处理
def on_generate_request(event):
    handler.submit(
        func=generate_content,
        args=(event.data,),
        callback=lambda r: event_bus.publish("generation.complete", r),
        error_callback=lambda e: event_bus.publish("generation.error", str(e))
    )

event_bus.subscribe("generation.request", on_generate_request)
```

## 注意事项

1. **回调线程安全**：所有回调自动在主线程执行，无需额外同步
2. **root窗口**：必须在Tkinter窗口创建后初始化handler
3. **超时控制**：长时间任务建议设置合理的timeout值
4. **资源清理**：应用退出前调用`handler.shutdown()`
