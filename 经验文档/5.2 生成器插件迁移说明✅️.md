## 一、迁移概述

将V5项目的3个核心生成器脚本迁移为Agent Pro项目的插件系统格式，实现`GeneratorPlugin`接口规范。

### 迁移清单

| 原脚本 | 目标插件 | 插件ID | 版本 |
|--------|----------|--------|------|
| `scripts/context_builder.py` | `plugins/context-builder-v1/` | `context-builder-v1` | 1.0.0 |
| `scripts/iterative_generator_v2.py` | `plugins/iterative-generator-v2/` | `iterative-generator-v2` | 2.0.0 |
| `scripts/optimized_generator_v2.py` | `plugins/novel-generator-v3/` | `novel-generator-v3` | 3.0.0 |

## 三、各插件迁移详情

### 3.1 上下文构建器 (context-builder-v1)

**原功能**:
- 构建优化的提示词（基于七要素提示词工程框架）
- 智能检索相关世界观片段
- 智能检索相关人物
- 压缩风格描述
- 构建前情提要
- 提取冲突钩子

**迁移变更**:
- 封装为`ContextBuilderPlugin`类，继承`GeneratorPlugin`
- 添加`build_optimized_prompt()`方法
- 保留完整的七要素提示词工程框架
- 强制要求章节结束必须添加【本章完】标记

**依赖**: 无外部依赖

### 3.2 迭代生成器 (iterative-generator-v2)

**原功能**:
- 迭代优化的章节生成
- 评分反馈循环优化
- 支持多种生成策略（创意/平衡/精确）
- 温度衰减控制

**迁移变更**:
- 封装为`IterativeGeneratorPlugin`类，继承`GeneratorPlugin`
- 添加`generate_with_iteration()`方法
- 保留完整的7步迭代流程：
  1. 整理打包请求内容
  2. 向大模型发送请求
  3. 接受返回文章
  4. 从多维度评分
  5. 判断是否满足停止条件
  6. 构建反馈
  7. 输出保存
- 保留核心常量（MAX_ITERATIONS=5, QUALITY_THRESHOLD=0.8）

**依赖**: weighted-validator（可选）

### 3.3 小说生成器 (novel-generator-v3)

**原功能**:
- 整合上下文构建、迭代生成、加权验证的完整流程
- 支持多种生成策略
- 自动补充【本章完】标记

**迁移变更**:
- 封装为`NovelGeneratorPlugin`类，继承`GeneratorPlugin`
- 添加`generate_chapter()`方法
- 依赖context-builder-v1和iterative-generator-v2
- 保留完整的生成流程和核心规则

**依赖**: context-builder-v1, iterative-generator-v2, weighted-validator

## 四、核心规则保留验证

### 4.1 【本章完】强制要求
- ✅ **已保留**
- `build_optimized_prompt()` 方法中强制添加【本章完】要求
- `_build_request_prompt()` 方法在每轮迭代中强制附加
- `_evaluate_content()` 方法检查【本章完】标记
- `generate_with_iteration()` 逻辑中作为停止条件之一

### 4.2 迭代上限
- ✅ **已保留**
- `MAX_ITERATIONS` 常量（默认5次）
- 迭代循环中使用 `for iteration in range(self.max_iterations)`

### 4.3 温度衰减
- ✅ **已保留**
- `GenerationStrategy` 枚举定义三种策略
- `temperature_map` 定义不同温度值
- `CREATIVE`: 0.9, `BALANCED`: 0.7, `PRECISE`: 0.5

### 4.4 评分阈值
- ✅ **已保留**
- `QUALITY_THRESHOLD` 常量（默认0.8）
- 停止条件：`total_score >= self.quality_threshold and has_chapter_end`

## 八、UI适配方法（重要）

### 8.1 进度回调

生成器插件支持进度回调，用于更新UI进度条和状态：

```python
# 方法1：通过回调函数
def generate_with_progress(
    prompt: str,
    progress_callback: Callable[[int, int, float, str], None]
) -> Tuple[str, Dict]:
    """
    progress_callback 参数：
    - iteration: 当前迭代次数
    - max_iterations: 最大迭代次数  
    - score: 当前评分
    - message: 状态消息
    """
    # 示例
    content, stats = generator.generate_with_progress(
        prompt=prompt,
        progress_callback=lambda it, max_it, score, msg: 
            self.update_progress_bar(it, max_it, score, msg)
    )

# 方法2：通过EventBus事件
# 插件会自动发布 "generation.progress" 事件
# UI订阅事件：
self.event_bus.subscribe("generation.progress", self._on_progress_update)

def _on_progress_update(self, event_data):
    iteration = event_data["iteration"]
    max_iterations = event_data["max_iterations"]
    score = event_data["score"]
    message = event_data["message"]
    
    # 更新进度条
    progress = (iteration / max_iterations) * 100
    self.progress_bar["value"] = progress
    
    # 更新状态标签
    self.status_label.config(text=message)
```

### 8.2 日志回调

生成器插件支持日志回调，用于更新UI日志显示：

```python
def generate_with_logging(
    prompt: str,
    log_callback: Callable[[str], None]
) -> Tuple[str, Dict]:
    """
    log_callback 参数：
    - message: 日志消息
    """
    # 示例
    content, stats = generator.generate_with_logging(
        prompt=prompt,
        log_callback=lambda msg: self.log_text.insert("end", msg + "\n")
    )
```

### 8.3 异步生成（推荐）

所有生成操作都应通过AsyncHandler异步执行，避免阻塞UI：

```python
# 在UI中使用异步处理器
def _on_generate_click(self):
    # 禁用按钮，显示加载状态
    self.generate_btn.config(state=tk.DISABLED, text="生成中...")
    
    # 使用异步处理器执行生成任务
    self.async_handler.execute(
        task_func=self._generate_task,
        on_complete=self._on_generate_complete
    )

def _generate_task(self):
    """异步生成任务（在工作线程执行）"""
    generator = self.plugin_registry.get_plugin("novel-generator-v3")
    generator.set_api_client(self.api_client)
    generator.set_config(quality_threshold=0.8, max_iterations=5)
    
    # 执行生成（带进度回调）
    content, stats = generator.generate_with_progress(
        prompt=prompt,
        progress_callback=self._create_progress_callback()
    )
    
    return content, stats

def _on_generate_complete(self, result):
    """生成完成回调（在主线程执行）"""
    content, stats = result
    
    # 更新UI（线程安全：通过root.after调度）
    self.root.after(0, lambda: self._update_result(content, stats))
    
    # 恢复按钮状态
    self.generate_btn.config(state=tk.NORMAL, text="开始生成")

def _create_progress_callback(self):
    """创建进度回调（返回结果到主线程）"""
    def callback(iteration, max_iterations, score, message):
        # 使用root.after确保线程安全
        self.root.after(0, lambda: self._update_progress(
            iteration, max_iterations, score, message
        ))
    return callback
```

### 8.4 线程安全

**重要**：所有UI更新必须通过 `root.after(0, callback)` 调度到主线程执行：

```python
# 错误方式（在工作线程直接操作UI）
self.progress_bar["value"] = 50  # ❌ 可能导致崩溃

# 正确方式（通过root.after调度）
self.root.after(0, lambda: self.progress_bar.config(value=50))  # ✅ 线程安全
```

## 九、注意事项

1. **依赖安装**: 使用前需确保`requirements.txt`中的依赖已安装
2. **API配置**: 需要配置API客户端才能调用生成功能
3. **核心规则**: 严禁修改【本章完】强制要求、迭代上限、质量阈值等核心规则
4. **版本兼容**: V5脚本与V6插件并存期间，功能保持一致
5. **线程安全**: 所有UI操作必须通过 `root.after(0, callback)` 调度
6. **异步执行**: 所有生成操作都应通过AsyncHandler异步执行
7. **错误处理**: 生成失败时应提供友好的错误消息，不要直接崩溃
5. **扩展性**: 可通过`dependencies`字段声明插件依赖关系

### 10.2 功能概述

快捷创作生成器插件是Agent Pro项目的新增核心生成器，提供快速生成各类设定的能力，包括世界观、大纲、人物设定、关键情节等。

### 10.3 核心能力

| 能力 | 描述 |
|------|------|
| 世界观生成 | 基于关键词生成完整世界设定 |
| 大纲生成 | 生成章节规划和剧情走向 |
| 人物生成 | 生成立体丰满的人物形象 |
| 情节生成 | 生成关键剧情和转折点 |
| 参考文本解析 | 从已有文本提取设定元素并融合 |
| 结果存储 | 保存到新项目或导入当前项目 |

### 10.4 目录结构

```

```

### 10.5 生成详细程度

| 类型 | Token限制 | 说明 |
|------|-----------|------|
| quick | 2000-3000 | 快速生成，较少细节 |
| standard | 3000-5000 | 标准生成，平衡细节和速度 |
| detailed | 5000-8000 | 详细生成，丰富细节 |

### 10.6 参考文本解析

支持自动识别以下文本类型：

| 类型 | 枚举值 | 提取内容 |
|------|--------|----------|
| 小说文本 | novel | 风格、人物、世界观元素 |
| 大纲文本 | outline | 章节规划、关键事件 |
| 人物设定 | character | 姓名、性格、能力、关系 |
| 世界观设定 | worldview | 时代、力量体系、势力 |
| 混合类型 | mixed | 多种元素综合 |

### 10.7 结果存储功能

**冲突处理策略**：

| 策略 | 枚举值 | 说明 |
|------|--------|------|
| 保留原有设定 | keep_original | 默认策略，保护用户已有设定 |
| 覆盖原有设定 | overwrite | 直接覆盖原有文件 |
| 合并设定 | merge | 智能合并新旧内容 |
| 重命名新设定 | rename_new | 新文件添加时间戳后缀 |

def generate_all(request: QuickCreationRequest) -> QuickCreationResult

def parse_reference_text(text: str) -> Dict[str, Any]

def save_result_to_project(

def import_to_current_project(

### 10.9 使用示例

**世界观生成**：

```python
from plugins.quick_creator_v1 import QuickCreationPlugin

# 创建插件实例
plugin = QuickCreationPlugin()
plugin.initialize(context)
plugin.set_api_client(openai_client)

# 生成世界观
worldview = plugin.generate_worldview(
    keywords=["修仙", "仙界", "灵气"],
    genre="仙侠",
    generation_type="standard"
)

print(f"世界名称: {worldview.setting_name}")
print(f"力量体系: {worldview.power_system}")
```

**带参考文本生成**：

```python
reference_text = """
世界观设定：
时代：上古修仙时期
力量体系：修仙境界，从练气到化神
势力：青云宗、天剑门、魔教

人物设定：
姓名：李云
角色定位：主角
性格：坚韧不拔、重情重义
"""

# 生成世界观（融合参考文本）
worldview = plugin.generate_worldview(
    keywords=["修仙", "复仇"],
    reference_text=reference_text,
    genre="仙侠"
)
```

**保存和导入**：

```python
from core.models import QuickCreationRequest

# 生成全部设定
request = QuickCreationRequest(
    keywords="修仙,逆袭,系统",
    target="all",
    genre="玄幻",
    character_count=3,
    chapter_count=20,
    detailed_level="standard"
)

result = plugin.generate_all(request)

if result.success:
    # 保存到新项目
    project_path = plugin.save_result_to_project(
        result,
        project_name="修仙之路",
        output_path="E:/小说项目"
    )
    
    # 或导入到当前项目
    import_result = plugin.import_to_current_project(
        result,
        conflict_strategy="keep_original"
    )
```
