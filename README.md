## 项目简介

本项目是一个自动化AI写作Agent，通过多维度评分反馈机制实现中长篇小说的连续生成与质量优化。

> 作者背景：社科人文方向研究者，非专业程序员。本项目基于WorkBuddy平台自动化构建，是一次AI辅助开发的探索实践。

---

## 核心功能

### 1. 文档解析与知识提取

自动解析用户上传的设定文档，提取核心创作要素：

- **世界观设定** — 故事背景、规则体系、地理环境
- **故事大纲** — 情节脉络、章节规划、冲突设计
- **人物档案** — 角色性格、关系网络、行为动机
- **风格画像** — 叙事风格、语言特征、情感基调

### 2. 评分反馈循环优化生成

这是本项目的核心创新点。不同于单次生成，我们采用**多维度评分 + 自动反馈优化**的迭代机制：

```
┌─────────────────────────────────────────────────────────┐
│                    生成循环流程                          │
├─────────────────────────────────────────────────────────┤
│  1. 上下文构建  →  整合世界观/人设/风格/大纲/历史记忆    │
│  2. 内容生成    →  调用LLM API生成章节内容              │
│  3. 多维评分    →  6维度加权评估生成质量                │
│  4. 达标判断    →  评分≥0.8 且含【本章完】→ 保存       │
│  5. 反馈优化    →  评分<0.8 → 构建建议，重新生成        │
│  (最多迭代5次)                                          │
└─────────────────────────────────────────────────────────┘
```

**六维度加权评分体系：**

| 维度 | 权重 | 评估内容 |
|------|------|----------|
| 风格一致性 | 25% | 与学习风格的匹配程度 |
| 人设一致性 | 25% | 角色行为与设定的吻合度 |
| 世界观一致性 | 20% | 设定规则的遵循程度 |
| 大纲符合性 | 15% | 情节走向与大纲的一致性 |
| 字数符合性 | 10% | 目标字数的达成情况 |
| 自然度 | 5% | 语言流畅度与可读性 |

**强制约束：** 每章必须以 `【本章完】` 标记结束，否则视为未完成。

### 3. 上下文记忆机制

- 保持前5章的情节记忆
- 确保人物行为、情节发展的连续性
- 自动处理伏笔与呼应

## 技术架构

```
Novel Writing Assistant-Agent Pro/
├── agents/          # Agent系统（MasterAgent + 专家Agent协作）
├── core/            # 核心模块（事件总线、插件系统、配置管理）
├── infrastructure/  # 基础设施（日志、监控、熔断器）
├── plugins/         # 插件扩展系统
├── services/        # LLM服务与API客户端
├── scripts/         # V5核心算法（解析、生成、验证）
└── prompts/         # 提示词模板库
```

**技术栈：** Python 3.12 + Tkinter + SQLite (WAL) + OpenAI SDK

## 使用说明

### 环境要求

- Python 3.12+
- 外部LLM API（支持OpenAI标准接口，如DeepSeek/OpenAI/Anthropic/Ollama）

### 快速启动

```bash
# 安装依赖
pip install -r requirements.txt

# 配置API
# 编辑 config.yaml，填入您的API密钥

# 启动应用
python gui_main.py
```

### 基本流程

1. **上传设定文档** — 大纲、人设、世界观、风格参考文本
2. **点击解析** — 系统自动提取结构化知识
3. **开始生成** — 选择起始章节，启动自动化生成
4. **查看结果** — 检查评分详情与迭代历史
