## 一、快速开始

### 1.1 30秒创建你的第一个插件

**步骤1：创建插件目录**

```bash
mkdir -p plugins/my-first-plugin
cd plugins/my-first-plugin
```

**步骤2：创建plugin.json清单**

```json
{
    "id": "my-first-plugin",
    "name": "我的第一个插件",
    "version": "1.0.0",
    "description": "一个示例插件",
    "author": "你的名字",
    "plugin_type": "tool",
    "api_version": "1.0",
    "priority": 100,
    "enabled": true,
    "dependencies": [],
    "conflicts": [],
    "permissions": ["file.read"],
    "min_platform_version": "6.0.0",
    "entry_class": "MyFirstPlugin"
}
```

**步骤3：创建插件主类**

```python
# plugin.py
from core.plugin_interface import BasePlugin, PluginMetadata, PluginType, PluginContext

class MyFirstPlugin(BasePlugin):
    """我的第一个插件"""
    
    @classmethod
    def get_metadata(cls) -> PluginMetadata:
        return PluginMetadata(
            id="my-first-plugin",
            name="我的第一个插件",
            version="1.0.0",
            description="一个示例插件",
            author="你的名字",
            plugin_type=PluginType.TOOL
        )
    
    def initialize(self, context: PluginContext) -> bool:
        """初始化插件"""
        self._context = context
        print(f"插件 {self.metadata.name} 初始化成功")
        return True
    
    def shutdown(self) -> bool:
        """关闭插件"""
        print(f"插件 {self.metadata.name} 已关闭")
        return True
```

**步骤4：创建入口文件**

```python
# __init__.py
from .plugin import MyFirstPlugin

__all__ = ["MyFirstPlugin"]

def register_plugin():
    """供插件加载器调用"""
    return MyFirstPlugin
```

**完成！** 你的插件已经可以加载了。

## 二、插件类型选择

### 2.1 7种插件类型一览

| 类型 | 用途 | 典型场景 | 专用接口 |
|------|------|----------|----------|
| **PROTOCOL** | 格式转换 | 导入/导出功能 | `import_data()`, `export_data()` |
| **AI** | AI能力封装 | LLM调用、模型集成 | `call()`, `stream_call()` |
| **STORAGE** | 数据持久化 | 数据库、文件存储 | `save()`, `load()`, `delete()` |
| **ANALYZER** | 内容分析 | 大纲解析、风格学习 | `analyze()` |
| **GENERATOR** | 内容生成 | 章节生成、上下文构建 | `generate()` |
| **VALIDATOR** | 质量验证 | 内容评分、一致性检查 | `validate()` |
| **TOOL** | 工具功能 | 热榜抓取、统计分析 | `execute()` |

### 2.2 类型选择决策树

```
你的插件要做什么？
│
├─ 转换文件格式（导入/导出）
│  └─ 选择 PROTOCOL
│
├─ 调用AI模型
│  └─ 选择 AI
│
├─ 存储数据
│  └─ 选择 STORAGE
│
├─ 分析内容（不生成新内容）
│  └─ 选择 ANALYZER

├─ 生成内容
│  └─ 选择 GENERATOR
│
├─ 验证内容质量

│
└─ 其他工具功能
   └─ 选择 TOOL
```

## 四、接口详解

### 4.1 BasePlugin基础接口

所有插件必须继承`BasePlugin`类：

```python
from core.plugin_interface import BasePlugin, PluginMetadata, PluginType, PluginContext

class MyPlugin(BasePlugin):
    """插件基类实现"""
    
    @classmethod
    def get_metadata(cls) -> PluginMetadata:
        """获取插件元数据（类方法，必须实现）"""
        return PluginMetadata(
            id="my-plugin",
            name="我的插件",
            version="1.0.0",
            description="插件描述",
            author="作者",
            plugin_type=PluginType.TOOL
        )
    
    def initialize(self, context: PluginContext) -> bool:
        """初始化插件（必须实现）
        
        Args:
            context: 插件上下文，包含核心服务引用
            
        Returns:
            True表示初始化成功，False表示失败
        """
        self._context = context
        
        # 订阅事件（可选）
        self._context.event_bus.subscribe(
            "generation.completed",
            self._on_generation_completed
        )
        
        return True
    
    def shutdown(self) -> bool:
        """关闭插件（可选重写）
        
        用于优雅关闭，释放资源
        
        Returns:
            True表示关闭成功
        """
        # 取消事件订阅
        self._context.event_bus.unsubscribe(
            f"{self.metadata.id}.gen_completed"
        )
        
        return True
    
    def cleanup(self) -> bool:
        """强制资源回收（可选重写）
        
        与shutdown区分，用于强制清理
        
        Returns:
            True表示清理成功
        """
        self._cleanup_done = True
        return True
```

**重要说明**：
- ❌ `BasePlugin` **不包含** `execute()` 方法
- ✅ 使用专用接口：`AnalyzerPlugin.analyze()`、`GeneratorPlugin.generate()`、`ValidatorPlugin.validate()`

### 4.2 AnalyzerPlugin分析器接口

用于内容分析类插件：

```python
from core.plugin_interface import AnalyzerPlugin, PluginMetadata, PluginType
from typing import Dict, Any, List, Optional

class OutlineParserPlugin(AnalyzerPlugin):
    """大纲解析器插件示例"""
    
    @classmethod
    def get_metadata(cls) -> PluginMetadata:
        return PluginMetadata(
            id="outline-parser-v3",
            name="大纲解析器 V3",
            version="3.0.0",
            description="基于LangChain的智能大纲解析",
            author="项目组",
            plugin_type=PluginType.ANALYZER
        )
    
    def initialize(self, context: PluginContext) -> bool:
        self._context = context
        return True
    
    def analyze(self, content: str, 
                options: Optional[Dict[str, Any]] = None) -> Dict[str, Any]:
        """分析内容并返回结果
        
        Args:
            content: 待分析的内容文本
            options: 分析选项（可选）
            
        Returns:
            分析结果字典，格式由插件定义
        """
        chapters = self._extract_chapters(content)
        
        return {
            "chapters": chapters,
            "total_count": len(chapters),
            "format": options.get("format", "auto") if options else "auto"
        }
    
    def get_supported_formats(self) -> List[str]:
        """获取支持的输入格式"""
        return ["txt", "json", "yaml", "md"]
    
    def get_analysis_types(self) -> List[str]:
        """获取支持的分析类型"""
        return ["outline", "structure", "chapters"]
    
    def _extract_chapters(self, content: str) -> List[Dict]:
        """内部方法：提取章节列表"""
        chapters = []
        lines = content.split("\n")
        for line in lines:
            if line.startswith("第") and "章" in line:
                chapters.append({"title": line.strip(), "content": ""})
        return chapters
```

### 4.3 GeneratorPlugin生成器接口

```python
from core.plugin_interface import GeneratorPlugin, PluginMetadata, PluginType
from core.models import GenerationRequest, GenerationResult
from typing import Dict, Any, List, Tuple

class NovelGeneratorPlugin(GeneratorPlugin):
    """小说生成器插件示例"""

            description="迭代优化的小说章节生成",
            author="项目组",

        self._context = context
        return True
    
    def generate(self, request: GenerationRequest) -> GenerationResult:
        """生成内容
        
        Args:
            request: 生成请求（Pydantic模型）
            
        Returns:
            生成结果（包含内容和评分）
        """
        content = ""
        iteration_count = 0
        
        # 迭代生成逻辑
        for i in range(request.max_iterations):
            iteration_count = i + 1
            # 调用LLM生成...
            # 验证评分...
            # 如果评分达标则break
            
        return GenerationResult(
            request_id=request.request_id,
            content=content,
            word_count=len(content),
            iteration_count=iteration_count,
            validation_scores=None
        )
    
    def validate_request(self, request: GenerationRequest) -> Tuple[bool, List[str]]:
        """验证请求是否有效
        
        Returns:
            (是否有效, 错误列表)

        errors = []
        
        if not request.title:
            errors.append("章节标题不能为空")
        if request.word_count < 500 or request.word_count > 10000:
            errors.append(f"字数范围应为500-10000，当前为{request.word_count}")
        if not request.outline:
            errors.append("章节大纲不能为空")
            
        return len(errors) == 0, errors
    
    def get_generation_options(self) -> Dict[str, Any]:
        """获取生成选项定义"""
        return {
            "word_count": {
                "type": "integer",
                "default": 2000,
                "min": 500,
                "max": 10000,
                "description": "目标字数"
            },
            "max_iterations": {
                "type": "integer",
                "default": 5,
                "min": 1,
                "max": 10,
                "description": "最大迭代次数"
            }
        }
    
    def cancel(self, request_id: str) -> bool:
        """取消生成（可选实现）"""
        # 实现取消逻辑
        return False
```

### 4.4 ValidatorPlugin验证器接口

用于质量验证类插件：

```python
from core.plugin_interface import ValidatorPlugin, PluginMetadata, PluginType
from core.models import ValidationScores
from typing import Dict, Any, List, Optional

class WeightedValidatorPlugin(ValidatorPlugin):
    """加权验证器插件示例"""
    
    @classmethod
    def get_metadata(cls) -> PluginMetadata:
        return PluginMetadata(
            id="weighted-validator",
            name="加权验证器",
            version="1.0.0",
            description="6维度加权评分验证",
            author="项目组",
            plugin_type=PluginType.VALIDATOR
        )
    
    def initialize(self, context: PluginContext) -> bool:
        self._context = context
        return True
    
    def validate(self, content: str, 
                 context: Optional[Dict[str, Any]] = None) -> ValidationScores:
        """验证内容并返回评分
        
        Args:
            content: 待验证内容
            context: 验证上下文（包含大纲、风格等）
            
        Returns:
            ValidationScores评分对象
        """
        scores = ValidationScores()
        
        # 字数评分
        word_count = len(content)
        target_count = context.get("word_count_target", 2000) if context else 2000
        scores.word_count_score = min(1.0, word_count / target_count)
        
        # 其他维度评分...
        scores.style_score = self._evaluate_style(content, context)
        scores.character_score = self._evaluate_character(content, context)
        # ...
        
        # 检查章节结束标记
        scores.has_chapter_end = "【本章完】" in content
        
        # 计算总分
        scores.calculate_total()
        
        return scores
    
    def get_validation_dimensions(self) -> List[str]:
        """获取验证维度"""
        return [
            "word_count",      # 字数（10%）
            "outline",         # 大纲（15%）
            "style",           # 风格（25%）
            "character",       # 人设（25%）
            "worldview",       # 世界观（20%）
            "naturalness"      # 自然度（5%）
        ]
    
    def _evaluate_style(self, content: str, context: Optional[Dict]) -> float:
        """风格评分（内部方法）"""
        # 实现风格评分逻辑
        return 0.8
```

## 五、数据模型

### 5.1 Pydantic模型定义

所有数据模型使用Pydantic v2定义：

```python
from pydantic import BaseModel, Field, ConfigDict
from typing import Dict, Any, List, Optional
from datetime import datetime

class GenerationRequest(BaseModel):
    """生成请求模型"""
    model_config = ConfigDict(from_attributes=True)
    
    request_id: str = Field(..., description="请求ID")
    title: str = Field(..., description="章节标题")
    outline: str = Field(..., description="章节大纲")
    word_count: int = Field(default=2000, ge=500, le=10000, description="目标字数")
    max_iterations: int = Field(default=5, ge=1, le=10, description="最大迭代次数")
    context_chapters: int = Field(default=5, ge=0, le=10, description="上下文章节数")

class GenerationResult(BaseModel):
    """生成结果模型"""
    model_config = ConfigDict(from_attributes=True)
    
    request_id: str = Field(..., description="请求ID")
    content: str = Field(..., description="生成内容")
    word_count: int = Field(..., description="实际字数")
    iteration_count: int = Field(..., description="迭代次数")
    validation_scores: Optional['ValidationScores'] = Field(None, description="验证评分")

class ValidationScores(BaseModel):
    """验证评分模型"""
    model_config = ConfigDict(from_attributes=True)
    
    word_count_score: float = Field(ge=0.0, le=1.0, description="字数评分")
    outline_score: float = Field(ge=0.0, le=1.0, description="大纲评分")
    style_score: float = Field(ge=0.0, le=1.0, description="风格评分")
    character_score: float = Field(ge=0.0, le=1.0, description="人设评分")
    worldview_score: float = Field(ge=0.0, le=1.0, description="世界观评分")
    naturalness_score: float = Field(ge=0.0, le=1.0, description="自然度评分")
    total_score: float = Field(ge=0.0, le=1.0, description="总分")
    has_chapter_end: bool = Field(description="是否包含章节结束标记")
    
    def calculate_total(self) -> float:
        """6维度加权计算"""
        self.total_score = (
            self.word_count_score * 0.10 +
            self.outline_score * 0.15 +
            self.style_score * 0.25 +
            self.character_score * 0.25 +
            self.worldview_score * 0.20 +
            self.naturalness_score * 0.05
        )
        return self.total_score
```

## 六、事件通信

### 6.1 事件总线机制

插件间通过EventBus进行解耦通信：

```python
def initialize(self, context: PluginContext) -> bool:
    self._context = context
    
    # 订阅事件
    self._context.event_bus.subscribe(
        "generation.started",
        self._on_generation_started,
        handler_id=f"{self.metadata.id}.gen_started"
    )
    
    return True

def _on_generation_started(self, event):
    """事件处理函数"""
    # Tkinter线程安全：使用after()调度到主线程
    def update_ui():
        print(f"生成开始: {event.data.get('chapter_title')}")
    
    # 如果需要操作UI
    if hasattr(self._context, 'root'):
        self._context.root.after(0, update_ui)
    else:
        update_ui()

def publish_event(self):
    """发布事件"""
    self._context.event_bus.publish(
        "outline.parse_completed",
        {
            "result": {"chapters": []},
            "request_id": "req_001"
        }
    )

def shutdown(self) -> bool:
    """取消订阅"""
    self._context.event_bus.unsubscribe(
        f"{self.metadata.id}.gen_started"
    )
    return True
```

### 6.2 标准事件类型

| 事件类型 | 说明 | 数据格式 |
|----------|------|----------|
| `plugin.loaded` | 插件加载完成 | `{plugin_id, version}` |
| `plugin.unloaded` | 插件卸载完成 | `{plugin_id}` |
| `plugin.error` | 插件错误 | `{plugin_id, error}` |
| `outline.parse_requested` | 大纲解析请求 | `{content, request_id}` |
| `outline.parse_completed` | 大纲解析完成 | `{result, request_id}` |
| `generation.started` | 生成开始 | `{request_id, chapter_title}` |
| `generation.completed` | 生成完成 | `{result}` |
| `validation.completed` | 验证完成 | `{scores, passed}` |

### 6.3 线程安全注意事项

**Tkinter线程安全原则**：
- ❌ 不要在EventBus回调中直接操作UI
- ✅ 使用 `widget.after(0, callback)` 调度到主线程

```python
def _on_event(self, event):
    """错误示例：直接操作UI"""
    # ❌ 危险：可能导致线程冲突
    self.ui_label.config(text="更新")
    
def _on_event_safe(self, event):
    """正确示例：线程安全"""
    # ✅ 安全：通过after()调度
    def update_ui():
        self.ui_label.config(text="更新")
    
    self._context.root.after(0, update_ui)
```

```python
def check_permission(plugin_id: str, permission: str) -> bool:
```

## 九、发布规范

### 9.1 版本号规范

使用语义化版本：`MAJOR.MINOR.PATCH`

- **MAJOR**：不兼容的API变更
- **MINOR**：向后兼容的功能新增
- **PATCH**：向后兼容的问题修复

示例：
- `1.0.0` → `1.0.1`：修复bug
- `1.0.0` → `1.1.0`：新增功能

## 十、常见问题

### 10.1 插件加载失败

**问题**：插件无法加载

**检查清单**：
- [ ] `plugin.json`格式是否正确
- [ ] `entry_class`是否与主类名一致
- [ ] `__init__.py`是否正确导出主类
- [ ] 依赖插件是否已加载
- [ ] Python版本是否满足要求（3.12+）

### 10.2 事件订阅失效

**问题**：订阅的事件没有触发

**解决方案**：

```python
# 错误：忘记返回True
def initialize(self, context):
    self._context = context
    self._context.event_bus.subscribe("event", self._handler)
    # 缺少 return True

# 正确：必须返回True
def initialize(self, context):
    self._context = context
    self._context.event_bus.subscribe("event", self._handler)
    return True  # ✅
```

### 10.3 线程安全问题

**问题**：UI操作导致崩溃

**原因**：在非主线程操作Tkinter组件

**解决方案**：

```python
# 错误：直接操作UI
def _on_event(self, event):
    self.label.config(text="更新")  # ❌

# 正确：使用after()调度
def _on_event(self, event):
    def update():
        self.label.config(text="更新")
    self._context.root.after(0, update)  # ✅
```

### 10.4 依赖循环

**问题**：插件间相互依赖导致加载失败

**解决方案**：

1. 检查`dependencies`声明
2. 使用EventBus解耦依赖
3. 使用ServiceLocator延迟获取

```python
# 错误：直接导入其他插件
from plugins.other_plugin import OtherPlugin

# 正确：通过ServiceLocator获取
def initialize(self, context):
    other = context.service_locator.get_service("other_plugin")
```

### 10.5 V5保护模块冲突

**问题**：尝试覆盖保护模块

**保护模块列表**：
- `outline-parser-v3`
- `style-learner-v2`
- `character-manager`
- `worldview-parser`
- `context-builder`
- `iterative-generator-v2`
- `weighted-validator`
- `optimized-generator-v2`

**解决方案**：
- 使用不同的插件ID
- 通过继承扩展功能

### 快速创建插件模板

```bash
# 创建目录结构
mkdir -p plugins/my-plugin/{core,tests,assets}

cat > plugins/my-plugin/plugin.json << 'EOF'

from .plugin import MyPlugin

EOF

from core.plugin_interface import BasePlugin, PluginMetadata, PluginType, PluginContext

class MyPlugin(BasePlugin):

        return PluginMetadata(
            id="my-plugin",

            description="插件描述",
            author="作者",
            plugin_type=PluginType.TOOL
        )

        self._context = context
        return True
    
    def shutdown(self) -> bool:
        return True
EOF
```
