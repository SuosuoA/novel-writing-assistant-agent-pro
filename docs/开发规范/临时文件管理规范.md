## 一、临时文件定义

### 1.1 临时文件类型

以下类型的文件被视为临时文件，**禁止在项目根目录创建**：

| 文件类型 | 匹配模式 | 说明 |
|---------|---------|------|
| 临时测试脚本 | `temp_*.py`, `test_*.py`, `*_test.py` | 测试完成后应删除 |
| 临时检查脚本 | `check_*.py`, `verify_*.py` | 验证完成后应删除 |
| 临时修复脚本 | `fix_*.py`, `debug_*.py` | 修复完成后应删除 |
| 临时工具脚本 | `quick_*.py`, `clear_*.py`, `find_*.py` | 使用完后应删除 |
| 临时输出文件 | `temp_*.txt`, `temp_*.json`, `temp_*.log` | 调试输出文件 |
| 临时批处理 | `*.bat`（除启动脚本外） | 临时批处理文件 |
| 临时文档 | `*报告.md`, `*说明.md`, `*总结.md`（根目录） | 应移至经验文档/ |

### 1.2 例外情况

以下文件**不是临时文件**，可以保留：

| 文件类型 | 说明 |
|---------|------|
| `一键启动.bat` | 主启动脚本 |
| `Novel Writing Assistant-Agent Pro启动.bat` | 项目启动脚本 |
| `经验文档/**` | 经验文档目录下的所有文档 |
| `docs/**` | 文档目录下的所有文档 |
| `tests/**` | 正式测试目录下的测试文件 |

## 四、清理流程

### 4.1 AI开发时的清理流程

**必须遵循的流程**：

1. **创建前确认位置**：
   - 测试脚本 → `tests/`
   - 工具脚本 → `tools/`
   - 经验文档 → `经验文档/`
   - 临时调试文件 → 使用后立即删除

2. **创建后立即清理**：
   - 临时测试脚本：测试完成后立即删除
   - 临时输出文件：使用后立即删除
   - 临时批处理：执行后立即删除

3. **项目瘦身命令**：
   当用户发送【项目瘦身】指令时，执行完整清理（见MEMORY.md ID:57669496）

### 4.2 临时文件自动清理脚本

**位置**：`tools/cleanup_temp_files.py`

```python
#!/usr/bin/env python3
"""临时文件自动清理脚本"""

import os
import re
from pathlib import Path

def cleanup_temp_files(project_root: Path):
    """清理项目根目录的临时文件"""
    
    # 临时文件模式
    temp_patterns = [
        r'^temp_.*\.py$',
        r'^test_.*\.py$',
        r'^.*_test\.py$',
        r'^check_.*\.py$',
        r'^verify_.*\.py$',
        r'^fix_.*\.py$',
        r'^quick_.*\.py$',
        r'^clear_.*\.py$',
        r'^debug_.*\.py$',
        r'^temp_.*\.(txt|json|log)$',
        r'^.*\.bat$',
    ]
    
    # 排除列表
    exclude_files = {
        '一键启动.bat',
        'Novel Writing Assistant-Agent Pro启动.bat'
    }
    
    # 清理根目录
    root_files = list(project_root.glob('*'))
    cleaned = []
    
    for file in root_files:
        if file.is_file() and file.name not in exclude_files:
            for pattern in temp_patterns:
                if re.match(pattern, file.name):
                    try:
                        file.unlink()
                        cleaned.append(file.name)
                    except Exception as e:
                        print(f"删除失败 {file.name}: {e}")
                    break
    
    print(f"已清理 {len(cleaned)} 个临时文件")
    for name in cleaned:
        print(f"  - {name}")

if __name__ == '__main__':
    project_root = Path(__file__).parent.parent
    cleanup_temp_files(project_root)
```

## 五、最佳实践

### 5.1 测试脚本

**推荐做法**：

```python
# 1. 在tests/目录创建正式测试
tests/
  test_api_integration.py
  test_knowledge_manager.py
  test_novel_generator.py

# 2. 使用pytest运行
pytest tests/ -v

# 3. 测试完成后不删除（作为项目资产）
```

```python
# ❌ 在根目录创建临时测试
temp_api_test.py  # 使用后忘记删除
test_quick.py     # 污染项目根目录
```

## 六、违反规范的处理

### 6.1 检测机制

- Git提交前检查：`.gitignore`自动忽略临时文件
- 项目瘦身命令：手动触发清理
- AI开发提醒：AI创建临时文件时主动提醒位置

### 6.2 修正流程

1. 发现临时文件在根目录
2. 判断文件类型（测试/工具/文档）
3. 移动到正确位置或删除
4. 更新`.gitignore`（如需要）
