## 一、安装与启动

### Q1.1: 支持哪些操作系统?

**A**: 支持:
- Windows 10/11(推荐)
- macOS 10.15+
- Linux(Ubuntu 20.04+, CentOS 8+)

暂不支持:
- Windows 7及以下
- macOS 10.14及以下
- 32位操作系统

### Q1.2: Python版本要求?

**A**: 
- 要求:Python 3.12.x
- 不支持:Python 3.13+和Python 3.11及以下
- 原因:3.12是当前稳定版本,3.13生态尚未完全成熟

### Q1.3: 如何安装依赖?

**A**: 
```bash
# 方式1:使用requirements.txt
pip install -r requirements.txt

# 方式2:使用国内镜像加速
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 方式3:使用conda
conda env create -f environment.yml
conda activate novel-assistant
```

### Q1.4: 启动时提示缺少模块?

**A**: 
检查依赖是否完整安装:
```bash
# 检查核心依赖
python -c "import pydantic; print(pydantic.__version__)"
python -c "import openai; print(openai.__version__)"
python -c "import lancedb; print(lancedb.__version__)"

# 重新安装依赖
pip install -r requirements.txt --force-reinstall
```

### Q1.5: 启动时提示权限错误?

**A**: 
- Windows:以管理员身份运行
- Linux/macOS:检查文件权限
```bash
chmod +x gui_main.py
chmod -R 755 .
```

## 二、配置问题

### Q2.1: 如何配置API密钥?

**A**: 
方式1:编辑config.yaml
```yaml
llm:
  default: deepseek
  deepseek:
    model: deepseek-chat
    api_key: sk-xxxxxxxxxxxxxxxx
    base_url: https://api.deepseek.com
```

方式2:使用环境变量
```bash
export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxx"
export OPENAI_API_KEY="sk-xxxxxxxxxxxxxxxx"
```

方式3:GUI界面配置
- 设置 → API设置 → 输入密钥 → 保存

**安全提示**:
- 不要将config.yaml上传到Git仓库
- 使用环境变量更安全
- 定期更换API密钥

### Q2.2: API密钥无效?

**A**: 
检查清单:
1. 密钥格式是否正确(以sk-开头)
2. 密钥是否过期或被禁用
3. 是否有足够的余额
4. base_url是否正确

测试API连接:
```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-xxxxxxxxxxxxxxxx",
    base_url="https://api.deepseek.com"
)

try:
    response = client.chat.completions.create(
        model="deepseek-chat",
        messages=[{"role": "user", "content": "你好"}]
    )
    print("API连接成功")
except Exception as e:
    print(f"API连接失败: {e}")
```

### Q2.3: 如何切换不同的API?

**A**: 
方式1:修改config.yaml
```yaml
llm:
  default: openai  # 切换为openai
```

方式2:GUI界面切换
- 设置 → API设置 → 选择默认API → 保存

方式3:代码中动态切换
```python
from core.config_manager import ConfigManager

config = ConfigManager()
config.set("llm.default", "openai")
```

### Q2.4: 如何配置代理?

**A**: 
方式1:系统代理
```bash
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"
```

方式2:config.yaml配置
```yaml
network:
  proxy:
    http: "http://127.0.0.1:7890"
    https: "http://127.0.0.1:7890"
```

方式3:OpenAI SDK配置
```python
from openai import OpenAI
import httpx

client = OpenAI(
    api_key="sk-xxx",
    http_client=httpx.Client(
        proxies="http://127.0.0.1:7890"
    )
)
```

### Q2.5: 配置文件丢失怎么办?

**A**: 
恢复步骤:
1. 使用config.yaml.example模板
```bash
cp config.yaml.example config.yaml
```

2. 编辑config.yaml,填入自己的配置
3. 重启应用

## 三、功能使用

### Q3.1: 大纲解析失败?

**A**: 
常见原因:
1. **格式不符合要求**
   - 确保使用标准格式
   - 卷次、章节、场景层级清晰
   - 示例:
     ```
     第一卷:序章
       第1章:开端
         场景1:城市夜景
     ```

2. **编码格式错误**
   - 确保使用UTF-8编码
   - 避免使用Word格式
   - 推荐使用纯文本编辑器

3. **内容不完整**
   - 确保大纲包含完整结构
   - 每个章节至少有一个场景
   - 场景描述不能为空

### Q3.2: 风格学习效果不好?

**A**: 
优化建议:
1. **样本质量**
   - 使用同一作者的作品
   - 确保风格统一
   - 样本数量:5-10个最佳

2. **样本大小**
   - 每个样本:1000-10000字
   - 太小:特征不明显
   - 太大:噪音增加

3. **样本处理**
   - 去除无关内容(版权信息等)
   - 统一编码格式
   - 检查文本完整性

### Q3.3: 人物一致性检查不准确?

**A**: 
改进方法:
1. **完善人物档案**
   - 补充性格特点
   - 添加行为示例
   - 标注语言习惯

2. **调整检查参数**
   - 提高严格度
   - 增加上下文窗口
   - 启用深度检查

3. **手动标注**
   - 标注关键行为
   - 标注语言风格
   - 标注决策模式

### Q3.4: 章节生成质量不高?

**A**: 
提升策略:
1. **优化提示词**
   - 提供详细上下文
   - 明确风格要求
   - 添加参考示例

2. **调整参数**
   - 提高迭代次数(3-5次)
   - 调整温度参数(0.7-0.9)
   - 增加上下文窗口

3. **完善配置**
   - 完整的大纲
   - 详细的人物设定
   - 完整的世界观

### Q3.5: 生成的章节缺少【本章完】标记?

**A**: 
原因:AI生成可能遗漏标记

解决:
1. **自动补充**(V5.6已修复)
   - 系统会自动检测并补充
   - 智能判断补充位置

2. **手动添加**
   - 在章节末尾添加【本章完】

3. **重新生成**
   - 点击"重新生成"按钮
   - 系统会在提示词中强调

### Q3.6: 如何使用长期记忆功能?

**A**: 
长期记忆自动启用:
- L1热记忆:自动维护会话状态
- L2温记忆:自动向量索引
- L3冷记忆:需手动添加Git-Notes

手动添加Notes:
```bash
git notes add -m "重要决策:主角性格调整" <commit-hash>
```

查询历史:
```bash
git log --notes --grep="关键词"
```

### Q3.7: 如何创建自定义知识库?

**A**: 
步骤:
1. 准备知识点数据(JSON格式)
```json
[
  {
    "category": "分类",
    "title": "标题",
    "content": "内容",
    "tags": ["标签1", "标签2"]
  }
]
```

2. 导入知识库
```python
from core.knowledge_manager import KnowledgeManager

manager = KnowledgeManager()
manager.import_from_json("custom_knowledge.json")
```

3. 使用知识库
- 自动在章节生成时检索
- 手动检索:知识库管理界面

## 四、性能问题

### Q4.1: 启动速度慢?

**A**: 
优化方法:
1. **减少插件加载**
   - 禁用不用的插件
   - 设置延迟加载

2. **清理缓存**
```bash
rm -rf cache/*
rm -rf __pycache__
```

3. **优化数据库**
```bash
sqlite3 data/novel.db "VACUUM;"
sqlite3 data/novel.db "ANALYZE;"
```

4. **增加硬件资源**
   - 使用SSD硬盘
   - 增加内存(8GB+)
   - 使用多核CPU

### Q4.2: 生成速度慢?

**A**: 
影响因素:
1. **API响应时间**
   - DeepSeek:2-5秒
   - OpenAI:3-10秒
   - 本地Ollama:5-30秒(取决于硬件)

2. **生成参数**
   - 字数越多越慢
   - 迭代次数越多越慢
   - 温度参数影响不大

3. **网络延迟**
   - 使用国内API更快
   - 配置代理可能加速

4. **系统资源**
   - 检查CPU占用
   - 检查内存占用
   - 关闭其他应用

### Q4.3: 内存占用过高?

**A**: 
排查方法:
1. **检查缓存大小**
```python
from core.cache_manager import CacheManager
cache = CacheManager()
print(cache.get_size())
```

2. **调整缓存配置**
```yaml
cache:
  l1_size: 128  # 减小缓存大小
```

3. **清理内存**
```python
import gc
gc.collect()
```

4. **检查向量数据库**
- LanceDB内存映射
- 定期清理无用向量

### Q4.4: 向量检索慢?

**A**: 
优化方法:
1. **创建索引**
```python
from infrastructure.vector_store import NovelVectorStore

store = NovelVectorStore()
store.create_index(
    num_partitions=256,
    num_sub_vectors=16
)
```

2. **调整参数**
```yaml
vector_store:
  nprobes: 20  # 搜索探测数量
  refine_factor: 1  # 精炼因子
```

3. **硬件优化**
- 使用SSD硬盘
- 增加内存
- 使用GPU加速

### Q4.5: 数据库文件过大?

**A**: 
清理方法:
1. **VACUUM操作**
```bash
sqlite3 data/novel.db "VACUUM;"
```

2. **清理历史数据**
```sql
DELETE FROM generation_history WHERE created_at < datetime('now', '-30 days');
```

3. **压缩向量**
- 定期重建向量索引
- 删除冗余向量

## 五、错误排查

### Q5.1: 错误代码E001:API密钥无效

**A**: 
解决步骤:
1. 检查密钥格式
2. 验证密钥有效性
3. 检查余额
4. 联系API提供商

### Q5.2: 错误代码E002:网络连接失败

**A**: 
检查清单:
1. 网络连接是否正常
2. 代理配置是否正确
3. 防火墙是否拦截
4. API服务是否可用

测试连接:
```bash
curl -I https://api.deepseek.com
ping api.deepseek.com
```

### Q5.3: 错误代码E003:文件格式错误

**A**: 
处理方法:
1. 检查文件编码(应为UTF-8)
2. 检查文件格式(应符合要求)
3. 检查文件完整性
4. 尝试重新保存文件

### Q5.4: 错误代码E004:内存不足

**A**: 
解决方法:
1. 关闭其他应用
2. 减小缓存大小
3. 清理内存
4. 增加虚拟内存

### Q5.5: 错误代码E005:插件加载失败

**A**: 
排查步骤:
1. 检查插件依赖
```bash
pip install -r plugins/my-plugin/requirements.txt
```

2. 检查插件配置
3. 查看错误日志
4. 联系插件开发者

### Q5.6: 错误代码E006:数据库错误

**A**: 
修复方法:
1. 检查数据库文件
```bash
sqlite3 data/novel.db "PRAGMA integrity_check;"
```

2. 备份数据
```bash
cp data/novel.db data/novel.db.backup
```

3. 恢复数据库
```bash
sqlite3 data/novel.db "RECOVER;"
```

### Q5.7: 错误代码E007:大纲解析失败

**A**: 
见Q3.1。

### Q5.8: 错误代码E008:风格学习失败

**A**: 
见Q3.2。

### Q5.9: 未知错误怎么办?

**A**: 
排查步骤:
1. 查看日志文件
```bash
tail -f logs/app.log
```

2. 启用调试模式
```yaml
logging:
  level: DEBUG
```

3. 重现错误
   - 记录操作步骤
   - 截图错误信息
   - 保存相关文件

4. 寻求帮助
   - 提交Issue
   - 附上日志文件
   - 描述复现步骤

## 六、最佳实践

### Q6.1: 如何提高生成质量?

**A**: 
建议:
1. **准备充分的素材**
   - 完整的大纲
   - 详细的人物设定
   - 完整的世界观
   - 高质量的风格样本

2. **优化配置**
   - 合适的迭代次数(3-5次)
   - 适当的温度参数(0.7-0.9)
   - 足够的上下文窗口

3. **人工干预**
   - 审查生成结果
   - 手动修改不足之处
   - 反馈改进建议

### Q6.2: 如何降低API成本?

**A**: 
策略:
1. **使用性价比高的API**
   - DeepSeek:¥0.14/¥0.28每百万token
   - 本地Ollama:免费

2. **优化提示词**
   - 精简上下文
   - 避免重复信息
   - 使用模板

3. **启用缓存**
   - 缓存常用查询
   - 复用生成结果

4. **批量处理**
   - 批量生成章节
   - 减少API调用次数

### Q6.3: 如何保证长篇连贯性?

**A**: 
方法:
1. **使用长期记忆**
   - L1热记忆:实时状态
   - L2温记忆:向量检索
   - L3冷记忆:历史决策

2. **维护人物档案**
   - 及时更新人物状态
   - 记录人物成长
   - 标注关系变化

3. **定期一致性检查**
   - 使用ConsistencyChecker Agent
   - 检查人物一致性
   - 检查世界观一致性

4. **人工审查**
   - 定期审查已生成章节
   - 标记不一致之处
   - 修正和补充
