## 一、系统概述

### 1.1 产品定位

Novel Writing Assistant-Agent Pro 是一款智能小说写作辅助工具,基于微内核架构设计,集成长期记忆系统和本地知识库,为作者提供从大纲构建到内容生成的全流程辅助。

### 1.2 核心特性

**智能创作辅助**
- 多维度风格学习与分析
- 人物人设一致性检查
- 世界观设定自动化解析
- 大纲驱动的章节生成

**长期记忆系统**
- OpenClaw mem9五层记忆架构
- L1热记忆:实时会话状态
- L2温记忆:LanceDB向量检索
- L3冷记忆:Git-Notes历史决策
- L4档案:持久化知识库

**本地知识库**
- 科幻知识库:490条
- 玄幻知识库:300条
- 通用知识库:200条
- 支持自定义扩展

**插件生态**
- 7种插件类型
- 13个已部署插件
- 支持热插拔
- 标准化接口

### 1.3 技术架构

**核心技术栈**
- Python 3.12.x
- Tkinter + sv_ttt (GUI)
- Pydantic v2 (数据验证)
- SQLite + WAL (数据库)
- LanceDB (向量数据库)
- OpenAI SDK (LLM客户端)

**架构设计**
- 微内核架构
- 事件驱动
- 插件系统
- Agent协作

## 三、核心功能详解

### 3.1 大纲解析

#### 3.1.1 大纲导入

**支持格式**:
- 纯文本文件(.txt)
- Markdown文件(.md)
- JSON格式(.json)

**大纲结构要求**:
```
第一卷:序章
  第1章:开端
    场景1:城市夜景
    场景2:主角登场
  第2章:相遇
第二卷:发展
  第3章:冲突
```

**自动解析功能**:
- 卷次识别
- 章节划分
- 场景拆分
- 剧情线标注

#### 3.1.2 大纲验证

**验证项**:
- 结构完整性
- 章节编号连续性
- 场景描述完整性
- 剧情线一致性

**输出结果**:
- 验证报告
- 问题列表
- 修复建议

### 3.2 风格学习

#### 3.2.1 样本上传

**样本要求**:
- 格式:纯文本(.txt)
- 大小:建议1000-10000字
- 数量:至少3个样本
- 类型:同一作者或同一风格

**上传方式**:
1. 拖拽文件到上传区
2. 点击"选择文件"按钮
3. 批量上传(支持多选)

#### 3.2.2 风格分析

**分析维度**:
1. **语言特征**
   - 句子长度分布
   - 词汇多样性
   - 标点符号使用
   - 修辞手法

2. **叙事风格**
   - 视角选择
   - 节奏控制
   - 描写比例
   - 对话占比

3. **情感倾向**
   - 情感基调
   - 情感变化曲线
   - 高潮点分布

#### 3.2.3 风格报告

**报告内容**:
- 风格特征摘要
- 量化指标
- 可视化图表
- 改进建议

### 3.3 人物管理

#### 3.3.1 人物档案

**档案字段**:
- 基本信息:姓名、年龄、性别
- 外貌特征:身高、体型、特点
- 性格特点:核心性格、优点、缺点
- 背景故事:出身、经历、动机
- 人物关系:家庭、朋友、敌人
- 成长轨迹:初期、发展、变化

#### 3.3.2 人设一致性检查

**检查项**:
- 性格行为一致性
- 语言风格一致性
- 能力成长合理性
- 关系发展连贯性

**检查时机**:
- 章节生成后自动检查
- 手动触发检查
- 批量检查整部作品

#### 3.3.3 关系图谱

**图谱类型**:
- 人物关系网
- 势力关系图
- 情感关系链

**操作说明**:
- 拖拽调整布局
- 点击查看详情
- 导出图片

### 3.4 世界观解析

#### 3.4.1 设定导入

**支持格式**:
- 结构化文本
- Markdown文档
- JSON配置

**解析内容**:
- 地理环境
- 历史背景
- 社会制度
- 力量体系
- 种族设定
- 科技水平

#### 3.4.2 自动检测

**检测项**:
- 设定完整性
- 逻辑一致性
- 冲突点识别

**冲突类型**:
- 时间线冲突
- 能力体系冲突
- 地理描述冲突
- 历史事件冲突

### 3.5 章节生成

#### 3.5.1 生成参数

**基础参数**:
- 目标字数:1000-10000字
- 生成模式:大纲驱动/自由创作
- 风格参考:已学习风格
- 人物视角:第一人称/第三人称

**高级参数**:
- 上下文窗口:前N章记忆
- 创意度:0.0-1.0
- 迭代次数:1-5次
- 质量阈值:0.8+

#### 3.5.2 生成流程

**标准流程**:
1. 选择大纲章节
2. 配置生成参数
3. 构建上下文
4. 调用AI生成
5. 质量评分
6. 迭代优化(如需)
7. 输出最终结果

**评分维度**:
- 字数符合度:10%
- 大纲吻合度:15%
- 风格一致性:25%
- 人设准确性:25%
- 世界观一致性:15%
- 自然度:10%

#### 3.5.3 迭代优化

**触发条件**:
- 总分低于0.8
- 缺少【本章完】标记
- 用户手动要求

**优化策略**:
- 针对性提示词调整
- 上下文补充
- 参数微调
- 最多迭代5次

### 3.6 质量验证

#### 3.6.1 自动验证

**验证时机**:
- 章节生成后
- 用户请求时
- 批量验证

**验证内容**:
1. **结构验证**
   - 章节完整性
   - 段落合理性
   - 场景连贯性

2. **内容验证**
   - 人物一致性
   - 世界观一致性
   - 剧情连贯性

3. **语言验证**
   - 语法正确性
   - 用词恰当性
   - 风格统一性

#### 3.6.2 验证报告

**报告格式**:
```markdown
## 验证报告

**章节**: 第X章 标题
**总分**: 0.85

### 各维度得分
- 字数符合度: 0.90
- 大纲吻合度: 0.88
- 风格一致性: 0.82
- 人设准确性: 0.85
- 世界观一致性: 0.87
- 自然度: 0.83

### 问题列表
1. [P1] 第3段人物性格描述与设定不一致
2. [P2] 第5段世界观描述存在轻微矛盾

### 改进建议
1. 调整第3段人物反应,增强性格一致性
2. 修正第5段世界观细节描述
```

## 五、AI辅助创作

### 5.1 API配置

#### 5.1.1 支持的API

**DeepSeek API** (默认)
- 模型:deepseek-chat
- 定价:输入¥0.14/百万token,输出¥0.28/百万token
- 特点:性价比高,中文理解好

**OpenAI API**
- 模型:GPT-4, GPT-3.5
- 定价:按官网标准
- 特点:功能全面,生态完善

**Anthropic API**
- 模型:Claude系列
- 定价:按官网标准
- 特点:安全可靠,长文本

**本地Ollama**
- 模型:开源模型
- 定价:免费
- 特点:隐私保护,离线使用

#### 5.1.2 API密钥管理

**安全规则**:
1. API密钥存储在config.yaml
2. config.yaml已从git追踪中排除
3. 提供config.yaml.example作为模板
4. 支持加密存储(可选)

**配置示例**:
```yaml
llm:
  default: deepseek
  deepseek:
    model: deepseek-chat
    api_key: sk-xxxxxxxxxxxxxxxx
    base_url: https://api.deepseek.com
  openai:
    model: gpt-4
    api_key: sk-xxxxxxxxxxxxxxxx
    base_url: https://api.openai.com/v1
```

#### 5.1.3 切换API

**切换方式**:
1. GUI设置界面选择
2. 修改config.yaml配置
3. 代码中动态切换

**注意事项**:
- 切换前确保密钥正确
- 不同API模型能力不同
- 注意成本控制

### 5.2 模型选择

#### 5.2.1 模型特点对比

| 模型 | 优势 | 劣势 | 适用场景 |
|------|------|------|----------|
| deepseek-chat | 性价比高,中文好 | 能力中等 | 日常创作 |
| GPT-4 | 能力强,功能全 | 成本高 | 复杂任务 |
| GPT-3.5 | 成本低,速度快 | 能力有限 | 简单生成 |
| Claude-3 | 长文本,安全可靠 | 生态较小 | 长篇创作 |
| Qwen-7B | 免费,隐私保护 | 需本地部署 | 离线使用 |

#### 5.2.2 模型参数调优

**温度(Temperature)**:
- 范围:0.0-2.0
- 默认:0.7
- 建议:创意内容0.8-1.0,严谨内容0.3-0.5

**Top-P**:
- 范围:0.0-1.0
- 默认:0.9
- 建议:与温度配合调整

**最大令牌数**:
- 范围:1-模型上限
- 建议:根据目标字数设置

**频率惩罚**:
- 范围:0.0-2.0
- 默认:0.0
- 建议:减少重复内容时0.5-1.0

### 5.3 提示词工程

#### 5.3.1 提示词模板

**基础模板结构**:
```
【角色定义】
你是一位专业的小说创作助手。

【任务描述】
根据以下大纲生成章节内容。

【上下文信息】
前文概要: {summary}

【输出要求】
- 字数: {word_count}字左右
- 风格: {style_guide}

- 结尾标记: 【本章完】

【大纲内容】
{outline}

【开始生成】
```

#### 5.3.2 提示词优化技巧

**清晰性**:
- 使用明确的指令
- 避免模糊表述
- 提供具体示例

**结构化**:
- 分段组织内容
- 使用标记区分
- 层次清晰

**上下文**:
- 提供必要背景
- 控制上下文长度
- 突出关键信息

**示例驱动**:
- 提供参考样本
- 展示期望输出
- 对比好坏示例

### 5.4 成本控制

#### 5.4.1 成本估算

**DeepSeek示例**:
- 目标字数:3000字
- 输入token:约2000(含上下文)
- 输出token:约4000
- 成本:(2000×0.14 + 4000×0.28) / 1000000 ≈ ¥0.0015

**优化建议**:
- 精简上下文
- 复用生成结果
- 批量处理

#### 5.4.2 缓存策略

**缓存层级**:
1. L1:会话缓存(内存)
2. L2:本地缓存(磁盘)
3. L3:向量检索(数据库)

**缓存效果**:
- 重复请求:100%命中
- 相似请求:80%命中
- 新请求:缓存未命中

## 八、最佳实践

### 8.1 大纲构建最佳实践

#### 8.1.1 大纲层次

**推荐结构**:
```
作品
├── 卷次(第一层)
│   ├── 章节(第二层)
│   │   ├── 场景(第三层)
│   │   │   └── 情节点(第四层)
```

#### 8.1.2 大纲粒度

**粗细平衡**:
- 太粗:缺乏指导性
- 太细:限制创作空间
- 建议:场景级描述,关键情节点明确

#### 8.1.3 剧情线管理

**剧情线类型**:
- 主线:核心剧情
- 副线:辅助剧情
- 暗线:伏笔剧情

**管理建议**:
- 主线贯穿始终
- 副线适时穿插
- 暗线前后呼应

### 8.2 风格学习最佳实践

#### 8.2.1 样本选择

**样本质量**:
- 同一作者或同一风格
- 代表性作品
- 内容完整
- 风格统一

**样本数量**:
- 最少:3个样本
- 建议:5-10个样本
- 最多:不超过20个样本

#### 8.2.2 样本处理

**预处理步骤**:
1. 去除无关内容(版权信息等)
2. 统一编码格式(UTF-8)
3. 检查文本完整性
4. 标注样本来源

#### 8.2.3 风格应用

**应用建议**:
- 初期:严格遵循学习风格
- 中期:适度调整
- 后期:形成独特风格

### 8.3 人物设定最佳实践

#### 8.3.1 人物深度

**立体人物构建**:
- 外在特征:外貌、穿着、习惯
- 内在特征:性格、价值观、动机
- 成长轨迹:变化过程、成长点
- 关系网络:社会关系、情感纽带

#### 8.3.2 人物一致性维护

**一致性检查点**:
- 行为是否符合性格
- 语言风格是否统一
- 能力成长是否合理
- 关系发展是否自然

#### 8.3.3 人物档案更新

**更新时机**:
- 章节生成后
- 剧情转折点
- 人物成长节点
- 定期检查

### 8.4 世界观构建最佳实践

#### 8.4.1 世界观完整性

**必要元素**:
- 地理环境:地形、气候、资源
- 历史背景:重要事件、时代背景
- 社会制度:政治、经济、文化
- 力量体系:规则、等级、限制
- 种族设定:特点、关系、分布

#### 8.4.2 世界观一致性

**一致性维护**:
- 设定文档化
- 定期检查冲突
- 版本控制
- 修改需同步更新

#### 8.4.3 世界观展示

**展示原则**:
- 渐进式展示
- 避免信息堆砌
- 与剧情结合
- 留有探索空间
