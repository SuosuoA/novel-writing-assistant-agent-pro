## 一、功能概述

### 1.1 什么是专家模式？

专家模式是一个智能创作质量优化系统，通过九维度智能评分和切实可执行的优化建议，帮助您提升小说创作的整体质量。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 九维度智能评分 | 世界观/人设/大纲/风格/知识库/写作技巧/字数/上下文衔接/AI感 |
| 强制质量检查 | 【本章完】标记缺失时自动拦截，确保章节完整性 |
| 切实可执行建议 | 针对低分维度生成具体修改示例，而非空泛描述 |
| Claw记忆优化 | 越用越聪明，自动记录优化历史，避免重复问题 |
| 本地模型辅助 | 调用本地AI模型辅助评分，提升准确性 |

### 1.3 与普通模式的区别

| 对比项 | 普通模式 | 专家模式 |
|--------|----------|----------|
| 评分维度 | 8维度 | 9维度（新增AI感检测） |
| 优化建议 | 通用建议 | 切实可执行 + 修改示例 |
| 质量检查 | 无强制检查 | 【本章完】强制检查 |
| 记忆优化 | 无 | Claw四层记忆架构 |
| 本地辅助 | 无 | sentence_transformers语义分析 |

## 二、快速开始

### 2.1 启动专家模式

1. 打开 Novel Writing Assistant-Agent Pro
2. 进入【工作台】标签页
3. 在创作区域顶部找到【专家选择器】下拉框
4. 选择【小说创作专家】
5. 点击【开始创作】按钮

### 2.2 第一次使用

**准备工作**:
- 确保已加载项目的基本设定（世界观/人物/大纲）
- 建议首次使用时选择较短章节（2000-3000字）测试效果

**操作流程**:
```
选择专家 → 配置参数 → 开始创作 → 查看评分 → 采纳建议 → 完成
```

## 三、操作详解

### 3.1 专家选择器

**位置**: 工作台 → 创作区域顶部

**可用专家**:
- 小说创作专家（当前可用）
- 更多专家类型（开发中）

**配置选项**:
| 选项 | 说明 | 默认值 |
|------|------|--------|
| 启用记忆优化 | 使用Claw记忆架构，越用越聪明 | 开启 |
| 启用本地模型 | 使用本地AI辅助评分 | 开启 |
| 评分阈值 | 低于此分数触发优化建议 | 0.8 |

### 3.2 创作流程

#### 步骤1：选择专家
在下拉框中选择【小说创作专家】

#### 步骤2：配置参数
- 章节标题
- 目标字数
- 大纲引用
- 人物引用
- 风格引用

#### 步骤3：开始创作
点击【开始创作】按钮，系统将：
1. 自动整合所有设定数据
2. 调用AI生成内容
3. 进行九维度评分
4. 检查【本章完】标记

#### 步骤4：查看评估结果
创作完成后，将弹出【专家评估对话框】，显示：
- 总分和各维度得分
- 优化建议
- 修改示例

#### 步骤5：决策
根据评估结果，您可以选择：
- **采纳建议**：系统将根据建议重新生成优化版本
- **保持原内容**：直接使用当前内容
- **手动修改**：根据建议自行调整

### 3.3 强制检查机制

**【本章完】标记检查**:
- 系统会自动检查章节结尾是否有【本章完】标记
- 支持的格式：【本章完】、[本章完]、（本章完）、(本章完)、本章完
- **缺失标记时**：总分强制为0，触发重新生成流程
- **自动补充**：检测到缺失时自动补充标记并重新评分

**注意**: 此机制为一票否决制，确保章节完整性。

## 四、评分体系

### 4.1 九维度权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 世界观 | 12% | 设定一致性、世界观融合度 |
| 人设 | 19% | 人物性格、行为、对话是否符合设定 |
| 大纲 | 13% | 情节推进是否符合大纲规划 |
| 风格 | 19% | 写作风格是否与参考作品一致 |
| 知识库 | 8% | 专业知识、背景设定的运用 |
| 写作技巧 | 8% | 叙事技巧、描写手法、修辞运用 |
| 字数 | 8% | 是否达到目标字数 |
| 上下文衔接 | 8% | 与前文衔接是否自然流畅 |
| AI感 | 5% | 文本是否显得自然，避免机械感 |

**权重总和**: 100%

### 4.2 评分标准

**各维度评分范围**: 0.0 - 1.0

**总分计算**:
```
总分 = Σ(维度得分 × 维度权重)
```

**质量等级**:
| 总分范围 | 等级 | 建议 |
|----------|------|------|
| ≥ 0.9 | 优秀 | 可直接使用 |
| 0.8 - 0.9 | 良好 | 可使用，可选优化 |
| 0.6 - 0.8 | 合格 | 建议优化后再使用 |
| < 0.6 | 待改进 | 必须优化 |

### 4.3 【本章完】强制检查

**检查规则**:
- 检查位置：章节结尾最后100字符
- 匹配模式：【本章完】、[本章完]、（本章完）、(本章完)、本章完
- **一票否决**：缺失标记 → 总分强制为0

**自动处理**:
1. 检测到缺失 → 补充标记 → 重新评分
2. 重新评分后仍不达标 → 触发优化流程

## 五、优化建议

### 5.1 建议类型

专家模式会针对低分维度（< 0.7）生成优化建议：

| 维度 | 建议示例 |
|------|----------|
| 人设 | "增加人物标志性动作描写"、"对话中体现人物性格特点" |
| 风格 | "保持句式多样性"、"注意用词的准确性" |
| 写作技巧 | "使用'展示而非告知'技巧"、"增加感官描写" |
| AI感 | "增加口语化表达"、"变化句式结构" |

### 5.2 修改示例

每个建议都附带具体的修改示例，例如：

**原文本**:
> 他走进房间，看到了那把剑。

**优化建议**: 增加感官描写，使用"展示而非告知"技巧

**修改示例**:
> 他推开门，一股陈腐的气息扑面而来。昏暗的房间角落，那把古剑静静躺在木架上，剑身映着微弱的光，似乎在诉说着过往的岁月。

### 5.3 迭代优化

**优化流程**:
```
评分不达标 → 生成建议 → 采纳建议 → 重新生成 → 重新评分 → 达标输出
```

**迭代上限**: 5次

**达标标准**: 总分 ≥ 0.8

## 六、常见问题

### Q1: 专家模式会消耗更多Token吗？

**A**: 会增加约20%的Token消耗，主要用于评分和建议生成。但通过本地模型辅助和Claw记忆优化，长期来看可以减少重复问题的Token消耗。

### Q2: 如果专家插件加载失败怎么办？

**A**: 系统会自动降级为普通模式，不影响基本创作功能。降级时会显示提示信息。

### Q3: 评分阈值可以调整吗？

**A**: 可以。在专家配置面板中调整【评分阈值】参数，范围0.5-0.95。建议保持默认值0.8。

### Q4: 如何关闭强制检查机制？

**A**: 此机制为强制执行，不可关闭。这是为了确保章节完整性。

### Q5: 本地模型是否必须？

**A**: 非必须。如果本地模型不可用，系统会自动使用Jaccard相似度算法作为降级方案。

### Q6: Claw记忆数据存储在哪里？

**A**: 存储在项目的 `.workbuddy/memory/` 目录下，包括：
- L1热记忆：`core/session_state.py`
- L2温记忆：`infrastructure/vector_store.py`
- L3冷记忆：`core/git_notes_manager.py`
- L4档案：`.workbuddy/memory/MEMORY.md`

### Q7: 如何查看历史���估记录？

**A**: 在【专家评估对话框】中点击【历史记录】按钮，可查看最近的评估记录和优化历史。

### Q8: 优化建议是否总是准确？

**A**: 建议基于当前数据源和评分算法生成，准确性与数据源质量相关。建议作为参考，结合自身判断使用。

### Q9: 多个专家可以同时使用吗？

**A**: 当前版本只支持选择一个专家。后续版本会支持专家组合。

### Q10: 如何反馈问题或建议？

**A**: 请在项目GitHub仓库提交Issue，或在软件内使用【反馈】功能。
